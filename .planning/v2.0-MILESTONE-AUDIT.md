## Cross-Phase Integration Issues

### Critical (Block Requirements)

1. **Missing adapter subpath exports** — `packages/integrations/package.json` missing exports for `notion-adapter`, `confluence-adapter`, `google-calendar-adapter`, `outlook-calendar-adapter`. Blocks DOCS-01, DOCS-02, CAL-01, CAL-02.

2. **`time` permission not registered** — `packages/auth/src/permissions.ts` missing `time: ["read", "approve"]`. All 10 admin time router procedures fail compilation. Blocks TIME-02.

3. **`ctx.userId` → `ctx.user.id`** — `packages/api/src/routers/calendar.ts` uses wrong context field in 5 places. Runtime crash on all calendar procedures. Blocks CAL-01, CAL-02.

4. **`contract.name` → `contract.title`** — `packages/api/src/routers/calendar.ts` uses wrong field name. Runtime crash on syncContractDeadline. Blocks CAL-01.

5. **`DocLinksSection` orphaned** — Created but never imported into `task-card-run.tsx` or `workflow-side-panel.tsx`. No UI path to attach docs to workflow steps. Blocks DOCS-01.

6. **`CalendarTaskConfig` orphaned** — Created but never imported into `template-builder/task-card.tsx`. No UI to configure per-task calendar events. Blocks CAL-02.

7. **Calendar auto-push not wired** — `syncContractDeadline` and `syncPaymentDeadline` mutations exist but are never called from contract or invoice lifecycle routers. CAL-01 is manual-only.

### Build Blockers

8. **`time-entry.ts` `$transaction` type** — `(tx: PrismaClient)` callback annotation incompatible with Prisma's expected type. Causes API package build failure.

9. **`CredentialBlob` cast** — `doc-link-service.ts` casts `CredentialBlob` directly to `Record<string, unknown>` without `as unknown` intermediate. Type error.
