### JIRA-02 Gap Detail

**Requirement:** Workflow steps can auto-create Jira issues with configurable project/type mapping

**What exists:**
- `createJiraIssue` service in `packages/api/src/services/jira-issue-sync.ts` (line 84) — fully implemented
- `JiraTaskConfig` UI component mounted in `task-card.tsx` — admin can configure project/issue type per task
- Unit tests covering `createJiraIssue` behavior

**What's missing:**
- `workflow.ts` `startRun` (or task activation point) never calls `createJiraIssue` for tasks with `jiraEnabled: true`
- The service exists, the UI config exists, but the trigger wiring is absent
- Pattern needed: fire-and-forget call mirroring `transitionJiraIssue` at workflow.ts lines 1081-1108
