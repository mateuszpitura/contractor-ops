> Tracks risks accepted (deferred fixes, intentional gaps, operational blockers) that survived the post-migration parity restoration. Each row carries an ID, owner, review cadence, and cross-reference to the originating audit row.

## Schema

| Field | Meaning |
|-------|---------|
| ID | `RISK-<AREA>-<NNN>` â€” stable handle for cross-reference |
| Origin | Audit row (`GAP-<AREA>-<NNN>`) or other source |
| Description | One-line statement of the risk |
| Severity | `low` / `medium` / `high` |
| Owner | Role responsible for next-step decision |
| Review cadence | Quarterly / monthly / before-next-release |
| Mitigations | Existing controls that reduce impact |
