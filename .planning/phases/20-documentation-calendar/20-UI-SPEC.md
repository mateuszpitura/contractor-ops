## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps within doc chips, provider icon inline spacing, chip vertical padding |
| sm | 8px | Doc chip horizontal padding, doc link list row gaps, calendar event row spacing |
| md | 16px | Form field vertical spacing, card content padding, Cmd+K group padding |
| lg | 24px | Section padding within dialogs, card internal padding, settings section spacing |
| xl | 32px | Layout gaps between settings sections, dialog section breaks |
| 2xl | 48px | Page-level vertical rhythm |
| 3xl | 64px | Not used in Phase 20 |

Exceptions: Doc link chips use 4px vertical padding (`py-1`) and 8px horizontal padding (`px-2`) for compact inline display, consistent with JiraIssueChip. Calendar event config uses 8px vertical padding per row.

## Layout Contract

### Focal Point

The primary screen focal points for Phase 20 are:

1. **Workflow task card (template + run)**: DocLinksSection shows attached documents as chips. CalendarTaskConfig shows calendar event toggle and config summary. These render below the existing JiraTaskConfig section.
2. **Cmd+K command palette**: New "Docs" group appears in search results alongside existing Contractors, Contracts, Invoices, Pages, Actions groups.
3. **Settings > My Calendar**: New personal settings page for per-user calendar OAuth connection (Google/Outlook).
4. **Settings > Integrations tab**: Notion and Confluence provider cards for org-level doc search connection. OrgCalendarSection for shared org calendar.

### Doc Links on Workflow Task Card (DOCS-01, D-01, D-02, D-03)

- Location: within each workflow task card (template editor and run view), below JiraTaskConfig, above CalendarTaskConfig
- DocLinksSection renders a header row with FileText icon + "Documents" label + "Attach" ghost button
- Attached docs display as flex-wrap DocLinkChip list with 8px (`gap-2`) between chips
- Each chip is a clickable link opening the Notion/Confluence page in a new tab (D-07)
- Remove button (X, 12px) on each chip is visible on hover only, triggers AlertDialog confirmation
- "Attach" button opens AttachDocDialog for search-and-select
- Template view: attach/remove available (editable)
- Run view: read-only display of inherited doc links (no attach/remove buttons)
- Multiple links per step supported (D-02)

### Attach Document Dialog (DOCS-01, D-04, D-06)

- Trigger: "Attach" ghost button in DocLinksSection
- Dialog width: `sm:max-w-lg` (480px)
- Search input at top with placeholder "Search Notion and Confluence pages..."
- Provider filter as inline toggle group below search: "All" (default) | "Notion" (with icon) | "Confluence" (with icon)
- Results rendered in ScrollArea (max-height 320px)
- Each result: provider icon (14px) + page title (text-sm font-medium, flex-1 truncated) + workspace/space name (text-xs text-muted-foreground)
- Click on result immediately attaches and closes dialog (single-click attach, no separate confirm)
- Loading state: 3 Skeleton rows (icon skeleton 14px + text skeleton 60% width + text skeleton 30% width)
- Empty search state: "No pages found matching your search."
- No results when no providers connected: "Connect Notion or Confluence in Settings to search for pages."

### Docs in Cmd+K (DOCS-02, D-05, D-06, D-07)

- New "Docs" CommandGroup added to CommandPalette search results
- Group heading: "Docs" (matches existing "Results", "Pages", "Actions" pattern)
- Results fetched from tRPC alongside existing global search, debounced at 200ms
- Each result row: provider icon (14px) + page title (text-sm font-medium) + workspace/space name (text-xs text-muted-foreground, right-aligned)
- Type badge: Badge with `bg-chart-3/10 text-chart-3 border-transparent` -- new "docs" type in TYPE_BADGE_CLASSES
- Selecting a doc result opens the page URL in a new browser tab (D-07) -- does NOT navigate within the app
- Results appear after entity results, before Pages group
- Maximum 5 doc results shown in Cmd+K (full search available in Attach dialog)

### My Calendar Settings Page (CAL-01, CAL-02, D-12, D-14)

- New page at `/settings/calendar` (personal settings, not org admin)
- Page header: "My Calendar" (text-xl font-semibold) + description "Connect your personal calendar to sync deadlines and events from Contractor Ops." (text-sm text-muted-foreground)
- Two CalendarProviderCards stacked vertically with 16px gap
- Google Calendar card: Google Calendar icon (32px, multi-color) + "Google Calendar" title + connection status
- Outlook Calendar card: Outlook icon (32px, #0078D4 fill) + "Outlook Calendar" title + connection status
- Below cards: "Active Synced Events" section header (text-sm font-semibold) + count badge showing "{N} events synced" + "Events are automatically created for contract expiry, approval SLA deadlines, and payment due dates." helper text (text-xs text-muted-foreground)
- Only one calendar provider can be connected at a time (connecting a second disconnects the first) -- enforced in backend, UI shows both cards but only one can show "Connected" status

### Calendar Task Config (CAL-02, D-09)

- Location: within each workflow task template card, below DocLinksSection
- Single-row flex layout: Switch (left) + config summary text (center, flex-1) + "Configure" ghost button (right)
- Switch label: "Create calendar event when task activates"
- When configured: summary shows event title template + duration (e.g., "Onboarding kickoff - 1h") in text-sm
- When not configured: shows "Not configured" in text-sm text-muted-foreground
- "Configure" ghost button opens CalendarEventConfigDialog
- Switch disabled when no configuration exists (must configure first)
- Section separator (Separator component) between DocLinksSection and CalendarTaskConfig

### Calendar Event Config Dialog (D-09)

- Trigger: "Configure" ghost button in CalendarTaskConfig
- Dialog width: `sm:max-w-md` (448px)
- "Event Title" Input with placeholder "e.g., Onboarding kickoff - {contractor}"
- Below input: muted hint text "Available variables: {contractor}, {contract}, {task}" in text-xs text-muted-foreground
- "Duration" Select with options: "30 minutes", "1 hour", "2 hours", "4 hours", "Full day"
- "Attendees" Textarea with placeholder "email@example.com, another@example.com" and helper "Comma-separated email addresses. The contractor and task assignee are included automatically." in text-xs text-muted-foreground
- Footer: "Cancel" outline button (left) + "Save" primary button (right)

### Notion/Confluence Provider Cards (D-13, settings)

- Location: Settings > Integrations tab, alongside existing Jira/KSeF/Slack cards
- Standard ProviderConnectionCard pattern for each
- Notion card: Notion icon (32px, monochrome) + "Notion" title + status badge. Scopes displayed on detail sheet.
- Confluence card: Confluence icon (32px, #0052CC Atlassian blue) + "Confluence" title + status badge. Scopes displayed on detail sheet.
- These are org-level connections (admin only) -- enables doc search for all org users

### Org Shared Calendar (D-12)

- Location: Settings > Integrations tab, new "Calendar" subsection
- OrgCalendarSection renders ProviderConnectionCard for Google Calendar + Outlook (org-level shared calendar)
- Separate from per-user "My Calendar" page
- Description text: "Events will be pushed to this shared calendar for the entire organization."

## Calendar Event Title Format (D-15)

System-generated calendar events follow this title format:

| Event Type | Title Format | Example |
|------------|-------------|---------|
| Contract expiry | `[Contractor Ops] Contract expiry: {contractor} - {contract name}` | `[Contractor Ops] Contract expiry: Jan Kowalski - Frontend Dev Q2 2026` |
| Approval SLA | `[Contractor Ops] Approval deadline: {item type} - {item name}` | `[Contractor Ops] Approval deadline: Invoice - INV-2026-042` |
| Payment due | `[Contractor Ops] Payment due: {contractor} - {invoice number}` | `[Contractor Ops] Payment due: Jan Kowalski - INV-2026-042` |
| Task event | User-configured title template with variable substitution | `[Contractor Ops] Onboarding kickoff - Jan Kowalski` |

Event description contains a deep link to the relevant entity in Contractor Ops. Format: plain text summary + URL on separate line.
