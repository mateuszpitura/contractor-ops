<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tRPC input → DB | Untrusted client data hits the Leitweg-ID table; must be Zod + tenant-scoped. |
| Cross-tenant read | Leitweg-IDs encode relationships with public-sector buyers — potentially sensitive. |
| RBAC gate | Only users with contractors:write may CRUD Leitweg-IDs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-04-01 | Information Disclosure | Cross-tenant leak via `update`/`delete`/`setDefault` taking an id parameter | mitigate | Every `where` clause includes `organizationId: ctx.organizationId`; test #6 + #8 + #10 assert NOT_FOUND for cross-tenant ids. Prisma client extension enforces at runtime. |
| T-61-04-02 | Tampering | Malformed Leitweg-ID bypass check digit | mitigate | Zod `leitwegIdSchema` with Modulo-11-10 refine runs at router boundary (input schema); DB rejects via unique constraint + NOT NULL. Test #2 asserts BAD_REQUEST. |
| T-61-04-03 | Elevation of Privilege | Non-admin user creates Leitweg-IDs | mitigate | `requirePermission({ contractors: ['write'] })` on every mutation; test #11 asserts FORBIDDEN. |
| T-61-04-04 | Repudiation | Who set this Leitweg-ID as default? | accept | Prisma `updatedAt` + row-level columns suffice for v5.0; audit-log integration deferred to Phase 60 notification plumbing. |
| T-61-04-05 | Tampering | Race condition: two concurrent `setDefault` calls leave two rows with default=true | mitigate | Both writes inside `$transaction` with SELECT+UPDATE ordering; Prisma's `$transaction` defaults to serializable on Postgres. |
| T-61-04-06 | Information Disclosure | Enumeration via NOT_FOUND vs FORBIDDEN response-timing differences | mitigate | Always return NOT_FOUND for missing-or-wrong-tenant (never FORBIDDEN) — prevents leak via response-code oracle. |
| T-61-04-07 | Denial of Service | Abuse of `create` with invalid check-digit hot loop | accept | Zod rejects before DB hit; no cost amplification. Existing tRPC rate-limits cover the per-user side. |
</threat_model>
