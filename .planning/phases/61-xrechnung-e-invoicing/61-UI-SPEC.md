# Phase 61 — UI Design Contract

> Visual and interaction contract for the XRechnung E-Invoicing phase. Generated by gsd-ui-researcher, consumed by gsd-planner / gsd-executor / gsd-ui-checker.
>
> **This phase reuses the locked Precision Craft design system** (Deep Teal primary, OKLCh tokens, Outfit/Bricolage Grotesque/JetBrains Mono fonts, shadcn `base-nova` preset). Phase 61 surfaces — invoices-list compliance column + filter chips + summary tile, per-invoice "E-invoice" tab, Settings → E-invoicing page (Peppol participant + Leitweg-ID management), Leitweg-ID forms on contractor + contract pages — extend existing patterns. No new tokens, no new fonts, no new chromatic palette, no third-party shadcn registries.

## Spacing Scale

All values inherit the existing 4-pt-multiple scale via Tailwind v4 (`gap-1=4px`, `gap-2=8px`, `gap-4=16px`, `gap-6=24px`, `gap-8=32px`, `gap-12=48px`, `gap-16=64px`).

| Token | Value | Phase-61 Usage |
|-------|-------|----------------|
| xs | 4px | Icon-to-label gap inside status pills (validation layer pill, transmission pill), between check-digit segment and refinement segment captions in Leitweg-ID display |
| sm | 8px | Row padding inside SVRL issue list; chip + icon vertical rhythm; gap between filter chips on the invoice list |
| md | 16px | Default card padding (`p-4`); spacing between E-invoice-tab sub-sections (Generation → Validation → Transmission); grid gap in Leitweg-ID list rows |
| lg | 24px | Card section padding (`p-6`); gap between the compliance summary tile and the invoices table; gap between Peppol-participant card and Leitweg-ID list card on Settings page |
| xl | 32px | Vertical gap between major E-invoice-tab sections |
| 2xl | 48px | Page top padding under breadcrumb on `/settings/e-invoicing/` |
| 3xl | 64px | Reserved for empty-state vertical centering (no Leitweg-IDs registered / no e-invoices generated) |

**Touch targets:** All interactive controls (Finalize + validate, Download XML, Download full report, Send via Peppol, Validate now, Register Peppol participant, Create Leitweg-ID, Set default, Delete Leitweg-ID) must be ≥44px tall (WCAG 2.5.5). Inherits shadcn `button` default `h-10` (40px) → use `size="default"` minimum; primary CTAs on the E-invoice tab and Settings page use `size="default"` with `py-2.5` pad when a larger target is needed. Never `size="sm"` (`h-9` = 36px) for primary actions.

**Compliance filter chips:** 32px pill height (`h-8`) with 12px horizontal padding is acceptable for secondary filter controls because they are non-primary navigation; they remain keyboard-reachable and carry text labels meeting 4.5:1 contrast.

Exceptions: none.

Inherits the locked fluid-typography scale from `globals.css:306-323`. Phase 61 declares exactly **4 sizes** and **2 weights** for its surfaces.

| Label / heading (tile / row) | `text-base` | 14px → 15px (clamp) | 600 (semibold) | `1.5rem` | E-invoice tab sub-section titles ("Validation", "Transmission", "Generation"), table column headers, form-field labels, Leitweg-ID row primary text ("1234567890-AGENCY-56"), per-layer validation status row ("Layer 2 — EN 16931 Schematron"), Peppol participant status row |

**Total: 4 sizes (sm, base, xl, 3xl) and 2 weights (400 regular, 600 semibold) — strictly within the ≤4 sizes / ≤2 weights design-system budget.**

**Font families (locked by `globals.css`):**
- Body / labels / chrome / KPIs → `var(--font-sans)` = Outfit

**Explicitly not counted as additional scale steps:** `text-xs` (11–12px) is NOT used anywhere in Phase 61 chrome. All meta captions use `text-sm` at muted foreground colour for visual de-emphasis, not a smaller size. This keeps the phase-61 scale strictly at 4 sizes.

## Color

Inherits the full Precision Craft palette from `globals.css:20-121` (light) and `globals.css:127-224` (dark). Phase 61 maps semantic intent to existing tokens — does NOT introduce new color values.

| Role | Token | Usage in Phase 61 |
|------|-------|-------------------|
| Dominant (60%) | `--background` (warm off-white / deep warm slate) | Page surface for `/settings/e-invoicing/`, invoices list background, E-invoice tab canvas |
| Secondary (30%) | `--card`, `--surface-1`, `--surface-2` | Peppol participant card, Leitweg-ID list card, compliance summary tile, per-layer validation status panels inside E-invoice tab, Leitweg-ID form dialog container |
| Accent (10%) | `--primary` (Deep Teal `oklch(0.44 0.145 178)`) | **Reserved for:** primary CTAs ("Finalize + validate", "Send via Peppol", "Register Peppol participant", "Create Leitweg-ID"); compliance-summary KPI numeral when org is ≥95% compliant; active filter chip background; selected-tab indicator on the per-invoice tab row (`E-invoice` tab when active); focus rings (`--ring`) on every interactive control |
| Destructive | `--destructive` (refined crimson `oklch(0.58 0.23 18)`) | **Reserved for:** `INVALID` validation status pill; `FAILED` transmission status pill; "Delete Leitweg-ID" destructive confirmation button; "Deregister Peppol participant" destructive confirmation button; SVRL `<svrl:failed-assert>` rows with `@flag="fatal"` |
| Success (semantic, not part of 60/30/10) | `--success` (`oklch(0.55 0.17 150)`) | `VALID` validation status pill; `DELIVERED` transmission status pill; `ACTIVE` Peppol participant status pill; "This invoice is EN 16931 compliant" inline attest |
| Warning (semantic) | `--warning` (`oklch(0.7 0.16 65)`) | `WARNINGS` validation status pill; `LEITWEG_ID_MISSING` soft-gate inline alert on finalize result; SVRL rows with `@flag="warning"`; `PENDING` / `SUSPENDED` Peppol participant status pill; "Validation layer 1 passed, layers 2/3 have warnings" inline copy |
| Info (semantic) | `--info` (`oklch(0.55 0.18 260)`) | `QUEUED` / `SENT` transmission status pill (in-flight, awaiting delivery ack); SVRL rows with `@flag="info"`; "Last validated {time}" caption icon |
| Neutral / muted | `--muted`, `--muted-foreground` | `NOT_GENERATED` / `NOT_VALIDATED` / `NOT_SENT` / `NOT_REGISTERED` pills; caption text; empty-state illustrations |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. Primary CTAs on the E-invoice tab: **Finalize + validate**, **Send via Peppol**, **Validate now**
2. Primary CTAs on the Settings → E-invoicing page: **Register Peppol participant**, **Create Leitweg-ID**
3. Compliance-summary KPI numeral foreground when org is ≥95% compliant (below that threshold: `--warning`; zero or negative drift: `--destructive` with `--warning` band)
4. Active filter chip background (e.g. selected `Invalid` filter on invoices list) with `--primary-foreground` text
5. Focus rings via `--ring` on buttons, chip triggers, tab triggers, form inputs, table-row keyboard focus
6. Selected-tab indicator for the `E-invoice` tab on `/invoices/[id]/` (left border rule, 4px, matches Phase-58 verdict-banner pattern)

**Never use accent for:** card backgrounds, body text, decorative dividers, validation status pills (use semantic tokens), transmission status pills, or non-actionable iconography.

**Semantic triad rule (WCAG 1.4.1):** every status communicator (validation status pill, transmission status pill, participant status pill, compliance band chip, SVRL issue severity marker) MUST combine **colour + icon + text** — never colour alone. Reuses the Phase-58 / Phase-60 semantic-triad pattern (`verdict-banner.tsx:28-44`). Phase 61 lucide icons:

- `VALID` / `DELIVERED` / `ACTIVE` participant / compliant → `CircleCheck`
- `WARNINGS` / `LEITWEG_ID_MISSING` / `PENDING` / `SUSPENDED` / SVRL warning → `ShieldAlert`
- `INVALID` / `FAILED` transmission / SVRL fatal → `ShieldX`
- `QUEUED` / `SENT` / in-flight / SVRL info → `CircleDashed` (for in-flight feel) or `Info` (for informational)
- `NOT_GENERATED` / `NOT_VALIDATED` / `NOT_SENT` / `NOT_REGISTERED` / unknown → `Circle` (hollow neutral)

**Do NOT introduce a chart palette** — the compliance summary progress bar uses a single `--primary` fill on a `--muted` track (shadcn `progress` primitive). No stacked bars, no second hue.

### Error state copy (rule for the whole phase)

All tRPC query errors render with the existing `<Alert variant="destructive">` shadcn primitive. Copy pattern: **"{Problem one-liner}. {Recovery action}."**

| Error code | End-user copy |
|------------|---------------|
| `LEITWEG_ID_MISSING` | Leitweg-ID missing for this German public-sector buyer. Add one to the contractor or contract before sending via Peppol. |
| `PARTICIPANT_NOT_REACHABLE` | The recipient isn't registered on Peppol or doesn't support XRechnung. Confirm the buyer's Peppol ID and try again. |
| `KOSIT_VALIDATION_FAILED` | This e-invoice didn't pass KoSIT validation. Review the validation report and re-finalize after correcting the invoice data. |
| `PEPPOL_PARTICIPANT_NOT_ACTIVE` | Register your organisation on Peppol before sending e-invoices. Go to Settings → E-invoicing to start. |
| `STORECOVE_TRANSMISSION_FAILED` | Peppol transmission failed. We'll retry automatically — you can also retry manually from the E-invoice tab. |
| Generic tRPC error fallback | Something went wrong. Try refreshing or contact support if this persists. |

Never expose raw tRPC error codes, SVRL XPath, or KoSIT internal messages to end users in toast / alert chrome. The full SVRL issue list IS shown — but in a dedicated SVRL-issue-list panel where the developer / compliance reviewer expects technical detail. Chrome strings outside that panel remain human-friendly.

## Interaction Contracts

|---------|-------------|-----------|
| Invoices list compliance summary tile | Mount | Fetches `einvoice.summaryForOrg` query; renders `Skeleton` while pending |

| Invoices list filter chips | Click | Toggles the chip as the active filter (single-select within a row); URL-query-param backed (`?einvoiceStatus=invalid`) so links share filter state |

## Accessibility Contract (WCAG 2.2 AA — locked)

- **Colour contrast:** all text on `--background`, `--card`, and on `--primary` / `--destructive` / `--success` / `--warning` / `--info` swatches meets ≥4.5:1 for body, ≥3:1 for large text. Confirmed by the Precision Craft OKLCh palette reused unchanged from Phase 58–60.
- **Semantic triad:** every validation layer pill, transmission status pill, participant status pill, compliance filter chip, and SVRL severity marker combines colour + icon + text (1.4.1).
- **Keyboard:** all E-invoice-tab CTAs, filter chips, SVRL issue row toggles, Leitweg-ID row menus, participant register / deregister dialogs are reachable via Tab; activatable via Enter/Space. Focus ring uses `--ring` (Deep Teal).
- **Focus order:** logical — breadcrumb → page H1 → Peppol participant card (header → CTA) → Leitweg-ID list card (header → rows → Create CTA); on invoice-detail page: Details tab → E-invoice tab → Generation → Validation → Transmission.
- **Tooltip vs visible label:** tooltips on status pills are supplementary; raw status + count also visible as pill text. Information is never conveyed by tooltip alone (1.3.1).
- **Live regions:** "Finalize + validate" success announces "E-invoice finalized — validation {status} with {N} issues" via `aria-live="polite"`. Webhook-driven transmission status updates announce silently (the visible text change alone is sufficient; `aria-live="polite"` on the transmission pill ensures screen-reader users are notified).
- **Form validation:** Leitweg-ID + Peppol participant forms use inline `role="alert"` for validation errors (label, message, field association via `aria-describedby`).
- **Modal traps:** all create / edit / delete dialogs use shadcn `dialog` / `alert-dialog` (Radix) — focus trap and ESC handling inherited.
- **RTL:** all new layouts use `border-s-` / `start/end` Tailwind utilities. Leitweg-ID value keeps LTR direction inside mono cell via `<bdi dir="ltr">` wrapping (matches existing pattern in `apps/web/src/components/ui/bdi.tsx`).
- **i18n:** every visible string keyed under `EInvoice.*` in all four locales (en/de/pl/ar). Numbers, percentages, dates rendered via `Intl.NumberFormat` / `Intl.DateTimeFormat` with the active locale. German translation uses formal register (Sie) consistent with Phase 56 locked terminology.
- **No ambiguous iconography:** status-pill icons always paired with text; never icon-only.
