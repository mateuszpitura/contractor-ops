## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialized in Phase 1) |

**shadcn components needed for Phase 7 (new additions beyond Phase 6):**
- None. All required shadcn primitives are already installed.

**Custom components built in Phase 7 (not from any registry):**
- Notification popover -- bell icon dropdown with scrollable notification list. Built with shadcn Popover, ScrollArea, Button, Badge.
- Notification item -- individual notification row with icon, title, body, relative timestamp, unread dot. Built with custom CSS + shadcn Avatar, Badge.
- Notification center page -- full /notifications page with filters and bulk actions. Built with shadcn Tabs, Badge, Button, ScrollArea.
- Notification preference matrix -- settings grid with event type rows and channel toggle columns. Built with shadcn Switch, Table, Label.
- Reminder rule editor -- form for creating/editing reminder rules. Built with shadcn Dialog, Select, Input, Switch, Button.
- Reminder rules list -- card list of active rules with edit/delete actions. Built with shadcn Card, Badge, Switch, Button.
- Slack integration card -- settings card showing connection status, OAuth flow, user mapping. Built with shadcn Card, Button, Badge, Avatar, Table.
- Slack user mapping table -- table of auto-matched and manually linked users. Built with shadcn Table, Badge, Button, Command.

## Spacing Scale

Inherited from Phase 1. No changes.

| Token | Value | Usage in Phase 7 |
|-------|-------|-------------------|
| xs | 4px | Unread dot size, icon-to-text gap in notification items, inline padding in channel toggle cells |
| sm | 8px | Gap between notification items, gap between preference matrix rows, icon-to-label gap in reminder rule cards |
| md | 16px | Popover internal padding, notification item horizontal padding, form field gaps in reminder rule editor, Slack card section padding |
| lg | 24px | Notification center page content padding, settings tab section padding, gap between reminder rule cards |
| xl | 32px | Gap between notification preferences section and reminder rules section, gap between Slack connection card and user mapping table |
| 2xl | 48px | Page horizontal padding (desktop) |
| 3xl | 64px | Not used in Phase 7 |

Exceptions:
- Notification popover width: 384px (w-96)
- Notification popover max height: 480px (scrollable via ScrollArea)
- Unread indicator dot: 8px diameter, `--primary` color, positioned top-right of bell icon
- Unread count badge on bell: 18px height, min-width 18px, `--destructive` background, white text, 10px font size, positioned -4px top -4px right of bell icon
- Notification item height: minimum 64px (icon 32px + 16px top/bottom padding)
- Notification item icon circle: 32px diameter (contains Lucide icon matching event type)
- Notification center page max-width: 768px centered (reading-optimized)
- Preference matrix cell width: 80px per channel column (In-app, Email, Slack)
- Slack integration card: full-width within settings content area
- User mapping table: full-width within Slack integration card

## Color

Inherited from Phase 1. All color tokens from `globals.css` apply. Phase 7 adds usage patterns only.

| Role | Token | Usage in Phase 7 |
|------|-------|-------------------|
| Dominant (60%) | `--background` | Page background, popover background, notification center content area, dialog backgrounds |
| Secondary (30%) | `--muted` | Unread notification item background, preference matrix header row, reminder rule card backgrounds, Slack card background, filter chip inactive state |
| Accent (10%) | `--primary` | See reserved list below |
| Destructive | `--destructive` | See destructive usage below |

**Accent (`--primary`) reserved for in Phase 7:**
- Unread indicator dot on bell icon
- "Mark all read" link text in popover header
- "View all notifications" link in popover footer
- Active filter chip on notification center page (selected type filter)
- "Save preferences" primary CTA on notification preferences
- "Create rule" primary CTA on reminder rules section
- "Save rule" primary CTA in reminder rule editor dialog
- "Connect Slack" primary CTA on integrations card (disconnected state)
- Focus ring on all interactive elements
- Notification type icon circle for approval_request and invoice_received types

**Destructive (`--destructive`) usage in Phase 7:**
- Unread count badge background on bell icon (red badge with count)
- "Disconnect Slack" destructive action on integrations card
- "Delete rule" action on reminder rule cards
- Delete rule confirmation dialog CTA
- Error status badge on Slack connection card (when connection fails)

## Interaction Patterns

### Notification Popover (Bell Icon)

- **Trigger:** Click bell icon in top bar. Bell icon already rendered in `top-bar.tsx`.
- **Unread badge:** When unread count > 0, show red badge (`--destructive` bg, white text, 10px font) on bell icon. Count capped at "99+" display. Badge is 18px min-height/min-width, rounded-full, positioned absolute -4px top -4px right.
- **Open:** Popover anchored to bell icon, opens downward-left (align end). 384px width, max 480px height. Animation: 150ms fade-in + scale-up from 95%.
- **Header:** "Notifications" label (14px semibold) left-aligned + "Mark all read" link (12px, primary text) right-aligned. "Mark all read" only visible when unread count > 0. Marks all as read via single API call. Updates unread badge to 0.
- **List:** ScrollArea containing up to 10 most recent notifications. Each item is a clickable row (full-width). Unread items have `--muted` background. Read items have transparent background. Hover: `--accent` background.
- **Item layout:** Left: 32px icon circle (event type colored, see color table). Center: title (14px semibold, single line truncated) + body (14px regular, `--muted-foreground`, single line truncated) stacked. Right: relative timestamp (12px, `--muted-foreground`). Unread dot: 6px circle, `--primary`, positioned left of icon.
- **Click notification:** Navigates to entity page (uses entityType + entityId to construct URL). Marks as read. Closes popover.
- **Footer:** "View all notifications" link centered, 12px, primary text. Navigates to /notifications page.
- **Empty:** `BellOff` icon (32px, `--muted-foreground`) centered + "No notifications" text below.
- **Polling:** Fetch unread count every 30 seconds via tRPC query. Full list fetched on popover open.
- **Close:** Click outside, press Escape, or navigate away.

### Notification Center Page (/notifications)

- **URL:** /notifications. Added to sidebar navigation under existing items (before Settings).
- **Layout:** Max-width 768px, centered. Page title "Notifications" as Heading.
- **Filter bar:** Horizontal row of Badge-style chips: All / Approvals / Tasks / Contracts / Invoices. Active chip: `--primary` at 10% opacity bg, `--primary` text. Inactive: `--muted` bg, `--muted-foreground` text. Each chip shows count. Filter synced to URL via nuqs `parseAsString` with `?type=approvals` etc.
- **Read/unread toggle:** "Unread only" toggle (Switch) to the right of filter chips. Synced to URL via nuqs `parseAsString` with `?unread=true`.
- **Mark all read:** Button top-right of list. Disabled when no unread notifications.
- **List:** Same notification item component as popover but full-width. Click navigates + marks read.
- **Pagination:** Bottom of list. 10 items per page. Previous/Next buttons. "Page X of Y" label.
- **Loading:** 6 skeleton notification items.
- **Empty (filtered):** "No {type} notifications" with description. No CTA.

### Settings > Notifications Tab

- **Location:** New tab in Settings page Tabs component. Positioned after "Approvals" tab.
- **Notification preferences section:**
  - Matrix table with 6 rows (event types) and 3 channel columns (In-app, Email, Slack).
  - In-app switches are always checked and disabled. Tooltip on hover: "In-app notifications are always enabled."
  - Slack column switches are disabled when Slack is not connected. Tooltip: "Connect Slack in Integrations to enable."
  - Changes tracked in React Hook Form state. "Save preferences" primary CTA below matrix. Disabled until dirty.
  - On save: Sonner toast "Notification preferences saved".
  - Loading: Skeleton matrix (6 rows of switch placeholders).
- **Reminder rules section (below preferences):**
  - Section heading + description + "Create rule" primary CTA top-right.
  - List of ReminderRule cards, vertical stack with 16px gap.
  - Each card: Name (14px semibold), description line ("7 days before contract end, notify entity owner via Email"), channel Badge, recipient Badge, active Switch toggle on right. Edit/Delete ghost buttons in card footer.
  - Active toggle: Immediate optimistic update (no save button). Sonner toast on toggle.
  - Delete: AlertDialog confirmation. See destructive actions section.
  - Empty state: See visual focal points.

### Reminder Rule Editor Dialog

- **Trigger:** "Create rule" CTA or "Edit" on existing rule card.
- **Size:** 560px width, max 70vh height with internal scroll.
- **Form:** React Hook Form + Zod resolver.
  1. Rule name (Input).
  2. Trigger type (Select with 6 options).
  3. Offset (conditional: shown when trigger type supports offset. Number Input with "days" or "hours" suffix label).
  4. Entity type (Select: Contract, Invoice, Task, Document).
  5. Channel (Select: In-app, Email, Slack. Slack disabled when not connected).
  6. Recipient mode (Select: Entity owner, Finance team, Assignee, Specific user, Role).
  7. Conditional: User picker (Command, 300ms debounce) when "Specific user". Role picker (Select) when "Role".
  8. Active toggle (Switch, default true for new rules).
- **Footer:** "Discard changes" ghost + "Save rule" primary. Save disabled until form is valid.
- **Validation:** Inline errors below each field. See form validation table.

### Settings > Integrations Tab

- **Location:** New tab in Settings page Tabs component. Positioned after "Notifications" tab.
- **Admin-only:** Tab visible only to users with admin permission (via `usePermissions().can('organization', ['update'])`).
- **Slack card (disconnected):**
  - Card with Slack logo (32px SVG), "Slack" heading, description text.
  - "Connect Slack" primary CTA. Clicking initiates Slack OAuth redirect flow.
  - OAuth redirect returns to /settings?tab=integrations with success/error query param.
  - On success: Sonner toast "Slack workspace connected". Card updates to connected state.
  - On error: Sonner toast with error message.
- **Slack card (connected):**
  - Card header: Slack logo + "Slack" heading + "Connected" success badge.
  - Body: Workspace name (from IntegrationConnection.displayName), connected by user name, connected date.
  - "Disconnect Slack" destructive outline button. Triggers AlertDialog confirmation.
  - "Reconnect Slack" warning outline button (shown only when status = REAUTH_REQUIRED).
- **User mapping section (shown when connected):**
  - Table below Slack card. Columns: User (avatar + name), Email, Slack User (Slack display name or "---"), Status (badge), Action.
  - Auto-matched rows: Green "Auto-matched" badge. "Unlink user" ghost action.
  - Manually linked rows: Blue "Manually linked" badge. "Unlink user" ghost action.
  - Unmatched rows: Amber "Unmatched" badge. "Link user" primary ghost action. Clicking "Link user" opens inline Command popover to search Slack workspace users.
  - Summary line above table: "{matched} of {total} users matched" in `--muted-foreground`.
  - Loading: 5 skeleton table rows.

### Loading States

- **Notification popover:** 4 skeleton notification items (icon circle + text bars).
- **Notification center page:** 6 skeleton notification items (full-width).
- **Notification preferences matrix:** 6 skeleton rows with switch placeholders per column.
- **Reminder rules list:** 3 skeleton cards with header + body bars.
- **Slack integration card:** Skeleton card with logo placeholder + text bars + button placeholder.
- **User mapping table:** 5 skeleton table rows.

### Responsive Behavior

- **>= 1280px:** Full layout. Notification popover 384px. Notification center 768px max-width centered. Settings full-width.
- **1024-1279px:** Same layout, slight padding reduction. Preference matrix columns may tighten.
- **< 1024px:** Notification popover becomes full-width sheet from bottom (not popover). Notification center full-width. Preference matrix: horizontal scroll if needed. Slack user mapping table: horizontal scroll. Reminder rule cards stack naturally.
