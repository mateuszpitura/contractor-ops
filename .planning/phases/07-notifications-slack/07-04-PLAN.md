**NotificationPreferences (apps/web/src/components/settings/notification-preferences.tsx):**

Per D-05 (per-type per-channel matrix) and UI-SPEC:

"use client" component.

Load preferences via `api.notification.getPreferences.useQuery()`.
Check Slack connection status via `api.integration.getSlackStatus.useQuery()`.

Layout per UI-SPEC visual focal points:
- Section heading "Notification preferences" (heading size) + description text.
- Matrix table using shadcn Table:
  - Header row: "Event" column (wide) + "In-app" + "Email" + "Slack" columns (80px each).
  - 6 rows, one per event type. Each row has:
    - Event type icon (same icon/color mapping as notification-item) + label from UI-SPEC copywriting.
    - In-app Switch: always checked, disabled=true. Tooltip on hover: "In-app notifications are always enabled" per UI-SPEC.
    - Email Switch: bound to channelEmail state.

- Track dirty state via React Hook Form with Zod resolver (local schema mirroring notificationPreferenceUpdateSchema per project convention 02-02).
- "Save preferences" primary Button below matrix. Disabled until form is dirty.
- On save: call updatePreferences mutation. Sonner toast "Notification preferences saved".
- Loading: 6 skeleton rows per UI-SPEC.
- Use `useTranslations('Settings')` for all labels.
