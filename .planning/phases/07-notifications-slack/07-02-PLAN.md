**Slack client service (packages/api/src/services/slack-client.ts):**

Per D-09 and research Pattern 2:

1. Token encryption/decryption using Node.js `crypto` module with AES-256-GCM:
   - `encryptToken(token: string): string` -- encrypts with SLACK_TOKEN_ENCRYPTION_KEY env var, returns iv:authTag:ciphertext as hex string
   - `decryptToken(encrypted: string): string` -- reverse
   - Per research pitfall 2: NEVER store raw xoxb- tokens

2. `getSlackClient(organizationId: string): Promise<WebClient | null>`:
   - Find IntegrationConnection where provider=SLACK, status=CONNECTED, organizationId
   - If not found or not connected, return null
   - Decrypt credentialsRef to get bot token
   - Return new WebClient(token)

3. `getSlackUserIdForUser(organizationId: string, userId: string): Promise<string | null>`:
   - Query ExternalLink where organizationId, entityType=USER, entityId=userId, externalType=SLACK_USER
   - Return externalId (Slack user ID) or null

4. `sendApprovalCard(params: { organizationId, slackUserId, invoiceNumber, contractorName, amount, currency, slaDeadline, invoiceId, flowId })`:
   - Build Block Kit blocks per research code example: header, section with fields, actions with approve/reject buttons
   - Button values encode JSON with invoiceId, flowId
   - Call client.chat.postMessage({ channel: slackUserId, text: fallback, blocks })

5. `updateMessageToResult(params: { organizationId, channel, ts, result: 'approved'|'rejected', actorName, comment? })`:
   - Build updated blocks showing result (green checkmark for approved, red X for rejected)
   - Call client.chat.update({ channel, ts, blocks })

6. `sendReminderDM(params: { organizationId, slackUserId, text, blocks })`:
   - Generic DM sender for reminder notifications

7. `syncWorkspaceUsers(organizationId: string, integrationConnectionId: string): Promise<{ matched: number, total: number }>`:
   - Call client.users.list() to get all workspace members (per D-10)
   - For each member with email, find matching User in org by email
   - Create ExternalLink for matches (entityType=USER, externalType=SLACK_USER)
   - Per research pitfall 3: must have users:read.email scope to get email field
   - Return match stats

**Slack OAuth callback (apps/web/src/app/api/slack/oauth/route.ts):**

Per research Pattern 2 and pitfall 6 (CSRF protection):

Export `GET` handler:
1. Extract `code` and `state` query params from request URL.
2. Verify state parameter: HMAC-SHA256 verify using SLACK_SIGNING_SECRET. State contains JSON with orgId, userId, timestamp. Reject if signature invalid or timestamp > 10 minutes old.
3. Exchange code for token: POST to `https://slack.com/api/oauth.v2.access` with code, client_id (SLACK_CLIENT_ID env), client_secret (SLACK_CLIENT_SECRET env), redirect_uri.
4. Extract bot access token (`data.access_token`), team name (`data.team.name`), team id (`data.team.id`).
5. Encrypt bot token using `encryptToken()` from slack-client service.
6. Upsert IntegrationConnection: provider=SLACK, status=CONNECTED, displayName=team name, credentialsRef=encrypted token, connectedByUserId=userId from state, configJson={teamId}.
7. Call `syncWorkspaceUsers()` to auto-match users by email per D-10.
8. Redirect to `/settings?tab=integrations&slack=connected`.
9. On any error: redirect to `/settings?tab=integrations&slack=error`.

**Slack interactivity webhook (apps/web/src/app/api/slack/interactivity/route.ts):**

Per research Pattern 3, pitfall 1 (3-second timeout), and state of the art (use chat.update not response_url):

Export `POST` handler:
1. Read form data, parse `payload` field as JSON.
2. Verify Slack request signature using HMAC-SHA256 with SLACK_SIGNING_SECRET and X-Slack-Signature + X-Slack-Request-Timestamp headers. Return 401 if invalid.
3. Respond immediately with 200 OK (empty body) -- critical for 3-second window per research pitfall 1.
4. Process async (fire-and-forget pattern, or use Vercel waitUntil if available):

For `block_actions` (payload.type === "block_actions"):
- Extract action: payload.actions[0]
- If action_id === "approve_invoice":
  - Parse value JSON to get invoiceId, flowId
  - Resolve actorUserId from Slack user ID (payload.user.id) via ExternalLink lookup
  - Call approval engine advanceFlow with decision=APPROVED
  - Update original Slack message via chat.update to show "Approved by {name}" per D-09
- If action_id === "reject_invoice":
  - Open reject modal via views.open with trigger_id per research code example
  - private_metadata encodes invoiceId, flowId, channel, message_ts

For `view_submission` (payload.type === "view_submission"):
- Extract comment from payload.view.state.values.comment_block.comment_input.value
- Extract invoiceId, flowId, channel, messageTs from private_metadata JSON
- Resolve actorUserId from Slack user ID
- Call approval engine advanceFlow with decision=REJECTED, comment
- Update original message via chat.update to show "Rejected by {name}: {comment}"
- Return 200 with response_action: clear (close modal)

**Cron reminder route (apps/web/src/app/api/cron/reminders/route.ts):**

Per research Pattern 5 and pitfall 7 (double-send protection):

Export `GET` handler:
1. Verify CRON_SECRET from Authorization header (Bearer token). Return 401 if invalid.
2. Find all active ReminderRules across all organizations.
3. For each rule:
   - Query entities matching the rule's entityType and triggerType:
     - BEFORE_CONTRACT_END: contracts where endDate - offsetDays <= now AND endDate > now
     - BEFORE_DUE_DATE: invoices/tasks where dueDate - offsetDays <= now AND dueDate > now
     - AFTER_DUE_DATE: invoices/tasks where dueDate + offsetDays <= now
     - BEFORE_DOCUMENT_EXPIRY: documents with expiryDate if tracked
   - For each matching entity:
     - Check if ReminderInstance already exists for (ruleId + entityId + entityType + scheduledFor date). Skip if exists per pitfall 7.
     - Create ReminderInstance with status=PENDING.
     - Resolve recipients based on recipientMode (ENTITY_OWNER -> look up owner field, FINANCE_TEAM -> find users with finance role, ASSIGNEE -> task assignee, SPECIFIC_USER -> from configJson, ROLE -> users with that role).
     - Call notificationService.dispatch() with appropriate type and recipients.
     - Update ReminderInstance status=SENT, sentAt=now().
4. Return JSON { processed: number, sent: number }.
