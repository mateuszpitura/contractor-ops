Entity URL helper (inline or separate util):
```typescript
function getEntityUrl(entityType: string, entityId: string): string {
  const routes: Record<string, string> = {
    INVOICE: `/invoices/${entityId}`,
    CONTRACT: `/contracts/${entityId}`,
    CONTRACTOR: `/contractors/${entityId}`,
    WORKFLOW_RUN: `/workflows/${entityId}`,
    APPROVAL_FLOW: `/approvals`,
  };
  return routes[entityType] ?? "/notifications";
}
```

**NotificationCenter component (apps/web/src/components/notifications/notification-center.tsx):**

Per D-04 (full page with filters) and UI-SPEC interaction patterns:

"use client" component.

URL state via nuqs (following project pattern per decision 02-02):
- `type` filter: parseAsString, maps to notification type (approvals, tasks, contracts, invoices). Maps filter chip values to notification types: approvals -> APPROVAL_REQUEST,APPROVAL_DECISION; tasks -> TASK_ASSIGNED,TASK_OVERDUE; contracts -> CONTRACT_EXPIRING; invoices -> INVOICE_RECEIVED.
- `unread` filter: parseAsString, "true" for unread only.
- `page`: parseAsInteger, default 1.

Wrap in Suspense for nuqs SSG prerendering (per decision 02-02).

Layout per UI-SPEC:
- Max-width 768px centered (mx-auto max-w-3xl).
- Page heading: "Notifications" (20px font-semibold).
- Filter bar: Horizontal row of Badge-style chips: All, Approvals, Tasks, Contracts, Invoices. Each shows count from the query data. Active chip: bg-primary/10 text-primary. Inactive: bg-muted text-muted-foreground. Clicking a chip updates URL ?type param.
- "Unread only" Switch toggle right of filter chips. Synced to ?unread URL param.
- "Mark all read" Button top-right of list. Disabled when no unread items.
- List: NotificationItem components (same component from Task 1, not compact mode). Full width. On click: markRead + navigate.
- Pagination: Bottom of list. Previous/Next buttons + "Page X of Y" label.
- Loading: 6 skeleton notification items.
- Empty (with filters): "No {type} notifications" heading + description. No CTA.
- Empty (no filters): BellOff icon (48px text-muted-foreground) + "No notifications yet" heading + body text per UI-SPEC.

Query: `api.notification.list.useQuery({ type: mappedTypes, unreadOnly: unreadParam === 'true', page, perPage: 10 })`.
