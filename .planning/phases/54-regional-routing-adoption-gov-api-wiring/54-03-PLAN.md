6. **Refactor `post()` method** to use inherited `this.fetch()` from GovApiClient, gaining retry and timeout:
```typescript
private async post<T>(
  path: string,
  body: Record<string, unknown> | ZatcaSubmissionPayload,
  organizationId?: string,
): Promise<T> {
  const response = await this.fetch(
    path,
    {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        Accept: "application/json",
        Authorization: this.authHeader,
        "Accept-Language": "en",
        "Accept-Version": "V2",
      },
    },
    { organizationId, skipAudit: !organizationId },
  );
  // Note: GovApiClient.fetch() sets default headers but we override them above.
  // The auth header is ZATCA-specific Basic auth, NOT the GovApiClient Bearer cert.

  if (!response.ok) {
    throw await this.createError(response, path);
  }
  return response.json() as Promise<T>;
}
```

Wait — re-reading `GovApiClient.fetch()`: it sets Authorization as Bearer only if `this.certificate` is loaded. Since we never call `loadCertificate()`, the Bearer auth won't interfere. But we need to pass body in the `fetch()` call too. Let me check the parent fetch signature again.

Actually, `GovApiClient.fetch()` accepts `RequestInit` as second arg, which includes `body`. So pass the body there:

```typescript
private async post<T>(
  path: string,
  body: Record<string, unknown> | ZatcaSubmissionPayload,
  organizationId?: string,
): Promise<T> {
  const response = await this.fetch(
    path,
    {
      method: "POST",
      headers: {
        Accept: "application/json",
        Authorization: this.authHeader,
        "Accept-Language": "en",
        "Accept-Version": "V2",
      },
      body: JSON.stringify(body),
    },
    { organizationId, skipAudit: !organizationId },
  );

  if (!response.ok) {
    throw await this.createError(response, path);
  }
  return response.json() as Promise<T>;
}
```

Note: GovApiClient.fetch() sets Content-Type to application/json by default if not present, so we don't need to set it explicitly.
