2. **Replace** the r2.ts import:
   - **Remove:** `import { createPresignedDownloadUrl, createPresignedUploadUrl, deleteObject, generateStorageKey, headObject } from "../services/r2.js"`
   - **Add:** `import { createRegionalPresignedUploadUrl, createRegionalPresignedDownloadUrl, headRegionalObject, deleteRegionalObject } from "../services/regional-storage.js"`

4. **Replace** storage function calls:
   - `createPresignedUploadUrl(key, contentType)` → `createRegionalPresignedUploadUrl(key, contentType)`
   - `createPresignedDownloadUrl(key)` → `createRegionalPresignedDownloadUrl(key)`
   - `headObject(key)` → `headRegionalObject(key)`
   - `deleteObject(key)` → `deleteRegionalObject(key)`
5. **In `scanAndUpdate` helper function:** This function uses `prisma` directly outside a handler. Refactor it to accept a db client parameter:
   - Change signature to `async function scanAndUpdate(db: PrismaClient, documentId: string, storageKey: string): Promise<void>`
   - Replace internal `prisma.document.update(...)` with `db.document.update(...)`
   - At each call site in handlers, pass `ctx.db` as the first argument
   - Also update the `R2_BUCKET_NAME` reference in scanAndUpdate to use the regional bucket. Since scanAndUpdate reads from R2 for virus scanning, it should use `getRegionalBucket()` from regional-storage.ts or the legacy bucket (both share the same R2 client). For now, keep the bucket reference as-is since scanAndUpdate reads the file — it accesses the correct bucket because files are already uploaded to the regional bucket by the upload URL.
