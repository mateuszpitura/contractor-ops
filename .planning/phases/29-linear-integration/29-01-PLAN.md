**2. Create LinearAdapter** at `packages/integrations/src/adapters/linear-adapter.ts`:
Mirror `JiraAdapter` structure exactly. Key differences from Jira:

- OAuth config:
  ```
  clientIdEnvVar: "LINEAR_CLIENT_ID"
  clientSecretEnvVar: "LINEAR_CLIENT_SECRET"
  authorizationUrl: "https://linear.app/oauth/authorize"
  tokenUrl: "https://api.linear.app/oauth/token"
  scopes: ["read", "write"]
  redirectPath: "/api/oauth/linear/callback"
  ```
- No `extraAuthParams` needed (Linear does not require audience or prompt params)
- `exchangeCodeForTokens`: POST to `https://api.linear.app/oauth/token` with `Content-Type: application/x-www-form-urlencoded` using `URLSearchParams` body (NOT JSON like Jira). Fields: `grant_type=authorization_code`, `client_id`, `client_secret`, `code`, `redirect_uri`. Response: `{ access_token, refresh_token, expires_in, token_type, scope }` where `scope` is an array of strings (join with comma).
- `refreshToken`: POST to same URL with `grant_type=refresh_token`, `client_id`, `client_secret`, `refresh_token`. Preserve old refresh token if new one not returned.
- `verifyWebhookSignature`: Read `linear-signature` header (lowercase). Compute HMAC-SHA256 of rawBody using webhook secret from `x-webhook-secret` header or `LINEAR_WEBHOOK_SECRET` env. Compare hex digests using `timingSafeEqual`. Extract eventType as `${parsed.type}.${parsed.action}` from payload.
- `handleWebhook`: Empty method (processing delegated to _process route, same as Jira).
- `getHealthStatus`: Copy from JiraAdapter but change provider string to `"linear"`.
- `getRequiredScopes`: return `["read", "write"]`.
- Export `LinearAdapter` class.
- Add `discoverWorkspace` method: after token exchange, query Linear GraphQL at `https://api.linear.app/graphql` with `Authorization: Bearer {accessToken}`, query: `{ teams { nodes { id name key states { nodes { id name type color position } } } } organization { id name urlKey } }`. Return `{ organizationId, organizationName, urlKey, teams }`.

**1. Create Linear tRPC router** at `packages/api/src/routers/linear.ts`:
Mirror `packages/api/src/routers/jira.ts` structure. This is the SHELL -- sync service calls will be wired in Plan 02.

Define `linearRouter` using `router({...})` with these procedures:

- `teams` (query, `tenantProcedure`): Fetch Linear teams for the connected workspace. Read IntegrationConnection where provider=LINEAR and status=CONNECTED for the org. Decrypt credentials via `decryptCredentials`. Call Linear GraphQL `{ teams { nodes { id name key states { nodes { id name type color position } } } } }` using fetch to `https://api.linear.app/graphql` with Bearer token. Return the teams array. If no connection found, throw TRPCError NOT_FOUND.

- `getStatusMapping` (query, `tenantProcedure`, input: `z.object({ teamId: z.string() })`): Read IntegrationConnection for LINEAR, extract `configJson.statusMappings[teamId]` and return it. Return empty array if no mapping exists for the team.

- `saveStatusMapping` (mutation, `tenantProcedure` + `requirePermission("integration:manage")`): Input uses `saveLinearStatusMappingInputSchema`. Update the IntegrationConnection's `configJson.statusMappings[teamId]` with the provided mappings. Also update `configJson.stateCache[teamId]` with the state ID to name/type map for webhook lookup. Return success.

- `saveTaskConfig` (mutation, `tenantProcedure` + `requirePermission("workflow:manage")`): Input uses `saveLinearTaskConfigInputSchema`. Update WorkflowTaskTemplate.configJson with the Linear config. Return success.

- `getLinkedIssue` (query, `tenantProcedure`, input: `z.object({ taskRunId: z.string() })`): Find ExternalLink where entityType=TASK_RUN, entityId=taskRunId, externalType="LINEAR_ISSUE". Return the link with parsed metadataJson (as `LinearIssueMetadata`) or null.

- `getLinkedIssues` (query, `tenantProcedure`, input: `z.object({ taskRunIds: z.array(z.string()) })`): Batch version of getLinkedIssue. Return map of taskRunId to LinearIssueMetadata or null.

Use same helper pattern as jira.ts (`plain<T>()` helper, `buildLinearApiContext()` helper that extracts accessToken from decrypted credentials).
