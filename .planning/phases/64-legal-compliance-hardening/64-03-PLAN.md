**A. Add `ConsentEvent` model to `consent.prisma` (D-30):**

The existing `ConsentRecord` model tracks PDPL per-purpose consent. The new `ConsentEvent` model is a separate, simpler model specifically for Terms of Service re-acceptance (distinct from PDPL consent records).

Append to `consent.prisma` after the existing `ConsentPurpose` enum:

```prisma
// ---------------------------------------------------------------------------
// Phase 64 Â· LEGAL-07 â€” Terms of Service acceptance events (D-30)
//
// Distinct from ConsentRecord (PDPL per-purpose consent â€” Phase 51).
// ConsentEvent records a user's explicit acceptance of the platform ToS.
// Used to enforce re-acceptance when the ToS version is bumped.
//
// Multi-tenant scoped: each org tracks its users' acceptance state.
// Append-only: no update or delete procedures.
// ---------------------------------------------------------------------------

enum ConsentEventScope {
  TOS      // Terms of Service acceptance
}

model ConsentEvent {
  id             String           @id @default(cuid())
  organizationId String
  userId         String
  scope          ConsentEventScope
  version        String           // e.g. "2026.1.0" â€” must match TOS_CURRENT_VERSION
  acceptedAt     DateTime
  ipAddress      String?
  userAgent      String?          @db.Text
  createdAt      DateTime         @default(now())

  organization Organization @relation("OrganizationConsentEvents", fields: [organizationId], references: [id])
  user         User         @relation("UserConsentEvents", fields: [userId], references: [id])

  // Append-only â€” no updatedAt.

  @@index([organizationId, userId, scope])
  @@index([organizationId, scope, createdAt])
}
```
