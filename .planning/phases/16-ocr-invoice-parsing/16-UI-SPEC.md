# Phase 16 — UI Design Contract

> Visual and interaction contract for OCR invoice parsing: auto-extract invoice fields from PDFs with confidence scores, color-coded field borders, and side-by-side PDF + form review. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Confidence | Border Color | Icon | Tooltip |
|------------|-------------|------|---------|

Accent reserved for: "Accept & Save" primary CTA on review form, "Re-run OCR" secondary CTA, extraction complete checkmark, focus ring on form inputs during edit-in-place, active PDF page indicator in viewer.

## Layout Contract

### Focal Point

The primary screen focal point is the right-side form panel in the split view. On page load after extraction completes, visual attention is directed to the first low- or medium-confidence field (scrolled into view if needed), drawing the user to the data most likely to need manual correction. If all fields are high-confidence, the focal point falls to the "Accept & Save" CTA in the sticky action footer.

### Split Panel Review (D-06)

- Container: full width of content area, CSS grid `grid-cols-2` with 16px gap on desktop
- Left panel (PDF): Card with muted background. PdfViewer fills card body. Min-height 400px. Sticky position (top: 80px) on desktop so PDF stays visible while scrolling form.
- Right panel (Form): Card with white background. Pre-filled form fields organized in sections.
- Mobile (<768px): stacks vertically -- PDF on top (max-height 300px with scroll), form below
- ExtractionStatusBar spans full width above both panels

### Form Field Layout (Right Panel)

- Section 1 -- Invoice Header: 2-column grid (16px gap). Fields: invoice number, issue date, due date, currency. Each field wrapped in ConfidenceFieldWrapper.
- Section 2 -- Parties: 2-column grid. Fields: seller NIP (+ NipValidationBadge), buyer NIP (+ NipValidationBadge), seller name, buyer name. Separator below.
- Section 3 -- Amounts: 3-column grid. Fields: total net, tax amount, total gross. Each with ConfidenceFieldWrapper. Separator below.
- Section 4 -- Bank Account: Full-width field. Bank account number with ConfidenceFieldWrapper. Separator below.
- Section 5 -- Line Items: Full-width. "Line Items" subheading + item count badge. LineItemsTable below. "Add line item" ghost button at bottom.
- Section 6 -- Actions: Sticky footer (bottom: 0) on desktop. "Accept & Save" primary button (right-aligned) + "Discard Extraction" outline button + "Re-run OCR" ghost button (left-aligned).

### Admin Invoice Upload Integration

- After PDF upload in admin invoice creation: if file is PDF, trigger OCR automatically (D-03)
- Show OcrProcessingOverlay on form while extraction runs
- On completion: populate form fields, show ExtractionStatusBar with results
- User can switch to split panel view via "View PDF" toggle button in form header

### Portal Invoice Submit Integration (D-05)

- After PDF upload in portal submit form: trigger OCR automatically
- Show OcrProcessingOverlay on submit form while extraction runs
- On completion: pre-fill existing form fields with extracted data
- Confidence indicators shown inline next to pre-filled fields
- Portal form does NOT get full split panel (simpler layout) -- PDF thumbnail + "View PDF" link instead

### Extraction Status States

| State | Visual | User Action |
|-------|--------|-------------|
| Idle (no PDF) | Standard empty form | Upload PDF |
| Processing | OcrProcessingOverlay with progress bar + skeleton fields | Wait (can cancel) |
| Complete | All fields pre-filled, green ExtractionStatusBar, confidence badges visible | Review and accept |
| Partial | Some fields pre-filled, warning ExtractionStatusBar showing "{N} of {M} fields extracted" | Fill missing fields manually |
| Failed | Red ExtractionStatusBar with error message, empty form | Re-run OCR or fill manually |

## Loading States

| Element | Skeleton Pattern |
|---------|-----------------|
| OcrProcessingOverlay | Full form area: centered Loader2 spinner (24px, animate-spin) + progress bar below + "Analyzing invoice..." text. Background: card with 60% opacity overlay. Below overlay: skeleton field rows (8 rows of label + input skeleton pairs). |
| Split panel initial load | Left: PDF skeleton (full-height muted rectangle). Right: 8 field skeleton rows (label line + input rectangle each) + table skeleton (header row + 3 body rows). |
| Line items loading | Table header skeleton + 3 row skeletons (6 cells each, alternating widths). |
| Portal form pre-fill | Each field shows a brief shimmer animation (200ms) as the extracted value populates, top-to-bottom cascade with 50ms stagger per field. |

All skeletons use the existing `Skeleton` component with pulse animation.
