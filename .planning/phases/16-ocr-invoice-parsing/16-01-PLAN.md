**2. Create `packages/integrations/src/types/ocr.ts`** (per D-01, D-08):

Define the OcrAdapter interface and supporting types exactly as in 16-RESEARCH.md Pattern 1:
- `OcrExtractionRequest`: pdfBase64, fileName, pageCount?, locale? (default "pl")
- `OcrExtractionField`: key, value (string | number | null), confidence (0-100), boundingBox? (page, x, y, w, h)
- `OcrLineItem`: description, quantity, unit, unitPriceGrosze, netAmountGrosze, vatRate, vatAmountGrosze, grossAmountGrosze, confidence
- `OcrExtractionResult`: status ("EXTRACTED" | "PARTIAL" | "FAILED"), fields (Record<string, OcrExtractionField>), lineItems (OcrLineItem[]), rawResponse?, processingTimeMs, pageCount, overallConfidence
- `OcrAdapter` interface: `extractInvoice(request: OcrExtractionRequest): Promise<OcrExtractionResult>`, readonly `providerName: string`, readonly `supportedDocumentTypes: string[]`

Also export `validateNip(nip: string): { valid: boolean; formatted: string }` utility using the modulo-11 checksum (weights 6,5,7,2,3,4,5,6,7 -- remainder 10 = invalid) per D-09. Strip dashes/spaces, check 10 digits.

Also export `adjustConfidences(result: OcrExtractionResult): OcrExtractionResult` that:
- Cross-validates net + tax = gross (cap all amount confidences to 60 if mismatch exceeds 0.01)
- Validates NIP checksums (cap NIP confidence to 40 if invalid)
- Returns adjusted result

**3. Create `packages/integrations/src/adapters/claude-ocr-adapter.ts`** (per D-01, D-02):

Implement OcrAdapter using @anthropic-ai/sdk. Use Claude native PDF document support (type: "document", media_type: "application/pdf", base64 data) per 16-RESEARCH.md Pattern 2. Do NOT convert PDF to images -- Claude handles this internally.

Use tool_use with JSON schema for structured extraction per 16-RESEARCH.md Pattern 2. Tool name: "extract_invoice_data". Use the INVOICE_EXTRACTION_TOOL_SCHEMA and POLISH_INVOICE_EXTRACTION_PROMPT from 16-RESEARCH.md code examples.

Model: `claude-sonnet-4-5-20250514` (configurable via constructor param `modelId?`). Max tokens: 4096. Parse the tool_use response, normalize amounts from decimal to grosze (multiply by 100, round to int), call adjustConfidences, compute overallConfidence as weighted average of all field confidences.

Constructor accepts optional `apiKey?: string` (falls back to ANTHROPIC_API_KEY env var).

Set `providerName = "claude"` and `supportedDocumentTypes = ["application/pdf"]`.

Handle errors: if Claude returns no tool_use block, return status FAILED. If extraction is partial (< 50% of required fields), return PARTIAL.

Check PDF size before sending -- if > 30MB, return FAILED with errorMessage about size limit.

**1. Create `packages/api/src/services/ocr-extraction.ts`** (per D-03 background queue):

Export `triggerOcrExtraction(params: { organizationId, documentId, storageKey, invoiceId? }): Promise<{ extractionId: string }>`:
- Create OcrExtraction record in DB with status PENDING
- Dispatch QStash job to `${process.env.NEXT_PUBLIC_APP_URL}/api/ocr/_process` with body `{ extractionId, organizationId, storageKey }`, retries: 2, timeout: "60s"
- Return the extractionId

Export `processOcrExtraction(params: { extractionId, organizationId, storageKey }): Promise<void>`:
- Update OcrExtraction status to PROCESSING
- Fetch PDF from R2 using presigned download URL (getPresignedDownloadUrl from r2 service), then fetch the URL to get binary, convert to base64
- Call `extractInvoice({ provider: "CLAUDE", pdfBase64, fileName: storageKey, locale: "pl" })` from ocr-service
- On success: update OcrExtraction with status from result, resultJson, overallConfidence, pageCount, processingTimeMs, completedAt
- On failure: update OcrExtraction with status FAILED, errorMessage, increment retryCount

Export `getExtractionResult(params: { organizationId, extractionId }): Promise<OcrExtraction | null>`:
- Fetch OcrExtraction by id scoped to organizationId

Export `getExtractionByDocument(params: { organizationId, documentId }): Promise<OcrExtraction | null>`:
- Fetch latest OcrExtraction for a document, ordered by createdAt desc

**2. Create `packages/api/src/routers/ocr.ts`**:

Use tenantProcedure for all endpoints (admin). Add:
- `trigger` mutation: input { documentId: z.string(), storageKey: z.string(), invoiceId: z.string().optional() }. Calls triggerOcrExtraction. Returns { extractionId }.
- `getResult` query: input { extractionId: z.string() }. Calls getExtractionResult. Returns extraction record with parsed resultJson.
- `getByDocument` query: input { documentId: z.string() }. Calls getExtractionByDocument. Returns extraction or null.
- `retrigger` mutation: input { extractionId: z.string() }. Finds existing extraction, creates new OcrExtraction record with same documentId, dispatches new QStash job. Returns new extractionId.

Also add portal-accessible endpoints using portalProcedure (per D-05):
- `portalTrigger` mutation: same as trigger but uses portal session's organizationId
- `portalGetResult` query: same as getResult but scoped to portal session's organizationId
- `portalGetByDocument` query: same as getByDocument but scoped to portal session
