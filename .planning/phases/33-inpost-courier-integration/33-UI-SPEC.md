## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, table cell padding |
| md | 16px | Default element spacing, card padding, Geowidget modal body padding |
| lg | 24px | Section padding, form group gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions: Geowidget iframe container uses 0px padding (map fills entire modal body). Touch targets for Paczkomat selection confirmation button are minimum 44px height. InPost brand logo in modal header renders at 24px height.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | text-sm (clamp 13-14px) | 400 (regular) | 1.25rem |
| Label | text-xs (clamp 11-12px) | 400 (regular) | 1rem |
| Heading | text-lg (clamp 16-18px) | 600 (semibold) | 1.75rem |

Notes: Consistent with Phase 30 equipment UI typography. Paczkomat name and address in selection confirmation use Body size. Tracking numbers and shipment IDs use font-mono (JetBrains Mono) at text-xs for scanability, consistent with Phase 30 serial number pattern.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` oklch(0.985 0.003 85) | Page background, portal background |
| Secondary (30%) | `--card` oklch(1 0 0) | Shipment cards, detail sections, modal surfaces, portal equipment cards |
| Accent (10%) | `--primary` oklch(0.44 0.145 178) | Primary CTA buttons, selected Paczkomat highlight, active return flow step |
| Destructive | `--destructive` oklch(0.58 0.23 18) | Cancel return action |

Accent reserved for: "Create InPost shipment" button, "Select Paczkomat" button, "Request return" button (portal), "Confirm return" button, selected Paczkomat card border/highlight, active step indicator in return flow, download/print label button.

### InPost-Specific Status Badge Colors

Inherits Phase 30 Shipment Status Badge Colors exactly. No new statuses introduced -- ShipX API statuses map to existing ShipmentStatus enum.

### InPost Brand Color

InPost brand yellow (#FFCD00) is NOT used in the app UI. The Geowidget iframe handles its own InPost branding internally. The app stays on the project design system.

## Interaction Contracts

### InPost Shipment Creation

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Open Paczkomat picker | "Select Paczkomat" button in shipment form | Dialog opens with Geowidget iframe | Map loads with nearby lockers |
| Select Paczkomat | Click locker marker in Geowidget | Geowidget fires postMessage event; selected locker info appears below map | "Confirm locker" button becomes enabled |
| Confirm Paczkomat | "Confirm locker" button | Picker modal closes, PaczkomatDisplay card appears in form | Form updates with locker ID and address |
| Create shipment | "Create InPost shipment" button | Loading spinner on button, form disabled | Sonner toast "InPost shipment created", dialog closes, timeline updates with CREATED + LABEL_GENERATED events |
| View label | Click shipment in timeline or "View label" link | ShipmentLabelView expands/navigates in detail | QR code / label renders with download/print buttons |
| Download label | "Download label" button | Browser file save dialog | File downloads (PDF or PNG) |
| Print label | "Print" button | Browser print dialog opens with label content | Print dialog |

### Contractor Return Flow (Portal)

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Start return | "Return equipment" button on portal equipment card | Multi-step modal opens at step 1 | Items listed, Geowidget loads |
| Select drop-off | Pick locker in Geowidget + confirm | Same as admin Paczkomat picker flow | "Next" button becomes enabled |
| Request return | "Request return" at step 2 | Loading spinner, request submitted | Modal shows "Return pending approval" state, sonner toast "Return requested" |
| Cancel return | "Cancel return" in portal equipment card | AlertDialog confirmation | Sonner toast "Return cancelled", card reverts to normal state |
| View label (after approval) | Revisit portal equipment tab or click notification link | Step 3 of return flow loads with label | QR/label visible with download/print |

### Admin Return Approval

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Approve return | "Approve return" on banner | Loading spinner, InPost shipment created server-side | Sonner toast "Return approved -- shipment created", banner removed, timeline shows new return shipment |
| Reject return | "Reject return" on banner | AlertDialog confirmation | Sonner toast "Return rejected", banner removed, equipment status reverts |

### Geowidget Communication

| Event | Direction | Payload | Handler |
|-------|-----------|---------|---------|
| Locker selected | Geowidget -> App | postMessage with locker ID, name, address, lat/lng | `window.addEventListener('message', ...)` with origin check |
| Widget ready | Geowidget -> App | postMessage ready signal | Hide skeleton loader, show map |
| Widget error | Geowidget -> App | postMessage error | Show error state with retry button |

Origin validation: Only accept postMessages from `https://geowidget.inpost.pl` domain. Reject all others silently.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Paczkomat picker modal traps focus; "Confirm locker" is first focus target after selection; return flow steps navigable with Tab |
| Screen reader | Geowidget iframe has `title="InPost Paczkomat locker map"` for screen readers; selected locker announced via `aria-live="polite"` region |
| Focus visible | All interactive elements show `ring` on focus (project default via `outline-ring/50`) |
| Color contrast | All badge text meets 4.5:1 against badge background; approval banner uses both color and icon (warning triangle) for status indication |
| Reduced motion | No custom animations introduced; Geowidget handles its own motion internally |
| Alternative to map | Selected Paczkomat name and full address displayed as text; screen reader users get locker details without needing map interaction |

### External Script: InPost Geowidget

The InPost Geowidget loads via iframe from `https://geowidget.inpost.pl`. This is a first-party InPost service, not a third-party npm package. Security considerations:
- iframe is sandboxed with `sandbox="allow-scripts allow-same-origin allow-popups"` -- no `allow-top-navigation`
- Communication via postMessage with strict origin checking (`https://geowidget.inpost.pl` only)
- No InPost JavaScript SDK loaded into the app bundle -- iframe isolation only
- CSP frame-src must include `https://geowidget.inpost.pl`
