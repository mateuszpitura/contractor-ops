## Layout Contract

### Focal Point

The primary screen focal points for Phase 17 are:

1. **Settings > Integrations tab**: The KSeF provider card draws attention via the "Connect KSeF" primary CTA when disconnected, or the "Sync Now" button and last sync info when connected.
2. **Invoice detail page**: The KsefMetadataSection appears above the existing metadata form for KSeF-sourced invoices, giving immediate KSeF provenance context.
3. **Invoice list table**: The KsefSourceBadge in the source column provides at-a-glance identification of KSeF-pulled invoices.

### KSeF Provider Card (D-02, D-06, D-07)

- Follows existing ProviderConnectionCard pattern exactly (consistent with Slack/DocuSign/Autenti cards)
- Card header: KSeF icon (ShieldCheck, 32px) + "KSeF" title + status badge (Connected/Disconnected/Error)
- Card body when disconnected: description text + "Connect KSeF" primary button
- Card body when connected: last sync time (relative) + invoice count from last sync + "Sync Now" outline button + "Manage" text button
- Expandable sync history section (Collapsible): chevron toggle, last 10 syncs in compact rows
- "Sync Now" button shows Loader2 spinner while syncing, disabled during active sync

### KSeF Setup Dialog (D-01, D-03, D-04)

- Trigger: "Connect KSeF" button on provider card
- Dialog width: `sm:max-w-lg` (480px)
- Header: "Connect to KSeF" title + "Configure your KSeF credentials to auto-fetch invoices" description
- NIP display: read-only field showing org NIP from settings, with info tooltip "NIP is pulled from your organization settings"
- Auth method tabs:
  - **Token tab**: Label "API Token" + textarea (rows=4, monospace font) + helper text "Generate this token in the KSeF portal at ksef.mf.gov.pl"
  - **Certificate tab**: Label "Certificate File (.p12 or .pem)" + file upload drop zone (dashed border, 80px height) + optional password field for encrypted .p12
- "Verify Credentials" outline button: triggers test API call. Shows inline result: green checkmark + "Credentials verified" or red X + error message. Button shows Loader2 while verifying.
- Footer: "Cancel" outline button (left) + "Save" primary button (right, disabled until verification passes)

### Invoice Detail -- KSeF Metadata (D-13)

- Position: above the existing invoice metadata form, below the DuplicateWarning if present
- Container: Card with subtle left border in primary color (2px)
- Header row: ShieldCheck icon (16px, primary color) + "KSeF Data" heading (text-sm font-semibold) + "View in KSeF" external link (right-aligned, text-sm, primary color, opens new tab)
- Content: 2-column grid (16px gap) on desktop, single column on mobile
  - Row 1: "KSeF Reference" label + CopyableField with reference number (monospace)
  - Row 2: "UPO Receipt" label + CopyableField with UPO number (monospace)
  - Row 3: "Fetched" label + relative timestamp + exact date tooltip
  - Row 4: "Source" label + KsefSourceBadge

### Invoice Table -- Source Indicator (D-13)

- Existing invoice table source filter already supports KSEF enum value
- KSeF-sourced rows show KsefSourceBadge in the source column cell
- Badge is compact: 14px ShieldCheck icon + "KSeF" text, primary color at 80% opacity
- On hover: tooltip with "Fetched from KSeF on {date}"

### Duplicate Detection UI (D-11, D-12)

- On manual invoice detail page when KSeF duplicate found: KsefDuplicateBanner renders above metadata form
- On KSeF invoice detail page when manual duplicate found: standard DuplicateWarning component renders (already exists from Phase 5)
- KsefDuplicateBanner uses warning color scheme (not destructive) because KSeF data is authoritative
- Both banners link to the other version for easy comparison

### Sync History (D-07)

- Location: expandable Collapsible section within ProviderDetailSheet for KSeF
- Compact list layout (not table) for visual density
- Each row: flex row with timestamp (left, text-muted-foreground), invoice count badge (center), status badge (right)
- Status badges: "Success" (success variant), "Partial" (warning variant), "Failed" (destructive variant), "No new invoices" (secondary variant)
- Empty state: "No sync history yet. Click Sync Now to fetch invoices from KSeF."
- Maximum 10 entries shown, sorted newest first
