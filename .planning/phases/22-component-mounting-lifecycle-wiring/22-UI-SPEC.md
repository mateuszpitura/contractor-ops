## Component Placement Contract

This is the core of the Phase 22 UI contract -- where existing components mount and how they behave.

### DocLinksSection in task-card-run.tsx

| Property | Value |
|----------|-------|
| Mount location | Inside `CollapsibleContent > div.space-y-4`, after TaskAttachments, before TaskComments |
| Visibility | Only when task card is expanded (inside CollapsibleContent) |
| Props | `workflowTaskRunId={task.id}` and `readOnly={["DONE", "SKIPPED", "CANCELLED"].includes(task.status)}` |
| Section ordering | TaskAttachments, DocLinksSection, TaskComments (files first, external docs second, discussion last per D-01) |

**Interaction states:**
- **Default:** Shows list of attached doc links as DocLinkChip components with detach capability
- **Empty:** FileText icon with "No documents linked" text and "Attach Document" button
- **Read-only:** Attach/detach buttons hidden when task status is DONE, SKIPPED, or CANCELLED
- **Loading:** Skeleton placeholders matching existing section loading pattern

### CalendarTaskConfig in template-builder/task-card.tsx

| Property | Value |
|----------|-------|
| Mount location | Directly below JiraTaskConfig (after line ~489), before Actions div |
| Conditional guard | `{task?.id && <CalendarTaskConfig taskTemplateId={task.id} />}` -- identical to JiraTaskConfig pattern |
| Props | `taskTemplateId={task.id}` |

**Interaction states:**
- **Default:** Switch toggle (off) with "Calendar Event" label. When enabled, shows duration and "Configure" button.
- **Configured:** Switch on, summary line showing duration label (e.g., "1 hour"), "Configure" button opens CalendarEventConfigDialog
- **Loading:** Skeleton matching JiraTaskConfig loading state
- **Unsaved task:** Component not rendered (conditional guard prevents mount for templates without persisted ID)

## Lifecycle Wiring Contract (Non-Visual)

These are backend-only hooks with no direct UI manifestation, but they produce side effects visible in the user's external calendar.

| Trigger | Service Call | User-Visible Effect |
|---------|-------------|-------------------|
| Contract created with endDate | `void syncContractExpiryDeadline(...)` | Calendar event appears on user's Google/Outlook calendar |
| Contract updated with endDate | `void syncContractExpiryDeadline(...)` | Calendar event updated (upsert) |
| Contract endDate cleared | `void deleteCalendarEvent(...)` | Calendar event removed |
| Contract deleted | `void deleteCalendarEvent(...)` | Calendar event removed |
| Invoice approved with dueDate | `void syncPaymentDueDeadline(...)` | Payment deadline event appears on calendar |
| Invoice voided | `void deleteCalendarEvent(...)` | Payment deadline event removed |
| Approval chain started with SLA | `void syncApprovalSlaDeadline(...)` | SLA deadline event appears on calendar |

**Error handling pattern:** Fire-and-forget (`void` async). Calendar push never blocks or shows errors to the user on the contract/invoice save flow. Errors are logged server-side only.
