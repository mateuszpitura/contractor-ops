## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, table cell padding, summary bar vertical padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, form group gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions: Summary bar uses 8px (py-2) vertical padding for compact density. Avatar circles are 32px diameter in the directory table (consistent with existing user avatar pattern).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | text-sm (clamp 13-14px) | 400 (regular) | 1.25rem |
| Label | text-xs (clamp 11-12px) | 400 (regular) | 1rem |
| Heading | text-lg (clamp 16-18px) | 600 (semibold) | 1.75rem |
| Display | text-2xl (clamp 22-30px) | 600 (semibold) | 2.25rem |

Notes: All sizes use the project's existing fluid clamp() scale defined in globals.css. Email addresses in the directory table use `text-muted-foreground` at Body size. Department and org unit labels use Label size. Google group names in the mapping step use Body size with `font-semibold` (600) for visual grouping.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` oklch(0.985 0.003 85) | Page background, wizard surface |

## Interaction Contracts

### Connection Flow

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Connect | "Connect Google Workspace" button | OAuth redirect to Google admin consent screen | On return: provider card shows "Connected", redirect to import wizard with preview loaded (D-04) |
| Disconnect | Row action "Disconnect" | Confirmation dialog | Sonner toast "Google Workspace disconnected", card reverts to disconnected state |

### Import Wizard Flow

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Open wizard | "Import users" button or post-OAuth redirect | Dialog opens, directory fetch starts | Skeleton table during loading, summary bar populates on load |
| Search users | Search input in preview table | Client-side filter, debounced 300ms | Table filters immediately, summary bar updates counts |
| Filter by org unit | Org unit dropdown | Client-side filter | Table filters, summary bar updates |
| Select users | Checkbox per row or "Select all" | Row highlights, count updates | Summary bar shows selected count |
| Set default role | Role select dropdown (step 2) | All selected users without group override get this role | Preview updates in confirm step |
| Map group to role | Role dropdown per group (step 2) | Users in that group get the mapped role | Confirm step reflects role breakdown |
| Confirm import | "Import {N} users" button (step 3) | Button shows loading spinner, disables | Progress: sonner toast "Importing users...", on complete: "Successfully imported {N} users", dialog closes, integrations tab refreshes |
| Partial failure | Some users fail during import | Dialog stays open showing error state | Error banner with count of failures + retry button for failed users |
| Dismiss wizard | [X] close button in dialog header | Dialog closes, no import occurs | No confirmation needed -- no destructive action has been taken |

### Sync Controls

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Manual sync | "Sync now" button | Button shows loading spinner | Sonner toast "Directory sync started", on complete: "Sync complete -- {N} changes detected" or "No changes detected" |

### Notification Interactions

| Trigger | Behavior | UI Element |
|---------|----------|------------|
| Sync detects new hire | Dashboard notification appears | Notification with "New team member" copy + "Import" action button linking to wizard |
| Sync detects departure | Dashboard notification appears | Notification with departure copy + "Review" action linking to member list |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Desktop (>1024px) | Full directory table with all columns, dialog at max-width 720px |
| Tablet (768-1024px) | Table hides org unit column, dialog fills available width |
| Mobile (<768px) | Table hides department + org unit columns, shows name/email/status only; dialog becomes full-screen sheet; group mapping cards stack vertically |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Table rows focusable with Space to toggle checkbox; Tab moves between search/filter/table; Dialog traps focus; Step navigation via Tab |
| Screen reader | Summary bar uses `role="status"` with `aria-live="polite"` for count updates; "Already exists" badge has `aria-label="Already imported"`; Step indicator uses `aria-current="step"` |
| Focus visible | All interactive elements show `ring` on focus (project default via `outline-ring/50`) |
| Color contrast | "Already exists" greyed rows maintain 4.5:1 text contrast against row background; Badge text meets WCAG AA |
| Select all | "Select all" checkbox has `aria-label="Select all users"` and operates on visible (filtered) rows only |
