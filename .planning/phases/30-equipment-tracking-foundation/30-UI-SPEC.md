## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, table cell padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, form group gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions: Timeline connector line uses 2px width. Timeline node circles are 12px diameter (consistent with courier-tracking patterns). Touch targets for timeline event expansion are minimum 44px height.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | text-sm (clamp 13-14px) | 400 (regular) | 1.25rem |
| Label | text-xs (clamp 11-12px) | 400 (regular) | 1rem |
| Heading | text-lg (clamp 16-18px) | 600 (semibold) | 1.75rem |
| Display | text-2xl (clamp 22-30px) | 600 (semibold) | 2.25rem |

Notes: All sizes use the project's existing fluid clamp() scale defined in globals.css. Equipment type labels and serial numbers use font-mono (JetBrains Mono) at text-xs for scanability. Label role differentiates from Body through size (text-xs vs text-sm), not weight.

## Page Layouts

### Equipment List Page (`/equipment`)

```
+--------------------------------------------------+
| [Breadcrumb: Equipment]                          |
| Page title: "Equipment"        [+ Add equipment] |
+--------------------------------------------------+
| [Search] [Type filter] [Status filter] [Assignee]|
+--------------------------------------------------+
| Table                                             |
| Icon | Name | Serial | Type | Status | Assignee  |
| ...  | ...  | ...    | ...  | Badge  | Avatar+nm |
+--------------------------------------------------+
| Pagination (matches contractor table pattern)     |
+--------------------------------------------------+
```

Layout follows the exact pattern of `/contractors` and `/invoices`: breadcrumb, title row with primary CTA, filter toolbar, data table, pagination.

### Equipment Detail Page (`/equipment/[id]`)

```
+--------------------------------------------------+
| [Breadcrumb: Equipment > {name}]                 |
| [Icon] Equipment Name           [Edit] [Actions] |
| Serial: ABC-123  Type: Laptop  Status: Badge     |
+--------------------------------------------------+
| [Info] [Assignments] [Shipments] [Workflows]     |
+--------------------------------------------------+
| Tab content area (min-height: 400px)             |
| ...                                               |
+--------------------------------------------------+
```

Matches the contractor profile pattern: header with metadata, tabbed content below. Tabs use `variant="line"` consistent with `profile-tabs.tsx`.

### Shipment Timeline (within Shipments tab)

```
+-- Shipment #SHP-001 ---- InPost ---- TRK-123 --+
|                                                   |
|  o  CREATED          2026-04-02 09:00            |
|  |                                                |
|  o  PICKED_UP        2026-04-02 14:30            |
|  |  Note: Collected from office                   |
|  |                                                |
|  *  IN_TRANSIT       2026-04-03 08:15  (current) |
|  :                                                |
|  .  OUT_FOR_DELIVERY (pending)                    |
|  .                                                |
|  .  DELIVERED        (pending)                    |
|                                                   |
+---------------------------------------------------+
```

- Filled circle (`o`) = completed event, uses `text-muted-foreground`
- Star/filled circle (`*`) = current status, uses `text-primary` with `bg-primary/10` highlight
- Dot (`.`) = future/pending, uses `text-muted-foreground/40`
- Connector line: 2px solid `border` color, dashed for pending segments
- Each event row: timestamp right-aligned, optional notes below in `text-muted-foreground text-xs`

## Interaction Contracts

### Equipment CRUD

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Create | "Add equipment" button | Dialog opens with form | Sonner toast "Equipment added", table refreshes |
| Edit | Row action or detail "Edit" button | Same dialog, pre-filled | Sonner toast "Equipment updated" |
| Retire | Row action "Retire" | Confirmation dialog | Sonner toast "Equipment retired", status badge updates |

### Assignment Flow

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Assign | "Assign to contractor" button (detail page) | Dialog with Command-based contractor picker (search by name) | Sonner toast "Equipment assigned to {name}", status changes to ASSIGNED |
| Unassign | "Unassign equipment" button (detail page, visible only when assigned) | Confirmation dialog | Sonner toast "Equipment unassigned", status changes to AVAILABLE |

### Shipment Flow

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Create | "Create shipment" button | Dialog with carrier select, tracking number input, expected delivery calendar | Sonner toast "Shipment created", timeline updates |
| Update status | "Add status update" on timeline | Inline form at timeline top: status select + optional notes | New event appears at timeline top |
| Delete | Shipment row action | Confirmation dialog | Sonner toast "Shipment deleted" |

### Workflow Integration

| Trigger | Behavior | UI Element |
|---------|----------|------------|
| Onboarding EQUIPMENT task starts | Shipment creation dialog opens pre-filled with contractor address | Banner on task: "Ship equipment to {contractor}" with "Create shipment" button |
| Offboarding EQUIPMENT task starts | Return request created, status changes to RETURN_REQUESTED | Banner on task: "Request equipment return from {contractor}" |
| Shipment reaches DELIVERED | Outbound task auto-completes | Task status badge updates, sonner toast "Equipment delivered -- task completed" |
| Shipment reaches RETURNED | Return task auto-completes | Task status badge updates, sonner toast "Equipment returned -- task completed" |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Desktop (>1024px) | Full table with all columns, side-by-side layout on detail page |
| Tablet (768-1024px) | Table hides assignee column, stacks to single column on detail |
| Mobile (<768px) | Table switches to card list view, timeline stays vertical (natural fit), dialogs go full-screen sheet |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Table rows focusable, Enter opens detail; Dialog trap focus; Timeline events focusable |
| Screen reader | Timeline uses `role="list"` with `role="listitem"` per event; Status badges have `aria-label` with full status text |
| Focus visible | All interactive elements show `ring` on focus (project default via `outline-ring/50`) |
| Color contrast | All badge text meets 4.5:1 against badge background; timeline icons supplemented with shape differentiation (filled/outline/dashed) |
| Reduced motion | Timeline connector animations respect `prefers-reduced-motion` |
