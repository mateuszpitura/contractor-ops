# Plan 50-06: Complete CSS Logical Property Conversion

**Goal:** Convert all remaining physical CSS directional properties to logical equivalents across the entire `apps/web/src` tree, ensuring correct RTL layout for Arabic locale.

## Context

UAT Test 12 found 125 physical CSS property occurrences (`pl-`/`pr-`/`ml-`/`mr-`/`text-left`/`text-right`) across 66 files that were not converted by plans 03-05. Additionally, the UI primitive `alert.tsx` was missed in plan 01.

## Conversion Rules

Apply these Tailwind class substitutions throughout:

| Physical | Logical |
|----------|---------|
| `pl-` | `ps-` |
| `pr-` | `pe-` |
| `ml-` | `ms-` |
| `mr-` | `me-` |
| `text-left` | `text-start` |
| `text-right` | `text-end` |
| `left-` (positional) | `start-` |
| `right-` (positional) | `end-` |

**Exceptions (DO NOT convert):**
- Centering patterns: `left-1/2 -translate-x-1/2` (axis-based, not directional)
- `data-[side=left/right]` attribute selectors (reference prop values, not layout)
- Animation slide directions: `slide-in-from-left`, `slide-in-from-right` (these are visual motion, not layout)
