## Component Mounting Contract

This is the core of Phase 36's UI work. All components already exist and are fully implemented. This contract specifies WHERE and HOW they mount.

### 1. DPD Provider Section — Settings > Integrations

| Property | Value |
|----------|-------|
| Component | New `DpdProviderSection` wrapper |
| Mount point | `integrations-tab.tsx` grid, after existing provider sections |
| Pattern | Follows `TeamsProviderSection` pattern exactly |
| Inner components | `ProviderConnectionCard` + `Dialog` containing `CarrierCredentialForm` |
| Icon | `Truck` from lucide, `size-8`, color `text-red-600` (DPD brand red) |
| Status badge | Show "Connected" (green) when DPD credentials exist in org settings, "Not configured" (muted) otherwise |
| Dialog trigger | "Configure" button on card (outline variant, sm size) |
| Dialog title | "DPD Configuration" |
| Dialog content | `CarrierCredentialForm carrier="dpd"` |

### 2. UPS Provider Section — Settings > Integrations

| Property | Value |
|----------|-------|
| Component | New `UpsProviderSection` wrapper |
| Mount point | `integrations-tab.tsx` grid, after DPD section |
| Pattern | Identical to DPD section |
| Inner components | `ProviderConnectionCard` + `Dialog` containing `CarrierCredentialForm` |
| Icon | `Truck` from lucide, `size-8`, color `text-amber-700` (UPS brand brown approximation) |
| Status badge | Show "Connected" (green) when UPS credentials exist, "Not configured" (muted) otherwise |
| Dialog trigger | "Configure" button on card (outline variant, sm size) |
| Dialog title | "UPS Configuration" |
| Dialog content | `CarrierCredentialForm carrier="ups"` |

### 3. Carrier Shipment Button — Equipment Detail Page

| Property | Value |
|----------|-------|
| Mount point | `equipment/[id]/page.tsx`, action bar alongside existing "Create Shipment" button |
| Button label | "Ship via Carrier" |
| Button variant | `outline` (to differentiate from existing "Create Shipment" primary button) |
| Button icon | `Truck` from lucide, size 16, before label |
| Visibility | Only show when at least one carrier has configured credentials (D-07) |
| Dialog | Opens `CarrierShipmentForm` in a `Dialog` (not Sheet — matches equipment page existing dialog pattern) |
| Dialog title | "Create Carrier Shipment" |
| Dialog width | `max-w-lg` (matches existing ShipmentForm dialog) |
| Pre-selection | If only one carrier configured, auto-select it in form dropdown |

### 4. FeatureGate Wrappers

Based on PLAN_CONFIG excludedFeatures for STARTER tier:

| Gated Section | Mount Point | Gate Props | Approach |
|--------------|-------------|------------|----------|
| Integrations tab content | `integrations-tab.tsx` around provider grid | `requiredTier="Pro" featureName="Integrations"` | Inline wrapper |
| OCR trigger button | OCR invoice section | `requiredTier="Pro" featureName="OCR invoice parsing"` | Inline wrapper |
| Advanced workflow features | Workflow template creation / conditional logic | `requiredTier="Pro" featureName="Advanced workflows"` | Inline wrapper |
| Audit log export | Settings audit log tab or export button | `requiredTier="Enterprise" featureName="Audit log export"` | Page-level or button-level |
| API access | API key management section (if exists) | `requiredTier="Enterprise" featureName="API access"` | Page-level redirect or no-op if page doesn't exist |

### 5. Global TIER_REQUIRED Error Handler

| Property | Value |
|----------|-------|
| Mount point | tRPC QueryClient configuration (`query-client.ts` or provider wrapper) |
| Trigger | `FORBIDDEN` error with `JSON.parse(message).type === "TIER_REQUIRED"` |
| UI response | Sonner toast with error variant |
| Toast message | "This feature requires {requiredTier} plan." |
| Toast action | "Upgrade" button linking to `/settings?tab=billing` |
| Fallthrough | Non-tier FORBIDDEN errors pass through to default error handling |
| Parse safety | `try/catch` around `JSON.parse` — malformed messages fall through silently |

## Interaction States

### Provider Card States

| State | Visual |
|-------|--------|
| Not configured | Muted status badge "Not configured", "Configure" button enabled |
| Configured/Connected | Green status badge "Connected", "Configure" button still enabled (for editing) |
| Loading | Skeleton pulse on status badge area |

### FeatureGate States

| State | Visual |
|-------|--------|
| Loading (subscription query) | Render children normally (no flash) — per Phase 35 decision |
| Tier sufficient | Render children normally |
| Tier insufficient | Replace children with `UpgradeInlineBanner` — left teal border, Gem icon, tier requirement text, "Upgrade Plan" primary button |

### Carrier Shipment Button States

| State | Visual |
|-------|--------|
| No carriers configured | Button hidden entirely |
| One or more carriers configured | Button visible, outline variant |
| Dialog open | Standard Dialog overlay with CarrierShipmentForm |

### Global Tier Error Toast

| State | Visual |
|-------|--------|
| Toast appears | Sonner error toast, bottom-right, auto-dismiss after 5s |
| User clicks "Upgrade" | Navigate to `/settings?tab=billing` |
| User dismisses | Toast closes, no side effects |
