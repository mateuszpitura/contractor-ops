**Create `packages/api/src/services/skonto.ts` (per D-21, D-24):**

Types:
```typescript
export type SkontoTermData = {
  discountPercent: number;
  discountPeriodDays: number;
  netPeriodDays: number;
};

export type SkontoEligibilityInput = {
  invoiceTotalMinor: number;
  invoiceIssueDate: Date;
  skontoTerm: SkontoTermData | null;
  paidAt: Date | null;
  asOf: Date;
};

export type SkontoEligibilityResult = {
  eligible: boolean;
  eligibilityReason: 'ELIGIBLE' | 'PAST_DISCOUNT_WINDOW' | 'NO_SKONTO_CONFIGURED' | 'UNPAID_IN_WINDOW';
  discountedAmountMinor: number;
  discountAmountMinor: number;
  netAmountMinor: number;
  discountDeadline: Date | null;
};
```

`export function resolveSkontoTerm(invoiceTerm: SkontoTermData | null, profileDefault: SkontoTermData | null): SkontoTermData | null`:
- Return `invoiceTerm ?? profileDefault ?? null` (per D-21 cascade).

`export function evaluateSkontoEligibility(input: SkontoEligibilityInput): SkontoEligibilityResult`:
1. If `skontoTerm === null` -> return `{ eligible: false, eligibilityReason: 'NO_SKONTO_CONFIGURED', discountedAmountMinor: input.invoiceTotalMinor, discountAmountMinor: 0, netAmountMinor: input.invoiceTotalMinor, discountDeadline: null }`.
2. Calculate `discountDeadline = invoiceIssueDate + discountPeriodDays`.
3. Calculate `discountAmountMinor = Math.floor(invoiceTotalMinor * discountPercent / 100)`.
4. Calculate `discountedAmountMinor = invoiceTotalMinor - discountAmountMinor`.
5. Determine reference date: `paidAt ?? asOf`.
6. If reference date <= discountDeadline -> `{ eligible: true, eligibilityReason: 'ELIGIBLE', ... }`.
7. Else -> `{ eligible: false, eligibilityReason: 'PAST_DISCOUNT_WINDOW', ... }`.

2. When `skontoTerm` is set:
   - Compute the German description by interpolating `SKONTO_DESCRIPTION_TEMPLATE_DE` from `@contractor-ops/validators`:
     ```typescript
     import { SKONTO_DESCRIPTION_TEMPLATE_DE } from '@contractor-ops/validators';
     
     const germanDescription = SKONTO_DESCRIPTION_TEMPLATE_DE
       .replace('{percent}', skontoTerm.discountPercent.toFixed(2))
       .replace('{discountDays}', String(skontoTerm.discountPeriodDays))
       .replace('{netDays}', String(skontoTerm.netPeriodDays));
     ```
   - Compute the structured Skonto extension per XRechnung 3.0.2 Anhang E:
     ```typescript
     const structuredSkonto =
       `#SKONTO#TAGE=${skontoTerm.discountPeriodDays}` +
       `#PROZENT=${skontoTerm.discountPercent.toFixed(2)}` +
       `#BASISBETRAG=${(totalMinor / 100).toFixed(2)}#`;
     ```
   - Set `ram:Description` as `${germanDescription}\n${structuredSkonto}` in the `ram:SpecifiedTradePaymentTerms` block.

3. When `skontoTerm` is NOT set: BG-20 block behavior is identical to current (omit or emit only due date — no behavior change for non-Skonto invoices).

4. The `ram:DueDateDateTime` element should use `issueDate + netPeriodDays` when skontoTerm is set, formatted as `udt:DateTimeString format="102"` (YYYYMMDD).

5. The structured Skonto string goes in the TEXT content of `ram:Description` via fast-xml-parser's `#text` property — NOT in an XML attribute. This ensures `#` and decimal numbers appear literally without XML escaping issues (per Pitfall 6 from RESEARCH).
