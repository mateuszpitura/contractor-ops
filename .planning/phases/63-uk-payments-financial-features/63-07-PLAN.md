- Gated on PAY_SKONTO_ENABLED.
- Three inputs: Discount % (`z.number().gt(0).lte(50)`), Discount period days, Net period days.
- Default cascade UI:
  - Profile default exists + no invoice term: "Using contractor default: 3% / 7 days / net 30 days" pill + "Customize for this invoice" toggle (`<Switch>`).
  - Invoice term exists: show inputs + "Reset to contractor default" / "Remove Skonto" button.
  - No default, no term: "Add Skonto" button reveals inputs.
- Inline validation on blur. Error text in destructive tint.
- Live preview line: interpolated SKONTO_DESCRIPTION_TEMPLATE_DE below inputs (body font, not mono).
- Save calls `trpc.skonto.upsertForInvoice.useMutation(...)`. Toast "Skonto term saved".
- Delete calls `trpc.skonto.deleteForInvoice.useMutation(...)`.

**2. Create `skonto-banner.tsx` (per D-25, UI-SPEC):**
- Rendered below invoice-status card on detail page for DE invoices with resolved SkontoTerm.
- Fetches `trpc.skonto.evaluateForInvoice.useQuery({ invoiceId })`.
- State machine per UI-SPEC:
  - ELIGIBLE: success-tinted banner "Save {discountAmount} if paid by {date} -- discounted total {discountedTotal}"
  - PAST_DISCOUNT_WINDOW: muted banner "Discount window expired on {date}"
  - TAKEN_AT_PAYMENT: success-tinted with CheckCircle2 "Skonto applied at payment: {discountAmount} saved on {paidDate}"
  - NOT_TAKEN_AT_PAYMENT: muted "Paid after discount window -- no Skonto applied"
  - NO_SKONTO_CONFIGURED: banner hidden
- Loading: skeleton single-line.
