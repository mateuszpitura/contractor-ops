Implement `export function generateBacsStandard18(items: BacsExportItem[], orgBank: BacsOrgBankInfo, runRef: string, processingDate: Date): BacsGenerateResult`:

1. **Header records** (per BACS Std 18 MIG):
   - VOL1: `VOL1` + 76 spaces (80 chars total)
   - HDR1: `HDR1A` + SUN (6) + `S` + `  001` + `  001` + ` ` + julian(processingDate, 'YYDDD') (6) + ` ` + julian(processingDate, 'YYDDD') (6) + `000000` + rest spaces (80 total)
   - HDR2: `HDR2F0200000100` + 65 spaces (80 total)
   - UHL1: `UHL1` + julian(processingDate, 'YYDDD') (6) + rest per spec (varies; fill with spaces up to 80)

2. **Detail records** (106 chars each, per D-03 and RESEARCH code example):
   - Pos 1-6: dest sort code (6 digits)
   - Pos 7-14: dest account number (8 digits)
   - Pos 15: space (type of account default)
   - Pos 16-17: '99' (Direct Credit)
   - Pos 18-23: originator sort code (6 digits)
   - Pos 24-31: originator account number (8 digits)
   - Pos 32-35: 4 spaces (free)
   - Pos 36-46: amount in pence, 11 digits zero-padded
   - Pos 47-64: originator name/ref (18 chars, space-padded, transliterated uppercase ASCII)
   - Pos 65-82: user reference (18 chars, from paymentReference truncated, space-padded)
   - Pos 83-100: destination account name (18 chars, contractor name transliterated, space-padded)
   - Pos 101-106: processing date YYDDD or 6 spaces

   For each item: run `transliterateToBacs(item.contractorName)` and collect warnings. Run `modulusCheck(item.sortCode, item.accountNumber, VOCALINK_MODULUS_TABLE_V840)` from `@contractor-ops/validators` and collect warnings.

3. **Trailer records:**
   - EOF1: `EOF1A` + SUN (6) + rest spaces (80 total)
   - EOF2: `EOF2F0200000100` + rest spaces (80 total)
   - UTL1: `UTL1` + total amount across all items (11 digits zero-padded) + rest spaces (80 total)

4. Join all records with `\r\n`. Return as Buffer (no BOM).

**Validate:** Assert each detail record is exactly 106 characters. Throw if `amountMinor >= 100_000_000_000` (11-digit pence overflow).

**Julian date helper:** `function toJulianDate(date: Date): string` returns `YYDDD` format (2-digit year + 3-digit day-of-year, zero-padded).
