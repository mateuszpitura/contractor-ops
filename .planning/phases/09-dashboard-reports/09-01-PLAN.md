Create `packages/api/src/routers/dashboard.ts` with dashboardRouter containing these procedures, all using `tenantProcedure.use(requirePermission({ report: ["read"] }))`:

1. **kpis** query (no input): Returns 5 KPI values with trend data. Use `Promise.all` with parallel Prisma queries:
   - `activeContractors`: count Contractor where status="ACTIVE", deletedAt=null. Trend: compare count vs previous month (contractors created before start of current month with status ACTIVE).
   - `pendingApprovals`: count ApprovalStep where status="PENDING". Trend: compare vs count at start of previous month.
   - `readyToPayTotal`: aggregate Invoice where paymentStatus="READY", deletedAt=null, sum amountToPayGrosze. Trend: compare vs previous month total.
   - `expiringContracts`: count Contract where status IN ["ACTIVE","EXPIRING"], endDate between now and now+30 days, deletedAt=null. No trend (point-in-time metric, show neutral).
   - `openTasks`: count WorkflowTaskRun where status IN ["PENDING","IN_PROGRESS"]. No trend (show neutral).
   - Return shape: `{ activeContractors: { value: number, prevValue: number }, pendingApprovals: { value: number, prevValue: number }, readyToPayTotal: { valueGrosze: number, prevValueGrosze: number }, expiringContracts: { value: number }, openTasks: { value: number } }`. Use `plain()` wrapper.
   - For trend "previous month" comparison on activeContractors/pendingApprovals/readyToPayTotal: query the same metrics but scoped to the date range of the previous calendar month.

2. **spendTrend** query (input: `{ months: z.enum(["6","12","ytd"]) }`): Uses `prisma.$queryRaw` to group paid invoices by `date_trunc('month', "paidAt")` and currency. Cast bigint SUM to `::int` in SQL to avoid BigInt serialization issues. Return array of `{ month: string (ISO), currency: string, totalGrosze: number }`. Compute startDate: "6" = 6 months ago, "12" = 12 months ago, "ytd" = Jan 1 of current year. Filter: paymentStatus='PAID', paidAt >= startDate, deletedAt IS NULL.

3. **deadlines** query (no input): Returns upcoming deadlines combining 3 sources via `Promise.all`:
   - Contracts expiring in next 90 days: Contract where endDate between now and now+90d, status IN ["ACTIVE","EXPIRING"], deletedAt=null. Select id, title, endDate. Map to `{ type: "CONTRACT_EXPIRING", entityId, entityName, dueDate, daysRemaining }`.
   - Overdue workflow tasks: WorkflowTaskRun where status IN ["PENDING","IN_PROGRESS"] and dueDate < now. Select id, title, dueDate. Map to `{ type: "TASK_OVERDUE", entityId, entityName, dueDate, daysOverdue }`.
   - Invoices due in next 30 days: Invoice where dueDate between now and now+30d, paymentStatus NOT IN ["PAID","CANCELLED"], deletedAt=null. Select id, invoiceNumber, dueDate. Map to `{ type: "INVOICE_DUE", entityId, entityName, dueDate, daysRemaining }`.
   - Merge all, sort by urgency: overdue first (most overdue at top), then soonest upcoming. Limit 20. Use `plain()`.

4. **activity** query (no input): Query AuditLog where organizationId = ctx.organizationId, order by createdAt DESC, take 20. Return `{ items: Array<{ id, actorName, actorType, action, resourceType, resourceId, resourceName, createdAt }> }`. Use `plain()`.

Create `packages/api/src/routers/report.ts` with reportRouter containing these procedures, all using `tenantProcedure.use(requirePermission({ report: ["read"] }))`:

1. **spendByContractor** query: Input `{ dateFrom: z.string(), dateTo: z.string(), page: z.number().min(1).default(1), pageSize: z.number().min(1).max(100).default(20), sortBy: z.enum(["totalSpend","invoiceCount","contractorName"]).default("totalSpend"), sortOrder: z.enum(["asc","desc"]).default("desc"), contractorId: z.string().optional() }`. Use `prisma.$queryRaw` joining Invoice to Contractor, filtering paymentStatus='PAID', paidAt in date range, deletedAt IS NULL. Group by contractor. Cast SUM/COUNT/AVG to `::int`. Return `{ items: Array<{ contractorId, contractorName, invoiceCount, totalGrosze, avgGrosze, lastPaidAt }>, totalCount }`. Include total count via separate COUNT query.

2. **spendByTeam** query: Same input shape as spendByContractor but `sortBy: z.enum(["totalSpend","invoiceCount","teamName"])`. Join Invoice -> Contractor -> Team (via Contractor.primaryTeamId). Group by team. Return `{ items: Array<{ teamId, teamName, contractorCount, invoiceCount, totalGrosze }>, totalCount }`.

3. **expiringContracts** query: Input `{ days: z.enum(["30","60","90"]).default("30"), page, pageSize, sortBy: z.enum(["endDate","contractorName","title"]).default("endDate"), sortOrder }`. Query Contract where endDate between now and now+N days, status IN ["ACTIVE","EXPIRING"], deletedAt=null. Include contractor relation for name. Return `{ items: Array<{ contractId, contractTitle, contractorId, contractorName, endDate, daysRemaining, status }>, totalCount }`.

4. **overdueInvoices** query: Input `{ page, pageSize, sortBy: z.enum(["dueDate","amount","contractorName"]).default("dueDate"), sortOrder }`. Query Invoice where dueDate < now, paymentStatus NOT IN ["PAID","CANCELLED"], deletedAt=null. Include contractor. Return `{ items: Array<{ invoiceId, invoiceNumber, contractorId, contractorName, amountGrosze, currency, dueDate, daysOverdue, status }>, totalCount }`.

5. **complianceGaps** query: Input `{ page, pageSize, sortBy: z.enum(["health","contractorName","missingDocs"]).default("health"), sortOrder }`. Query contractors where complianceHealth IN ["YELLOW","RED"] (or compute from existing compliance health logic in contractor router). Return `{ items: Array<{ contractorId, contractorName, missingDocuments: number, contractStatus: string, overdueTasks: number, health: string }>, totalCount }`.

6. **spendByContractorChart** query: Input `{ dateFrom, dateTo }`. Returns top 10 contractors by spend for bar chart. No pagination. Return array of `{ contractorId, contractorName, totalGrosze }`.

7. **spendByTeamChart** query: Input `{ dateFrom, dateTo }`. Returns all teams with spend for bar chart. No pagination.

8. **expiringContractsChart** query: Input `{ days: "30"|"60"|"90" }`. Returns counts grouped by 30-day buckets: `{ bucket: string, count: number }[]`.

9. **complianceGapsChart** query: No date input. Returns `{ critical: number, warning: number, ok: number }` for pie chart.

Wire both routers into `packages/api/src/root.ts`: add imports for `dashboardRouter` from `./routers/dashboard.js` and `reportRouter` from `./routers/report.js`, register as `dashboard: dashboardRouter` and `report: reportRouter`.

Follow exact patterns from invoice.ts: `tenantProcedure`, `requirePermission()`, `plain()` wrapper on returns. Use `Prisma.sql` for conditional raw SQL fragments. Cast all `bigint` to `Number()` or use `::int` in SQL.
