# Phase 9 — UI Design Contract

> Visual and interaction contract for Dashboard & Reports. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialized in Phase 1) |
| Preset | base-nova with Zinc neutral + Indigo accent override |
| Component library | Base UI (via shadcn/ui base-nova style) |
| Icon library | Lucide React |
| Font | Inter (variable, via `next/font/google`) |

**shadcn components needed for Phase 9 (new additions beyond Phase 8):**
- None. All required shadcn primitives are already installed from prior phases.

**New dependency for Phase 9:**
- `recharts` -- declarative React charting library for spend area chart, report bar/pie charts. Per decision D-05.

**Custom components built in Phase 9 (not from any registry):**
- KPI card -- clickable metric card with title, value, trend indicator (arrow + percentage change), optional sparkline. Built with shadcn Card, Lucide icons.
- Spend area chart -- Recharts AreaChart with month-over-month spend data, stacked by currency, 6m/12m/YTD toggle. Built with Recharts + shadcn ToggleGroup.
- Deadlines widget -- grouped list of upcoming events (contract expirations, overdue tasks, due invoices) with date, entity link, and type badge. Built with shadcn Card, Badge, ScrollArea.
- Approval queue widget -- compact list of top pending approvals with contractor name, amount, SLA indicator. Built with shadcn Card, Badge.
- Activity feed -- grouped event list (today/yesterday/earlier) with actor, action, entity link. Built with shadcn Card, ScrollArea.
- Report sidebar -- vertical nav listing report types with active state indicator. Built with custom nav items.
- Report summary chart -- Recharts BarChart/PieChart per report type with click-to-drill-down. Built with Recharts.
- Report detail table -- TanStack Table with server-side pagination, sorting, CSV export. Mirrors contractor-table pattern.
- Report breadcrumb trail -- filter path display with clear button for drill-down state. Built with shadcn Breadcrumb, Button.
- Audit log table -- TanStack Table with expandable rows for before/after diffs, full-text search, structured filters. Built with shadcn Table, Input, Select, Badge, Collapsible.
- Audit log diff viewer -- inline old/new value comparison for expanded rows. Built with custom diff rendering.
- Date range filter bar -- preset buttons (This month / Last 3 months / Last 6 months / YTD / Custom) + custom DatePicker range. Built with shadcn ToggleGroup, Calendar, Popover.

**Reused from Phase 1-8 (already installed):**
- `table`, `badge`, `card`, `dialog`, `sheet`, `button`, `input`, `label`, `select`, `skeleton`, `tabs`, `checkbox`, `popover`, `progress`, `scroll-area`, `separator`, `breadcrumb`, `dropdown-menu`, `tooltip`, `sonner`, `alert-dialog`, `calendar`, `collapsible`, `toggle-group`

## Spacing Scale

Inherited from Phase 1. No changes.

| Token | Value | Usage in Phase 9 |
|-------|-------|-------------------|
| xs | 4px | Icon-to-text gap in KPI cards, gap between trend arrow and percentage, inline gaps in activity feed entries |
| sm | 8px | Gap between KPI card elements (title/value/trend), gap between activity feed items, gap between deadline list items, gap between report sidebar items |
| md | 16px | KPI card internal padding, widget card padding, report detail table cell padding, audit log table cell padding, chart container padding |
| lg | 24px | Gap between KPI row and two-column section, gap between widgets in a column, report page sidebar-to-content gap, settings section padding |
| xl | 32px | Gap between dashboard sections (KPI row, widget grid), page content top margin, report page horizontal gap |
| 2xl | 48px | Page horizontal padding (desktop) |
| 3xl | 64px | Not used in Phase 9 |

Exceptions:
- KPI card minimum width: 200px (ensures readable content at all breakpoints)
- Spend chart height: 280px (sufficient for 6-12 month trend visibility)
- Report summary chart height: 240px (compact but readable)
- Report sidebar width: 220px (accommodates longest report label)
- Deadlines widget max height: 320px with internal ScrollArea
- Activity feed max height: 400px with internal ScrollArea
- Approval queue widget max height: 280px with internal ScrollArea
- Audit log expanded row diff section padding: 16px with 1px left border accent

## Color

Inherited from Phase 1. All color tokens from `globals.css` apply. Phase 9 adds usage patterns only.

| Role | Token | Usage in Phase 9 |
|------|-------|-------------------|
| Dominant (60%) | `--background` | Page background, widget card backgrounds, report page background, audit log background |
| Secondary (30%) | `--muted` | KPI card backgrounds, chart container backgrounds, report sidebar background, date range filter bar background, audit log expanded row background, table header rows |
| Accent (10%) | `--primary` | See reserved list below |
| Destructive | `--destructive` | See destructive usage below |

**Accent (`--primary`) reserved for in Phase 9:**
- KPI card clickable hover state (border highlight)
- Spend chart area fill (at 20% opacity) and line stroke
- Active report type in sidebar (left border + text color)
- Active date range preset button
- "Export" primary CTA buttons on report pages
- Chart bar/segment fill for primary data series
- Drill-down breadcrumb active segment
- Focus ring on all interactive elements
- Audit log search input focus border

**Destructive (`--destructive`) usage in Phase 9:**
- Overdue indicators in deadlines widget (overdue badge, overdue date text)
- SLA breach indicators in approval queue widget
- Compliance gap severity "critical" badge in compliance report
- Negative trend indicators in KPI cards (only when trend is explicitly negative)

**Chart color palette (Recharts):**

| Series | Color Token | Usage |
|--------|-------------|-------|
| Primary | `--primary` | First/default data series (PLN spend, primary metric) |
| Secondary | `--chart-2` or `hsl(220 70% 55%)` | Second data series (EUR spend, secondary metric) |
| Tertiary | `--chart-3` or `hsl(160 60% 45%)` | Third data series (if needed) |
| Muted | `--muted-foreground` at 30% | Grid lines, axis lines, inactive chart elements |

**KPI trend indicator colors:**

| Trend | Color | Icon |
|-------|-------|------|
| Positive | `--success` (green) | `TrendingUp` |
| Negative | `--destructive` (red) | `TrendingDown` |
| Neutral | `--muted-foreground` | `Minus` |

**Deadline type badge colors:**

| Type | Badge BG | Badge Text |
|------|----------|------------|
| Contract expiring | `--warning` at 10% | `--warning` |
| Task overdue | `--destructive` at 10% | `--destructive` |

**Compliance gap severity colors:**

| Severity | Badge BG | Badge Text |

## Interaction Patterns

### Dashboard Page

- **URL:** / (root authenticated route, replaces existing placeholder). Dashboard is the landing page after login.
- **Layout:** Full-width content area within app shell. Page title "Dashboard" as Heading, hidden visually (page is self-evident), present in document title and breadcrumb.
- **KPI row:** 5 cards in a CSS grid with `grid-cols-5` at desktop, wrapping to `grid-cols-3` + `grid-cols-2` on tablet, `grid-cols-1` on mobile. Each card: Label (12px, `--muted-foreground`), Value (28px semibold), trend row (arrow icon + "{+/-X}% vs last month" in 12px semibold, colored by trend). Cards have `cursor-pointer`, hover: border transitions to `--primary` at 50% opacity. Click navigates per D-03: active contractors -> /contractors?status=active, pending approvals -> /approvals?tab=my&status=pending, ready to pay -> /payments?status=ready, expiring contracts -> /contracts?status=expiring, open tasks -> /workflows?tab=my-tasks.
- **Two-column layout:** Below KPI row with 24px gap. `grid-cols-2` at desktop, `grid-cols-1` on mobile (widgets stack vertically).
- **Spend chart widget:** Card with "Monthly spend" heading + 6m/12m/YTD ToggleGroup in card header right. ToggleGroup synced to URL via nuqs `parseAsString` with `?spend=6m`. Recharts AreaChart inside card body, 280px height. Area fill: `--primary` at 20% opacity. Line stroke: `--primary` at 100%, 2px width. If multi-currency: stacked areas with secondary color for EUR. X-axis: month labels (MMM format). Y-axis: formatted PLN amounts. Tooltip on hover: month, total amount formatted with Intl.NumberFormat, breakdown by currency if stacked. Grid lines: `--muted-foreground` at 15%.
- **Deadlines widget:** Card with "Upcoming deadlines" heading + "See all deadlines" link in card header right (navigates to /reports with expiring contracts report). ScrollArea body, max 320px. Items sorted by urgency (overdue first, then soonest). Each item: type badge (colored per deadline type table), entity name (link), time remaining or overdue text. Overdue items: `--destructive` text for days count.
- **Approval queue widget:** Card with "Approval queue" heading + "See all approvals" link (navigates to /approvals). ScrollArea body, max 280px. Shows top 5 pending approvals. Each item: contractor name, amount formatted, SLA badge (green/yellow/red per existing SLA pattern from Phase 6). Click item navigates to invoice detail.
- **Activity feed widget:** Card with "Recent activity" heading + "See all activity" link (navigates to /settings?tab=audit-log). ScrollArea body, max 400px. Shows last 20 events grouped by today/yesterday/earlier (D-04). Group headers: 12px `--muted-foreground` uppercase. Each event: two-line card -- line 1: actor name + action verb (14px, action semibold), line 2: entity type badge + entity name (link) + relative timestamp (12px `--muted-foreground`). Click event navigates to entity.
- **Loading:** 5 skeleton KPI cards (pulse animation). 4 widget skeleton cards with internal content skeletons (3 lines each).
- **Error:** If any widget fails to load, show inline error card with `AlertCircle` icon + error message + "Try again" link. Other widgets continue to load independently.

### Reports Page

- **URL:** /reports. Sidebar navigation item with `BarChart3` icon, after "Payments" and before "Settings". Permission: `report.read`.
- **Layout:** Sidebar (220px, fixed) + main content area. Sidebar has vertical list of report types with Lucide icons: `Users` Spend by contractor, `UsersRound` Spend by team, `FileWarning` Expiring contracts, `Clock` Overdue invoices, `ShieldAlert` Compliance gaps. Active item: left 3px border `--primary`, text `--primary`, bg `--primary` at 5%. Inactive: text `--muted-foreground`, hover bg `--muted`.
- **Report type selection:** Synced to URL via nuqs `parseAsString` with `?report=spend-contractor`. Default: spend-contractor.
- **Date range filter bar:** Horizontal bar below page title. ToggleGroup with preset buttons: This month, Last 3 months, Last 6 months, YTD, Custom. Custom shows DatePicker range popover. Active preset: `--primary` bg at 10%, `--primary` text. Date range synced to URL via nuqs. Persistent across report type switches (D-10).
- **Summary chart:** Below filter bar, 240px height. Chart type varies: BarChart for spend reports (horizontal bars, sorted by total descending, top 10), BarChart for expiring contracts (grouped by 30/60/90 day buckets), PieChart for compliance gaps (critical/warning/ok segments). Click bar/segment filters the detail table below (D-11). Clicked segment highlighted, others dimmed to 30% opacity.
- **Drill-down breadcrumb:** Appears above detail table when a chart segment is clicked. Shows: "All > {clicked entity name}". "All" is clickable to clear filter. "Clear filter" ghost button at end. Breadcrumb uses existing shadcn Breadcrumb component.
- **Detail table:** TanStack Table with server-side pagination (20 per page), sorting. Columns vary by report type (see copywriting tables). Row click navigates to entity detail page.
- **Export buttons:** Top-right aligned with page title. "Export page" outline button (exports current page of filtered data). "Export all" primary button (exports full dataset matching date range, ignoring pagination) (D-12). Both trigger browser download via base64 Blob pattern (reuse from Phase 8).
- **Loading:** Sidebar always visible. Chart: skeleton rectangle 240px. Table: 8 skeleton rows.
- **Empty state per report type:** Centered in main content area with relevant icon (48px, `--muted-foreground`), heading, and body text (see copywriting tables).

### Settings > Audit Log Tab

- **URL:** /settings?tab=audit-log. New tab added after "Integrations" in Settings page tabs. Permission: admin role only (D-13).
- **Layout:** Full-width within Settings content area.
