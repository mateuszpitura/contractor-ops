## Spacing Scale

Inherited from Phase 1. No changes.

| Token | Value | Usage in Phase 6 |
|-------|-------|-------------------|
| xs | 4px | Icon gaps in chain tracker steps, inline padding in SLA badge, timeline entry icon spacing |
| sm | 8px | Gap between chain level cards, gap between timeline entries, inter-column gap in condition builder rows |
| md | 16px | Default form field gaps in chain editor, gap between approval queue table rows, chain tracker step internal padding |
| lg | 24px | Section padding within chain editor card, audit timeline section padding, approval queue page content padding |
| xl | 32px | Gap between major sections on approval queue page, gap between chain tracker and audit timeline on invoice detail |
| 2xl | 48px | Page horizontal padding (desktop) |
| 3xl | 64px | Not used in Phase 6 |

Exceptions:
- Chain tracker stepper connector line: 2px height, `--border` color, spans between step circles
- Chain tracker step circle: 32px diameter (contains step number or status icon)
- SLA countdown badge minimum width: 72px (prevents layout shift between "22h left" and "OVERDUE 3h")
- Floating bulk action toolbar height: 56px, 16px vertical padding, fixed to bottom of viewport with 16px horizontal margin
- Floating bulk action toolbar shadow: `shadow-lg` with 80% opacity backdrop blur
- Timeline entry left gutter: 40px (avatar 32px + 8px gap to content)
- Timeline connector line: 2px width, `--border` color, centered under avatar column
- Side panel width: 480px (consistent with Phase 2-5)
- Side panel animation: 200ms ease-out slide from right (consistent with previous phases)
- Condition builder row height: 40px per row (input + operator + value aligned horizontally)

## Interaction Patterns

### Approvals Queue Page -- Tabs

- **Layout:** Two tabs at top of page: "My Approvals" (default) and "All" (admin only, hidden for non-admin roles via `usePermissions().can('approval', ['manage'])`).
- **Tab state:** Synced to URL via nuqs `parseAsString` with `?tab=my` / `?tab=all` for deep-linking.
- **"My Approvals" tab:** Shows only ApprovalSteps where `approverUserId` matches the current user and status is PENDING.
- **"All" tab:** Shows all ApprovalSteps across the organization. Admin-only visibility.

### Approvals Queue Page -- Status Chip Bar

- **Layout:** Horizontal row of Badge-style chips above the table. Each chip: label + count. All / Pending / Overdue / Approved / Rejected.
- **Active state:** Selected chip uses `--primary` at 10% opacity background with `--primary` text. Unselected chips use `--muted` background with `--muted-foreground` text.
- **Click:** Filters the table. "All" clears the filter. Active filter synced to URL via nuqs `parseAsString`.
- **Counts:** Each chip shows live count. Counts update after any approval action.
- **Overflow:** On smaller screens (< 1280px), chips scroll horizontally with fade gradient on edges.

### Approvals Queue Page -- Data Table

- **Columns:** Checkbox (selection), Invoice # (monospace, link to invoice detail), Contractor (name), Amount (right-aligned, formatted with locale, monospace), Submitted (relative date), SLA remaining (countdown badge), Priority (derived: overdue items get `--destructive` row tint).
- **Default sort:** Overdue first, then SLA remaining ascending (soonest due at top).
- **Row click:** Opens side panel (Sheet from right, 480px). Does NOT navigate.
- **Row hover actions:** Right side of row reveals "Approve" (ghost button with `CheckCircle2` icon, primary text) and "Reject" (ghost button with `XCircle` icon, destructive text). Actions appear on hover only, hidden by default.
- **Approve action:** Single click approves immediately. Sonner toast confirms. Row animates out of list (200ms fade).
- **Reject action:** Opens Popover anchored to button. Textarea for mandatory comment (min 10 chars). "Reject invoice" destructive CTA. "Keep pending" to dismiss.
- **Search:** Debounce 300ms, minimum 2 characters. Server-side search across invoice number, contractor name, NIP.
- **Pagination:** Bottom of table. Page size (10, 25, 50). Previous/Next. "Page X of Y".
- **Overdue highlighting:** Rows where SLA is breached have `--destructive` at 5% opacity background. SLA badge shows "OVERDUE {n}h" in destructive.

### Approvals Queue -- Bulk Actions
