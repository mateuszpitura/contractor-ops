    Export `TIER_CREDIT_ALLOWANCE` constant (re-export or co-locate):
    ```typescript
    export const TIER_CREDIT_ALLOWANCE: Record<SubscriptionTier, number> = {
      STARTER: 20,
      PRO: 100,
      ENTERPRISE: 500,
    } as const;
    ```

    Export `getCreditBalance(organizationId: string)`:
    - Query current subscription to get tier, currentPeriodStart, currentPeriodEnd
    - If no active/trialing subscription, return `{ balance: 0, allowance: 0, used: 0, tier: null }`
    - Aggregate OcrCreditLedger for the current billing period: `SUM(credits)` where organizationId matches and periodStart/periodEnd match subscription's current period
    - The net sum is the balance (positive allocations + negative deductions)
    - Return `{ balance: netSum, allowance: TIER_CREDIT_ALLOWANCE[tier], used: Math.abs(negativeSum), tier }`

    Export `checkAndDeductCredit(organizationId: string)`:
    - Must be atomic to prevent race condition (Pitfall 3 from RESEARCH.md)
    - Use `prisma.$transaction` with isolation level `Serializable`:
      1. Fetch subscription (must be `ACTIVE` or `TRIALING`)
      2. If no subscription: return `{ allowed: false, remaining: 0, reason: "no_subscription" as const }`
      3. Aggregate credits for current period
      4. Calculate remaining = allowance - used (where used = absolute value of negative credits sum)
      5. If remaining <= 0: return `{ allowed: false, remaining: 0, reason: "credits_exhausted" as const }`
      6. Insert negative ledger entry: `{ organizationId, credits: -1, reason: "OCR_EXTRACTION", periodStart: subscription.currentPeriodStart, periodEnd: subscription.currentPeriodEnd }`
      7. Return `{ allowed: true, remaining: remaining - 1 }`
    - After successful deduction (outside transaction), fire-and-forget Stripe Meter event:
      ```typescript
      stripe.billing.meterEvents.create({
        event_name: "ocr_extraction",
        payload: {
          stripe_customer_id: subscription.stripeCustomerId,
          value: "1",
        },
      }).catch((err) => console.error("[billing] Meter event failed:", err));
      ```
    - Return type: `Promise<{ allowed: boolean; remaining: number; reason?: "no_subscription" | "credits_exhausted" }>`

    Export `allocateTopUpCredits(params: { organizationId: string; credits: number; stripeEventId?: string })`:
    - Look up current subscription to get periodStart/periodEnd
    - Insert positive ledger entry with reason "TOP_UP"
    - Return `{ balance: newBalance }` after allocation
