    Dedicated Stripe webhook endpoint per D-20 (NOT through the [provider] route).

    ```typescript
    import { NextRequest, NextResponse } from "next/server";
    import { stripe } from "@contractor-ops/api/services/stripe-client";
    import { routeStripeEvent } from "@contractor-ops/api/services/billing-webhook";
    import { prisma } from "@contractor-ops/db";
    ```

    Export `POST` handler:
    1. Read raw body via `request.text()`
    2. Get `stripe-signature` header
    3. Call `stripe.webhooks.constructEvent(rawBody, signature, process.env.STRIPE_WEBHOOK_SECRET!)` -- if verification fails, return 400
    4. Check idempotency: `prisma.stripeEvent.findUnique({ where: { stripeEventId: event.id } })` -- if `processedAt` is set, return `{ received: true }` with 200
    5. Process in transaction:
       - Upsert StripeEvent record (create with stripeEventId, eventType, payloadJson)
       - Call `routeStripeEvent(event, tx)`
       - Update StripeEvent with `processedAt: new Date()`
    6. Return 200 `{ received: true }`
    7. Catch all errors: log with `console.error("[stripe-webhook]", error)`, return 500

    **2. Create `packages/api/src/services/billing-webhook.ts`:**

    Export `routeStripeEvent(event: Stripe.Event, tx: PrismaTransactionClient)`:

    Switch on `event.type`:
    - `checkout.session.completed`: Extract subscription ID from session, call `handleSubscriptionUpdated` if session.mode === "subscription"
    - `customer.subscription.created`: Call `handleSubscriptionUpdated(event.data.object, tx)`
    - `customer.subscription.updated`: Call `handleSubscriptionUpdated(event.data.object, tx)`
    - `customer.subscription.deleted`: Call `handleSubscriptionDeleted(event.data.object, tx)`
    - `customer.subscription.trial_will_end`: Call `handleTrialWillEnd(event.data.object, tx)`
    - `invoice.paid`: Call `handleInvoicePaid(event.data.object, tx)`
    - `invoice.payment_failed`: Call `handlePaymentFailed(event.data.object, tx)`
    - Default: log `[stripe-webhook] Unhandled event type: ${event.type}`

    Helper `handleSubscriptionUpdated(subscription: Stripe.Subscription, tx)`:
    - Resolve `organizationId` from `subscription.metadata.organizationId`
    - Map Stripe status to SubscriptionStatus enum: `trialing` -> `TRIALING`, `active` -> `ACTIVE`, `past_due` -> `PAST_DUE`, `canceled` -> `CANCELED`, `unpaid` -> `UNPAID`, `incomplete` -> `INCOMPLETE`, `incomplete_expired` -> `INCOMPLETE_EXPIRED`, `paused` -> `PAUSED`
    - Determine tier from price ID by looking up price metadata or a config constant `PRICE_TO_TIER_MAP`
    - `tx.subscription.upsert` with `where: { stripeSubscriptionId: subscription.id }`, update and create with all fields

    Helper `handleSubscriptionDeleted(subscription, tx)`:
    - Update subscription status to CANCELED via `tx.subscription.update`

    Helper `handleTrialWillEnd(subscription, tx)`:
    - Look up org by stripeSubscriptionId
    - Create notification via existing notification service pattern (import from notification-service.ts) -- "Your trial ends in 3 days"

    Helper `handleInvoicePaid(invoice: Stripe.Invoice, tx)`:
    - If invoice is for subscription renewal (not first invoice), allocate monthly credits:
      - Determine tier from subscription
      - Insert positive OcrCreditLedger entry: STARTER=20, PRO=100, ENTERPRISE=500 credits per D-06
      - Set periodStart/periodEnd from subscription billing period

    Helper `handlePaymentFailed(invoice, tx)`:
    - Update subscription status to PAST_DUE
    - Create notification for billing admin: "Payment failed. Update your payment method."

    Export a `TIER_CREDIT_ALLOWANCE` constant: `{ STARTER: 20, PRO: 100, ENTERPRISE: 500 }` per D-06.
    Export a `PRICE_TO_TIER_MAP` as a Record<string, SubscriptionTier> that maps Stripe price IDs to tiers. Use env vars: `STRIPE_PRICE_STARTER`, `STRIPE_PRICE_PRO`, `STRIPE_PRICE_ENTERPRISE` so prices are configurable without code changes.
