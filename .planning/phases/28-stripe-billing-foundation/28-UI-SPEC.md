## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 14px | 600 (semibold) | 1.4 |
| Heading | 22px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Two declared weights: 400 (regular) for body text and 600 (semibold) for labels, headings, and display. Labels are differentiated from Body by weight alone (semibold vs regular at the same 14px size), creating a clear visual hierarchy without a third weight.

Source: Existing codebase patterns. Headings use `font-display text-[22px] font-semibold leading-tight tracking-tight` throughout the app. Display size (28px) reserved for pricing amounts in plan comparison cards.

## Layout Contract

### Settings > Billing Tab

The Billing tab is added to the existing Settings page TabsList (after "Integrations", before "Audit Log"). URL param: `?tab=billing`. Permission gated: only visible to users with `organization.update` permission.

**Layout structure:**

```
[CurrentPlanSummary]        [CreditUsageCard]
(full-width on mobile, 2-col grid on md+)

[PlanComparisonGrid]
(3-col grid on lg+, vertical stack on mobile, 2-col on md)

[Manage Billing Portal Link]
(secondary button, redirects to Stripe-hosted portal)
```

### Plan Comparison Grid

Each `PlanCard` contains:
1. Tier name (Label role, 14px semibold)
2. Price display: flat fee in Display size (28px semibold), per-seat price in Body (14px muted)
3. Included seats count
4. Feature list with check/x icons (lucide `Check` / `X`)
5. CTA button at card bottom
6. "Recommended" badge on Pro tier (accent ring highlight)
7. Current plan indicated with accent border ring and "Current plan" badge

### Trial Banner

- Position: top of dashboard layout, below nav, above page content
- Full-width, muted warning background with left accent border
- Copy: "Your trial ends in {N} days. Upgrade to keep your data and features."
- Action: "Choose a plan" button (primary variant)
- Appears only during last 7 days of trial (D-17)
- Dismissible per session but reappears on next login

### Soft-Block Modal

- Full viewport overlay, not dismissible via backdrop click or Escape
- Contains: heading, brief explanation, `PlanComparisonGrid` (compact variant), Stripe Checkout redirect buttons
- Close button visible but triggers re-show on next navigation (persistent nudge per D-09)

### Credit Exhausted Inline

- Appears inline at the OCR trigger point (upload area or extraction button)
- Replaces the normal extraction CTA when credits are zero
- Contains: warning icon, message, two buttons (Upgrade plan / Buy credits)

### Premium Badge (Diamond Icon)

- Lucide `Gem` icon at 14px, muted-foreground color
- Placed inline next to feature labels in navigation, settings, or feature entry points
- Tooltip on hover: "Available on {required_tier} plan"

## Interaction States

### Plan Selection Flow

1. User clicks "Choose a plan" or "Upgrade plan" on any PlanCard
2. If upgrading/downgrading: `ProrationPreview` shown inline in a confirmation step
3. User confirms -> redirect to Stripe Checkout (hosted page, not embedded)
4. On success: Stripe redirects back to `/settings?tab=billing` with success toast
5. On cancel: Stripe redirects back to `/settings?tab=billing` with no change

### Credit Top-Up Flow

1. User clicks "Buy credits" on `CreditUsageCard` or `CreditExhaustedInline`
2. `TopUpDialog` opens with bundle size selection
3. User confirms -> redirect to Stripe Checkout for one-time purchase
4. On success: credits added, toast confirmation, progress bar updates

### Auto-Renewal Toggle (Pro/Enterprise only)

1. Toggle switch within `CreditUsageCard` for auto-renewal
2. On enable: show threshold and bundle size selectors (inline expand)
3. On save: mutation updates org billing settings, success toast

### Stripe Billing Portal

1. User clicks "Manage billing" button
2. Server creates Stripe billing portal session
3. User redirected to Stripe-hosted portal (payment methods, invoices, cancel)
4. On return: redirected back to `/settings?tab=billing`

### Loading States

- `CurrentPlanSummary`: Skeleton with 3 lines (plan name, renewal, seats)
- `CreditUsageCard`: Skeleton with progress bar placeholder
- `PlanComparisonGrid`: 3 skeleton cards matching card dimensions
- All loading states use existing `Skeleton` component

### Error States

- Failed to load billing data: Card with error message + retry button
- Stripe Checkout redirect failure: Toast error with "Try again" action

## Accessibility Contract

- Plan comparison cards: Use `role="radiogroup"` with `role="radio"` per card for keyboard plan selection
- Trial banner: `role="alert"` with `aria-live="polite"` for screen reader announcement
- Soft-block modal: Focus trap, `aria-modal="true"`, `role="alertdialog"`
- Credit progress bar: Uses existing `Progress` component with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-label="OCR credits: {used} of {total} used"`
- Premium diamond badge: `aria-label="Requires {tier} plan"` on the icon, not just tooltip
- All pricing displayed with `tabular-nums` for alignment
- Destructive cancel action: requires explicit confirmation via `AlertDialog`
