## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / `oklch(0.985 0.003 85)` | Page background, wizard content area |
| Secondary (30%) | `var(--card)` / `oklch(1 0 0)` | Source selection cards, preview table container, step cards |
| Accent (10%) | `var(--primary)` / `oklch(0.44 0.145 178)` | Primary CTA buttons, active step indicator, progress bar fill, selected card border |
| Destructive | `var(--destructive)` / `oklch(0.58 0.23 18)` | Conflict badges, error states, skip-all destructive action |

Accent reserved for: primary CTA buttons, wizard step progress indicator, selected source card ring/border, "Import" toggle active state, progress bar fill during async import.

Additional semantic colors used in this phase:
- `var(--success)` / `oklch(0.55 0.17 150)`: "New" status badge (importable person), completed step checkmark, successful import count
- `var(--warning)` / `oklch(0.70 0.16 65)`: "Conflict" status badge, unresolved conflict count indicator
- `var(--info)` / `oklch(0.55 0.18 260)`: "Exists" status badge (already a member), source badge icons
- `var(--muted)` / `oklch(0.955 0.004 250)`: Skipped rows, disabled source cards, inactive step indicators

## Component Inventory

### Wizard Layout
- **Full-page layout** at `/onboarding/import` (D-02). No sidebar, centered content container max-width 960px.
- **Step indicator**: Horizontal numbered step bar at top. Pattern: reuse `StepIndicator` from `directory-import-wizard.tsx`. Steps: 1. Select Sources, 2. Review People, 3. Import Projects, 4. Confirm & Import.
- **Navigation**: "Back" ghost button (left) + primary "Continue" button (right) in sticky footer. Step 1 also shows "Skip -- I'll invite people manually" text link (D-04).
- **Progress bar**: `<Progress>` component below step indicator showing overall wizard completion (25/50/75/100%).

### Step 1: Source Selection (ONBD-01)
- **Source cards**: 4 cards in a 2x2 grid (Jira, Linear, Google Workspace, Slack). Each card:
  - Provider icon (existing `react-icons` brand icons: `SiJira`, `SiLinear`, `SiGoogleworkspace`, `SiSlack`)
  - Provider name (Label weight)
  - Connection status: "Connected" with green checkmark OR "Not connected" with muted text
  - Action: Connected shows toggle switch for "Import from this source"; unconnected shows "Connect" button triggering OAuth popup
  - Selected state: `ring-2 ring-primary` border treatment
- Card grid responsive: 2x2 on desktop, single column on mobile.

### Step 2: Review People (ONBD-02, ONBD-04)
- **Summary bar**: Counts for New / Conflicts / Existing / Total (reuse `DirectorySummaryBar` pattern)
- **Filter tabs**: All | New | Conflicts | Existing (D-11). Use existing `Tabs` component.
- **Preview table** (D-07, D-08, D-10):
  - Columns: Checkbox | Name | Email | Sources | Status | Role | Action
  - Checkbox column with select-all in header (D-10)
  - Sources column: Row of small colored badges showing which tools found this person (e.g., Jira chip, Slack chip)
  - Status column: Badge component -- green "New", amber "Conflict", blue "Exists"
  - Role column: Dropdown select defaulting to "Member" for new, showing current role for existing
  - Action column: "Skip" button for individual rows
  - Existing members pre-set to Skip with muted/strikethrough row styling (D-10)
- **Conflict resolution** (D-09): Clicking amber "Conflict" badge opens inline dropdown (Popover) showing values from each source with radio selection. Unresolved conflicts show warning icon and block the Continue button.
- **Batch toolbar** (D-10): Appears when 1+ rows selected. Contains: "Import Selected", "Skip Selected", "Assign Role to Selected" (dropdown).
- **Empty state**: When no sources returned people.

### Step 3: Import Projects (ONBD-03)
- **Project cards** (D-12, D-13): One card per Jira project / Linear team. Each card:
  - Project name + source icon badge
  - Status steps listed as reorderable chips/tags
  - "Edit steps" button opening inline editor (add/remove/rename/reorder steps)
  - "Skip this project" text button
- Auto-generated workflow template name shown as editable text input (D-12)
- Pre-configured sync mapping note: "Bidirectional sync will be configured automatically" (D-14)

### Step 4: Confirm & Import (ONBD-04, ONBD-05)
- **Summary cards**: Two cards side by side -- "People to import" (count + role breakdown) and "Projects to create" (count + step count)
- **Confirm button**: Primary CTA "Start Import"
- **Async progress** (ONBD-05): After confirming:
  - Full-width progress bar with percentage and "Importing 12 of 47 people..." label
  - Live item list below showing completed (green check), in-progress (spinner), failed (red X with "Retry" button)
  - Individual retry: failed items show "Retry" ghost button inline (ONBD-05)
  - Completion state: Success summary card with counts + "Go to Dashboard" primary CTA

### Shared Patterns
- **Loading states**: Skeleton placeholders matching card/table layout during source data fetching
- **OAuth error**: Toast notification (sonner) with error message + "Try again" action
- **Step validation**: Continue button disabled with tooltip when step has unresolved requirements (e.g., unresolved conflicts in step 2)

## Interaction States

| Interaction | Behavior |
|-------------|----------|
| Source card hover | Subtle elevation lift (`var(--elevation-sm)` to `var(--elevation-md)`) |
| Source card selected | `ring-2 ring-primary` + slight background tint `bg-teal-50/50` |
| Source card disabled (no OAuth) | Muted opacity 0.5, cursor-not-allowed |
| Continue button disabled | Reduced opacity, tooltip explaining what needs resolution |
| Table row hover | `bg-muted/50` background |
| Table row selected (checkbox) | `bg-teal-50` background |
| Table row skipped | `opacity-50` + strikethrough on name |
| Conflict badge click | Opens popover with source comparison dropdown |
| Progress bar animation | Smooth CSS transition on width, 300ms ease |
| Step transition | No page navigation; state-managed steps with content swap |
| OAuth popup | `window.open` centered popup; parent polls for callback completion |
| Retry button | Inline spinner while retrying; success replaces with checkmark |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Step indicator | `aria-current="step"` on active step, `role="navigation"` on container |
| Source cards | `role="checkbox"` with `aria-checked`, keyboard toggle with Space/Enter |
| Preview table | Standard `<table>` with `<thead>` scope attributes, checkbox `aria-label` per row |
| Conflict popover | Focus trap, `aria-expanded`, Escape to dismiss |
| Progress bar | `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax` |
| Skip link | Visible text link, not hidden. Keyboard focusable in tab order after source cards |
| Batch toolbar | `aria-live="polite"` announcing selection count changes |
| Color-only indicators | All status badges include text labels, not color alone |
