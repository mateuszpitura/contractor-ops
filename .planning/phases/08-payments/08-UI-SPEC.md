**shadcn components needed for Phase 8 (new additions beyond Phase 7):**
- `date-picker` -- date range filter for invoice selection and payment run history filtering. Built with shadcn Calendar + Popover.

**Custom components built in Phase 8 (not from any registry):**
- Payment run history table -- TanStack Table with run number, status, date, invoice count, total by currency, export format. Mirrors invoice-table pattern.
- Payment run side panel -- Sheet showing run summary, invoice list preview, status, and quick actions. Mirrors existing side panel pattern.
- New payment run dialog -- multi-step dialog (Select Invoices -> Review -> Lock & Export). Built with shadcn Dialog, Table, Checkbox, Button, Badge, Select.
- Invoice selection table -- TanStack Table with checkboxes for picking invoices into a run. Filters for currency, due date range, contractor. Built with shadcn Table, Checkbox, Select, DatePicker, Input.
- Run review summary -- read-only summary card showing invoice count, totals by currency, run number, export format selector. Built with shadcn Card, Badge, Select, Separator.
- Payment status toolbar -- floating bulk action bar for Mark All Paid, Mark Failed. Mirrors contractor-table bulk action toolbar pattern.
- Bank statement import dialog -- file upload + parsing progress + match results table. Built with shadcn Dialog, Progress, Table, Badge, Button.
- Payment run status badge -- status-colored badge component for DRAFT, LOCKED, EXPORTED, COMPLETED, CANCELLED states.
- Transfer title template editor -- small form in Settings for configuring payment title pattern. Built with shadcn Input, Label, Card.
- Contractor payments tab -- payment history filtered to contractor, replacing TabPlaceholder. Built with shadcn Table, Badge.

**Reused from Phase 1-7 (already installed):**

## Interaction Patterns

### /payments Page (Payment Run History)

- **URL:** /payments. Added to sidebar navigation after "Approvals" and before "Settings".
- **Layout:** Full-width content area. Page title "Payments" as Heading. "New payment run" primary CTA top-right, aligned with title.
- **Filter bar:** Horizontal row of Badge-style chips: All / Draft / Locked / Exported / Completed / Cancelled. Active chip: `--primary` at 10% opacity bg, `--primary` text. Inactive: `--muted` bg, `--muted-foreground` text. Each chip shows count. Filter synced to URL via nuqs `parseAsString` with `?status=exported`.
- **Date range filter:** DatePicker range to the right of status chips. Synced to URL via nuqs.
- **Table:** TanStack Table with server-side pagination (20 per page), sorting (default: created descending). Columns: Run # (semibold, clickable to open side panel), Status (colored badge), Created (relative date with tooltip for absolute), Invoices (count integer), Total (formatted grosze with currency, multiple lines if multi-currency), Format (plain text badge), Actions (dropdown: Download export, Mark all paid, Cancel run -- visibility per D-15 status rules).
- **Row click:** Opens side panel for that run.
- **Loading:** 8 skeleton table rows.
- **Pagination:** Bottom of table. Previous/Next with page indicator.

### New Payment Run Dialog

- **Trigger:** "New payment run" CTA on /payments page.
- **Size:** 720px width, max 80vh height.
- **Step indicator:** 3 dots horizontally centered below title. Active: `--primary` fill. Inactive: `--muted` fill. Completed: `--primary` fill with check icon. 8px dot diameter, 16px gap between dots.

**Step 1 -- Select Invoices:**
- Filter row: Currency Select (All / PLN / EUR / USD), Due date DatePicker range, Contractor search Input (300ms debounce).
- Table: TanStack Table with checkbox column (header checkbox for select-all-on-page). Columns: Checkbox, Invoice # (14px semibold), Contractor, Amount (formatted grosze), Currency, Due date, Contract #.
- "Select all matching" link above table (selects all invoices matching current filters, not just visible page). Shows total count.
- Footer summary: left-aligned "{N} invoices selected -- {amount} {currency}" (updates live). If multi-currency: shows each currency on separate line.
- "Group by currency" Switch toggle. When enabled, system will create one run per unique currency in the selection. Label updates to show "Will create {N} runs".
- Invoices already in another DRAFT run: shown with `--warning` background and "In run {PR-XXXX}" badge. Checkbox disabled.
- Invoices with no IBAN on contractor billing profile: shown with `--warning` background and "Missing IBAN" badge. Selectable but warned.
- "Cancel" ghost CTA closes dialog. "Review selection" primary CTA advances to Step 2 (disabled until >= 1 invoice selected).
- Empty state when no invoices ready: centered `FileSearch` icon (48px), heading "No invoices ready for payment", body text directing to approval queue.

**Step 2 -- Review:**
- Auto-generated run number (PR-{year}-{seq}) displayed prominently: 20px semibold.
- Optional name Input and description Textarea below run number.
- Invoice list grouped by currency. Each group: currency heading with count and subtotal (20px semibold for amount). Below: compact read-only table (Invoice #, Contractor name, Amount). Max 10 rows visible per group, scroll for more.
- Grand total row at bottom with thicker top border separator.
- Export format Select below totals: CSV (universal) / Elixir (PLN domestic, shown only if PLN invoices present) / SEPA XML (EUR, shown only if EUR invoices present).
- "Back" ghost returns to Step 1 preserving selection. "Lock & Export" primary CTA proceeds to lock + generate file.
- While processing: button shows spinner + "Locking..." text. Dialog not closable during lock operation.

**Step 3 -- Confirmation:**
- Success icon: `CheckCircle2` (48px, `--success`) centered.
- Heading: "Payment run {runNumber} locked and exported".
- Summary stats below: invoice count, total by currency, export format.
- "Download export file" primary outline button (triggers browser download).
