  - Reference (payment reference string or "--")
- Data source: Query payment run items filtered by contractorId. Use a new tRPC query or filter client-side. The simplest approach: add a `listByContractor` query to the payment router that accepts contractorId and returns PaymentRunItems with joined run number and invoice number. Alternatively, if the payment router from Plan 01 already supports filtering by contractor via the list query, use that. If neither exists, create a simple prisma query inline:
  ```
  trpc.payment.listByContractor.useQuery({ contractorId })
  ```
  The query should return items: { id, paymentRunId, runNumber, invoiceId, invoiceNumber, amountGrosze, currency, status, paymentReference, markedPaidAt, createdAt }.
- Sorted by createdAt descending (client-side, contractor payment count typically < 100).
- Simple prev/next pagination if > 10 items (client-side slice).
- Empty state: Banknote icon (32px, text-muted-foreground), "No payments for this contractor" heading, body text per UI-SPEC.
- Use useTranslations("Payments") for all strings.

**NOTE:** If the payment tRPC router from Plan 01 does not have a `listByContractor` procedure, add one to `packages/api/src/routers/payment.ts`:
```typescript
listByContractor: tenantProcedure
  .use(requirePermission({ payment: ["read"] }))
  .input(z.object({ contractorId: z.string().cuid() }))
  .query(async ({ ctx, input }) => {
    const items = await ctx.prisma.paymentRunItem.findMany({
      where: { organizationId: ctx.organizationId, contractorId: input.contractorId },
      include: {
        paymentRun: { select: { runNumber: true } },
        invoice: { select: { invoiceNumber: true } },
      },
      orderBy: { createdAt: "desc" },
      take: 100,
    });
    return items.map(plain);
  }),
```
