1. Create `packages/api/src/services/esign-orchestrator.ts`:

**sendForSignature(params):**
- Input: `{ organizationId, userId, contractId?, documentId, connectionId, provider: "DOCUSIGN" | "AUTENTI", signers: { name, email, role, routingOrder }[], message?, expiresInDays?, reminderIntervalDays? }`
- Fetch the Document record to get storageKey, generate presigned download URL, fetch the PDF content, convert to base64
- Call `createSigningEnvelope()` from esign-service with the request
- In a transaction:
  a. Create SigningEnvelope record with all fields, status SENT, sentAt = now
  b. Create SigningRecipient records for each signer (map externalRecipientId from result)
  c. Create SigningEvent with type ENVELOPE_CREATED, description "Signing envelope created", occurredAt = now
  d. Create SigningEvent with type ENVELOPE_SENT, description "Sent to {first signer name} for signature", occurredAt = now
  e. If contractId provided: update Contract.status to "PENDING_SIGNATURE"
  f. Create ExternalLink mapping contractId/documentId to externalEnvelopeId
- Return the created SigningEnvelope with recipients

**getSigningUrl(params):**
- Input: `{ organizationId, envelopeId, recipientEmail, returnUrl }`
- Fetch SigningEnvelope with provider and connectionId
- Call `getEmbeddedSigningUrl()` from esign-service
- If result is null (provider doesn't support embedded), return `{ embedded: false, redirectUrl: null }` — the provider sends email links instead
- Return `{ embedded: true, url: result.url, expiresAt: result.expiresAt }`

**handleSigningCompletion(envelopeId, connectionId, provider):**
- Download signed PDF via `downloadSignedDocument()`
- Generate storage key: `generateStorageKey("esign", "{contractTitle}-signed.pdf")`
- Upload to R2 via presigned URL
- Create Document record with source = "ESIGN", documentType matching original
- Create DocumentLink with entityType "CONTRACT", entityId = envelope.contractId, linkRole "SIGNED_COPY"
- Create SigningEvent with type SIGNED_PDF_SAVED
- Return created Document

**voidEnvelope(params):**
- Input: `{ organizationId, envelopeId, userId, reason }`
- Fetch envelope, call `voidSigningEnvelope()` from esign-service
- Update SigningEnvelope: status = VOIDED, voidedAt = now, voidReason = reason
- If contractId: check if contract was PENDING_SIGNATURE, if so revert to DRAFT (or leave as-is if user prefers — use DRAFT for simplicity)
- Create SigningEvent with type ENVELOPE_VOIDED

**resendToRecipient(params):**
- Input: `{ organizationId, envelopeId, recipientEmail }`
- Fetch envelope, call `resendSigningNotification()` from esign-service
- Create SigningEvent with type ENVELOPE_SENT, description "Resent to {recipientEmail}"

2. Create `packages/api/src/routers/esign.ts`:

Use `router` from init.ts, `tenantProcedure` from tenant middleware, `requirePermission` from rbac middleware.

Procedures:

a. `sendForSignature` — mutation. Input: `{ contractId?: string, documentId: string, connectionId: string, provider: z.enum(["DOCUSIGN", "AUTENTI"]), signers: z.array(z.object({ name: z.string(), email: z.string().email(), role: z.enum(["signer", "countersigner"]), routingOrder: z.number().int().positive() })).min(1), message: z.string().optional(), expiresInDays: z.number().int().min(1).max(90).optional().default(14), reminderIntervalDays: z.number().int().min(1).max(30).nullable().optional() }`. Requires "contract:write" permission. Calls orchestrator.sendForSignature.

b. `getSigningUrl` — query. Input: `{ envelopeId: z.string(), recipientEmail: z.string().email(), returnUrl: z.string().url() }`. Calls orchestrator.getSigningUrl. Generate URL on-demand (never cache — DocuSign URLs expire in 5 min per RESEARCH.md Pitfall 3).

c. `voidEnvelope` — mutation. Input: `{ envelopeId: z.string(), reason: z.string().min(1).max(500) }`. Requires "contract:write". Calls orchestrator.voidEnvelope.

d. `resendToRecipient` — mutation. Input: `{ envelopeId: z.string(), recipientEmail: z.string().email() }`. Requires "contract:write". Calls orchestrator.resendToRecipient.

e. `getEnvelopeDetail` — query. Input: `{ envelopeId: z.string() }`. Fetches SigningEnvelope with recipients and events (ordered by occurredAt desc). Returns full envelope detail.

f. `listEnvelopes` — query. Input: `{ contractId: z.string() }`. Fetches all SigningEnvelopes for a contract, ordered by createdAt desc. Includes recipient summary (count signed/total).

g. `listPendingForContractor` — query (uses portalProcedure if available, or tenantProcedure with contractor context). Input: none. Fetches SigningEnvelopes where a recipient matches the current user's email and status is PENDING or SENT. Returns list for portal "Pending Signatures" section.

1. Create `packages/integrations/src/services/esign-webhook-handler.ts`:

Export `handleSigningWebhook(params: { provider: "DOCUSIGN" | "AUTENTI", payload: unknown, organizationId: string, connectionId: string }): Promise<void>`

Implementation follows RESEARCH.md webhook handler pattern exactly:
a. Call `normalizeSigningEvent(provider, payload)` to get NormalizedSigningEvent
b. Find SigningEnvelope by externalEnvelopeId + organizationId. If not found, throw error (will be re-queued by webhook infrastructure)
c. Idempotency check: query SigningEvent by signingEnvelopeId + providerEventId. If exists, return early.
d. In a prisma.$transaction:
   - Create SigningEvent record with all fields from normalized event
   - If event has recipientEmail: update matching SigningRecipient status. Set signedAt on RECIPIENT_SIGNED, declinedAt on RECIPIENT_DECLINED, viewedAt on RECIPIENT_VIEWED.
   - If event has envelopeStatus: update SigningEnvelope status. Set completedAt on COMPLETED, voidedAt on VOIDED.
   - If envelope has contractId and envelopeStatus is terminal: update Contract.status. Map COMPLETED -> "ACTIVE" (+ set signedAt), DECLINED -> "SIGNATURE_DECLINED", EXPIRED -> "SIGNATURE_EXPIRED".
e. Post-transaction: if envelopeStatus is COMPLETED, call `handleSigningCompletion(envelope.id, connectionId, provider)` from esign-orchestrator (import from api package or inline the logic)
f. Post-transaction: if event is RECIPIENT_DECLINED, ENVELOPE_EXPIRED, or ENVELOPE_COMPLETED, create notification (use existing notification service pattern from Phase 7)
