- Opens SendForSignatureDialog on click (manages dialog open state)
- Per UI-SPEC: primary variant, PenLine icon from lucide-react

2. **SendForSignatureDialog** (`send-for-signature-dialog.tsx`):
- Props: `{ open: boolean; onOpenChange: (open: boolean) => void; contractId?: string; documentId: string; contractParties: { name: string; email: string; role: "signer" | "countersigner" }[] }`
- Uses Dialog component, max-width 640px, ScrollArea body max-height 80vh
- Section 1 — Provider: Label "Signing Provider" + Select (ProviderPicker). Options fetched from `trpc.integration.listConnections` filtered to DOCUSIGN/AUTENTI with status CONNECTED. Each option shows provider icon + name. Disabled options show "Not connected" sublabel with hint "Connect {provider} in Settings > Integrations".
- Section 2 — Signers: Label "Signers" + ordered list. Auto-populated from contractParties prop (contractor first per D-09). Each row: GripVertical drag handle (aria-label="Drag to reorder"), avatar/initials circle, name, email, role label Badge ("Contractor"/"Countersigner"). Use @dnd-kit/sortable for drag reorder. "Add countersigner" link appears if no countersigner. CountersignerPicker: Select from org members via `trpc.user.list` or similar.
- Section 3 — Message: Label "Message to Signers" + Textarea (3 rows, placeholder "Optional message included in the signing email")
- Section 4 — Document: Label "Document" + preview card with muted background, file name, page count
- Section 5 — Options: Two fields side by side (gap-2). "Expires After" Select: 7/14/30/60 days (default 14). "Send Reminders" Select: None/Every 3 days/Every 7 days (default Every 7 days).
- Footer: "Send for Signature" primary button (right) shows Loader2 spinner when loading + "Discard" outline button
- On submit: call `trpc.esign.sendForSignature.useMutation()`. On success: toast "Document sent for signature", close dialog, invalidate queries. On error: toast "Failed to send for signature. Please try again."

3. **SigningProgressBar** (`signing-progress-bar.tsx`):
- Props: `{ envelope: { status: string; recipients: { name: string; email: string; role: string; status: string; routingOrder: number }[] } }`
- Card with p-4. Horizontal layout on lg, stacked on mobile.
- Left: step indicators for each signer ordered by routingOrder. Each step: circle (32px) with initial letter. States per UI-SPEC:
  - PENDING/SENT: muted border, muted-foreground text
  - Current (next to sign): primary border with `animate-pulse` ring, primary text
  - SIGNED: green-600 bg, white CheckIcon
  - DECLINED: red-500 bg, white XIcon
- Connector lines between steps: 2px solid, muted (green for completed segments)
- Right: status text. "Waiting for {name} to sign" or "All parties have signed" or "{signed} of {total} signed"
- "View Signing History" button that opens SigningAuditTrail sheet
- Actions dropdown: "Void Envelope" (destructive), "Resend to {name}" (for pending signers)

4. **SigningStatusBadge** (`signing-status-badge.tsx`):
- Props: `{ status: string }`
- Maps status to Badge variant per UI-SPEC Status Badge Mapping:
  - PENDING_SIGNATURE -> warning variant, label "Pending Signature"
  - SIGNATURE_DECLINED -> destructive variant, label "Signature Declined"
  - SIGNATURE_EXPIRED -> destructive variant, label "Signature Expired"
- For per-signer statuses: SIGNED -> success, AWAITING -> secondary, SENT -> info

- Header: "Signing History" at text-xl font-semibold

- Chronological list (newest first). Each row: 8px vertical padding, border-bottom
  - Left: event icon (16px, mapped per UI-SPEC: Send, Eye, PenLine green, XCircle red, Ban red, CheckCircle2 green, FileDown)
  - Center: event description at text-sm, timestamp at text-sm text-muted-foreground (relative time, full datetime on Tooltip hover)
  - Right: actor name at text-sm text-muted-foreground
- Empty state: "No Signing History" heading + "Signing events will appear here once a document is sent for signature."

6. **EmbeddedSigningModal** (`embedded-signing-modal.tsx`):
- Props: `{ envelopeId: string; recipientEmail: string; documentTitle: string; provider: "DOCUSIGN" | "AUTENTI"; open: boolean; onOpenChange: (open: boolean) => void; onComplete: () => void }`
- Full-viewport overlay (fixed inset-0 z-50 bg-background)
- Top bar: h-14 (56px), border-b, flex justify-between items-center px-4. Left: document title text-sm font-semibold. Right: close button (X icon, ghost variant, 32x32, aria-label="Close signing modal")
- Fetches signing URL from `trpc.esign.getSigningUrl.useQuery({ envelopeId, recipientEmail, returnUrl: window.location.origin + "/signing-complete" })` on open
- If response.embedded = true: render iframe, width 100%, height calc(100dvh - 56px), src = url. Listen for postMessage from DocuSign (events: "signing_complete", "decline", "exception"). On signing_complete: close modal, toast "Document signed successfully", call onComplete.
- If response.embedded = false (Autenti): render centered Card (max-w-[480px]) per UI-SPEC fallback:
  - Provider logo or text "Autenti"

  - "Continue to Autenti" primary Button (opens Autenti signing URL in new tab)
  - "Return to Contract" outline Button (closes modal)
- Loading state: full-height Skeleton with centered spinner

7. **VoidEnvelopeDialog** (`void-envelope-dialog.tsx`):
