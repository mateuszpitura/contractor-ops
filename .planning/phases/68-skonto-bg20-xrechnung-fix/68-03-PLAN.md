<objective>
Wire the Skonto cascade resolution into `finalizeEInvoice` so that finalized DE invoices emit BG-20 Payment Terms when a Skonto term is set on either the invoice or the contractor's default billing profile.

Purpose: Phase 68 D-03 + D-04. The CII-level emission and the `XRechnungDEProfile` wrapper are correct after Plan 02 — the remaining gap is that `loadInvoiceWithRelations` doesn't fetch the Skonto records and `finalizeEInvoice` never resolves the effective term. This plan extends the Prisma include to fetch `invoice.skontoTerms[0]` + `contractor.billingProfiles[0].skontoTerms[0]`, calls the existing single-source-of-truth resolver `services/skonto.ts:resolveSkontoTerm` (Phase 63 D-21), maps Prisma rows to `SkontoTermData` via an inline helper (per RESEARCH.md Pitfall 2 — premature DRY avoided), and forwards the resolved term as `opts.skontoTerm` into `profile.generateAndValidate`.

Output:
- Extended `loadInvoiceWithRelations` Prisma include matching the `payment.ts:1213-1222` shape verbatim.
- Inline `toSkontoTermData(prismaRow)` helper that does `Number(row.discountPercent)` + plucks the three structural fields.
- Cascade-resolution block in `finalizeEInvoice` body immediately after the Leitweg-ID resolution and before `profile.generateAndValidate`.
- `{ leitwegId, skontoTerm: effectiveSkonto }` opts passed to `profile.generateAndValidate`.
- Layer B test (extending `einvoice-finalize.test.ts`) with three cascade-branch assertions.
</objective>

```ts
/**
 * Map a Prisma `SkontoTerm` row (or null) into the structural shape consumed
 * by `services/skonto.ts:resolveSkontoTerm`.
 *
 * Per Phase 68 D-04 + RESEARCH.md Pitfall 2 — kept inline (NOT extracted to
 * services/skonto.ts) because there are exactly three call sites today
 * (this file, `routers/payment.ts:1239-1253`, and `routers/einvoice.ts`
 * `generateZugferdPdf` after Plan 05). Extracting a helper for two new
 * callers is premature DRY.
 *
 * Per RESEARCH.md Pitfall 3 — `Number(row.discountPercent)` is REQUIRED:
 * the Prisma column is Decimal; the cascade resolver expects a number.
 * Failing to coerce produces `#PROZENT=Decimal(3.00)#` instead of
 * `#PROZENT=3.00#` in the emitted CII XML.
 */
function toSkontoTermData(
  row: { discountPercent: unknown; discountPeriodDays: number; netPeriodDays: number } | null
    | undefined,
): SkontoTermData | null {
  if (!row) return null;
  return {
    discountPercent: Number(row.discountPercent),
    discountPeriodDays: row.discountPeriodDays,
    netPeriodDays: row.netPeriodDays,
  };
}
```
