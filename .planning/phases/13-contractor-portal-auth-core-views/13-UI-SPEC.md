## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, form field gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card content spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical rhythm |
| 3xl | 64px | Hero/centered layout vertical padding |

Exceptions: Mobile touch targets minimum 44px height for all tappable elements (buttons, nav links, list rows). Portal top bar height: 56px (14 * 4).

## Layout Contract

### Portal Top Bar (D-01)
- Height: 56px
- Left: Org logo (32px square, rounded-md) + org name (14px semibold)
- Center: Nav links -- Overview, Contracts, Invoices, Documents, Payments (14px medium, 24px horizontal gap)
- Right: Contractor name + avatar (32px) + dropdown (logout)
- Active nav: Indigo underline (2px) below active link
- Mobile (<768px): Logo + hamburger icon. Sheet slides from right with full nav + profile section
- Background: `--card` with bottom border `--border`

### Overview Dashboard (D-02)
- Greeting: "Welcome back, {firstName}" at 28px semibold
- Summary cards: 2x2 grid on desktop, single column on mobile. Each card: Lucide icon (20px, muted-foreground) + label (13px, muted-foreground) + value (28px, semibold)
- Cards: Active Contracts (FileText icon), Pending Invoices (Clock icon), Recent Payments (Banknote icon), Upcoming Deadlines (CalendarDays icon)
- Quick actions below cards: "Submit Invoice" (primary button) + "View Contracts" (outline button)
- Below quick actions: Recent activity log (last 5 entries) with "View all" link

### Invoice List (D-10)
- Table layout on desktop (>768px): Invoice Number, Contract, Amount (gross), Date Submitted, Status badge, Actions (view)
- Card layout on mobile: stacked cards with invoice number + amount prominent, status badge, date secondary
- Status badges use semantic colors: Submitted (info/blue), In Review (warning/amber), Approved (success/green), Payment Scheduled (success/green outline), Paid (success/green solid), Rejected (destructive/red)
- Sort by date submitted (newest first), no filtering in Phase 13 (Phase 14 scope)

### Invoice Detail (D-10, D-11, D-12)
- Top section: Invoice number (20px heading) + status badge + submission date
- StatusTimeline: horizontal 5-step progress indicator below heading
- Details card: contract name, amounts (net/gross), dates (issue/due), PDF attachment (downloadable)
- Activity log card below: scrollable list of contractor-visible events with timestamps
- Payment section (when paid): payment date + amount. No batch IDs, no org bank details

### Invoice Submission (D-06, D-07, D-08, D-09)
- Single-page form (not wizard steps, but logical sections with separators)
- Section 1: Contract selection -- Select dropdown of active contracts. Auto-select if only 1. Shows expected amount hint when contract selected
- Section 2: Upload -- DropZone for single PDF. Shows filename + size after upload. Progress bar during upload
- Section 3: Metadata -- Invoice number (text), Issue date (date input), Due date (date input), Net amount (number with currency), Gross amount (number with currency)
- Section 4: Review summary before submit (collapsible, default open)
- Submit button: "Submit Invoice" (primary, full-width on mobile)
- After submit: redirect to SubmissionSuccess page

### Contracts List (D-02, PORT-02)
- Card grid (2 columns desktop, 1 column mobile)
- Each card: Contract title (16px medium), contractor role, date range, monthly rate, status badge (Active/Expired/Pending)
- Click card to view contract detail (read-only)

### Contract Detail (PORT-02)
- Read-only view: contract title, parties, dates, rate, payment terms, scope description
- Document download section: attached files (contract PDF, amendments)
- Back navigation to contracts list

### Documents List (PORT-05)
- Table layout: Document name, Type (Contract/NDA/Tax Form), Date Added, Actions (download)
- Type column uses badge with neutral styling
- Download button per row (outline variant, Download icon)

### Payments List (PORT-04, D-12)
- Table layout: Invoice Number, Amount, Payment Date, Status
- Minimal columns -- no internal IDs exposed
- Click row to navigate to associated invoice detail

### Portal Login (D-16)
- Centered card (max-width 400px) on muted background
- Heading: "Contractor Portal" at 28px semibold
- Subheading: "Enter your email to receive a sign-in link" at 14px muted
- Email input + "Send Magic Link" primary button (full-width)
- Below form: "Are you an admin?" link to admin login

### Org Picker (D-15)
- Shown after magic link verification when contractor has 2+ orgs
- Centered layout (max-width 480px)
- Heading: "Select Organization" at 20px semibold
- Card per org: org logo (40px) + org name (16px medium) + contractor role (14px muted). Hover state with accent border
- Click card to enter portal for that org

## Status Timeline Steps

Horizontal step indicator with 5 steps:

1. **Submitted** -- Circle + label. Filled with `--info` when active/past
2. **In Review** -- Circle + label. Filled with `--warning` when active, `--success` when past
3. **Approved** -- Circle + label. Filled with `--success` when active/past
4. **Payment Scheduled** -- Circle + label. Filled with `--success` when active/past
5. **Paid** -- Circle + label. Filled with `--success` when active/past

Connector lines between steps: solid `--border` for future, solid `--success` for completed. Active step pulses subtly (ring animation). Rejected state: step 2 turns `--destructive` with X icon, remaining steps stay muted.

Mobile: Steps stack vertically with connector lines on the left (vertical timeline).
