- `skeleton` — loading states for all data views
- `card` — settings sections, auth forms

- `switch` — dark mode toggle, notification defaults
- `breadcrumb` — top bar navigation context
- `tooltip` — collapsed sidebar icon labels, action button hints

## Color

Uses shadcn CSS custom properties for light/dark mode support. All colors defined as HSL values in `globals.css`.

| Role | Light Mode | Dark Mode | Usage |
|------|------------|-----------|-------|
| Dominant (60%) | `--background: 0 0% 100%` (white) | `--background: 240 10% 3.9%` (near-black) | Page background, content area |
| Secondary (30%) | `--muted: 240 4.8% 95.9%` (cool gray) | `--muted: 240 3.7% 15.9%` (dark gray) | Sidebar background, card backgrounds, table header rows, input backgrounds |
| Accent (10%) | `--primary: 238 73% 56%` (Indigo-600) | `--primary: 238 73% 67%` (Indigo-400) | See reserved list below |
| Destructive | `--destructive: 0 84% 60%` (Red-500) | `--destructive: 0 62% 54%` (Red-400) | Deactivate user button, error toasts, validation errors |

Accent reserved for:
- Primary CTA buttons (Create organization, Sign in, Send invite)
- Active sidebar navigation item indicator (left border or background)
- Active breadcrumb segment
- Links in body text
- Focus ring on form inputs (`ring-primary`)
- Toggle/switch active state
- Badge for "Active" status

All other interactive elements use `--foreground` or `--muted-foreground` with hover states using `--accent` (subtle background shift).

### Additional Semantic Colors

| Token | Value | Usage |
|-------|-------|-------|
| Success | `--success: 142 71% 45%` (Green-600) | Success toasts, active status dot |
| Warning | `--warning: 38 92% 50%` (Amber-500) | Expiring status, SLA warnings (future phases) |
| Info | `--info: 217 91% 60%` (Blue-500) | Informational banners, tooltips |

## Interaction Patterns

### Loading States
- **Page-level:** Full skeleton matching the target layout (sidebar skeleton renders instantly, content area shows skeleton cards/tables matching the expected shape).
- **Table loading:** Skeleton rows (5 rows) with column-width-matched gray bars.
- **Button loading:** Spinner replaces button text, button disabled, width preserved (no layout shift).
- **Form submission:** Primary CTA shows spinner + "Saving..." text, all form fields disabled.

### Dark Mode
- System preference detection on first visit via `prefers-color-scheme`.
- Manual toggle in Settings > General (stored in Zustand, persisted to `localStorage`).
- CSS class strategy: `dark` class on `<html>` element. All color tokens defined as CSS custom properties in `:root` and `.dark`.
- No flash of wrong theme: inline script in `<head>` reads `localStorage` before paint.

### Information Density
- Two modes: **comfortable** (default) and **compact**.
- Comfortable: 16px table cell padding, 40px row height, 14px body text.
- Compact: 8px table cell padding, 32px row height, 13px body text.
- Toggle in Settings > General. Stored in Zustand, persisted to `localStorage`.
- Applied via CSS class `density-compact` on `<html>`, with Tailwind utility overrides.

### Sidebar Behavior
- Default: expanded (240px) on viewports >= 1280px, collapsed (48px) on 1024-1279px.
- User can toggle manually; preference persisted to `localStorage`.
- Collapse animation: 200ms ease-out transition on width.
- Collapsed state: icons only, tooltips on hover showing label.
- Below 1024px: sidebar hidden, accessible via hamburger menu (Sheet component).

### Auth Flow Transitions
- Register: form -> loading spinner on CTA -> redirect to email verification screen -> after verification, redirect to dashboard.
- Login: form -> loading spinner on CTA -> redirect to dashboard (or last visited page).
- Invite: landing page with org name and inviter -> form (pre-filled email) -> loading -> redirect to dashboard.
- Social auth: button click -> redirect to provider -> callback -> redirect to dashboard.

### Responsive Breakpoints
- Desktop: >= 1280px (sidebar expanded by default)
- Tablet landscape: 1024-1279px (sidebar collapsed by default)
- Below 1024px: not a primary target, but sidebar hidden behind hamburger, content fills viewport.
