## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, channel mapping row vertical padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, form group gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions: Channel mapping rows use 8px (py-2) vertical padding for compact density, consistent with notification preferences table row density.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | text-sm (clamp 13-14px) | 400 (regular) | 1.25rem |
| Label | text-xs (clamp 11-12px) | 400 (regular) | 1rem |
| Heading | text-lg (clamp 16-18px) | 600 (semibold) | 1.75rem |
| Display | text-2xl (clamp 22-30px) | 600 (semibold) | 2.25rem |

Notes: Channel names in the mapping table use Body size. Notification category labels use Body size with `font-semibold` (600). The "channelTeams" column header in notification preferences uses Label size, matching existing "channelEmail" and "channelSlack" headers.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` oklch(0.985 0.003 85) | Page background, settings surfaces |
| Secondary (30%) | `--card` oklch(1 0 0) | Provider card, channel mapping card, notification preferences table |
| Accent (10%) | `--primary` oklch(0.44 0.145 178) | "Connect" CTA, "Save" CTA for channel mapping, active toggles |
| Destructive | `--destructive` oklch(0.58 0.23 18) | "Disconnect" action only |

Accent reserved for: "Connect Microsoft Teams" button, "Save channel mapping" button, active Switch toggles in notification preferences, "Refresh channels" icon button active state.

### Connection Status Badge Colors

| Status | Variant | Rationale |
|--------|---------|-----------|
| Connected | `bg-emerald-500/10 text-emerald-500` | Active integration, matches existing ProviderConnectionCard pattern |
| Disconnected | `bg-muted text-muted-foreground` | Inactive, matches existing pattern |
| Error | `bg-destructive/10 text-destructive` | Broken connection, matches existing pattern |
| Reauth Required | `bg-amber-500/10 text-amber-500` | Token expired, matches existing pattern |

### Adaptive Card Colors (Teams-native, not web UI)

Adaptive Cards use Teams' built-in theming. No custom hex values. Specific styling:
- Approval card: `attention` style for amount, `good` container style for approved result, `attention` container style for rejected result
- Alert card: `default` style, compact layout
- Reminder card: `warning` style container header for overdue duration

## Interaction Contracts

### Connection Flow

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Connect | "Connect Microsoft Teams" button | OAuth redirect to Azure AD consent screen | On return: provider card shows "Connected", channel mapping section appears below |
| Disconnect | "Disconnect" button | AlertDialog confirmation | Sonner toast "Microsoft Teams disconnected", card reverts to disconnected state, Teams column in notification prefs disables |

### Channel Mapping Flow

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Load channels | Section mounts (Teams connected) | Channel list fetched via Graph API | Skeleton dropdowns during loading, populate with channel names on load |
| Refresh channels | Refresh icon button | Re-fetches channel list from Graph API | Button shows Loader2 spinner, dropdowns repopulate |
| Select channel | Select dropdown per notification category | Stores selection locally | No immediate save -- user commits with "Save mapping" |
| Save mapping | "Save mapping" button | Mutation to tRPC endpoint | Button shows Loader2 spinner, Sonner toast "Channel mapping saved" on success |
| No channels | Bot not installed in any team | Empty state in dropdown area | "No channels available" message with guidance text |

### Notification Preferences (Teams column)

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Toggle Teams | Switch component in Teams column | Toggles `channelTeams` boolean for that notification type | Dirty form, saved with existing "Save preferences" button |
| Teams not connected | Teams integration absent | Switch disabled with reduced opacity | Tooltip on hover: "Connect Microsoft Teams in Integrations to enable Teams notifications." |

### Adaptive Card Interactions (Teams-native)

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Approve | "Approve" Action.Submit on card | Direct submit, no confirmation needed | Card replaces in-place with ApprovalResultCard showing "Approved" + approver name |
| Reject | "Reject" Action.Submit on card | Opens task module (modal) for mandatory comment | After submit: card replaces with ApprovalResultCard showing "Rejected" + approver name + reason |
| View in app | "View in Contractor Ops" Action.OpenUrl | Opens web app at relevant resource URL | Standard browser navigation |
| Stale card action | Action on already-decided invoice | Server rejects with 409 | Card updates to show current status: "This invoice was already {status} by {name}." |
| Auth failure | Action by user without approval permission | Server rejects with 403 | Card shows error: "You don't have permission to approve this invoice." |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Desktop (>1024px) | Full channel mapping with inline dropdowns, notification preferences table with all 3 channel columns |
| Tablet (768-1024px) | Channel mapping unchanged (simple layout), notification preferences unchanged |
| Mobile (<768px) | Channel mapping stacks label above dropdown; notification preferences table scrolls horizontally (existing behavior) |

Adaptive Cards are inherently responsive via Teams' built-in card renderer -- no custom breakpoints needed.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Select dropdowns navigable via arrow keys; Tab moves between mapping rows; AlertDialog traps focus |
| Screen reader | Channel mapping uses `aria-label="{category} notification channel"` per Select; disabled Teams switches have `aria-disabled="true"` with tooltip text as `aria-label` |
| Focus visible | All interactive elements show `ring` on focus (project default via `outline-ring/50`) |
| Color contrast | Status badges maintain 4.5:1 text contrast, matching existing ProviderConnectionCard pattern |
| Disabled states | Disabled Switch uses `opacity-50` + `cursor-not-allowed`, tooltip provides explanation |
