2. **TeamsAdapter** (`packages/integrations/src/adapters/teams-adapter.ts`):
   Per D-12, extend BaseAdapter following the LinearAdapter pattern. Per checker issue #5, use `credential-service.ts` for AES-256-GCM encryption of stored tokens (same pattern as DocuSignAdapter and AutentiAdapter — import `decryptCredentials` from `../services/credential-service.js`):
   ```typescript
   export class TeamsAdapter extends BaseAdapter {
     readonly slug = "microsoft_teams"; // underscore per Phase 31 convention -> MICROSOFT_TEAMS enum
     readonly displayName = "Microsoft Teams";
     readonly supportsOAuth = true;
     readonly supportsWebhooks = false; // Bot Framework handles its own messaging, not generic webhooks
   }
   ```
   - `getOAuthConfig()`: Return OAuthConfig with:
     - `clientIdEnvVar: "AZURE_BOT_APP_ID"`
     - `clientSecretEnvVar: "AZURE_BOT_APP_SECRET"`
     - `authorizationUrl: "https://login.microsoftonline.com/common/oauth2/v2.0/authorize"`
     - `tokenUrl: "https://login.microsoftonline.com/common/oauth2/v2.0/token"`
     - `scopes: ["https://graph.microsoft.com/Team.ReadBasic.All", "https://graph.microsoft.com/Channel.ReadBasic.All", "https://graph.microsoft.com/User.Read", "offline_access"]`
     - `redirectPath: "/api/oauth/microsoft_teams/callback"`
   - `exchangeCodeForTokens(code, redirectUri)`: Standard OAuth2 token exchange using `fetch` to tokenUrl with `grant_type=authorization_code`, client_id, client_secret, code, redirect_uri, scope. Return CredentialBlob with `accessToken`, `refreshToken`, `expiresIn`, `scope`, `tokenType`.
   - `refreshToken(credentials)`: Use `decryptCredentials` from credential-service.ts to decrypt stored credentials before refreshing. Fetch tokenUrl with `grant_type=refresh_token`. Return updated CredentialBlob.
   - `getHealthStatus(connectionId)`: Query `prisma.integrationConnection.findUnique` for lastSuccessAt/lastErrorAt/status. Return `{ status: connection.status, lastChecked: connection.lastSyncAt ?? connection.connectedAt, message }`.
