## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, KPI card internal gaps, upgrade inline banner vertical padding |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, form group gaps, upgrade banner horizontal padding |
| xl | 32px | Layout gaps between KPI card row and detail sections |
| 2xl | 48px | Page-level section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions: Upgrade inline banner uses 8px vertical padding (compact form factor to not dominate the gated section; uses the standard `sm` token). KPI cards use equal 16px internal padding on all sides.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | text-sm (clamp 13-14px) | 400 (regular) | 1.25rem |
| Label | text-xs (clamp 11-12px) | 400 (regular) | 1rem |
| Heading | text-lg (clamp 16-18px) | 600 (semibold) | 1.75rem |
| Display | text-2xl (clamp 22-30px) | 600 (semibold) | 2.25rem |

Notes: KPI card values (plan name, seat count number, credit count, billing date) use text-2xl font-semibold. KPI card labels use text-xs text-muted-foreground. Progress bar percentage labels use text-sm. Upgrade banner text uses text-sm. Feature name in upgrade prompt uses font-semibold inline.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` oklch(0.985 0.003 85) | Page background, usage dashboard background |
| Secondary (30%) | `--card` oklch(1 0 0) | KPI cards, plan comparison cards, upgrade banner surface, credential setup cards |
| Accent (10%) | `--primary` oklch(0.44 0.145 178) | Upgrade CTA buttons, active plan highlight ring, progress bar fill (healthy state) |
| Destructive | `--destructive` oklch(0.58 0.23 18) | Cancel subscription destructive action, exhausted credit bar |

Accent reserved for: "Upgrade Plan" button in upgrade banners, "Create shipment" button (carrier forms), active plan card border in plan comparison, progress bar fill at healthy credit levels, "Save credentials" button in carrier setup, "Buy credits" button.

### Credit Progress Bar Color Thresholds

| Credits Remaining | Bar Color | CSS Variable |
|-------------------|-----------|--------------|
| >50% remaining | `--success` oklch(0.55 0.17 150) | Green |
| 20-50% remaining | `--warning` oklch(0.70 0.16 65) | Yellow |
| <20% remaining | `--destructive` oklch(0.58 0.23 18) | Red |

### Upgrade Banner Color

Upgrade inline banners use `--primary` at 5% opacity as background with `--primary` left border (4px). The Gem icon uses `--primary` color. This matches the existing `PremiumBadge` component pattern from Phase 28.

- KPI cards row: 4 cards in `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4`

### Upgrade Inline Banner (replaces gated UI element)

```
+--------------------------------------------------+
| [Gem]  Time tracking requires Pro. [Upgrade Plan]|
+--------------------------------------------------+
```

- Full-width within the parent container where the gated feature would appear
- Left border 4px `--primary`, background `--primary` at 5% opacity
- Vertical padding 8px (`sm` token), horizontal padding 24px (`lg` token)

- Copy pattern: "{Feature name} requires {Tier}." -- always a single sentence

## Interaction Contracts

### Feature Gating

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Page-level gate (middleware) | User navigates to gated page without required tier | Redirect to `/settings/billing` with `?upgrade={feature}` query param | Billing page opens with upgrade context banner at top |
| Inline gate (tRPC) | User sees gated sub-feature within accessible page | UpgradeInlineBanner replaces the gated section | Banner visible immediately, no loading state |
| Upgrade from banner | Click "Upgrade Plan" on inline banner | Navigate to `/settings/billing` | Billing tab opens with plan comparison |
| Mid-session tier change | Org tier changes (downgrade, trial expiry) | Next action against gated feature triggers the gate (D-03) | Upgrade banner appears on next load/action |

### Usage Dashboard

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| View dashboard | Navigate to Settings > Billing | KPI cards load with skeleton states | Cards populate with live data |
| Buy more credits | Click "Buy more" on OCR credits card | TopUpDialog opens (existing) | Stripe checkout flow |
| View plan details | Scroll to plan comparison section | Current plan highlighted with accent border | Upgrade/downgrade CTAs on other plans |
| Manage billing | Click "Manage billing" | Stripe portal opens in same window | Redirect to Stripe hosted portal |

### Carrier Shipment Form

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Open form | "Create shipment" on equipment detail | CarrierShipmentForm dialog opens with carrier dropdown | Only configured carriers shown |
| Select carrier | Choose from carrier dropdown | Form fieldset swaps to carrier-specific fields | Smooth transition, previous field values cleared |
| Select InPost | Choose InPost in dropdown | InPost fieldset appears (Paczkomat picker + size) | Reuses existing InPost flow |
| Select DPD | Choose DPD in dropdown | DPD fieldset appears (address + size) | Address fields empty, ready for input |
| Select UPS | Choose UPS in dropdown | UPS fieldset appears (address + size + service type) | Address fields empty, service defaults to Standard |
| Submit shipment | Click "Create shipment" | Loading spinner on button, form disabled | Sonner toast "{Carrier} shipment created", dialog closes, timeline updates |
| Validation error | Submit with missing required fields | Inline field-level errors appear | Fields highlighted with destructive border, first error scrolled to |

### Carrier Credential Setup

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Enter credentials | Type in credential fields | Standard input behavior, password masked | Show/hide toggle on credential fields |
| Test connection | Click "Test connection" | Loading spinner on button | Sonner toast: "Connection verified" (success) or "Connection failed" (error) |
| Save credentials | Click "Save credentials" | Loading spinner on button | Sonner toast: "Credentials saved successfully", status badge changes to "Connected" |
| Remove credentials | Clear fields and save | Confirmation dialog | Status reverts to "Not configured", carrier removed from shipment dropdown |

### Seat Count Auto-Sync

| Action | Trigger | Interaction | Feedback |
|--------|---------|-------------|----------|
| Add contractor | Create new contractor via admin UI | Seat count syncs to Stripe automatically (fire-and-forget) | Usage dashboard reflects new count on next load |
| Archive contractor | Archive/deactivate contractor | Seat count syncs to Stripe automatically (fire-and-forget) | Usage dashboard reflects reduced count on next load |
| Overage display | Active contractors exceed plan included seats | Seat count card shows overage with additional billing info | "{overage} additional seats billed at {price}/seat" |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Carrier dropdown navigable with arrow keys; upgrade banner "Upgrade Plan" button focusable via Tab; credential fields follow standard tab order |
| Screen reader | KPI cards use `aria-label` describing the metric (e.g., "OCR credits: 73 of 100 used"); progress bars use `aria-valuenow`, `aria-valuemin`, `aria-valuemax`; upgrade banners use `role="status"` |
| Focus visible | All interactive elements show `ring` on focus (project default via `outline-ring/50`) |
| Color contrast | Credit progress bar colors all meet 4.5:1 against card background; upgrade banner text meets 4.5:1 against tinted background |
| Reduced motion | No custom animations introduced; progress bar color changes are instant (no transition) |
| Feature gate announcements | Upgrade inline banners include `aria-live="polite"` so screen readers announce the gate when it appears |
