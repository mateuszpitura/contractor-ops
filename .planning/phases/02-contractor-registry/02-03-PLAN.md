**Step A: Create the profile page (contractors/[id]/page.tsx)**

Server component shell that receives `params.id` and renders the client component:
- Breadcrumb: Contractors / {contractor name}
- Uses `useQuery(trpc.contractor.getById.queryOptions({ id }))` to fetch contractor
- Loading state: header skeleton (name bar 200px + 2 badge placeholders) + tab bar renders immediately + skeleton content
- Error state: "Could not load contractor profile. Try again." with retry button
- 404 state: "Contractor not found" with link back to /contractors
- Layout: full-width header, then flex row with main content (flex-1) and right rail (280px, sticky)
- Below 1024px: right rail stacks below main content

**Step B: Create profile-header.tsx**

Full-width header component:
- Contractor displayName in Heading (20px semibold)
- Status badge (lifecycle stage) with colors from UI-SPEC:
  - DRAFT: bg-muted text-muted-foreground border-border
  - ONBOARDING: bg-blue-500/10 text-blue-500
  - ACTIVE: bg-green-600/10 text-green-600
  - OFFBOARDING: bg-amber-500/10 text-amber-600
  - ENDED: bg-muted text-muted-foreground border-border
- Type badge: bg-secondary text-secondary-foreground (per UI-SPEC: no color differentiation by type)
- Owner avatar + name (Avatar component with fallback initials)
- Action buttons right-aligned in a DropdownMenu:
  - Primary visible actions: "Edit contractor", "Add contract" (disabled, tooltip "Coming in Phase 3")
  - Overflow menu items: "Upload invoice" (disabled, "Coming in Phase 5"), "Start onboarding" / "Start offboarding" / "Mark as inactive" (lifecycle actions, conditionally shown based on current stage)
  - Lifecycle action handlers: call `trpc.contractor.updateLifecycleStage.mutationOptions`. Legal transitions per current stage:
    - DRAFT: show "Start onboarding" (-> ONBOARDING)
    - ONBOARDING: show "Activate" (-> ACTIVE)
    - ACTIVE: show "Start offboarding" (-> OFFBOARDING), "Mark as inactive" (-> ENDED)
    - OFFBOARDING: show "Complete offboarding" (-> ENDED)
    - ENDED: show "Archive" (calls archive mutation)
  - On success: toast, invalidate query. On error: toast error.
- All labels via `useTranslations("ContractorProfile")`
