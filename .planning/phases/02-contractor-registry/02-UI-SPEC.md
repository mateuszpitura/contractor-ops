## Spacing Scale

Inherited from Phase 1. No changes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, table cell padding (compact mode) |
| md | 16px | Default element spacing, form field gaps, table cell padding (comfortable mode) |
| lg | 24px | Section padding within cards, sidebar section gaps, wizard step content padding |
| xl | 32px | Card padding, layout gaps between major sections, profile header padding |
| 2xl | 48px | Page horizontal padding (desktop) |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- Side panel width: 480px (40% of 1280px viewport, capped)
- Side panel animation: 200ms ease-out slide from right
- Profile tab content min-height: 400px (prevents layout jump between tabs)
- Wizard step content min-height: 320px (consistent step container height)
- Compliance health badge: 8px horizontal padding, 4px vertical padding (pill shape)
- Table toolbar height: 48px (search + filter bar)

Inherited from Phase 1. All color tokens from `globals.css` apply. Phase 2 adds usage patterns only.

| Role | Token | Usage in Phase 2 |
|------|-------|-------------------|
| Dominant (60%) | `--background` | Page background, table background, profile content area |
| Secondary (30%) | `--muted` | Table header row, side panel background, card backgrounds on profile, filter bar background |
| Accent (10%) | `--primary` | See reserved list below |
| Destructive | `--destructive` | Archive contractor button, delete draft confirmation |

**Accent (`--primary`) reserved for in Phase 2:**
- "Add contractor" primary CTA button (page-level and top bar)
- "Save" / "Next step" buttons in wizard
- Active tab indicator on profile page
- Row hover + selected row background tint (subtle `--primary` at 5% opacity)
- Compliance health "green" checkmark icons (uses `--success` not `--primary`)
- Links in contractor profile (e.g., email, website)
- Focus ring on all interactive elements

**Compliance health semantic colors:**
| Health State | Token | Badge Color | Usage |

| Red (critical) | `--destructive` | Red-500 bg at 10% opacity, Red-500 text | Critical docs missing, expired contract, critical overdue |

## Interaction Patterns

### Contractor List Table

- **Column visibility:** Dropdown menu (triggered by columns icon button) with checkboxes for each column. Persisted per user via `localStorage`.
- **Sorting:** Click column header to toggle ascending/descending/none. Single-column sort only. Default: created date descending.
- **Row selection:** Checkbox in first column. Select-all checkbox in header selects current page only. Bulk action bar appears above table when rows selected.
- **Row click:** Opens side panel (Sheet from right, 480px). Does NOT navigate away from the list.
- **Search:** Debounce 300ms, minimum 2 characters. Search triggers server-side PostgreSQL full-text search. Loading indicator in search input during query.
- **Filters:** Popover panels for each filter. Multi-select for status, owner, team, billing model. Date range picker for contract end date. Single-select for compliance health. Active filters shown as badges below the filter bar with "x" to remove individual filters. "Clear all" link when any filter active.
- **Pagination:** Bottom of table. Page size selector (10, 25, 50). Previous/Next buttons. "Page X of Y" indicator.
- **Bulk actions:** Toolbar appears above table when 1+ rows selected. Contains: assign owner (opens select popover), export contractors (dropdown: CSV/XLSX), archive (confirmation dialog), launch workflow (opens workflow picker).
- **Export:** Downloads file immediately via browser. Toast confirms "Exported {count} contractors".

### Side Panel

- **Open:** Slide from right, 200ms ease-out. Backdrop overlay at 20% opacity.
- **Close:** Click backdrop, press Escape, or click X button.
- **Content:** Contractor name (Heading), status badge, type badge. Compact card with key fields: NIP, email, billing model, rate, owner, team. Recent activity list (last 5 items). "Open full profile" CTA at bottom.

### Add Contractor Wizard

- **Trigger:** "Add contractor" button on list page or top bar quick action. Opens as Dialog (not full page).
- **Step navigation:** Progress indicator at top showing 3 steps. Completed steps show checkmark. Current step highlighted with `--primary`. Future steps grayed out.
- **Step validation:** Each step validates on "Next" click. Invalid fields show inline error below the field. Step does not advance until valid.
- **GUS autofill:** User enters NIP, clicks "Fetch from GUS". Button shows spinner during fetch. On success: auto-fills legal name, address, REGON. Fields are editable after autofill. On failure: toast with error, fields remain empty for manual entry.
- **Dismiss:** If no fields have been modified, the dismiss button reads "Close". If any field has been modified, the button reads "Discard changes" and triggers a confirmation dialog ("Discard changes? All entered data will be lost." with "Discard changes" as the destructive CTA and "Keep editing" as the secondary action).
- **Success:** Dialog closes, toast "Contractor added", list refreshes, new contractor appears at top (if sorted by created date).

### Contractor Profile Page

- **URL:** `/contractors/{id}` (slug-based routing with numeric ID)
- **Header:** Full-width. Contractor name (Heading), status badge, type badge, owner avatar + name. Action buttons right-aligned in a dropdown menu (primary actions visible, secondary in overflow).
- **Tabs:** Horizontal tab bar below header. Active tab underlined with `--primary` (2px bottom border). Tab content below. URL updates to `/contractors/{id}?tab={tabName}` for deep linking.
- **Placeholder tabs:** Contracts, Documents, Workflows, Invoices, Payments tabs show centered placeholder with icon, "Coming in Phase X" heading, and brief description.
- **Right rail:** Sticky (position: sticky, top: 80px). 280px width on desktop. Below 1024px, right rail collapses below the main tab content.
- **Activity timeline:** Vertical timeline with dots. Each entry: icon + action text + relative timestamp ("2 hours ago"). Shows contractor creation, status changes, profile edits.

### Compliance Health Card (Overview Tab)

- **Layout:** Card with health badge (green/yellow/red pill) in top-right. Four checklist items below, each with status icon (checkmark/warning/x) and label.
- **Each item:** Green checkmark for passing, yellow warning triangle for non-critical, red X for critical.
- **Click item:** Navigates to relevant tab or section (e.g., clicking "Documents complete?" switches to Compliance tab).

### Loading States

- **Contractor list:** 8 skeleton rows with column-width-matched bars. Search bar and filter bar render immediately.
- **Side panel:** Skeleton card with 6 field placeholders.
- **Profile page:** Header skeleton (name bar + badge placeholders) + tab bar renders immediately + skeleton content for active tab.
- **Wizard:** No skeleton needed — form renders empty.
- **GUS fetch:** Spinner on the "Fetch from GUS" button. NIP field disabled during fetch.

### Responsive Behavior

- **>= 1280px:** Full table with all visible columns. Side panel 480px. Profile: main content + 280px right rail.
- **1024-1279px:** Table with horizontal scroll if needed. Side panel 400px. Profile: right rail 240px.
- **< 1024px:** Table shows condensed view (name, status, health only). Tap row navigates to profile (no side panel). Profile: right rail below main content. Wizard dialog becomes near-full-screen.
