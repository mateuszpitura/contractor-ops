> Visual and interaction contract for Workflow Engine. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialized in Phase 1) |
| Preset | base-nova with Zinc neutral + Indigo accent override |

**shadcn components needed for Phase 4 (new additions beyond Phase 3):**
- None. All required primitives already installed.

**Custom components built in Phase 4 (not from any registry):**
- Sortable task list — drag-to-reorder vertical list using dnd-kit (or equivalent). Built with shadcn Card as task card container.
- Condition rule builder — inline AND/OR rule editor using shadcn Select, Input, Button, and Badge primitives.
- Progress checklist — vertical task checklist with status icons, inline actions, and expandable detail sections.
- Template picker dialog — searchable list of active templates, opened from multiple entry points.

**Reused from Phase 1, 2, and 3 (already installed):**
- `table`, `badge`, `card`, `dialog`, `sheet`, `button`, `input`, `label`, `select`, `skeleton`, `tabs`, `avatar`, `dropdown-menu`, `tooltip`, `sonner`, `separator`, `breadcrumb`, `command`, `checkbox`, `popover`, `progress`, `scroll-area`, `form`, `textarea`, `alert-dialog`, `collapsible`

## Spacing Scale

Inherited from Phase 1. No changes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding, task status icon margin |
| sm | 8px | Compact element spacing, task card internal padding between icon and text, comment avatar gap |
| md | 16px | Default element spacing, form field gaps, task card content padding, gap between task cards in list |
| lg | 24px | Section padding within cards, workflow run header padding, template builder section gaps |
| xl | 32px | Card padding, layout gaps between major sections, workflow detail page content padding |
| 2xl | 48px | Page horizontal padding (desktop) |
| 3xl | 64px | Not used in Phase 4 |

Exceptions:
- Task card collapsed height: 56px (sufficient for title + status icon + inline action buttons)
- Task card expanded state: auto-height with 16px internal padding
- Drag handle width: 24px (grip icon area on left of task card)
- Drag placeholder: 2px solid `--primary` horizontal line at drop position
- Condition badge max-width: 240px with text-overflow ellipsis when collapsed
- Comment thread indent: 0px (flat, not nested) with 8px gap between comments
- Progress bar height: 8px (consistent with shadcn Progress component)
- Template picker dialog max-width: 480px
- Side panel width: 480px (consistent with Phase 2 and Phase 3)
- Side panel animation: 200ms ease-out slide from right (consistent with previous phases)

## Interaction Patterns

### Workflows Page — Active Runs Table

- **Columns:** Workflow name, Contractor, Template, Status, Progress (X/Y), Started by, Started date, Due date. Default sort: due date ascending (soonest due first), overdue pinned to top.
- **Sorting:** Click column header to toggle ascending/descending/none. Single-column sort.
- **Row click:** Opens side panel (Sheet from right, 480px). Does NOT navigate.
- **Row double-click or "Open" button in side panel:** Navigates to `/workflows/[id]`.
- **Search:** Debounce 300ms, minimum 2 characters. Server-side full-text search across workflow name, contractor name, template name.
- **Filters:** Status (multi-select), Template (multi-select), Contractor (search-select), Overdue only (toggle). Active filters as badges below filter bar.
- **Pagination:** Bottom of table. Page size (10, 25, 50). Previous/Next. "Page X of Y".
- **Overdue highlighting:** Rows with overdue status have `--destructive` at 5% opacity background tint. Due date cell text in `--destructive`.

### Workflows Page — My Tasks Tab

- **Layout:** Flat task list (not table). Each task as a horizontal card.
- **Card layout:** Status icon (left) | Task title + workflow name link (center) | Assignee avatar (if not current user — for reassigned visibility) | Due date (right).
- **Sorting:** Overdue tasks first (sorted by most overdue), then by due date ascending.
- **Overdue styling:** Red `AlertCircle` icon, due date text in `--destructive`, subtle `--destructive` at 3% background.
- **Click:** Navigates to `/workflows/[runId]` and auto-scrolls to the task.
- **Empty state:** Centered illustration with message (see copywriting).

### Workflows Page — Templates Tab (Admin Only)

- **Visibility:** Tab only visible to users with workflow template management permission (admin, ops manager).
- **Columns:** Template name, Type (badge), Status (badge), Tasks count, Created by, Last updated.
- **Row click:** Navigates to `/workflows/templates/[id]` (template builder page).
- **Actions:** Dropdown per row: Edit, Duplicate, Activate/Archive (depends on current status), Delete (draft only).
- **Pagination:** Same pattern as runs table.

### Template Builder Page

- **URL:** `/workflows/templates/[id]` for edit, `/workflows/templates/new` for create.
- **Layout:** Full-width page. Header: template name (inline editable heading or large input), type select, description textarea, status badge. Below header: vertical task list.
- **Task list:** Vertical list of collapsible Card components. Left side: drag handle (GripVertical icon, 24px). Collapsed view: task title, task type badge, assignee mode label, due offset, required badge, condition badge. Click anywhere on collapsed card to expand.
- **Drag-to-reorder:** Drag handle on left activates drag. Drop indicator: 2px solid `--primary` line at target position. Reorder updates sortOrder in real-time (optimistic).
- **Expanded task card:** All editable fields visible inline. Fields: title (input), type (select), description (textarea), assignee mode (radio group), role/user (conditional select), due offset days/hours (number inputs), required (switch), dependency (select from other tasks in this template), conditions (rule builder section). "Done editing" button collapses the card. "Remove task" destructive link at bottom of expanded card.
- **Add task:** "Add task" button at bottom of task list. Inserts new expanded card at the end. Auto-scrolls to new card.
- **Condition badge (collapsed):** Inline badge on collapsed card showing "Only if: type = JDG" for single condition or "Only if: 2 conditions" for multiple. Click badge opens expanded view scrolled to conditions section.
- **Save:** "Save template" CTA in top-right. Saves all changes atomically. Toast on success.
- **Unsaved changes:** Track dirty state. Show dot indicator on Save button when unsaved. Browser beforeunload warning if navigating away with unsaved changes.

### Condition Rule Builder (Inline in Task Card)

- **Layout:** Vertical list of condition rows within the expanded task card, below other fields. Each row: field select (entity.field), operator select (equals/not equals/contains/starts with), value select or input (depending on field type — enum fields get select, string fields get input).
- **Combinator:** AND/OR toggle between rows. Default AND. Toggle button between consecutive condition rows.
- **Add condition:** "Add condition" secondary button below last condition row.
- **Remove condition:** X button on each row.
- **Empty state:** "No conditions — task will always be included." with "Add condition" button.
- **Preview:** When conditions exist, collapsed card shows badge summary: "Only if: {field} {operator} {value}" for single, "Only if: {count} conditions" for multiple.

### Workflow Run Detail Page

- **URL:** `/workflows/[id]`
- **Header:** Workflow name (Heading), status badge, template name (as link to template), contractor name (as link to profile). Started by, started date, due date metadata.
- **Progress bar:** Full-width below header. 8px height. `--primary` fill. Label: "{X} of {Y} tasks complete". Percentage calculated excluding auto-skipped conditional tasks.
- **Task checklist:** Vertical list of task cards. Each card shows:
  - Left: Status icon (see task status colors table above).
  - Center: Task title (Body weight), task type badge (small), assignee avatar + name (Label size), due date (Label size, `--destructive` if overdue).
  - Right: Inline action buttons (visible on hover on desktop, always visible on mobile). Actions depend on status and user permissions.
- **Expand task:** Click task card to expand. Shows: full description, comments thread, attachments section, task metadata (created, completed timestamp, completed by).
- **Task actions (inline):**
  - TODO/IN_PROGRESS tasks assigned to current user: "Complete" (primary button), "Skip" (ghost button, opens reason popover), "Reassign" (ghost button, opens user picker popover).
  - BLOCKED tasks: No actions. Shows tooltip "Waiting for: {dependency task title}".
  - DONE tasks: No actions. Shows completed by + timestamp.
  - SKIPPED (user): Shows "Skipped" label + skip reason on expand.
  - SKIPPED (condition): Shows "Skipped (condition not met)" label, entire card at 50% opacity. Excluded from progress bar count.
  - Tasks assigned to other users: "Reassign" only (if user has permission).
- **Comments (inline below expanded task):** Flat thread. Each comment: author avatar + name, timestamp (relative), body text. Input at bottom with "Post" button. Plain text only — no rich text.
- **Attachments (inline below expanded task):** Reuses Phase 3 DocumentCard component. "Add attachment" button opens DropZone (reused from Phase 3). Files linked via DocumentLink with entityType=WORKFLOW_TASK_RUN.
- **Cancel workflow:** Dropdown action in header. Opens AlertDialog confirmation.
