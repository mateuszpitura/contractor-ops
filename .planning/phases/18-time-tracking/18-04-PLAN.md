**1. Create `packages/api/src/routers/time.ts`:**

Use `tenantProcedure` from middleware (same pattern as approval.ts). Import validator schemas and service functions.

Router procedures:

- `listPending` query: no input (or optional cursor/limit). Fetches timesheets with status=SUBMITTED for the org, ordered by submittedAt ASC. Include contractor (name, email), entries count, totalMinutes. Returns paginated list.

- `listAll` query: input=listTimesheetsSchema. Fetches all timesheets for the org with optional filters (status, contractorId, date range). Include contractor relation. Cursor pagination, ordered by weekStartDate DESC.

- `getTimesheet` query: input=z.object({ timesheetId: z.string() }). Fetches single timesheet with all entries (including contract title), contractor info. For the review detail view.

- `approve` mutation: input=approveTimesheetSchema. Calls approveTimesheet service with ctx.session.user.id as reviewerUserId. Per D-08: standalone approval, not multi-level chain.

- `reject` mutation: input=rejectTimesheetSchema. Calls rejectTimesheet service. Per D-07: reason required, contractor gets notification.

- `bulkApprove` mutation: input=bulkApproveTimesheetsSchema. Calls bulkApproveTimesheets. Returns { count: number }.

- `bulkReject` mutation: input=bulkRejectTimesheetsSchema. Calls bulkRejectTimesheets. Returns { count: number }.

- `listContractors` query: no input. Fetches contractors with at least one timesheet, with summary stats (total submitted pending, total hours this month). For the admin time overview.

Export as `timeRouter`.

Use plain() helper to strip Prisma prototypes.
