# Phase 18 — UI Design Contract

> Visual and interaction contract for time tracking: manual hour logging in portal, weekly timesheet grid, manager approval queue, Clockify/Jira import with source badges, and invoice deviation flagging. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inline padding, timesheet cell internal padding |
| sm | 8px | Timesheet grid cell gaps, compact table row padding, source badge icon-text gap |
| md | 16px | Form field vertical spacing, card content padding, timesheet header row padding |
| lg | 24px | Section padding, approval queue card internal padding, reconciliation section margin |
| xl | 32px | Layout gaps between admin time sections, portal timesheet and history sections |
| 2xl | 48px | Page-level vertical rhythm |
| 3xl | 64px | Not used in Phase 18 |

Exceptions: Timesheet grid cells use 8px (sm) internal padding for density. Weekly grid day columns use 4px (xs) horizontal gap between cells. Touch targets on timesheet cells are 44px minimum height for mobile usability.

## Layout Contract

### Focal Point

The primary screen focal points for Phase 18 are:

1. **Portal > Time page**: The weekly TimesheetGrid is the primary interaction surface. The contractor's eyes land on the current week's grid with the "Submit Timesheet" CTA in the header.
2. **Admin > Time section**: The ApprovalQueueTable with pending timesheets. Batch action bar draws attention when items are selected.
3. **Invoice detail page**: The ReconciliationCard appears below the existing invoice metadata, providing immediate hours-vs-invoice context.

### Portal Time Page (D-01, D-02, D-03, D-04)

- New route: `/portal/time`
- New nav item in PortalTopBar: `{ label: "Time", href: "/portal/time", icon: Clock }` between Documents and Payments
- Page layout (top to bottom):
  1. **TimeSummaryStats**: 3 summary cards in a responsive grid (3 columns on desktop, stack on mobile)
  2. **TimesheetHeader**: Week selector + status + total hours + submit button
  3. **TimesheetGrid**: The primary weekly grid card
  4. **"Add Single Entry" button**: Below the grid, outline variant, Plus icon + "Add Entry" label. Opens SingleEntryForm dialog
  5. **External sync buttons**: Flex row with ExternalSyncButton for Clockify and Jira (only shown if provider is connected). Separated from grid by `xl` (32px) spacing
  6. **Time entry history**: Collapsible section showing past weeks' submissions with status badges. Table: Period, Total Hours, Status, Submitted date

### Timesheet Grid Details (D-01, D-02, D-03)

- Card container with rounded corners and border
- Header row: day abbreviations (Mon-Sun) as Label text, right-aligned "Total" column
- Project rows: left column shows project/contract name (truncate with tooltip at 200px), 7 day cells + row total
- Each cell: Input with `type="number"`, `step="0.25"`, `min="0"`, `max="24"`. Width: 64px. Center-aligned text. Empty cells show "0" in muted color on focus-out, blank when unfocused
- Grand total row: bottom border-t, bold font-semibold, accent-colored total value
- Submitted/Approved timesheets: all cells become read-only (disabled styling). Status badge updates in header
- Rejected timesheets: cells editable again. Rejection reason displayed as a warning Banner below header (amber left border, warning icon + reason text + "Resubmit" prompt)
- Tab navigation between cells (left-to-right, top-to-bottom). Enter key moves to next row same column

### Single Entry Form (D-01)

- Triggered by "Add Entry" outline button below timesheet grid
- Dialog opens with form fields stacked vertically (md gap):
  - Date: DatePicker (Popover + Calendar), defaults to today, required
  - Project: Select dropdown populated from contractor's active contracts, required
  - Hours: Number input (step="0.25", min="0.25", max="24"), required
  - Description: Textarea (optional, max 500 chars, 3 rows)
- Form validation: inline error messages below fields in destructive color
- On submit: entry added to the appropriate day in the current timesheet grid (or creates new entry if date is outside current week view)

### Admin Time Section (D-05, D-06, D-07, D-08)

- New sidebar nav item in "finance" group: `{ key: "time", label: "Time", href: "/time", icon: Clock, permission: { resource: "time", actions: ["read"] } }`
- Position: between Approvals and Payments in the finance group
- Page layout with Tabs:
  - **Tab 1: "Pending Reviews"** (default) -- ApprovalQueueTable
  - **Tab 2: "All Entries"** -- Full time entry table with filters (contractor, status, period, source)
  - **Tab 3: "Reconciliation"** -- ReconciliationTable

### Manager Approval Queue (D-05, D-07)

- Table with bulk selection checkboxes
- Each row represents one contractor's timesheet for one week
- Clicking contractor name or "Review" text button navigates to ContractorTimesheetReview
- Batch action bar: sticky bottom bar (card background, border-t, 16px padding). Appears with slide-up animation when 1+ checkboxes selected. Shows: "{N} timesheets selected" label + "Approve All" primary button + "Reject All" destructive outline button
- Individual row actions: "Approve" small primary button + "Reject" small destructive outline button (visible on row hover on desktop, always visible on mobile)

### Per-Contractor Timesheet Review (D-05, D-07)

- Accessed by clicking contractor name in approval queue
- Layout: Contractor info header (avatar + name + contract name) + read-only TimesheetGrid + entry descriptions below grid + action bar
- Each cell shows hours with source badge (tiny icon in cell corner: manual=pencil, clockify=clock, jira=ticket)
- Description entries listed below grid as compact rows: date + hours + description text + source badge
- Action bar (sticky bottom): "Back to Queue" ghost button (left) + "Reject" destructive outline (right) + "Approve" primary (right)

### External Import UI (D-09, D-10, D-11)

- ExternalSyncButton rendered in a flex row below the timesheet grid (portal view)
- Each button: provider icon (16px) + "Sync from Clockify" / "Sync from Jira" text
- Click opens Popover with date range selector:
  - "From" date picker (defaults to current week Monday)
  - "To" date picker (defaults to current week Sunday)
  - "Import" primary button
- During sync: button shows Loader2 spinner, "Importing..." text. Popover closes
- On complete: toast notification "{N} entries imported from {provider}"
- Imported entries appear in timesheet grid with TimeSourceBadge. Cells are read-only (D-11). Muted background tint to distinguish from manual entries
- If provider not connected: button disabled with tooltip "Connect {provider} in Settings > Integrations"

### Invoice Reconciliation (D-13, D-14, D-15, D-16)

- **On invoice detail page**: ReconciliationCard renders below invoice metadata section. Only shows when the invoice's contract has an hourly rate and approved time entries exist for the invoice billing period
- **In admin Time section**: ReconciliationTable as third tab content. Shows all invoices with time tracking data. Default sort: highest deviation first

### Reconciliation Card Layout (D-16)

- Card with subtle left border: color matches deviation severity (success/warning/destructive)
- Header: Clock icon (16px) + "Time Reconciliation" label (text-sm font-semibold) + DeviationFlag badge (right-aligned)
- Content: 3-stat row (responsive: 3 columns desktop, stack mobile)
  - "Approved Hours": value in Display size + "hours at {rate}/h" in muted Body
  - "Expected Amount": value in Display size (calculated: rate x hours)
  - "Invoiced Amount": value in Display size (from invoice)
- Visual comparison bar: horizontal bar (8px height, rounded), accent fill proportional to expected/actual ratio. If actual > expected, bar extends with destructive-colored overflow section
