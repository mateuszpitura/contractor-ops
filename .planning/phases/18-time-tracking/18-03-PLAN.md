**1. Create `packages/api/src/routers/portal-time.ts`:**

Use `portalProcedure` from `../middleware/portal-auth.js` (same as portal.ts). Import validator schemas from `@contractor-ops/validators`. Import service functions from `../services/time-entry.js`, `../services/clockify-sync.js`, `../services/jira-worklog-sync.js`.

Router procedures (all use portalProcedure for contractor-scoped auth):

- `getTimesheet` query: input=getTimesheetSchema. Calls getOrCreateTimesheet. Returns timesheet with entries and related contract names. Include `entries` relation with `contract: { select: { id: true, title: true } }`.

- `getActiveContracts` query: no input. Fetches contractor's active contracts (status=ACTIVE, not deleted) with id, title, rateType, rateValueGrosze. Needed for project picker in grid/form.

- `saveDraftEntries` mutation: input=saveDraftEntriesSchema. Calls saveDraftEntries service. Returns updated timesheet.

- `createSingleEntry` mutation: input=createSingleEntrySchema. Gets or creates timesheet for the entry's week (use startOfISOWeek from date-fns). Calls saveDraftEntries with single entry. Returns updated timesheet.

- `submitTimesheet` mutation: input=submitTimesheetSchema. Calls submitTimesheet service. Returns updated timesheet.

- `listTimesheets` query: input=listTimesheetsSchema (scoped to contractor). Fetches timesheets for current contractor with cursor pagination, ordered by weekStartDate DESC.

- `syncExternal` mutation: input=syncExternalEntriesSchema. Based on provider, finds the IntegrationConnection (CLOCKIFY or JIRA, status=CONNECTED) for the org. Gets or creates timesheet for the start date's week. Calls syncClockifyEntries or syncJiraWorklogs. Returns { imported, skipped }.

- `getConnectedProviders` query: no input. Returns which time-related providers (CLOCKIFY, JIRA) are connected for the org. Query IntegrationConnection where provider in [CLOCKIFY, JIRA] and status=CONNECTED. Return array of { provider, displayName }.

Export as `portalTimeRouter`.

Use the `plain()` helper pattern from portal.ts to strip Prisma prototypes from results:
```typescript
function plain<T>(data: T): T {
  return JSON.parse(JSON.stringify(data)) as T;
}
```
