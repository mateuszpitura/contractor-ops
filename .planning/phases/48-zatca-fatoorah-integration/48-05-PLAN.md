**generateZatcaCsr(attributes: ZatcaCsrAttributes): { csr: string; privateKey: string }**
1. Generate ECDSA P-256 key pair using Node.js crypto:
   ```typescript
   const { publicKey, privateKey } = crypto.generateKeyPairSync('ec', {
     namedCurve: 'prime256v1',
     publicKeyEncoding: { type: 'spki', format: 'pem' },
     privateKeyEncoding: { type: 'pkcs8', format: 'pem' },
   });
   ```
2. Build X.509 CSR with ZATCA-required subject attributes per research section 5:
   - CN = attributes.commonName (solution name, e.g., "contractor-ops")
   - O = attributes.orgName
   - OU = attributes.vatNumber (15-digit)
   - C = "SA"
   - SN = `1-${attributes.commonName}|2-ContractorOps|3-${crypto.randomUUID()}`
   - UID = attributes.vatNumber
   - title = attributes.title ("0100" for B2B, "1000" for B2C, "1100" for both)
   - registeredAddress = attributes.registeredAddress
   - businessCategory = attributes.businessCategory
   Note: Node.js crypto does not natively support CSR creation. Use the `pkijs` library or construct the CSR using ASN.1 DER encoding. If complexity is too high, use `node-forge` for CSR creation:
   `pnpm --filter @contractor-ops/einvoice add node-forge && pnpm --filter @contractor-ops/einvoice add -D @types/node-forge`
3. Return `{ csr: csrPem, privateKey: privateKeyPem }`

6. Simplified debit note (invoiceTypeCode="383", subtype="0200000")

Create `packages/api/src/services/zatca-onboarding.ts`:

**saveTaxDetails(organizationId, taxDetails: ZatcaTaxDetails): Promise<void>**
- Validate with zatcaTaxDetailsSchema
- Save to IntegrationConnection.configJson (create ZATCA connection if not exists)
- Set currentStep to "csr_generation"

**generateAndStoreCsr(organizationId): Promise<{ csrPem: string }>**
- Load taxDetails from IntegrationConnection.configJson
- Build CSR attributes from tax details
- Call generateZatcaCsr(attributes)
- Store private key in Infisical: `secretStore.setSecret({ path: '/zatca/{orgId}', secretName: 'PRIVATE_KEY', value: privateKey })`
- Store CSR PEM temporarily in configJson for next step
- Set currentStep to "compliance_csid"
- Return CSR PEM (public, safe to show in UI preview)

**requestComplianceCsid(organizationId): Promise<{ requestId: string }>**
- Load CSR from configJson
- Create ZatcaApiClient with sandbox base URL
- Call `apiClient.requestComplianceCsid(base64(csrPem))`
- Store compliance certificate in Infisical: `secretStore.setSecret({ secretName: 'X509_CERTIFICATE', value: response.binarySecurityToken })`
- Store API secret in Infisical: `secretStore.setSecret({ secretName: 'API_SECRET', value: response.secret })`
- Store requestId in Infisical: `secretStore.setSecret({ secretName: 'COMPLIANCE_REQUEST_ID', value: response.requestID })`
- Set currentStep to "compliance_checks"
- Return requestId

**runComplianceChecks(organizationId): Promise<Array<{ type: string; status: string }>>**
- Load tax details and compliance certificate/secret from Infisical
- Create ZatcaApiClient with compliance credentials
- Build 6 test invoices via buildComplianceTestInvoices(taxDetails)
- For each test invoice:
  1. Generate XML via ZatcaProfile
  2. Sign with compliance certificate
  3. Compute hash
  4. Submit via apiClient.submitComplianceInvoice()
  5. Record result (type, status: CLEARED/REPORTED/REJECTED)
- Set currentStep to "production_certificate" if all pass
- Return results array

**exchangeProductionCertificate(organizationId): Promise<void>**
- Load requestId from Infisical
- Create ZatcaApiClient with compliance credentials
- Call `apiClient.requestProductionCsid(requestId)`
- Overwrite certificate in Infisical with production cert
- Overwrite API secret with production secret
- Update IntegrationConnection: status=CONNECTED, environment=production
- Set currentStep complete, certificateStatus="production"

**getOnboardingState(organizationId): Promise<ZatcaOnboardingState>**
- Read configJson from IntegrationConnection
- Return current step and progress flags

Add tRPC procedures to `packages/api/src/routers/zatca.ts` (extend existing router from Plan 04):

- `generateCsr` — mutation: { organizationId } — generates CSR, returns PEM preview
- `requestComplianceCsid` — mutation: { organizationId } — requests compliance cert
- `runComplianceChecks` — mutation: { organizationId } — runs 6 test invoices, returns results
- `exchangeProductionCert` — mutation: { organizationId } — exchanges for prod cert
- `getOnboardingState` — query: { organizationId } — returns current wizard state
All mutations require org membership auth.
