# IRS Form 1042-S via IRIS (Pub 1187)

## Purpose

Transmit the IRS Copy A of Form 1042-S (chapter-3 withholding on US-source income paid to foreign
recipients) as IRIS (Information Returns Intake System) XML — the chapter-3 sibling of the 1099-NEC
IRIS surface (Pub 1220) documented in [[domains/us-tax-forms]]. The domain logic (box derivation,
§875(d) treaty gate, immutable snapshot) lives in the [[domains/us-tax-forms|US tax forms]] domain;
this page is the IRIS transmit/XSD integration seam.

The whole surface is dark behind `module.us-expansion`.

## Flow

```
form1042s.generateBatch (server-derived boxes, §875(d)-gated rate)
  → Form1042S immutable snapshot (FTIN last-4)
  → buildIris1042SXml   (Pub 1187 sibling record via fast-xml-parser XMLBuilder)
  → xsdValidate1042S    (form-parameterized libxmljs2 { nonet: true } against the pinned Pub 1187 XSD)
  → [HELD] transmit tail (download validated XML / upload IRIS ack) — reuses the P86 TaxFilingTransmitter seam
```

## Entry points

| Piece | Path |
|-------|------|
| 1042-S IRIS builder | `packages/iris/src/generator.ts` (`buildIris1042SXml`) — sibling builder (not a parameterized 1099 builder) |
| 1042-S XSD validator | `packages/iris/src/validator.ts` (`xsdValidate1042S`) — form-parameterized `ENTRY_MATCHERS` + per-form memoized loader |
| Pub 1187 XSD bundle | `packages/iris/src/schema-bundle/` — human-only IRS SOR download, SHA-256 pinned in `checksums.txt` (guarded by `pnpm --filter @contractor-ops/iris verify:schema-checksums`) |
| Domain core | `packages/api/src/services/form-1042s.service.ts` + `packages/api/src/routers/finance/form-1042s-router.ts` (us-expansion gated) |

## Invariants

- `buildIris1042SXml` is a **sibling** builder, not a parameterized 1099 builder — the Pub 1187 record
  layout differs materially (ch3/ch4 status/exemption codes, income codes, treaty fields). It shares the
  Transmission Manifest shape + the fast-xml-parser `XMLBuilder` instance, and **never string-concatenates**
  XML.
- The payee FTIN is emitted **masked last-4 only** — the full FTIN is never reconstructed or passed into
  the payload (asserted in the generator test).
- The withholding rate emits as a two-decimal percent string derived from basis points (`1500 → "15.00"`);
  amounts convert minor units to whole US dollars (shared `toUsAmount`). Element names + code formatting
  are adviser-verify against i1042s Appendix B/C when the pinned XSD lands.
- `xsdValidate1042S` is **form-keyed**: each form's payload XSD is selected by filename (`ENTRY_MATCHERS`)
  with per-form memoization; a missing 1042-S XSD reports missing rather than falling back to the 1099
  schema. SSRF/XXE-safe (`libxmljs2 { nonet: true }`, default `noent: false`). Until the human-only Pub 1187
  XSDs are placed + checksum-verified, `xsdValidate1042S` reports `XSD-BUNDLE-MISSING` (INVALID, never throws).
- **Cross-phase HOLD — the transmit tail:** the download-XML / upload-ack procedures reuse the P86
  `TaxFilingTransmitter` seam + `iris-ack-parser`, which are not on disk. No `form-1042s-transmit.service.ts`
  exists and no transmit procedures were added to `form-1042s-router.ts` / `root.ts` — they append verbatim
  once P86 lands, never rebuilt. ManualDownload is the default posture; A2A (automated) stays dark.

## Agent mistakes

- Do NOT parameterize the 1099 builder for 1042-S — it is a distinct Pub 1187 record layout; use the
  sibling `buildIris1042SXml`.
- Do NOT string-concatenate the IRIS XML — use the `@contractor-ops/iris` `XMLBuilder`.
- Do NOT pass a full FTIN into the payload (last-4 only), and do NOT treat `XSD-BUNDLE-MISSING` as a bug —
  it is the human-only IRS Pub 1187 XSD download checkpoint.
- Do NOT rebuild the transmit tail — it reuses the P86 `TaxFilingTransmitter` seam once that lands.
