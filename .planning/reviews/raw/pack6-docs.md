# PACK 6 — Documents/OCR/e-sign, Notifications/Reminders, Search, Reports, Dashboard, Audit (packages/api)

Production-grade business-logic review. Read-only. Cited to `file:line`.

Scope routers: `document`, `ocr`, `esign`, `notification`, `reminder`, `search`, `report`, `dashboard`, `audit`.
Scope services: `ocr-extraction`, `doc-link-service`, `pending-upload`, `virus-scanner`, `r2`, `regional-storage`, `exports/**`, `report-export`, `lib/csv`, `sanitize`, `status-aggregator`, `cache`, `resend-client`, `notification-service`, `messaging/**`, `rbac-recipients`, `audit-writer`, `esign-orchestrator`.

## Systemic patterns

- **Client-trusted storage/entity handles bypass the established server-derived pattern.** The codebase already solved cross-tenant R2 exfiltration with `pending-upload.ts` (server mints `documentId`, never returns/accepts `storageKey`) and `document.confirmUpload` correctly re-derives keys from an org-scoped DB row. OCR (`ocr.trigger`/`portalTrigger`) regresses this by accepting raw `storageKey`+`documentId`. Audit every remaining `z.string()` handle that becomes an R2 key or a `findFirst`/service call and enforce ownership via `ctx.db` before use.
- **Missing RBAC on read procedures.** Multiple `tenantProcedure` reads (`ocr.getResult/getByDocument`, `esign.listConnections/getEnvelopeDetail/listEnvelopes`, staff `esign.getSigningUrl`) rely on tenant scope alone with no `requirePermission`, exposing PII (invoice line items, recipient emails) and, worse, a signing capability. Mutations in the same routers are correctly gated — the reads were overlooked.
- **Two divergent CSV code paths with different safety guarantees.** `lib/csv.ts` correctly neutralizes formula injection for the async export framework; `report-export.ts` (ExcelJS) does not, yet still serves `audit.export`. Consolidate on the `lib/csv.ts` path so all CSV egress shares one hardened implementation.
- **Monetary aggregation cast inconsistency.** Dashboard KPIs use `::bigint`; report/export spend queries use `::int`. Standardize all `SUM(*Minor)` on `::bigint` to avoid overflow errors/silent truncation at scale.
- **Audit-write atomicity/coverage is uneven.** `document.delete` uses `auditedMutation` (audit inside tx); `reminder.*` calls `writeAuditLog` post-commit without `tx`, `reminder.toggleActive` omits it entirely, and `audit.export` is unaudited. Route all sensitive mutations/egress through the `tx`-bound audit helper.
- **Tenant scoping and raw-SQL parameterization are otherwise solid.** `search.global` sanitizes tsquery tokens and parameterizes via `Prisma.sql`/`to_tsquery` (no injection); report/dashboard raw SQL bind `organizationId` and use whitelisted `ORDER BY` columns; `document`/`reminder`/`notification` reads/writes consistently pin `organizationId`; `AuditLog` writer is create-only (append-only preserved).
