<objective>
Build the admin compliance dashboard at the TanStack-Router route `/<locale>/compliance/dashboard` in `apps/web-vite`, following Page->Container->Hook->Component (`apps/web-vite/ARCHITECTURE.md`). Surfaces COMPL-01 (at-risk count + clickable drilldown, blocked-payments queue) and COMPL-11 (per-locale doc names) over the SHIPPED Phase 72 backend + the Plan 73-05 dashboard query helpers.

Layout (D-01): a 3-up KPI summary card row (At risk / Upcoming renewals / Blocked payments) above a single tabbed table region. Clicking a card switches the active tab. Default landing tab = "At risk". Each table row links to the canonical per-contractor Compliance tab (`/contractors/{id}/compliance#item-{itemId}` — D-05). All three table reads are server-filtered + indexed (no N+1). The blocked-payments tab polls every 60s (D-04). Mandatory loading / empty / error states + WCAG (keyboard-activatable cards, aria, focus, contrast).

This is a UI-only plan (plus message-catalog keys). All backend queries are delivered by Plan 73-05; the override button cross-mounted on dashboard rows is delivered by Plan 73-08 (shared component), so this plan leaves a documented `renderRowActions?` prop slot for it but does NOT build the override modal.
</objective>
