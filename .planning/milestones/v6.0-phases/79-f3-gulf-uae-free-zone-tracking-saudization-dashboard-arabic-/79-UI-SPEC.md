# Phase 79 — UI Design Contract

> Visual and interaction contract for the F3 Gulf surfaces (UAE free-zone tracking, Saudization dashboard, Arabic + RTL). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase REUSES the existing `apps/web-vite` "Precision Craft" design system verbatim.** It introduces no new tokens, fonts, colors, or spacing values. Every contract value below is read from the live tree (`apps/web-vite/src/styles/globals.css`, `packages/ui/src/styles/tokens.css`, `packages/ui/components.json`) and is binding for the new Gulf surfaces. All copy goes through `useTranslations` with real values in en/de/pl/ar (D-16); statutory identifiers are locked code constants (D-14/D-15), PENDING legal review.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (already initialized — `packages/ui/components.json`) |
| Preset | `base-nova` (style), `baseColor: neutral`, `cssVariables: true`, `rtl: true` |
| Component library | radix-based shadcn primitives in `@contractor-ops/ui` (`@/components/shadcn`) |

**Architecture (binding — `apps/web-vite/ARCHITECTURE.md`, D-17):** every Gulf surface follows Page → Container (`*-container.tsx`) → Hook (`components/{domain}/hooks/use-*.ts` — the ONLY `useTRPC`/`useQuery`/`useMutation` boundary) → presentational Component. Enforced by `check:web-vite-{data-layer,page-shells,presentational,table-pattern,dialog-pattern}`. Dialogs use the `DialogBody` (scroll) + `DialogFooter` (sticky actions) convention. Tables use the canonical TanStack `table-pattern`.

**RTL contract (GULF-08, D-13 — binding, reuse v4.0 infra):** CSS **logical properties only** (`ms-`/`me-`/`ps-`/`pe-`/`start-`/`end-` — NEVER `ml-`/`mr-`/`pl-`/`pr-`/`left-`/`right-`). Charts wrap `useRtlChartConfig()` (`apps/web-vite/src/hooks/use-rtl-chart-config.ts`) for `reversed` x-axis, right-orientation y-axis, and `direction: rtl`. `isRtl(locale === 'ar')`. No new RTL machinery. The `ml-`/`mr-` ban guard must be verified-or-built per RESEARCH Pitfall 20 (planner owns; out of this visual contract but referenced for the checker).

## Spacing Scale

Declared values (multiples of 4 — `apps/web-vite/src/styles/globals.css` fluid scale + Tailwind 4 default 4px step). Gulf surfaces use ONLY these:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon-to-label gaps inside badges/buttons, inline-end badge padding |
| sm | 8px (`gap-2`, `p-2`) | Compact field spacing, dialog footer button gap, stat-card label-to-value |
| md | 16px (`gap-4`, `p-4`) | Default form-field vertical rhythm, card inner padding, banner content padding |
| lg | 24px (`gap-6`, `p-6`) | Section padding, dashboard card-grid gap, dialog body padding |
| xl | 32px (`gap-8`) | Major layout gaps between dashboard regions (headcount block ↔ band block) |
| 2xl | 48px (`gap-12`) | Top-level page-section breaks (free-zone form ↔ permitted-activities block) |
| 3xl | 64px (`gap-16`) | Page-level vertical spacing (rare; full-page empty states) |

Also available and preferred where present: semantic rhythm tokens `space-y-stack-tight`, `space-y-card-gap`, `space-y-section-gap` (from tokens.css) — use these for card stacks instead of raw values to match existing dashboard surfaces.

Exceptions: interactive touch targets (icon-only buttons, switch hit-areas, quarterly-prompt dismiss) must meet **44px minimum** hit-area (WCAG 2.5.5) even when the visual glyph is smaller — achieved via padding, not a new spacing token.

## Typography

Fluid `clamp()` scale from `tokens.css` `@theme`. Gulf surfaces declare exactly these four roles using exactly **two weights: 400 regular, 600 semibold**. The Display role uses 600 — its distinct `font-display` family (Bricolage Grotesque) plus the larger `text-2xl` size already provides the hero-stat hierarchy without a third weight. Arabic renders the same roles via the auto-swapped Noto Sans Arabic family — no per-locale size overrides.

| Role | Size (Tailwind / clamp) | Weight | Line Height |
|------|-------------------------|--------|-------------|
| Body | `text-sm` (~13–14px) | 400 regular | 1.25rem (~1.5) |
| Label | `text-base` (~14–15px) | 600 semibold | 1.5rem |
| Heading (card / dialog title, dashboard metric label) | `text-lg` (~16–18px) | 600 semibold | 1.75rem |
| Display (page title, hero stat value e.g. nationalisation rate %) | `text-2xl` (~22–30px) | 600 semibold (Bricolage Grotesque `font-display`) | 2.25rem |

Body line height ≈ 1.5; heading line height ≈ 1.2–1.5 (per `--text-*--line-height` tokens). Numerals in stat blocks (headcount, rate %, Iqama counts) use `tabular-nums` for column alignment. Do NOT introduce new font sizes or weights — clamp tokens cover responsive scaling and the 400/600 pair covers all hierarchy.

## Color

`oklch` semantic tokens from `tokens.css` (light) + `.dark` overrides + web-vite `globals.css` (charts/sidebar/viz). 60/30/10 split is the existing system; Gulf surfaces reserve accent strictly.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (`bg-background`) | Page background, form canvas, dashboard base surface |
| Secondary (30%) | `--card` / `--secondary` / `--surface-1..3` (`bg-card`) | Stat cards, free-zone form panels, dashboard region cards, sidebar/nav |
| Accent (10%) | `--primary` Deep Teal (`bg-primary`, `text-primary`, `ring-primary`) | See reserved-for list below |
| Destructive | `--destructive` (`text-destructive`, `bg-destructive/10`) | Destructive confirmations only (see Copywriting) — NOT for advisory banners |

**Accent (Deep Teal `--primary`) reserved ONLY for:**
1. The single primary CTA per surface (e.g. "Save free-zone assignment", "Update Nitaqat band") — exactly one teal-filled button per view.
2. Active/selected state of the 10-zone + Mainland `Select` and tab triggers.
3. Focus rings (`ring-primary`) on inputs, selects, switches (a11y, system-wide).
4. The Saudization band donut/progress primary series via `--chart-1` (teal) + `useRtlChartConfig`.
5. Key dashboard metric emphasis (nationalisation rate value, current-band label chip).

Accent is NEVER applied to every interactive element, to advisory banners, to badges, or to body links en masse.

**Semantic status colors (existing tokens — reuse, do NOT invent per-band colors):**
- `--warning` (amber) → advisory/non-blocking surfaces: scope-mismatch banner, NOC item (`WARNING`), offboarding band-trajectory banner, "Custom — verify with adviser" override badge, quarterly re-entry prompt, manual/advisory band-nature callout, Qiwa-auth coverage-gap count.
- `--destructive` (red) → BLOCKING/EXPIRED free-zone license state + destructive confirmation dialogs only.
- `--success` (green) → satisfied compliance items, in-coverage Qiwa rows.
- `--muted-foreground` → last-updated timestamps, platform-derived cross-check (secondary) numbers, helper text.

**Nitaqat band labels** (PLATINUM/HIGH_GREEN/MID_GREEN/LOW_GREEN/YELLOW/RED) are **locked statutory text** (D-14), rendered as a neutral `Badge` (`variant="outline"` or `secondary`) with the literal band label — the system MUST NOT colorize the badge to assert a band judgement (e.g. do not paint RED red as if authoritative). Band color, if any, is decorative-muted only; the manual/advisory nature must stay visually explicit (see Components). This preserves the never-auto-compute-band lock (D-12, RESEARCH Pitfall 8).

## Focal Points

Each primary Gulf surface declares ONE focal point — the single element the eye lands on first. Executors must compose the surface so this element wins the visual hierarchy (size, weight, position) with no competing accent.

| Surface | Focal point | Treatment |
|---------|-------------|-----------|
| Saudization dashboard | Nationalisation rate % (hero stat) | Display role, `text-2xl`, top-of-card hero stat (`tabular-nums`), `text-primary` metric emphasis — the first and largest value on the page |
| Free-zone assignment form | Zone `Select` (10 zones + Mainland) | First interactive element, top of form; everything else (expiry date, permitted-activities, save CTA) follows after the zone is chosen |
| Band edit / headcount entry | Primary CTA ("Update Nitaqat band" / "Save headcount") | Single teal-filled `button` in `DialogFooter`/form footer; the only accent in the dialog |
| Scope-mismatch advisory banner | Banner heading line | `--warning` `alert` at top of contract-creation flow; the advisory message is the focal element, NOT a competing CTA (creation still proceeds) |
