<objective>
Make `SlackAdapter implements Deprovisionable`: SCIM-deactivate (raw fetch, org-grid
token) + session-invalidate (admin API) + verify + describeImpact, plus the
SLACK_ORG_GRID connection sub-kind plumbing and the Enterprise-Grid pre-flight
detection. This is the Slack half of the wedge (SC#3 Slack two-step independence,
SC#5 Slack LIKELY_GONE, D-14/D-16 org-grid + Grid-detection). The connection
lifecycle UI (cards + toggle) is 77-05; the OAuth-start endpoint + token resolution
wiring is 77-04.
</objective>

</dependency_note>

<threat_model>
| ID | Threat | Mitigation |
|----|--------|-----------|
| T-77-03-01 | SCIM call uses the workspace bot token instead of the org-grid token → 403 / wrong-tenant write | The SCIM + admin.session methods take the org-grid token explicitly (resolved by 77-04 from the SLACK_ORG_GRID connection); the adapter NEVER falls back to the workspace token for these. Test injects two distinct tokens and asserts SCIM used the org-grid one. |
| T-77-03-02 | Non-Enterprise-Grid org reaches a real SCIM write and errors opaquely | Three-point detection (D-16): lookupByEmail `cannot_perform_operation` at connect-time, describeImpact `error: 'NOT_ON_ENTERPRISE_GRID'`, and saga-start eligibility (77-04). The adapter classifies the signal → PERMANENT_FORBIDDEN errorClass `slack_not_on_enterprise_grid`; never silently no-ops. |
| T-77-03-03 | Personal `xoxp-` token / impersonation creeps in (security hole, D-14 rejected) | Org-grid connection uses an org-level OAuth grant only; Phase 75 secret-shape detector rejects pasted user tokens. The adapter reads the token from the resolved SLACK_ORG_GRID connection, not from input. |
| T-77-03-04 | Slack web-API rate limits (Tier 2/3) abort describeImpact mid-preview | describeImpact reads are best-effort: a 429 on any single read degrades that metric to null rather than failing the whole preview; the deprovision step methods (SCIM/admin.session) re-throw on TRANSIENT for QStash retry. `withResilience({ provider: 'slack' })` honors Retry-After. |
| T-77-03-05 | SCIM hash embeds the org token or user email (PII/secret in audit) | Hashes computed on canonicalised + redacted payloads (token/authorization/externalUserId stripped). Test asserts redaction. |
| T-77-03-06 | `users.conversations` pagination loops forever for a 10k-channel user | Cap the channel fetch (first N pages / 1000 members) and surface "+more" — channelsMemberCount may be a capped count or null; documented (CONTEXT.md Discretion). |
</threat_model>
