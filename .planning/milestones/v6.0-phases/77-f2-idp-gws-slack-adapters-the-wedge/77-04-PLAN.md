<threat_model>
| ID | Threat | Mitigation |
|----|--------|-----------|
| T-77-04-01 | `_step-runner` invoked without QStash signature → attacker triggers a real provider deprovision | First line is `guardQStashRequest` (the Fastify QStash authn used by every `_sync`/`_submit` route). No signature → 401, no work. |
| T-77-04-02 | Preview cache serves a higher-privilege user's data to a lower-privilege one | `requirePermission` runs in the tRPC procedure BEFORE `getImpactPreview` calls `cached` (F-SCALE-09 invariant). Preview data is org-scoped, not user-scoped. |
| T-77-04-03 | `overrideStepFailure` is callable by non-OWNER/ADMIN, or with an empty reason | `.use(requirePermission({ idp: ['override_step_failure'] }))` (9 non-grantees → FORBIDDEN before DB work); server-side Zod `note.min(20)` + `category` enum re-validated (client modal is convenience only). |
| T-77-04-04 | Override on a step that is not actually exhausted (status FAILED but attempts < MAX) bypasses retry budget | Mutation asserts `status === 'FAILED' && attempts >= MAX_ATTEMPTS`; otherwise throws PRECONDITION_FAILED. |
| T-77-04-05 | Partial-write: status flips to MANUAL_COMPLETED but AuditLog / recompute fails → audit gap | Single `$transaction` wraps the 4 columns + status + AuditLog; recomputeRunStatus inside or immediately after the same tx (mirror Phase 74). Parent ACCESS_REVOKE auto-complete + its AuditLog are separate timestamped entries (D-12). |
| T-77-04-06 | Slack enabled for an org not on Enterprise Grid → jobs enqueue and all fail | Saga-start eligibility (D-15/D-16 point 3) excludes Slack when no SLACK_ORG_GRID connection or unavailableReason='not_on_enterprise_grid'; emits `idp.slack.org_grid_unavailable`; no QStash job enqueued for the skipped provider. |
| T-77-04-07 | Provider toggle enabled while the signoff flag is still PENDING → deprovisioning exposed pre-signoff | `enableProviderForOrg` refuses to set true unless the provider flag is APPROVED or FLAG_SIGNOFF_BYPASS=local. |
| T-77-04-08 | GWS three-audit-row mapping lost (only one row written) | The step-runner writes the suspend step row + reads the revoke result's two sub-action hash pairs and emits the second sub-action as an extra audit row (D-05). Test greps audit for 3 rows on a full GWS run. |
</threat_model>
