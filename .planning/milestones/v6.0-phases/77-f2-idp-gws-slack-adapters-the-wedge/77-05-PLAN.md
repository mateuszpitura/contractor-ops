| ID | Threat | Mitigation |
|----|--------|-----------|
| T-77-05-01 | 'Mark complete' button shown/enabled for users without the override permission | The button renders only when the caller holds `idp:override_step_failure` (read via the same permission-map pattern as `use-run-header.ts`); the server mutation independently gates (77-04) — UI hiding is convenience, not the gate. |
| T-77-05-02 | Override dialog submits an empty/short rationale | Client Zod `min(20)` disables submit until satisfied; server re-validates (authoritative). |
| T-77-05-03 | Direct tRPC calls leak into the container/component (architecture violation) | `pnpm check:web-vite-data-layer` enforces the layer boundary; only `use-*.ts` hooks import `useTRPC`. |
| T-77-05-04 | Provider toggle lets admin enable a PENDING-flag provider via the UI | The toggle is disabled (with an explanatory tooltip) when Flag Status != APPROVED; the server mutation refuses regardless (77-04). |
| T-77-05-05 | Missing loading/empty/error or a11y states (happy-path-only) | Each container ships explicit Skeleton (loading), empty, and error renders; tests assert all three; keyboard + focus-visible + contrast verified; ar RTL parity via i18n:parity. |
| T-77-05-06 | Preview shows stale data without the admin realizing | Freshness label ('Last refreshed: N min ago') + Refresh button (forceRefresh) make staleness explicit (D-02). |
</threat_model>
