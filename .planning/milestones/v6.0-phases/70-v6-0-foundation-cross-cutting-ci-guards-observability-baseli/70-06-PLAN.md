<objective>
Wire the three new lint scripts into CI (`.github/workflows/ci.yml`) and the husky pre-push hook (`.husky/pre-push`) per D-01 / D-04. Each new step runs independently — failure of one does NOT mask failures of the others (CI runs each as a separate step that GH Actions reports individually); the pre-push hook chains them with `&&` so first-failure short-circuits.

Purpose: The library/script work in Plans 02–04 is plumbing — without this wiring, no PR is actually gated. This plan is what turns "guards exist" into "engineers cannot land code that violates them" (the phase goal).

Output: Two file edits, both reviewable in isolation. No new scripts, no new packages.
