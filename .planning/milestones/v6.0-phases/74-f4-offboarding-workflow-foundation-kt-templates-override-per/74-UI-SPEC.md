# Phase 74 — UI Design Contract

> F4 Offboarding — Workflow Foundation + KT Templates + Override Permission. Visual and interaction contract for the override dialog, permanent override badge, Settings > Workflow Roles CRUD, workflow start screen template auto-selection + manual override dropdown, PTO admin-attention badge, `(English)` locale-fallback indicator, and Settings > Calendar PTO Keywords editor.

> Generated by gsd-ui-researcher; verified by gsd-ui-checker.

## Spacing Scale

All values are multiples of 4 and resolve to existing Tailwind utilities. Logical-property classes (`ms-*`, `me-*`, `ps-*`, `pe-*`) are mandatory; physical `ml-*`/`mr-*`/`pl-*`/`pr-*` are forbidden (Phase 79 ESLint guard, codebase-wide convention).

| Token | Value | Tailwind | Usage on Phase 74 surfaces |
|-------|-------|----------|----------------------------|
| xs | 4px | `gap-1` / `p-1` | Icon-to-text gap inside badges; `(English)` indicator suffix gap |
| sm | 8px | `gap-2` / `p-2` / `space-y-2` | Form-field label-to-input gap; dropdown-item internal padding |
| md | 16px | `gap-4` / `p-4` / `space-y-4` | Default form-field spacing in dialog body, settings forms; card content padding |
| lg | 24px | `gap-6` / `p-6` / `space-y-6` | Dialog content padding (`DialogContent` default); section breaks within Settings page |
| xl | 32px | `gap-8` / `p-8` | Settings page section gaps (between Role-template list and form) |
| 2xl | 48px | `gap-12` / `py-12` | Page-level top/bottom padding on Settings; empty-state vertical padding |
| 3xl | 64px | `gap-16` / `py-16` | Reserved (not used on these surfaces) |

**Exceptions:**
- Override dialog primary action button minimum tap target = 40px height (`h-10`, default shadcn `Button` size) — exceeds the 4-multiple grid via design-system primitive default; documented as primitive-driven, not custom.
- Permanent override badge minimum height = 20px (`h-5`, shadcn `Badge` default) — also primitive-driven.
- `WorkflowRoleTemplate` task-item form: 3 inputs per language stack vertically with 8px (`space-y-2`) inter-input gap, then 16px (`mt-4`) gap to the next task item; this is the only surface using sub-16 vertical rhythm between sibling form rows.

## Color

60/30/10 split per surface. All tokens are existing OKLCh CSS vars from `globals.css`; no raw hex introduced.

### Global palette (mapped to surfaces)

| Role | Token | Light value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--background` / `--surface-0` | `oklch(0.985 0.003 85)` (warm off-white) | Settings page surface, dialog overlay base, workflow-start-screen surface |
| Secondary (30%) | `--card` / `--surface-1` | `oklch(1 0 0)` (pure white) | `DialogContent`, `Card` in role-template list, Select dropdown panel |
| Accent (10%) | `--primary` (Deep Teal) | `oklch(0.44 0.145 178)` | Primary CTA only — see "Accent reserved for" below |
| Destructive | `--destructive` | `oklch(0.58 0.23 18)` | Override permanent badge background (10% tint), Override-button hover affordance NOT applied (Override is destructive intent but visually `variant="destructive"` per shadcn) |
| Muted text | `--muted-foreground` | `oklch(0.5 0.01 260)` | Helper copy, `(English)` suffix, badge metadata, empty-state secondary copy |
| Border | `--border` | `oklch(0.91 0.004 85)` | Card / Dialog / Input borders |
| Warning | `--warning` | `oklch(0.7 0.16 65)` (burnished amber) | PTO admin-attention badge background tint |
| Info | `--info` | `oklch(0.55 0.18 260)` | `(English)` indicator info-icon stroke (when icon variant chosen — see D-15 resolution) |

### Per-surface 60/30/10 application

**Override dialog**
- 60% — modal overlay backdrop (`bg-black/50` over `--background`) + dialog body content area (`--card`).
- 30% — `Textarea` background (`--input`), `Checkbox` chrome surface, footer separator (`--border`).
- 10% — Override button background (`--destructive`), focused-input ring (`--ring` = `--primary` teal). The Override CTA uses `Button variant="destructive"` because the action is destructive in nature; the teal `--primary` accent is reserved for Cancel/secondary actions only on this surface so the dual-validation gate is visually unmistakable.

**Permanent override badge (above offboarding header)**
- 60% — surface above which it floats (page `--background`).
- 30% — badge background `bg-destructive/10` (subtle red tint per `Badge variant="destructive"` from `badge.tsx:14-16`).
- 10% — badge foreground text `text-destructive`, `AlertOctagon` icon stroke (lucide).

**Settings > Workflow Roles list + form**
- 60% — page `--background`.
- 30% — `Card` per role template (`--card`), `Tabs` panel surface for en/pl/de language tabs.
- 10% — primary "Add role template" CTA (`Button` default = teal `--primary`), focused-input rings.

**Workflow-start-screen template dropdown**
- 60% — page `--background`.
- 30% — `Select` trigger surface (`--input`), Select content panel (`--popover`).
- 10% — Selected-option check icon and "Start offboarding" primary CTA (teal `--primary`).

**PTO admin-attention badge**
- 60% — surrounding card or table row.
- 30% — badge background `bg-amber-500/10` via `Badge variant="warning"` from `badge.tsx:22-23`.
- 10% — badge foreground (`text-amber-700`), `Clock` icon (lucide) stroke. Co-rendered text + icon (warning color is not the only signal — non-color a11y).

**`(English)` locale-fallback indicator**
- Visual treatment (D-15 resolution): **muted parenthetical suffix** (`text-muted-foreground text-xs`) immediately following the rendered English fallback string, plus an inline `Info` lucide icon (12px) with `aria-describedby` pointing to a hidden description (see A11y section). The icon is visible **only on hover/focus of the parent string** (`opacity-0 group-hover:opacity-100 group-focus-visible:opacity-100`) to avoid visual noise on dense lists. The `(English)` text suffix is always visible (text + spacing IS the always-on signal; the icon adds discoverability of the explanation).

**Accent reserved for:**
- Primary CTAs ("Add role template", "Start offboarding", "Save template", "Copy from English" helper button)
- Focused-input rings (Tailwind `focus-visible:ring-ring`)
- Active sidebar item indicator (existing pattern, untouched)
- NEVER on: badges (warning/destructive variants own those slots), headings, body copy, dividers, or hovered table rows.

## Per-Surface Interaction State Machines

### Override dialog state machine (most rigorous)

```
                       ┌────────────────────┐
                       │ closed              │
                       │ (no DOM)            │
                       └──────────┬──────────┘
                                  │ Owner clicks "Override IP verification"
                                  ▼
                       ┌────────────────────┐
                       │ idle                │
                       │ - reason = ""       │
                       │ - ack   = false     │
                       │ - submit DISABLED   │
                       │ - aria-invalid      │
                       │   NOT yet on field  │
                       │ - char counter 0/20 │
                       └──────────┬──────────┘
                                  │ user types in textarea
                                  ▼
                       ┌────────────────────┐
                       │ typing-invalid      │
                       │ - reason.length<20  │
                       │ - aria-invalid=true │
                       │ - inline error msg  │
                       │   shown ONLY after  │
                       │   first blur (UX:   │
                       │   no nag-while-     │
                       │   typing)           │
                       │ - submit DISABLED   │
                       └──────────┬──────────┘
                                  │ length crosses ≥20
                                  ▼
                       ┌────────────────────┐
                       │ typing-valid-no-ack │
                       │ - aria-invalid=false│
                       │ - error msg cleared │
                       │ - submit DISABLED   │
                       │   (ack still false) │
                       └──────────┬──────────┘
                                  │ user checks ack box
                                  ▼
                       ┌────────────────────┐
                       │ ready                │
                       │ - submit ENABLED    │
                       │ - aria-disabled=    │
                       │   false on Override │
                       │ - keyboard Enter    │
                       │   submits           │
                       └──────────┬──────────┘
                                  │ user clicks Override (or Enter)
                                  ▼
                       ┌────────────────────┐
                       │ submitting           │
                       │ - submit DISABLED + │
                       │   spinner inside    │
                       │ - cancel DISABLED   │
                       │ - all inputs        │
                       │   read-only         │
                       │ - aria-busy=true on │
                       │   DialogContent     │
                       └──────────┬──────────┘
                       ┌──────────┴──────────┐
                       │                     │
                  success                 server error
                       │                     │
                       ▼                     ▼
            ┌────────────────────┐  ┌────────────────────┐
            │ submitted-success   │  │ server-error        │
            │ - Sonner toast OK   │  │ - inline error      │
            │ - dialog closes     │  │   above CTA showing │
            │ - parent surface    │  │   server message    │
            │   re-renders with   │  │ - submit re-enables │
            │   permanent badge   │  │ - state = ready     │
            └────────────────────┘  │   (user can retry)   │
                                    └────────────────────┘
```

**Cancel / ESC behaviour:**
- `idle` (no input) + Cancel/ESC → close immediately, no confirm.
- Any state with `reason.length > 0` OR `ack === true` + Cancel/ESC → opens nested `AlertDialog` (`Offboarding.OverrideDialog.discardConfirm.*`); discarding closes both; "Keep editing" returns focus to textarea at the prior caret position.
- Backdrop click follows the same dirty-check rule as ESC.

**Server-side validation contract (load-bearing, not UI):**
- `requirePermission({ workflow: ['override_blocking_task'] })` middleware gates the mutation; client-side `getCurrentUserPermissions` UI-hide is decoration only.
- `zod.object({ reason: z.string().min(20).max(2000), acknowledgement: z.literal(true) })` re-validates server-side. UI shows `Offboarding.OverrideDialog.reasonServerError` on Zod failure (which is a defense-in-depth case, not the normal path).
- Mutation runs in a single `$transaction` writing `WorkflowRun.overrideMetadata` JSONB + `AuditLog` row + `WorkflowTaskRun.IP_VERIFICATION → SKIPPED` (per RESEARCH.md architecture diagram).

## RTL Safety Statement

`components.json` declares `"rtl": false` for the project, AND the OFFB scope explicitly excludes Arabic per ROADMAP. Despite that:

1. **All Tailwind margin/padding classes on Phase 74 surfaces MUST use logical properties** (`ms-*`, `me-*`, `ps-*`, `pe-*`) — never `ml-*`, `mr-*`, `pl-*`, `pr-*`. This is the codebase-wide convention being progressively enforced by a Phase 79 ESLint guard.
2. **Icon placement inside badges** uses the `data-icon="inline-start"` / `data-icon="inline-end"` slot attributes recognized by the existing badge variant CSS (`has-data-[icon=inline-start]:ps-1.5`, `has-data-[icon=inline-end]:pe-1.5`). Never raw `flex flex-row` with hardcoded `mr-1.5`.
3. **Form labels** sit `mb-2` (block-level, RTL-safe) above their inputs, never to the side.
4. **Override dialog footer** uses `gap-2` between Cancel and Override; the order of buttons in DOM is `[Cancel, Override]`. shadcn `DialogFooter` already applies `flex-col-reverse sm:flex-row sm:justify-end` so the primary CTA appears on the inline-end side in LTR — this is RTL-safe because `justify-end` flips logically.
5. **Tabs (en/pl/de)** order in DOM is alphabetical (`de`, `en`, `pl`) — visual order is determined by `flex` direction which is RTL-aware via the browser. Active-tab is keyboard-driven, not DOM-position-driven, so RTL flip is harmless.

A Phase 79 lint pass should report zero violations on the components introduced by Phase 74. The planner enforces this in code-review checklists.
