<objective>
Implement the Entra ID `Deprovisionable` adapter — the most complex of the three because it carries TWO novel pre-flight gates beyond the standard suspend/revoke/verify. Following the existing `OutlookCalendarAdapter` raw-Microsoft-Graph pattern (same IdP host, no new SDK), the adapter:
1. `suspendAccount` → `PATCH /users/{id} {accountEnabled:false}`, but FIRST runs a hybrid-AD pre-flight (`onPremisesSyncEnabled`) that HARD-BLOCKS the deprovision when on-prem AD is authoritative (CONTEXT.md D-02, SC#4).
2. `revokeAllSessions` → `POST /users/{id}/revokeSignInSessions`, then a single delayed `signInActivity` poll for forensic verification (CONTEXT.md D-03).
3. `verifyDeprovisioned` → `accountEnabled === false` / 404 → LIKELY_GONE (D-14).
4. `describeImpact` → enumerates Conditional Access policies and surfaces those that could override the revoke as a NON-BLOCKING warning (CONTEXT.md D-01, Pitfall 14), plus rich license/group/device/app-role metrics.

Output: 1 adapter class + Entra tests flipped GREEN.
</objective>

**suspendAccount(externalUserId):**
1. Pre-flight hybrid-AD read: `GET https://graph.microsoft.com/v1.0/users/{id}?$select=accountEnabled,onPremisesSyncEnabled` (use `withResilience`, read retries OK). On 404 → return SUCCEEDED with USER_NOT_FOUND-equivalent (LIKELY_GONE) and make no write.
2. If `onPremisesSyncEnabled === true` → return FAILED with `failureKind: 'HYBRID_AD_AUTHORITATIVE'` (add this kind to the `DeprovisionFailureKind` union if it doesn't already exist; if the union is locked by Phase 76, reuse the closest kind such as `PROVIDER_REFUSED`/`PRECONDITION_FAILED` and set a descriptive `errorMessage: 'On-prem AD authoritative — revoke at source'`). Make NO PATCH call. Record request/response hashes for the read so audit captures the block decision.
3. Otherwise `PATCH https://graph.microsoft.com/v1.0/users/{id}` body `{ accountEnabled: false }` with `client-request-id`. Idempotent → retries allowed. Compute request/response SHA-256 (canonicalize, excluding Authorization). Map status via `classifyError('ENTRA_ID', status, body)` → DeprovisionResult (2xx/204 → SUCCEEDED; 404 → LIKELY_GONE; 401/403/429 per classifier).

**revokeAllSessions(externalUserId):**
1. `POST https://graph.microsoft.com/v1.0/users/{id}/revokeSignInSessions` (no body). 2xx → proceed.
2. After a single short delay (configurable const, default ~2s; in tests use fake timers), `GET /users/{id}?$select=signInActivity` ONCE; record `signInActivity.lastSignInDateTime` in the result's audit detail. This poll is supplementary forensic data — never flip the step to FAILED based on it.
3. Return SUCCEEDED with hashes; classify non-2xx via `classifyError('ENTRA_ID', ...)`.

**verifyDeprovisioned(externalUserId):** `GET /users/{id}?$select=accountEnabled`; `accountEnabled === false` → true; 404 → true; else false (errors → false, never throw).

**describeImpact(externalUserId):** Build the `ENTRA_ID` ImpactPreview member:
- `GET /users/{id}?$select=accountEnabled,onPremisesSyncEnabled,assignedLicenses,signInActivity` for accountStatus + onPremisesSyncEnabled + assignedLicenseSkus.
- `GET /identity/conditionalAccess/policies`; for each policy with `state === 'enabled'`, compute `appliesToUser` = (`conditions.users.includeUsers` contains the user id OR `'All'`) AND not in `excludeUsers`; `hasSessionControls` = `sessionControls != null`. Push `{ displayName, state, appliesToUser, hasSessionControls }`.
- Best-effort counts (`groupMembershipCount` via `/users/{id}/memberOf/$count` or transitiveMemberOf, `registeredDeviceCount` via `/users/{id}/registeredDevices/$count`, `appRoleAssignmentCount` via `/users/{id}/appRoleAssignments/$count`); unreachable → 0.
- `commonMetrics`: externalUserId, displayName, accountStatus (ACTIVE/SUSPENDED from accountEnabled), sessionCount: null (Graph has no live session count).
- Set `fetchedAt` (ISO), `cacheKey` = `co:idp:preview:ENTRA_ID:{externalUserId}` (the saga/preview layer owns the actual Upstash caching per Phase 77 D-02/D-04; the adapter just produces the payload + cacheKey).
- On preview-fetch error → throw a typed error the saga maps to the Phase 77 D-03 "preview unavailable, proceed" flow (do not silently return empty metrics).

Emit structured audit via `getIdpAuditLogger()` for suspend/revoke/block decisions (allow-listed fields only; no token). Use `@contractor-ops/logger`, never `console.*`.
