
**suspendAccount:** verify first (`getUser` → status DEPROVISIONED or 404 → return LIKELY_GONE short-circuit). Else `await client.userApi.deactivateUser({ userId: externalUserId, sendEmail: false })`. SUCCEEDED on resolve; on SDK error extract status → `classifyError('OKTA', status, body)`. Compute request/response SHA-256 (canonicalize a logical request descriptor `{ op:'deactivate', userId }` and the response/error, never the token).

**revokeAllSessions:** `await client.userApi.revokeUserSessions({ userId: externalUserId })`; SUCCEEDED on resolve; classify errors.

**verifyDeprovisioned:** `const u = await client.userApi.getUser({ userId: externalUserId })`; return `u.status === 'DEPROVISIONED'`. On 404 → true. On other error → false (never throw).

**describeImpact:** populate OKTA ImpactPreview member:
- `getUser` → accountStatus (ACTIVE/SUSPENDED/NOT_FOUND from status), displayName.
- `client.userApi.listUserGroups`/equivalent → groupMembershipCount.
- factors list → enrolledFactorTypes.
- app links / `listAppLinks` → assignedAppCount.
- admin roles list → adminRoles.
- linked IdPs → linkedIdpCount.
All best-effort (unreachable → 0 / []). commonMetrics.sessionCount: null. cacheKey `co:idp:preview:OKTA:{externalUserId}`, fetchedAt ISO. On total preview failure → throw typed error for Phase 77 D-03 proceed-without-preview flow.
