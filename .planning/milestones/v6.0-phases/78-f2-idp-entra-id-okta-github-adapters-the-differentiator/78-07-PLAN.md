<ui_design_contract>
Frontend-design contract (no separate UI-SPEC.md — this block is the contract; follow the `frontend-design` skill).

**Layout & hierarchy:** Each provider section is a `Card` matching the existing GWS/Jira/Linear provider cards (brand icon `size-8`, title, description, status pill, primary action). Disconnected state: brand icon + one-line description + "Connect" primary button. Connected state: status pill (CONNECTED/REAUTH_REQUIRED/ERROR) + "Test connection" + "Disconnect" + (on 401) a reconnect banner identical in shape to `google-workspace-reconnect-banner.tsx`.

**Entra-specific:** when `describeImpact` data is available in a preview context, conditional-access policies render as a non-blocking warning list (informational icon, not destructive); the hybrid-AD hard-block renders as a destructive/blocking banner ("On-prem AD authoritative — revoke at source") with a link to the status panel. (The actual preview/saga UI is Phase 76/77's surface; this plan only ensures the connection card and any Entra-specific banner copy exist; do not rebuild the saga preview.)

**Compliance toggle table:** rows = providers; columns = Provider (icon+name) | Connection Status (pill) | Flag Status (PENDING/APPROVED pill) | Enable (Switch). The Switch is disabled with a tooltip "Awaiting compliance sign-off" when the flag is not APPROVED. Use the canonical patterns; if a `DataTable` is warranted use the workbench `DataTable` (client pagination, `hideFooter` if ≤ a handful of rows) per ARCHITECTURE.md — but a simple semantic table/row layout is acceptable for a fixed 5-row provider list (prefer the existing compliance-fields card style over a heavy DataTable for 5 rows).

**States (mandatory):** loading → section-shaped skeleton (not full-page spinner); empty → not-connected CTA card; error → inline error with retry. Mutations → pending (disabled + spinner) + toast on success/failure.

**Accessibility (WCAG AA):** keyboard nav + visible focus on all controls; Switch has `aria-label` "{Provider} enable deprovisioning"; status conveyed by text+icon, never color alone; external repo links `target="_blank" rel="noopener noreferrer"` with descriptive text; respects RTL (ar locale) and existing AnimateIn entrance convention.

**i18n:** no hardcoded strings; keys under `Settings.integrations.{entra|okta|github}` and `Settings.compliance.idpDeprovisioning`.
</ui_design_contract>

| T-78-07-01 | tRPC call leaks into a presentational view/dialog → breaks the data-layer contract | All `useTRPC`/`useQuery`/`useMutation` live ONLY in the `hooks/use-*.ts`; views are props-in/JSX-out. `check:data-layer` + `check:web-vite-presentational` enforce. |

| T-78-07-03 | Credential entry form echoes the secret back / stores in client state longer than needed | Secret inputs are write-only (`type=password`, not persisted to URL/state beyond submit); response never contains the secret (78-06). |
| T-78-07-04 | Color-only status pills fail WCAG | Pills use icon + text label. |
| T-78-07-05 | Missing loading/error/empty states → blank screen on slow/failed fetch | Container branches on hook flags for all three states (ARCHITECTURE.md decisive-container rule). |
| T-78-07-06 | External GitHub repo links open without rel=noopener (reverse-tabnabbing) | All external links use `rel="noopener noreferrer"`. |
</threat_model>
