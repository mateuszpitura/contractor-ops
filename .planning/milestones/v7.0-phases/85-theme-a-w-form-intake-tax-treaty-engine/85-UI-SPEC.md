## Spacing Scale

The repo already ships an 8-point scale plus three semantic vertical-rhythm tokens. Use the
semantic tokens for layout rhythm and the numeric scale for component-internal spacing.

| Token | Value | Tailwind | Usage in this phase |
|-------|-------|----------|---------------------|
| xs | 4px | `gap-1` / `p-1` | Icon тЖФ label gaps inside stepper items, pill icon gap |
| sm | 8px | `space-y-2` / `gap-2` | Label тЖФ control (`--space-stack-tight`); field error spacing |
| md | 16px | `space-y-4` / `p-4` | Rows inside a wizard-step card (`--space-card-gap`); card padding |
| lg | 24px | `space-y-6` / `gap-6` | Between top-level page sections (`--space-section-gap`); stepper nav gap |
| xl | 32px | `p-8` | Wizard container outer padding on тЙеsm |
| 2xl | 48px | `py-12` | Empty-state vertical breathing room |
| 3xl | 64px | `py-16` | Full-screen portal wizard top/bottom inset (mirrors `organization-onboarding`) |

Semantic tokens (preferred for rhythm): `space-y-stack-tight` (8px), `space-y-card-gap` (16px),
`space-y-section-gap` (24px) тАФ defined in `tokens.css`, bridged in `globals.css` `@theme inline`.

Exceptions:
- **Touch targets:** interactive controls (stepper next/back, reveal toggle, checkboxes) must be
  тЙе44├Ч44px effective hit area on touch (portal is contractor-facing/mobile). Use `size-9`+ wrappers
  or `min-h-11` on buttons; visual size may stay smaller but the hit area must meet 44px.
- **Radius:** `--radius: 0.625rem` (10px) base; cards use `rounded-xl`, fields `rounded-md`.
  No new radius values.

## Typography

The repo uses a **fluid `clamp()` scale** (`globals.css` `@theme inline`). Body base is 14px at
1.5 line-height (`body` rule). Declared roles for this phase (4 sizes, 2 weights):

| Role | Size token | Tailwind | Weight | Line Height | Usage |
|------|-----------|----------|--------|-------------|-------|
| Display (wizard title, step heading) | `--text-2xl` тЖТ `--text-3xl` (clamp ~22тАУ30px) | `text-2xl` / `text-3xl`, `font-display` | semibold (600) | 1.15 (`leading-tight`) | Wizard H1, success-receipt heading |
| Heading (step card title, section) | `--text-lg` (clamp ~16тАУ18px) | `text-lg`, `font-display` | semibold (600) | 1.75 | Per-step card `<h2>`, staff card title |
| Body (field values, help, instructions) | `--text-sm` / `--text-base` (clamp 13тАУ15px) | `text-sm` | regular (400) | 1.5 | Field help text, perjury-attestation paragraph, treaty explanation |
| Label / meta (field labels, status pill, last-4) | `--text-xs` тЖТ `--text-sm` (clamp 11тАУ14px) | `text-xs` / `text-sm` | medium (500) for labels, regular for meta | 1.25тАУ1.5 | Form `<Label>`, status badges, "Article 7" caption |

**Two weights only:** `font-normal` (400) for body/values, `font-semibold` (600) for headings and
the primary CTA. Labels may use `font-medium` (500) where the codebase already does (`Label`
default) тАФ this is the existing convention and counts as within contract.

**Numeric/monospace:** TIN/SSN last-4, FTIN, treaty rate %, and "Article N" render in
`font-mono tabular-nums` (matches `SsnMaskedReveal` and `hero-metric` precedent) so digits align.

## Color

60/30/10 split maps onto the existing token layer. **No new colors** тАФ the phase reuses the
teal-primary "Precision Craft" palette. Accent (teal `--primary`) is reserved, not sprayed.

| Role | Token | Tailwind | Usage |
|------|-------|----------|-------|
| Dominant (60%) | `--background` / `--surface-0` | `bg-background` | Page + wizard backdrop, form well |
| Secondary (30%) | `--card` / `--secondary` / `--muted` | `bg-card`, `bg-muted` | Wizard step cards, staff status cards, sidebar, read-only summary panels |
| Accent (10%) | `--primary` (Deep Teal `oklch(0.44 0.145 178)` light / `0.65` dark) | `bg-primary`, `text-primary` | See reserved list тЖУ |
| Destructive | `--destructive` | `bg-destructive`, `text-destructive` | Validation errors, "override the auto-rate" warning copy, supersede confirmation |

**Accent (teal `--primary`) reserved EXCLUSIVELY for:**
1. The single primary CTA per step (`Continue`, `Sign & submit`).
2. The active/current stepper indicator + the progress bar fill (`step-current-ring`, `[data-slot="progress-indicator"]`).
3. Focus-visible ring on inputs/buttons (`--ring` = teal; `focus-glow` / `[data-form-control]:focus-visible`).
4. The reveal toggle text/icon (`text-primary`, matching `SsnMaskedReveal`).
5. Inline links (e.g. "Why this rate?", treaty-article reference link).

Accent is **never** used for: every interactive element, body text, card borders, secondary
buttons (those use `variant="secondary"` / `variant="outline"` / `variant="ghost"`), or decoration
on the staff table rows.

**Semantic status colors (already defined тАФ use, don't redefine):**
- `--success` (green) тАФ form status `ACTIVE`, treaty `verified`, "submitted" receipt.
- `--warning` (amber) тАФ form `expiring`/`expired`, treaty rate defaulted to 30% statutory (no treaty), override-pending.
- `--info` (blue) тАФ `DRAFT` status, informational treaty notes.
- `--destructive` тАФ errors, superseded, validation failures, the override-reason requirement.

Status pills MUST reuse the `Badge` variants (`success` / `warning` / `secondary` / `destructive`)
exactly as `UspsAddressStatusPill` does тАФ same icon+label+tooltip idiom, same `data-status` attr.

**Adviser-deferred banner (D-04):** every seeded treaty rate/article shown to a user carries a
muted, non-alarming informational note ("Rates pending verification by a tax adviser") rendered in
`text-muted-foreground` with an `info`-toned icon тАФ NOT destructive/warning styling (it is a posture
note, not an error).

## Copywriting Contract

All strings are i18n keys across **en / en-US / de / pl / ar(RTL)** (D-13). en-US inherits en via
fallback-parity (P84 D-04); de/pl/ar require exact-parity keys in the same change set or `i18n:parity`
red-CIs. Copy below is the **en source-of-truth**; verb+noun and tone are the contract.

| Element | Copy (en source) |
|---------|------------------|
| Wizard entry CTA (portal landing / staff "request") | **Start tax form** (contractor) / **Request tax form** (staff) |
| Determination step heading | **Confirm your tax form** |
| Determination body | *"Based on your profile we'll prepare your **{formName}**. If this isn't right, you can change it below."* (formName = W-9 / W-8BEN / W-8BEN-E) |
| Step primary CTA (intermediate) | **Continue** |
| Final step primary CTA | **Sign & submit** |
| Back / secondary | **Back** (ghost/outline; never accent) |
| Save-and-finish-later | **Save draft** |
| Treaty auto-populate caption | *"Treaty rate applied automatically: **{rate}%** under **{article}** ({country})."* |
| Treaty no-match (30% default) | *"No tax treaty applies for **{country}** тАФ the default **30%** statutory rate is used."* (warning tone) |
| Treaty override affordance (staff) | **Override rate** тЖТ requires a reason field labeled *"Reason for override (required)"* |
| Perjury attestation (required checkboxes) | Reproduce IRS "under penalties of perjuryтАж" language verbatim per form (W-9 Part II / W-8BEN Part III / W-8BEN-E Part XXX); the **certification text is a domain string, not freely reworded copy** тАФ adviser-deferred (D-04/D-11). |
| Typed-name field label | **Type your full legal name to sign** |
| Submission success heading | **Tax form submitted** |
| Submission success body | *"Your {formName} is on file. We've recorded your signature on {date}. You can view or update it anytime."* (links to read-only receipt) |
| Empty state heading (portal, no forms yet) | **No tax form on file** |
| Empty state body (portal) | *"Complete your tax status form so we can pay you correctly. It takes a few minutes."* + **Start tax form** CTA |
| Empty state heading (staff, no forms for contractor) | **No tax form submitted yet** |
| Empty state body (staff) | *"This contractor hasn't completed a W-form. Request one to start the process."* + **Request tax form** CTA |
| Loading state | Skeleton (`animate-shimmer`) for the wizard shell + per-step card; `Loader2` spinner inside the CTA during submit ("SubmittingтАж"). Never a blank screen. |
| Error state (load failure) | **Couldn't load your tax form** тАФ *"Something went wrong. Refresh to try again."* + **Retry** action |
| Error state (submit failure) | Inline `role="alert"` under the CTA: *"We couldn't submit your form. Check your entries and try again."* тАФ preserves entered data, never wipes the wizard |
| Field validation error | Per-field `role="alert"` `text-destructive` text below the input (mirrors `organization-onboarding`) |

**Destructive / high-stakes actions + confirmation approach:**

| Action | Confirmation |
|--------|--------------|
| **Sign & submit** (creates an immutable, legally-attested record) | Final-step explicit gate: all perjury checkboxes checked + typed legal name matches + an explicit "I understand this is a legal signature" affirmation before the CTA enables. This is intentional friction, not a destructive-delete dialog. |
| **Re-certify / replace an active form** (supersedes the prior signed record) | `AlertDialog` confirmation: *"Submitting a new form will replace your current {formName} on file. The previous version is kept for records but no longer active. Continue?"* тАФ Confirm = `default`, Cancel = `outline`. (Append-only; never a hard delete.) |
| **Override the auto-resolved treaty rate** (staff only) | Inline gate: the **reason field is required** and the CTA stays disabled until a reason is entered; submit writes `writeAuditLog`. Warning-toned helper text explains the auto-detected value is preserved for audit. |
| **Reveal full SSN** (staff summary, W-9 records) | Reuse `SsnMaskedReveal` exactly тАФ gated by `contractorPii:read`, audit-logged on the server, never revealed on hover/focus/mount, control ABSENT (not disabled) without permission. |

No literal record-deletion exists in this phase (records are immutable + supersede-chained), so no
"delete" destructive dialog is specified.

## State & Accessibility Contract (mandatory per surface)

Every wizard step and the staff surface MUST specify all four states and meet WCAG:

| Surface | Loading | Empty | Error | A11y notes |
|---------|---------|-------|-------|-----------|
| Portal wizard shell | Skeleton card (`animate-shimmer`) while determination/profile loads | "No tax form on file" + Start CTA (pre-wizard) | "Couldn't load your tax form" + Retry | Stepper `aria-label` (wizard progress), `aria-current="step"` on active item; focus moves to the new step's heading on Continue/Back; each field `aria-invalid` + `aria-describedby` error wiring (mirror `organization-onboarding`) |
| Per-step form | Disabled inputs + CTA spinner during async validation/submit | n/a (always has fields) | Inline `role="alert"` per field + a step-level submit error region | Semantic `<form>`, `<Label htmlFor>`, `noValidate`, required-checkbox group has a group label |
| Attestation step | CTA spinner "SubmittingтАж" on submit | n/a | `role="alert" aria-live="polite"` submit-failure region, data preserved | Perjury checkboxes are real `<input type=checkbox>` with visible labels; typed-name field accessible name = visible label (Label-in-Name) |
| Treaty claim display | Inline skeleton on the rate/article line while resolving | n/a (auto-populated or 30% default) | Falls back to 30% statutory + warning note if lookup fails | Rate/article announced via `aria-live="polite"` when it auto-populates so SR users hear the change |
| Staff status surface | Table/card skeleton rows | "No tax form submitted yet" + Request CTA | Card-level error + Retry | Status pills carry `aria-label`; full SSN absent (not just hidden) without `contractorPii:read` |

**RTL (ar):** the wizard MUST work right-to-left. Use logical properties already in the codebase
(`ms-*`/`me-*`, `ps-*`/`pe-*`, `text-start`/`text-end`), and directional icons flip via `rtl:rotate-180`
(see `organization-onboarding` `ArrowRight`). Stepper separators and progress direction follow `dir`.
Noto Sans Arabic auto-applies under `:root[dir="rtl"]`. No `ml-/mr-/pl-/pr-/text-left/right` literals.

**Contrast:** all text/status combinations use the token pairs (`*-foreground` on its surface) which
already meet WCAG AA in this design system; status badges keep their paired foregrounds. Do not place
`text-primary` body text on `bg-primary`.
