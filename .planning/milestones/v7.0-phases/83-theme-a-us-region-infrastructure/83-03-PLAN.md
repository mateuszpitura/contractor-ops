`R2_BUCKET_NAME_US` to the env schema + `.env.example`, mirroring the `DATABASE_URL_US`
optional/lazy posture from Phase 82.

Purpose: US-INFRA-02's success criterion — US-org files (including tax archives) land
in a US-region R2 bucket for data residency. ONE US regional bucket holds all US-org
files (D-03 — NOT a separate dedicated tax bucket). The app boots/tests clean locally
with no US bucket (lazy-throw only on actual US-org file access).
Output: widened bucket map, optional `R2_BUCKET_NAME_US` env, and the regional-storage
test covering US resolution + lazy-throw.
