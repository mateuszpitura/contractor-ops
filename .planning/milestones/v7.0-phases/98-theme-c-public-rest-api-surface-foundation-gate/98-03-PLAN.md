Establish the scope taxonomy the whole write surface enforces against. Two moving parts: (1) widen `PUBLIC_API_SCOPES` to the granular `resource:action` strings for all 9 entities' reads + the 7 entities' writes, and (2) add the two missing READ-only auth resources (`classification`, `auditLog`) so the classification-read and audit-log-read gates type-check. This keeps the 4 existing read scopes valid and defers the display taxonomy to Phase 99.

**Locked entity write/read split:** the 7 write entities = contractors, invoices, payments, payment_runs, workflows, workflow_tasks, compliance_documents. classifications + audit_log are READ-ONLY (matches INTEG-AUTH-02 which lists `classifications:read` / `audit:read`, and the founder GTM posture that de-emphasizes classification-as-verdict). payment_runs reuse the `payment:*` resource; workflow_tasks reuse `workflow:*`; compliance_documents reuse `document:*`.

Output: the extended scope registry + the two read-only auth resources + the populated BFLA test table (RED until 98-09).
