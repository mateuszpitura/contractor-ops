<objective>
Build the reusable pagination + filtering + versioning primitives every list route (existing + net-new) will consume. These are pure, framework-thin helpers so the route plans (98-06/07/08) are assembly, not invention:
- **Opaque cursor** (`openapi-cursor.ts`) — a stateless base64url envelope over the internal row-id cursor (`pagination.ts` `cursorClause`), tamper-safe (BAD_REQUEST on garbage).
- **Bracket-filter/sort parser** (`parse-list-query.ts` extension) — turns flat `filter[status]=X` Hono query keys into a nested object BEFORE `.strict()` validation.
- **Shared `.strict()` list DTOs** (validators) — `publicListBaseSchema` (cursor+limit) + the per-entity filter/sort extension pattern + the `{data,meta:{nextCursor,hasMore}}` envelope, replacing the offset `paginationSchema` for public lists.
- **Version headers** (`version-headers.ts`) — RFC 8594 `Sunset`/`Deprecation`/`Link` mechanism (dormant for v1).

Honor the 98-01 SPIKE VERDICT for WHERE the DTOs live: VERDICT A → author in validators; VERDICT B → author the route-facing schemas in the app layer with the package `z`, keep the validators schemas for the tRPC `.input()`.
