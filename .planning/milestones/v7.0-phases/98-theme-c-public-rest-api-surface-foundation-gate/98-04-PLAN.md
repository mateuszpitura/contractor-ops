<objective>
Build the per-org `module.public-api` dark gate (D-05). Today `app.ts`/`apiKeyTenantProcedure` do NOT evaluate the flag per resolved tenant — any Enterprise key reaches the existing reads regardless of the flag. This plan closes that: a middleware chained into `apiKeyTenantProcedure` re-evaluates `module.public-api` for the caller's org/region and returns 404 when off, so the entire public surface (existing reads + the net-new reads + the dark writes) is invisible to any org not explicitly granted the module. This is what keeps the whole write surface dark until Phase 99 flips the flag on.

Output: the flag-gate middleware wired into the shared procedure + the Wave-0 flag security test turned GREEN.
</objective>
