# Phase 84 — UI Design Contract

> Visual and interaction contract for the US contractor profile section + en-US locale. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Consistency phase — NOT a new design language.** Everything below mirrors the existing `UkComplianceFields` / `DeComplianceFields` / `VatValidationStatusPill` surfaces inside `CountryComplianceSection`. The contract's job is to bind the *new* primitives (SSN gated reveal, USPS advisory states, EIN/SSN inline errors, en-US formatting) to the design system already in the tree. Do NOT invent tokens, components, or copy patterns the compliance section does not already use.

## Spacing Scale

Declared values (must be multiples of 4 — matches existing compliance section Tailwind usage):

| Token | Value | Usage (as used in `uk-compliance-fields.tsx`) |
|-------|-------|------------------------------------------------|
| xs | 4px | `gap-1` icon/label gaps; `me-1`/`ms-1` required-asterisk spacing |
| sm | 8px | `space-y-2` label→input stack; `gap-2` |
| md | 12px | `p-3` / `px-3 py-2` info-box + validation-section padding (Tailwind `3` = 12px) |
| md+ | 16px | `space-y-4` between fields; `gap-3`/`gap-4` |
| lg | 24px | `mt-6` block separators (classification / engagements) |

Exceptions:
- The compliance section uses the Tailwind 4-step scale natively (`2`=8, `3`=12, `4`=16, `6`=24). The **12px** step (`p-3`) is load-bearing for the info-box and the validation-status row — keep it; do not "round" the USPS suggestion box up to 16px.
- Interactive controls (`Input`, `select`, reveal button) are **36px tall** (`h-9`) per the shadcn `base-nova` control height. Keep all new controls at `h-9` for row alignment.

## Color

Inherit the "Precision Craft" semantic tokens (`@contractor-ops/ui/styles/tokens.css` + web-vite `globals.css`). **Use semantic CSS variables only — never raw hex/oklch in components.**

| Role | Token | Light value (oklch) | Usage |
|------|-------|---------------------|-------|
| Dominant (60%) | `--background` / `--card` | `0.985 0.003 85` / `1 0 0` | Page + card surface behind the US fields |
| Secondary (30%) | `--muted` / `--secondary` | `0.955 0.004 250` / `0.955 0.005 85` | Info box (`bg-muted/30`), USPS suggestion box, validation-status row |
| Accent (10%) | `--primary` (Deep Teal) | `0.44 0.145 178` | **Reserved list below** |
| Destructive | `--destructive` | `0.52 0.23 18` | Required-asterisk, `FieldError` text, EIN/SSN invalid-format errors |
| Success (status) | `--success` / Badge `success` | `0.55 0.17 150` | USPS **Verified** pill only |
| Warning (status) | `--warning` / Badge `warning` | `0.7 0.16 65` | USPS **Unverified / USPS-unavailable** pill; "incomplete fields" header badge |

**Accent (teal `--primary`/`--ring`) reserved for:**
- The single primary **Save** button (`<Button>` default variant) — one per section, already present.
- Focus rings on inputs/select/buttons (`focus-visible:ring-ring`).
- The SSN **"Reveal"** control as a low-emphasis affordance (`variant="ghost"` or `variant="link"` text button — teal text/icon, NOT a filled teal button; reveal must read as secondary to Save).

Accent is **never** applied to: field backgrounds, the masked SSN display, status pills (those use success/warning/destructive), the USPS suggestion box, or "Accept suggestion / Keep mine" — those are neutral/secondary or outline buttons.

## Component Contracts (phase-specific — the load-bearing additions)

### A. `UsComplianceFields` (US-FIELD-04, D-06)

- Layout + chrome identical to `UkComplianceFields`: top dashed `bg-muted/30` info box (`t('infoItem1..3')`), then a `space-y-4` field stack, all i18n via `useTranslations('Contractors.compliance.us')`.
- Fields in order: **Entity type** (`EntityTypeSelect`, US enum) → **EIN** (`Input`, `inputMode="numeric"`, `XX-XXXXXXX` placeholder) → **SSN** (masked-input + gated reveal, section B) → **US address** (street / city / state / ZIP + USPS status, section C).
- Every field: `RequiredLabel` (asterisk via `text-destructive` when required), `aria-required`, `aria-invalid`, `aria-describedby` → `FieldError` (`role="alert" aria-live="polite"`, `text-xs text-destructive`). Reuse the existing `RequiredLabel` / `FieldError` helpers verbatim — do not redefine.
- Register `case 'US'` in `CountryFieldsDispatch` and add `US: t('countries.US')` to the `COUNTRY_LABELS` map in `country-compliance-section.tsx`.

### B. SSN masked input + gated reveal (US-FIELD-02, D-01/D-02)

This is the highest-sensitivity surface in the phase. Three distinct render states:

| State | Who sees it | Render |
|-------|-------------|--------|
| **Masked (default)** | everyone with field access | Read-only display `•••-••-1234` (`font-mono`/`tabular-nums`, `text-sm`), a small `text-xs text-muted-foreground` hint ("Showing last 4 digits"). NO full value in the DOM. |
| **Reveal available** | roles with `CONTRACTOR_PII:READ` (owner / admin / finance) | Plus a low-emphasis **"Reveal SSN"** ghost/link button with a lucide `Eye` icon (teal text). |
| **Reveal absent** | manager / member / viewer / it_admin | Masked display only — the reveal control is **not rendered at all** (not disabled-greyed; absent). No tooltip teasing a hidden value. |
| **Revealed (transient)** | after explicit reveal click + successful audit-logged fetch | Full `XXX-XX-XXXX` shown (`font-mono`), button toggles to **"Hide SSN"** with lucide `EyeOff`. Revealing is an explicit, **audit-logged action** (server `writeAuditLog`) — never on hover, never on focus, never auto on mount. |
| **Reveal loading** | between click and server response | Button shows lucide `Loader2 animate-spin` + disabled (mirror the Save button's pending pattern). |
| **Reveal error** | server denies / fails | `FieldError`-style `text-xs text-destructive` inline: copy in Copywriting Contract. Fall back to masked display. |
| **Input (create/edit unset SSN)** | user entering a new SSN | Standard `Input` with `inputMode="numeric"`, `maxLength` for `XXX-XX-XXXX`, `aria-invalid` on range/format error. Once stored, returns to masked state on reload. |

A11y: reveal button has an explicit `aria-label` ("Reveal Social Security Number"); the masked value carries `aria-label="Social Security Number, last four digits 1234"` so screen readers do not read the bullet glyphs. The reveal toggle's pressed state uses `aria-pressed`. Keyboard: reveal/hide fully operable via Enter/Space; focus stays on the toggle after reveal.

### C. USPS address validation states (US-FIELD-03, D-03 — advisory, NEVER blocks save)

Mirror `VatValidationStatusPill` (same `Badge` + variant map + optional `Tooltip`). A new pill component (e.g. `UspsAddressStatusPill`) with these statuses:

| Status | Badge variant | lucide icon | Tooltip |
|--------|---------------|-------------|---------|
| `verified` | `success` | `CheckCircle2` | "USPS-verified (CASS) {relative} ago" |
| `unverified` | `warning` | `AlertTriangle` | "Not verified against USPS — saved as entered" |
| `validating` | `secondary` | `Loader2 animate-spin` | — (transient) |
| `unavailable` (USPS down / 60-req-hr throttle hit) | `warning` | `WifiOff` | "USPS check temporarily unavailable — saved unverified; you can re-validate later" |
| `not-validated` (never run) | `secondary` | `Minus` | — |

**Normalized-suggestion affordance** (when USPS returns a CASS-normalized form differing from input): a secondary `bg-muted/30 rounded-md border p-3` box below the address fields showing the normalized address as `text-sm`, with two buttons:
- **"Use USPS version"** — `Button variant="outline"` (neutral, NOT teal) — applies the normalized fields.
- **"Keep what I entered"** — `Button variant="ghost"` — dismisses, keeps `unverified`.

The suggestion box is informational; dismissing it or ignoring it **never prevents Save**. The validation-status row reuses the existing `flex flex-wrap items-center gap-3 rounded-md border bg-muted/30 p-3` pattern already used for VAT (`data-testid="vat-validation-section"` analog: `usps-validation-section`). Add a **"Re-validate address"** trigger mirroring `RevalidateVatButtonContainer`.

### D. EIN / SSN inline validation errors (US-FIELD-01 / US-FIELD-02, D-05)

All inline, non-blocking-of-typing, surfaced through the existing `FieldError` (`text-xs text-destructive`, `role="alert"`). Distinct messages (see Copywriting): invalid EIN format, invalid IRS prefix, invalid SSN format, invalid SSN range (area `000`/`666`/`900–999`, group `00`, serial `0000`). Field gets `aria-invalid="true"` + red ring (`border-destructive ring-2 ring-destructive/20` — same as `EntityTypeSelect` error styling).

### E. en-US locale rendering (US-LOC-01, D-04)

- Locale switcher gains **English (US)** as a selectable option (add `'en-US'` to `SUPPORTED_LOCALES` + `localeMeta`). Switcher item label: native "English (US)" with a US flag/region indicator consistent with how existing locales render in the switcher — do NOT redesign the switcher; add one entry.
- **Date formatting:** `MM/DD/YYYY` via `Intl.DateTimeFormat('en-US')`. Any date the US profile renders (e.g. validation timestamps via `date-fns`/Intl) uses American order.
- **Currency:** `$` prefix, `1,234.56` grouping via `Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' })`.
- **Measure/number:** comma thousands separator, period decimal.
- **Copy:** American spelling on divergent keys only (en-US is a thin override inheriting `en` via `fallbackLng: en-US → en → pl`). The contract: every new label/state/error string added this phase exists in `en` AND has a US-divergent entry in `en-US.json` **only where spelling/format actually differs** (e.g. "ZIP code" not "postcode"; "Social Security Number"). No string is hardcoded.
- **RTL invariant:** en-US is LTR; adding it must not regress the existing `ar` RTL. All new components use logical properties already in the codebase (`ms-*`/`me-*`/`ps-*`/`pe-*`, `text-start`) — never `ml-*`/`pl-*`/`text-left`.

| State | Contract |
|-------|----------|
