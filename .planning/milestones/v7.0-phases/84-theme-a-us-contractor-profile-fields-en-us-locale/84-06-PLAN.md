<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| browser → reveal mutation | the reveal control triggers a server-gated audit-logged decrypt; the client cannot bypass the gate |
| rendered DOM | the full SSN must not be in the DOM except transiently after an explicit reveal |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-84-06-01 | Information Disclosure | SSN in DOM | mitigate | Masked last-4 only by default; full value enters the DOM only after an explicit, server-audited reveal; reveal control ABSENT (not disabled) without permission (UI-SPEC §B; test asserts no-full-value-in-DOM) |
| T-84-06-02 | Elevation of Privilege | client-side reveal gate | mitigate | The control is a hint only — the server `revealSsn` (Plan 05) enforces contractorPii:read; client absence is UX, not the security boundary |
| T-84-06-03 | Denial of Service | USPS blocking save | mitigate | USPS pill + suggestion are advisory; dismiss/ignore/unavailable never prevents Save (UI-SPEC §C, D-03) |
| T-84-06-04 | Tampering | hardcoded/untranslated PII copy | mitigate | All copy via i18n keys (no hardcoded strings, CLAUDE.md); i18n:parity gate; en-US fallback-aware |
| T-84-SC | Tampering | npm/registry blocks | accept | No third-party registry blocks — first-party @contractor-ops/ui primitives only (UI-SPEC Registry Safety: gate not triggered) |
</threat_model>
