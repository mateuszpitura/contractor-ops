<objective>
Implement the ACH return-code subsystem's service layer (Gap C part 1): parse a NACHA return file into structured return entries, map each R-code / correction code to a disposition, and apply the returns idempotently to the live PaymentRunItems — flipping bounced credits to FAILED with an audited reason. This closes the return-code clause of ROADMAP Success Criterion #1 at the logic level; 88-11 wires the reachable entry point.

Purpose: A bounced ACH credit (R01/R02/R03…) must flip its PaymentRunItem back to FAILED so the operator can retry — today nothing does this. Money movement: correctness + idempotency + audit are non-negotiable. An operator must also be able to tell a clean no-bounce run from a mis-uploaded/wrong-run file — hence the `unmatched` signal.
Output: A fully implemented, unit-tested ach-return.service.ts; the 88-08 parse/map RED tests plus a new idempotent-apply RED→GREEN pair are green.
</objective>
