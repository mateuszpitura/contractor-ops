<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| external API key → mutation | Untrusted body; tenant from key; scope mandatory; invariants reused not reimplemented |
| request body → Prisma data | `.strict()` DTO + explicit allowlist prevent org/workerType/money mass-assignment |
| mutation → audit trail | Every external mutation audited as API_KEY + sourceIp/userAgent + actingUserId (non-repudiation) |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99-04-01 | Elevation of Privilege | Write reachable without a scope (BFLA) | mitigate | Mandatory `requirePermission` on EVERY write; live 403 matrix asserts 403-without-scope |
| T-99-04-02 | Tampering | Mass-assignment of organizationId/workerType/money | mitigate | `.strict()` DTOs omit them; explicit allowlist; money server-derived; tenant from ctx |
| T-99-04-03 | Tampering | Reimplemented FSM/money diverging from invariants (double-spend) | mitigate | Import + reuse payment-shared / workflow-shared; never reimplement VALID_TRANSITIONS/export |
| T-99-04-04 | Repudiation | External mutation without full audit | mitigate | writeAuditLog(API_KEY, apiKeyId, sourceIp, userAgent, actingUserId) inside the tx; lint-enforced |
| T-99-04-05 | Elevation of Privilege | Payout initiation exposed with wrong actor coupling | mitigate | `_initiatePayoutForRun` NOT exposed (deferred) |
| T-99-04-SC | Tampering | npm installs | accept | No new deps |
</threat_model>
