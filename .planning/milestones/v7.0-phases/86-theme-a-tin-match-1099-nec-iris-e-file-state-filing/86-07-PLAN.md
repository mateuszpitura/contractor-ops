<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| component → hook → tRPC | The hook is the only place client data crosses to the API; components never call tRPC |
| recipient input → consent step | The consent checkbox state is client-side, but the audited consent (ip/actorId/timestamp) is server-derived |
| TIN → DOM | Full TIN must never reach the DOM; last-4 only via SsnMaskedReveal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-86-07-01 | Information Disclosure | full TIN rendered in the UI/PDF preview | mitigate | `SsnMaskedReveal` (last-4, gated on `contractorPii:read`); full TIN never in the DOM (P84/P85 invariant). |
| T-86-07-02 | Tampering / Repudiation | client-supplied consent timestamp/IP | mitigate | The consent mutation input omits ip/actorId/timestamp; the server derives them (Plan 06) — the client cannot forge an attestation. |
| T-86-07-03 | Elevation | a UI affordance that hard-blocks 1099 generation on a TIN mismatch | mitigate | The mismatch surface is amber advisory with escalate/resolve only — NO blocking control (D-12, grep gate: no `destructive` on the mismatch list). |
| T-86-07-04 | Repudiation | a UI affordance that auto-files | mitigate | Generate and File are separate deliberate actions; the batch panel is review-before-file; no auto-file control (D-05). |
| T-86-07-05 | Compliance | Copy-B download offered without consent | mitigate | `copy-b-download` offers the download only when consent is stored; no consent → paper-copy message (D-09, Pub 1179 §4.6). |
| T-86-07-SC | Tampering | npm/pip/cargo installs | accept | No package installs this plan (RESEARCH § Package Legitimacy Audit + UI-SPEC Registry Safety: all primitives already installed; no new shadcn registry block pulled). No legitimacy checkpoint required. |
</threat_model>
