<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| Cookie → subject ctx | The middleware is the point where the session's subjectType becomes the request identity; an employee handler must never see `ctx.contractorId`. |
| Email → subject resolution | Resolution must not enumerate accounts and must never resolve a TERMINATED/deleted subject. |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-96-02-01 | Elevation of Privilege | a contractor token reaching an employee procedure (or vice versa) | mitigate | The middleware rejects on `subjectType` mismatch (Task 2); contractor regression test proves isolation. |
| T-96-02-02 | Spoofing | a TERMINATED employee re-authenticating | mitigate | `validatePortalSession` + `findEmployeesByEmail` reject TERMINATED/deleted (Tasks 1,3). |
| T-96-02-03 | Information Disclosure | account enumeration via the magic-link request | mitigate | `requestMagicLink` returns `{ success:true }` regardless of match (Task 3). |
| T-96-02-04 | Elevation of Privilege | a non-manager reaching the manager surface | mitigate | `portalManagerProcedure` asserts ≥1 report (Task 2); manager-IDOR test in 96-03/96-06. |
</threat_model>
