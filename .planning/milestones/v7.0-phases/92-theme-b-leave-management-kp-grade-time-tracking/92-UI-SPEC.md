# Phase 92 — UI Design Contract

> Visual and interaction contract for the Leave Management + KP-Grade Time Tracking staff/manager surfaces in `apps/web-vite`. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Binding stack:** `frontend-design` + `impeccable` + web-vite layering (Page = thin composer → wired section/`*PageContent` branches loading/empty/error → `hooks/use-*.ts` = sole tRPC boundary → `*View` presentational). This is a product dashboard — **never** apply `design-taste-frontend`. Reuse the existing `packages/ui` design system and tokens; do not invent a parallel visual language.
>

## Spacing Scale

Tailwind 4px base. Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | icon gaps (`gap-1`), badge internal padding |
| sm | 8px | `--space-stack-tight` — label+control, heading+body pairs (`space-y-2`) |
| md | 16px | `--space-card-gap` — rows inside a card/panel/section (`gap-4`, `space-y-4`) |
| lg | 24px | `--space-section-gap` — between top-level page sections (`gap-6`, `space-y-6`) |
| xl | 32px | major layout breaks |
| 2xl | 48px | page-level vertical rhythm |
| 3xl | 64px | rare hero spacing (not expected this phase) |

**Semantic rhythm tokens (use these, not ad-hoc `space-y-N` at page level):** `--space-stack-tight` (8), `--space-card-gap` (16), `--space-section-gap` (24). Raw `space-y-N` is allowed only for intrinsic component internals (table rows, dialog bodies, skeletons) — mirrors existing convention.

Fixed layout dimensions (inherited, not spacing tokens): review side-panel width `480px` (`w-[480px] sm:max-w-[480px]`); `DataTable` cap-mode body max-height `400px`; empty-state page min-height `60vh`; SLA/status badge min-width `72px`. Radius `--radius` = `0.625rem` (10px).

Exceptions: **team calendar (S2) day cells** — use a fixed min touch/click target of **44px** height per day cell for WCAG 2.5.5 pointer-target size (icon-dense capacity markers). This is the only new exception; record it in the calendar component.

## Typography

Contract weights: **2** — regular `400` (body) + semibold `600` (headings/emphasis, always paired with `font-display` for h1–h3 and KPI values).

| Role | Size | Weight | Line Height | Font | Where |
|------|------|--------|-------------|------|-------|
| Body | 14px (`text-sm`) | 400 | 1.5 (`leading-relaxed`/normal) | sans | table cells, panel copy, form values |
| Label / caption | 12px (`text-[12px]`) | 500* | 1.4 | sans, `uppercase tracking-wide` for group labels | field labels, muted section headings, badges, chain-step captions |
| Heading | 20px (`text-[20px]`) | 600 | 1.2 (`leading-tight`) | `font-display` | section/empty-state headings; **24px** for the page `<h1>` (`AtelierPageHeader`) |
| Display | 28px (`text-[28px]`) | 600 | 1.2 (`leading-[1.2]`) | `font-display` | KPI values (balance days, hours this week) in summary cards |

\* **Inherited exception:** the shared `packages/ui` primitives (badges, micro-labels, chips, chain captions) already use `font-medium` (500). This is an existing-system weight, **not introduced by this phase** — reuse it as-is on those primitives; do not add new 500-weight typography beyond what the primitives ship. New body/heading text uses 400/600 only.

Inherited size ramp also present and permitted where a reuse target already uses it: `10px` (uppercase eyebrow, bold), `11px` (group labels), `13px` (secondary body / chain rows). Do not introduce sizes outside this ramp.

## Color

OKLCh semantic tokens from `packages/ui/src/styles/tokens.css` + `status.css`. 60/30/10:

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--background` `oklch(0.985 0.003 85)` warm off-white | page background, surface-0 |
| Secondary (30%) | `--card` `oklch(1 0 0)` white + `--muted`/`--secondary` | cards, panels, sidebar/nav, table chrome, chip rests |
| Accent (10%) | `--primary` Deep Teal `oklch(0.44 0.145 178)` | see reserved list |
| Destructive | `--destructive` crimson `oklch(0.52 0.23 18)` (AA-tuned) | destructive actions only (reject, delete) |

**Accent (teal) reserved for — explicit list:**
- Primary CTA buttons: "Submit request", "Save entry", "Generate report", "Approve".
- Active/current approval-chain step circle + current-step marker (`bg-primary text-primary-foreground`).
- Active/selected filter chips and tabs (`border-primary/40 bg-primary/10`).
- Hyperlinks (`text-primary hover:underline`).
- Focus ring (`--ring` = teal).
- Emphasis on the primary balance figure in the leave side-panel.

Accent is **never** applied to: every interactive element, table rows, generic borders, or the whole calendar grid.

**Semantic status tokens (`--status-*` / `status.css`)** — the leave/time domain leans on these; they are NOT the 10% accent:

| Token | Value (light) | Domain meaning this phase |
|-------|---------------|---------------------------|
| `--status-success` | `oklch(0.55 0.17 150)` green | approved request, within-limit capacity, healthy balance |
| `--status-warning` | `oklch(0.7 0.16 65)` amber | SLA nearing, capacity near threshold, WT-limit **approaching**, night/overtime present |
| `--status-danger` | `oklch(0.58 0.23 18)` red | overdue SLA, WT-limit **breach**, calendar **conflict**, insufficient balance |
| `--status-info` | `oklch(0.55 0.18 260)` blue | informational (immutable/archived neutral state) |

Reuse the established badge idiom for status: `bg-{color}/10 text-{color}-800 dark:text-{color}-400` (see `sla-badge.tsx`, `side-panel.tsx` `statusBadgeColors`). **Never encode meaning by color alone** — always pair with icon + text label (conflict = `TriangleAlert` + "Conflict"; breach = icon + "Over limit"). WCAG AA is met by the `/10` background + `-800` foreground pairing already tuned in the codebase.

## Copywriting Contract

Top-level required contract (per template). Per-surface copy follows below.

| Element | Copy |
|---------|------|
| Primary CTA | S1 "Approve" / header "Record sick leave" · S3 "Save entry" · S4 "Generate report" |
| Empty state heading | "No leave requests to review" (S1) |
| Empty state body | "Approved and pending time-off requests from your team appear here. New submissions route to you automatically." |
| Error state | "We couldn't load the leave queue. Check your connection and try again." + Retry (`QueryErrorPanel`) |
| Destructive confirmation | Reject request: mandatory reason ≥10 chars in popover — "Reject request" / "Add a reason (min 10 characters)" |

### Per-surface copy (all strings i18n-keyed — en/de/pl/ar)

**S1 — Leave queue**
- Row/panel CTAs: `Approve` · `Reject` (destructive) · `Request clarification` · `Delegate` (reuse `Approvals` action keys; add `Leave.*` row-variant keys).
- Header action: `Record sick leave` (opens direct-entry dialog — NOT an approval; sick = notification).
- Empty (page): heading "No leave requests to review" / body as above.
- Sub-empty (filtered): "No requests match these filters." / "Clear filters".
- Error: `QueryErrorPanel` "We couldn't load the leave queue. Try again."
- Destructive — Reject: reason textarea, min 10 chars, `variant="destructive"` confirm (mirror invoice reject popover exactly).
- Destructive — Record sick leave writes a permanent absence record: dialog confirm "This records an absence directly and notifies the employee. It does not go through approval."

**S2 — Team calendar**
- View toggle: `Month` / `Quarter` (segmented control / tabs, teal active).
- Conflict marker: `TriangleAlert` + "Conflict" tooltip "N overlapping requests on this day for {team}".
- Capacity legend: "Available" / "Busy" / "Over capacity" (success/warning/danger bands).
- Empty (page): heading "No time off scheduled" / body "When your team books vacation or records absences it shows here. Overlapping requests are flagged automatically."
- Error: "We couldn't load the team calendar. Try again."
- Destructive: none.

**S3 — Employee time entry**
- Primary CTA: `Save entry` (busy: `Saving…`). Field labels: `Date`, `Hours worked`, `Night hours`, `Overtime`, `Weekend / holiday`, `On-call (dyżur)`, `Absence type`, `Notes (optional)`.
- Empty (page): heading "No hours logged yet" / body "Record daily working time, overtime, and night-shift hours here. Statutory limits are checked as you save."
- Error: "We couldn't save this time entry. Try again."
- On-save WT warning (inline, non-blocking — see S5): "Heads up — this puts {name} over the {limit} {jurisdiction} {daily|weekly} limit. You can still save." + `Save anyway`.
- Destructive: Delete a draft day entry: confirm "Delete this time entry?" (records are day-grain; deletion allowed only pre-lock).

**S4 — ewidencja report**
- Primary CTA: `Generate report`; on existing: `Regenerate` + `Export` (PDF/CSV).
- Immutable treatment copy: `Archived` badge + `Lock` icon; tooltip "Immutable KP §149 record — cannot be edited (retained 3 years)."
- Superseded rows: dimmed + chip "Superseded {date}".
- Empty (page): heading "No record generated for this period" / body "Generate the KP §149 working-time record (ewidencja czasu pracy) for {employee}, {period}. Once generated it is archived and cannot be edited."
- Error: "We couldn't load the ewidencja archive. Try again."
- Destructive — Regenerate supersedes: confirm "Regenerating creates a new immutable snapshot and marks the current one as superseded. The old record stays in the archive. Continue?"

**S5 — WT-limit alert surface**
- In-app banner (`--status-warning` approaching / `--status-danger` breach): "Working-time limit {reached|exceeded} — {name} logged {X}h this {period}, over the {limit}h {jurisdiction} {rule}." CTA "View time record".
- Notification (via `dispatch()`, in-app + email + Slack/Teams): title/body are **dotted i18n keys** resolved by `resolveEventCopy` (no hardcoded strings). Entity CTA needs `EMPLOYEE_TIME_RECORD` → `/employee-time` in `ENTITY_ROUTES`.

Destructive-action inventory (all confirm before commit): Reject leave request (reason ≥10); Record sick leave (direct, irreversible absence + notify); Regenerate ewidencja (supersedes prior); Delete draft time entry (pre-lock only). No hard-delete of ledger or archived snapshots (append-only).

## Team Calendar — visual spec (S2, new build)

- **Grid:** month = 7-column week grid; quarter = 3 stacked month strips (or 13-week horizontal strip). Day cell ≥ 44px tall. Cell shows date (top-start, `tabular-nums`) + capacity band + up to N avatar/initials chips, "+K more" overflow.
- **Capacity heatmap band:** thin `ProportionBar`-style fill OR cell background tint using `color-mix(in oklch, var(--status-*) …)` — reuse the `proportion-bar.tsx` / `composition-strip.tsx` `color-mix` idiom, not new colors. success → warning → danger by % of team out.
- **Conflict:** overlapping same-team approved/pending requests → `TriangleAlert` (`--status-danger`) in cell corner + accessible tooltip. Color never sole signal.
- **Holidays/weekends:** muted (`--muted`) cell background; public-holiday cells show a small `dot` + name tooltip (seeded reference data).
- **Legend:** always visible (`CapacityLegend`) mapping band → meaning.
- **[ASSUMPTION]** "Same team" grouping = org `Team` membership (RESEARCH Open Q4 unresolved). If the planner selects cost-center/manager grouping instead, the capacity aggregation key changes but the visual contract is unchanged.
- **[ASSUMPTION]** Calendar is built from `packages/ui` primitives + div grid (not a third-party calendar block) to guarantee RTL correctness and token consistency.

## Accessibility (WCAG 2.1 AA — mandatory)

- **Keyboard:** all actions reachable/operable — side-panel Approve/Reject/Delegate/Clarify, calendar cell arrow-key navigation + Enter to open day detail, date pickers, export menu. Focus visible via `--ring` teal.
- **Semantic HTML:** `<table>` (via `DataTable`) for queue + ewidencja; `<ol>/<li>` for approval chain; `<button aria-pressed>` for filter chips + proportion segments (existing pattern); calendar cells as focusable elements with `aria-label` = date + capacity + count.
- **Non-color cues:** status/conflict/breach always icon + text, never color alone.
- **Contrast:** reuse tuned badge pairings (`/10` bg + `-800`/`-400` fg); `--destructive` lightness 0.52 holds AA on light surfaces (documented in tokens.css). `prefers-contrast: more` overrides already defined.
- **Live regions:** WT-limit inline warning and dispatch toasts announced via `aria-live="polite"` (breach = `assertive`).
- **Reduced motion:** honor `prefers-reduced-motion`; entrance animation via existing `AnimateIn` only (no new motion system).
