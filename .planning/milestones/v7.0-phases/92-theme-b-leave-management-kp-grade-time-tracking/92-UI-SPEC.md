## Spacing Scale

Tailwind 4px base. Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | icon gaps (`gap-1`), badge internal padding |
| sm | 8px | `--space-stack-tight` â€” label+control, heading+body pairs (`space-y-2`) |
| md | 16px | `--space-card-gap` â€” rows inside a card/panel/section (`gap-4`, `space-y-4`) |
| lg | 24px | `--space-section-gap` â€” between top-level page sections (`gap-6`, `space-y-6`) |
| xl | 32px | major layout breaks |
| 2xl | 48px | page-level vertical rhythm |
| 3xl | 64px | rare hero spacing (not expected this phase) |

**Semantic rhythm tokens (use these, not ad-hoc `space-y-N` at page level):** `--space-stack-tight` (8), `--space-card-gap` (16), `--space-section-gap` (24). Raw `space-y-N` is allowed only for intrinsic component internals (table rows, dialog bodies, skeletons) â€” mirrors existing convention.

Fixed layout dimensions (inherited, not spacing tokens): review side-panel width `480px` (`w-[480px] sm:max-w-[480px]`); `DataTable` cap-mode body max-height `400px`; empty-state page min-height `60vh`; SLA/status badge min-width `72px`. Radius `--radius` = `0.625rem` (10px).

Exceptions: **team calendar (S2) day cells** â€” use a fixed min touch/click target of **44px** height per day cell for WCAG 2.5.5 pointer-target size (icon-dense capacity markers). This is the only new exception; record it in the calendar component.

## Typography

**Phase contract = exactly 4 sizes** (`12 / 14 / 20 / 28`) and **2 weights** (regular `400` + semibold `600`; `600` always paired with `font-display` for headings and KPI values). New and adapted surfaces S1â€“S5 use only these.

| Role | Size | Weight | Line Height | Font | Where |
|------|------|--------|-------------|------|-------|
| Label / caption | 12px (`text-[12px]`) | 400 | 1.4 | sans, `uppercase tracking-wide` for group labels | field labels, muted section headings, chain-step captions |
| Body | 14px (`text-sm`) | 400 | 1.5 (`leading-relaxed`/normal) | sans | table cells, panel copy, form values |
| Heading | 20px (`text-[20px]`) | 600 | 1.2 (`leading-tight`) | `font-display` | section + empty-state headings |
| Display | 28px (`text-[28px]`) | 600 | 1.2 (`leading-[1.2]`) | `font-display` | KPI values (balance days, hours this week) in summary cards |

**Page `<h1>` is OUT of this phase's typography contract.** The page title is owned entirely by the existing, unmodified `AtelierPageHeader` primitive (which ships its own 24px `<h1>`). This phase does not restyle it, so that size is neither declared nor counted here.

**Frozen inherited ramp â€” read-only, MUST NOT appear in S1â€“S5.** Already-shipped `packages/ui` sub-components this phase reuses *without restyling* (`sla-badge`, `chain-tracker`, `audit-timeline`, badges/chips/micro-labels) carry their own `10px`/`11px`/`13px` sizes and `font-medium` (500) weight. These are frozen existing-system values: reuse those primitives as-is, but do **not** introduce any `10/11/13px` size or new `500`-weight text in any new or adapted surface. New surface typography uses the 4 sizes + 2 weights above, exclusively.

**Accent (teal) reserved for â€” explicit list:**
- Primary CTA buttons: "Submit request", "Save entry", "Generate report", "Approve".
- Active/current approval-chain step circle + current-step marker (`bg-primary text-primary-foreground`).

- Hyperlinks (`text-primary hover:underline`).
- Focus ring (`--ring` = teal).
- Emphasis on the primary balance figure in the leave side-panel.

Accent is **never** applied to: every interactive element, table rows, generic borders, or the whole calendar grid.

**Semantic status tokens (`--status-*` / `status.css`)** â€” the leave/time domain leans on these; they are NOT the 10% accent:

| Token | Value (light) | Domain meaning this phase |
|-------|---------------|---------------------------|
| `--status-success` | `oklch(0.55 0.17 150)` green | approved request, within-limit capacity, healthy balance |
| `--status-warning` | `oklch(0.7 0.16 65)` amber | SLA nearing, capacity near threshold, WT-limit **approaching**, night/overtime present |

| `--status-info` | `oklch(0.55 0.18 260)` blue | informational (immutable/archived neutral state) |

Reuse the established badge idiom for status: `bg-{color}/10 text-{color}-800 dark:text-{color}-400` (see `sla-badge.tsx`, `side-panel.tsx` `statusBadgeColors`). **Never encode meaning by color alone** â€” always pair with icon + text label (conflict = `TriangleAlert` + "Conflict"; breach = icon + "Over limit"). WCAG AA is met by the `/10` background + `-800` foreground pairing already tuned in the codebase.
