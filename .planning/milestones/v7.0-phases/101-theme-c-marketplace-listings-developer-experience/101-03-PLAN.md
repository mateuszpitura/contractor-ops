<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| OpenAPI snapshot -> generated artifacts | The snapshot is the single source; a hand-edited artifact that drifts silently ships a wrong integration surface |
| staff -> listing state | Only an admin advances a listing's state; the state machine prevents an illegal jump that would misreport a listing as LIVE |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-101-03-01 | Tampering | A generated artifact drifting from the spec | mitigate | `generate --check` diff-check gates CI (mirror 98-11 snapshot) |
| T-101-03-02 | Spoofing | A listing marked LIVE without a real approval | mitigate | The validated state machine + the audited `update` — LIVE only from IN_REVIEW |
| T-101-03-03 | Information Disclosure | The generator embedding a secret/tenant value | mitigate | It reads only the public spec + the public event catalog; no tenant/secret input |
| T-101-03-SC | Tampering | New deps (none required for the generator itself) | accept | The generator is pure TS over JSON; platform SDK deps land in 101-06/101-07 under the 7-day floor |
</threat_model>
