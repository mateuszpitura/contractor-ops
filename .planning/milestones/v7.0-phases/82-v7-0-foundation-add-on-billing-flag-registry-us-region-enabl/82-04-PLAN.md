<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → grantAddOn mutation | The grant must be owner-gated; an org admin/member must not grant itself an add-on outside the intended RBAC. |
| session → entitlement read | `requireAddOn` reads entitlement keyed by session `organizationId` (never client-supplied); Subscription stays tenant-scoped (NOT in globalModels). |
| public-api → REST consumer | ADD_ON_REQUIRED is translated to a 403 JSON without leaking internals. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-82-04-01 | Elevation of Privilege (BFLA) | grantAddOn mutation | mitigate | Gate with the existing `adminProcedure` (= `requirePermission({organization:['update']})`) — the same owner-gate 4 sibling billing mutations use. A member without organization:update cannot self-grant. CONFIRMED adminProcedure is the right gate (RESEARCH/PATTERNS). Audit-logged via writeAuditLog for traceability. |
| T-82-04-02 | Spoofing / IDOR (tenant isolation) | requireAddOn entitlement read + Subscription | mitigate | Entitlement derived from session `ctx.organizationId` (tenant-from-session per CLAUDE.md), never client input. Subscription is correctly tenant-scoped and is NOT added to globalModels (verified IDOR landmine avoided). grantAddOn operates only on the caller's org subscription. |
| T-82-04-03 | Tampering (stale-grant) | subscription Redis cache | mitigate | grantAddOn invalidates `CacheKeys.subscription(orgId)` after the write (Pitfall 3) so the entitlement cannot be served stale (no 15-min window where a granted org is wrongly denied — or a revoked one wrongly allowed in future revoke work). |
| T-82-04-04 | Repudiation | grant action | mitigate | `writeAuditLog` records actor, old/new addOns, org, action — non-repudiable grant trail. |
| T-82-04-05 | Tampering | npm/pip/cargo installs | mitigate | No package installs — only an additive Prisma migration (RESEARCH § Package Legitimacy Audit: Not applicable). slopcheck N/A. |
</threat_model>
