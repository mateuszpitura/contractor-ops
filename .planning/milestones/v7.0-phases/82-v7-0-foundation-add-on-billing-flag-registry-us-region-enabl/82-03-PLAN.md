| flag declaration → flag evaluation | All 19 flags ship `default:false` PENDING; no v7.0 feature reads them yet (Unleash toggles absent until later phases). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-82-03-01 | Elevation of Privilege | boot gate bypass | mitigate | The gate must not be bypassable except via the documented `FLAG_SIGNOFF_BYPASS=local`. We reuse the existing `assertFlagSignoffsOrExit()` (which only downgrades on that exact env value) — no new bypass path introduced. Wiring it into all three boots closes the current gap where prod could ship with an unenforced gate. |
| T-82-03-02 | Tampering | premature flag enablement | mitigate | Every v7.0 flag is PENDING + `default:false`; gated prefixes ensure a missing registry entry exits boot. A flag cannot silently evaluate true (no Unleash toggle exists yet). |
| T-82-03-03 | Denial of Service | gate exits a healthy prod boot | accept | Intended fail-closed behavior: an un-signed-off gated flag SHOULD halt boot. All 19 are registered PENDING here, so boot is healthy; the gate fires only on a genuinely missing entry (a real misconfiguration). |
