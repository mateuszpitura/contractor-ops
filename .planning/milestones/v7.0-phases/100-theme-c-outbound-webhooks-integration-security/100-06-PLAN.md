<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| outbox drain → subscriber endpoint | Fan-out must not do network I/O; delivery is a separate, guarded drain |
| QStash → /webhooks-outbound/_deliver | Signed-callback boundary; `guardQStashRequest` is the authn |
| org config → live dispatch | `module.outbound-webhooks` is the kill switch; nothing egresses when off |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-100-06-01 | Denial of Service | A poison row stalls the whole delivery batch | mitigate | Per-attempt try/catch + terminal DLQ + `FOR UPDATE SKIP LOCKED`-style CAS claim |
| T-100-06-02 | Denial of Service | Fan-out amplification (one event → many endpoints) floods | mitigate | Per-sub 100/min dispatch limit (requeue-not-drop) + bounded retries |
| T-100-06-03 | Information Disclosure | Dispatch to a private IP via DNS rebind | mitigate | `assertWebhookUrlSafe` + `webhookHttpAgent` re-check at connect on EVERY delivery |
| T-100-06-04 | Elevation of Privilege | Unauthenticated call to the deliver route | mitigate | `guardQStashRequest` (QStash Receiver HMAC) on `/webhooks-outbound/_deliver` |
| T-100-06-05 | Spoofing | Dispatch before an org is enabled | mitigate | `module.outbound-webhooks` kill switch short-circuits before any POST |
</threat_model>
