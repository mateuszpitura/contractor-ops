<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| user input (url) → subscription | The create/update path must SSRF-check before persisting |
| stored secret → re-signing | The secret is recoverable (encrypted), not one-way — must never leak in a response/audit/log |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-100-08-01 | Information Disclosure | Webhook secret returned after create / in an audit row | mitigate | Reveal-once at create/rotate; encrypted at rest; never in audit/log/response otherwise |
| T-100-08-02 | Information Disclosure | SSRF via a subscription url entered in the UI | mitigate | `assertWebhookUrlSafe` on create + update (server-side, not client trust) |
| T-100-08-03 | Elevation of Privilege | Non-admin manages another user's/org's subscriptions | mitigate | Admin-gated procedure + tenant-scoped queries + `WEBHOOK_SUBSCRIPTION` audit |
| T-100-08-04 | Denial of Service | Unbounded subscription creation | mitigate | P99 `TIER_WEBHOOK_SUBSCRIPTION_CAP` enforced on create |
</threat_model>
