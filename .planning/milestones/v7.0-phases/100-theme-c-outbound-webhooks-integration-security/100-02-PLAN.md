default. It is consumed by the subscription service (100-05, subscribe-time) and the deliver drain
(100-06, dispatch-time). Turns the `webhook-ssrf` RED net green.

<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| customer webhook URL → our egress | Hostile input; the guard is the sole control against SSRF to internal services / cloud metadata |
| DNS resolver → socket connect | TOCTOU window a rebinding attacker exploits; closed by re-resolve + IP-pin at connect |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-100-02-01 | Information Disclosure | SSRF to `169.254.169.254` / internal services | mitigate | Block list at subscribe + connect; classify all resolved IPs |
| T-100-02-02 | Information Disclosure | DNS rebinding bypasses subscribe-time check | mitigate | `webhookHttpAgent` re-resolves + pins at connect; `maxRedirects:0` |
| T-100-02-03 | Information Disclosure | Redirect to a private/metadata host | mitigate | `redirect:'error'` / `maxRedirects:0` — redirects never followed |
| T-100-02-SC | Tampering | Malicious `request-filtering-agent` (typosquat/day-zero) | mitigate | 7-day age pin + name verify + `pnpm audit`/`security:scan`; fallback = no new dep |
</threat_model>
