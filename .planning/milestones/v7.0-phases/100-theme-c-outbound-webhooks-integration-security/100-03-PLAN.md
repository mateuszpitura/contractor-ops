5-minute window. Consumed by the deliver drain (100-06). Turns the `webhook-hmac` RED net green.

<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| our dispatcher → subscriber endpoint | The signature is how the subscriber authenticates the payload origin + integrity |
| captured delivery → replay | The signed timestamp + 5-min window is the replay defense |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-100-03-01 | Spoofing | Forged webhook without the secret | mitigate | HMAC-SHA256 over `{t}.{body}`; subscriber verifies with the shared secret |
| T-100-03-02 | Replay | Captured POST re-sent later | mitigate | Timestamp bound into the signed string; verifiers reject `|now-t|>5m` |
| T-100-03-03 | Information Disclosure | Timing side-channel on compare | mitigate | `timingSafeEqual`/`compare_digest`/`hash_equals` in signer + every verifier |
| T-100-03-04 | Tampering | Documented verifier drifts from the signer | mitigate | Round-trip test imports the TS verifier against the live signer |
</threat_model>
