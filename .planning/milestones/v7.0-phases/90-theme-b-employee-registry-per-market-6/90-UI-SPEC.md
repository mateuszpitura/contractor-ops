## Spacing Scale

The codebase uses Tailwind's 4px-base scale. Declared values (multiples of 4) — these are the only spacings the employee compliance components may use; they match the contractor compliance section verbatim:

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1`, `me-1` | Icon-to-text gap inside badges/buttons (e.g. `<AlertCircle className="me-1 h-3 w-3" />`) |
| sm | 8px | `space-y-2`, `gap-2` | Label-to-input stack inside a single field (the `space-y-2` field block — universal in `us-compliance-fields.tsx`) |
| md | 12px | `gap-3`, `p-3`, `pt-4` | Inter-field row gaps, dashed info-box padding, advisory boxes (`flex flex-wrap items-center gap-3`, `rounded-md border bg-muted/30 p-3`) |
| md+ | 16px | `space-y-4`, `mt-4` | Vertical rhythm between fields in a market block (`<div className="space-y-4">` wrapper); CardContent stack |
| lg | 24px | `mt-6` | Section breaks (e.g. the classification block `mt-6`); space between market block and PII block |
| section | `--space-section-gap` | `space-y-section-gap` | Semantic page-level section rhythm token (already exposed) — use for top-level page composition only |

> Note: the existing field stacks use `space-y-2` (8) within a field and `space-y-4` (16) between fields; `gap-3` (12) appears for inline pill rows. 12 and 16 are both legitimate established steps in this app (Tailwind 4px base). Do not introduce arbitrary values (no `gap-[10px]`, no `p-5` where the contractor pattern uses `p-3`).

Exceptions: `h-9` (36px) input height + `h-5` (20px) badge height are component-intrinsic heights from `@contractor-ops/ui`, not layout spacing — reuse, do not restyle. Touch targets: reveal/secondary buttons use `size="sm"`; do not shrink below the shadcn `sm` button (meets 32px+ min; keep accessible name = visible text).

## Typography

Fluid `clamp()` scale from `@theme inline` in `globals.css`. The employee compliance UI uses exactly the contractor compliance roles — only these four:

**Two spec weights only: regular (400) + semibold (600).** `font-medium` (500) is NOT a spec-introduced weight — it is the intrinsic weight baked into the existing `@contractor-ops/ui` `Label`/`RequiredLabel` + `Badge` base primitives (same component-intrinsic carve-out as the `h-9`/`h-5` heights below); this surface introduces no new 500 usage.

| Role | Size | Weight | Line Height | Token / class |
|------|------|--------|-------------|---------------|
| Body / input | `--text-sm` → clamp(13px → 14px) | 400 (regular) | 1.25rem (20px) | `text-sm` — field values, input text, suggestion text |
| Label / field label | `--text-sm` clamp(13px → 14px) | (primitive intrinsic) | 1.25rem | `<Label>`/`RequiredLabel` verbatim — its weight is the base-primitive's, not set by this spec |
| Section heading | `--text-sm` clamp(13px → 14px) | 600 (semibold) | 1.25rem | `text-sm font-semibold` — sub-section `<h3>`; CardTitle uses `text-base font-semibold` |
| Hint / error | `--text-xs` clamp(11px → 12px) | 400 (regular) | 1rem (16px) | `text-xs text-muted-foreground` (hints), `text-xs text-destructive` (errors); the `Badge` carries its own primitive weight |

Body base is 14px / line-height 1.5 (set on `body` in `globals.css`). Distinct sizes: `text-xs`, `text-sm`, plus `text-base` for the CardTitle (3 sizes, within the 4 cap). PII/ID values render in **mono** (`font-mono text-sm tabular-nums`) exactly as `ssn-masked-reveal.tsx` — apply the same to PESEL/SSN/Iqama/Emirates-ID masked + revealed values.

Card titles (the per-market card header) use `text-base font-semibold` per `CountryComplianceSectionView` — reuse for the `EmployeeComplianceSection` card title.

## Color

The 60/30/10 split maps to the established teal-on-warm-graphite "Precision Craft" tokens. All values are CSS variables (oklch); never hardcode hex.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` / `--surface-0` (`bg-background`) | Page + form background; the operational/workbench surface (calm, no mesh on this dense form per the workbench intensity tier) |
| Secondary (30%) | `--card` (`bg-card`) + `--muted`/`bg-muted/30` | The `<Card>` the compliance section lives in; the dashed info box + advisory boxes (`bg-muted/30`); inputs (`--input` border) |
| Accent (10%) | `--primary` (Deep Teal, `text-primary` / `bg-primary`) | Reserved ONLY for: the primary **Save / Register employee** CTA, the reveal toggle button text (`text-primary` per `ssn-masked-reveal.tsx`), focus rings (`--ring`), and required-field active state. NOT for body text, NOT for every interactive element. |
| Destructive | `--destructive` (`text-destructive`) | Inline field validation errors (`text-xs text-destructive`), the required-field asterisk (`*`), reveal-failed error text, and the (single) destructive confirmation if any. |
| Advisory / warning | `--warning` (Burnished Amber) via `Badge variant="warning"` (amber: `bg-amber-500/10 text-amber-700 dark:text-amber-400`) | **Emirates-ID checksum advisory** + **adviser-verify / statutory-seed-list notes** + **incomplete-fields count badge** (the contractor pattern uses `border-warning/20 bg-warning/5 text-warning` — reuse). Amber = "review / unverified", never red. Advisory NEVER blocks save (D-03/Pitfall-1). |
| Success | `Badge variant="success"` / `success` token | Optional: a "validated" / "checksum passed" affirmation pill (e.g. valid PESEL/Steuer-IdNr). Green; never the dominant signal. |

Accent reserved for: **(1) the primary Save/Register CTA, (2) the PII reveal toggle text, (3) focus-visible rings, (4) the required-field asterisk active emphasis.** Nothing else. Amber `warning` carries every "advisory / unverified / adviser-verify / incomplete" signal; red `destructive` carries only hard validation errors + the asterisk + destructive confirmations.

**Semantic mapping for this phase's three classes of feedback (load-bearing):**
- **Hard validation error** (bad PESEL checksum, bad NI format, bad Steuer-IdNr uniqueness) → `text-destructive` inline `FieldError`, `role="alert" aria-live="polite"`. Surfaces only after a complete-looking value is entered (never mid-keystroke), exactly as `us-compliance-fields.tsx` gates EIN/SSN errors.
- **Advisory** (Emirates-ID Luhn fails, GOSI/WPS lenient-length warning) → amber `Badge variant="warning"` or an amber info box; copy says "could not verify — save allowed". Never blocks the Save button. (D-03, Pitfall 1/2.)
- **Adviser-verify / statutory note** (seeded reference lists, statutory rules) → muted dashed info box (`border-dashed bg-muted/30 text-muted-foreground text-xs`) like the US info box, optionally an amber note. Local-only/legal-deferred posture (D-10).

## Copywriting Contract

All copy is **i18n keys**, never literals (CLAUDE.md: no hardcoded user-facing strings; `lint:no-breadcrumbs` + `i18n:parity`). Parity required across **en / en-US / de / pl / ar** (ar is RTL). en-US is fallback-aware (thin overrides over en, P84 pattern). The table gives the canonical en string + the key namespace `Employees.compliance.*` (mirror `Contractors.compliance.us` / `Contractors.countryCompliance`).

| Element | Copy (en) | i18n key (namespace `Employees.compliance` / `Employees.registry`) |
|---------|-----------|--------------------------------------------------------------------|
| Primary CTA | **Register employee** (on the registration form) / **Save details** (on an existing employee's compliance card — mirror contractor `saveButton`) | `registry.registerButton` / `compliance.saveButton` |
| Section card title | **{country} statutory details** (e.g. "Poland statutory details") | `compliance.cardTitle` (param `country`) |
| Incomplete-fields badge | **{count} fields to complete** | `compliance.incompleteFields` (count) |
| Empty state heading | **No employees yet** | `registry.emptyHeading` |
| Empty state body | **Register your first employee to capture their market-specific statutory details.** + primary CTA | `registry.emptyBody` |
| No-fields-for-country | (render nothing — `null` — when the selected market has no configured field set, exactly as `CountryComplianceSectionView` returns `null`) | n/a |
| Loading | Spinner (`Loader2 animate-spin`) inside the card, centered (`CountryComplianceLoadingCard` analog); `aria-busy` on the region; sr-only "Loading employee details" | `compliance.loading` |
| Error state (section load) | **Could not load employee details. Retry.** (problem + retry action) | `compliance.loadError` + `common.retry` |
| Validation error (field) | Per-field, e.g. **Enter a valid PESEL** / **Enter a valid National Insurance number** / **Tax ID format not recognised** — problem-specific, inline, non-blocking | `compliance.{pl.peselFormatError, uk.niFormatError, de.steuerIdNrFormatError, …}` |

| Adviser-verify note | **Statutory list — verify with a local payroll adviser before relying on it for filings.** | `compliance.adviserVerifyNote` |

| PII masked hint | **Only the last 4 digits are shown. Reveal is logged.** | `compliance.pii.maskedHint` (mirror `ssnMaskedHint`) |

## Accessibility Contract (WCAG — mandatory)

- Every input: associated `<Label htmlFor>` (via `useId()`), `aria-invalid` + `aria-describedby` wired to the `FieldError` id when present, `aria-required` on required fields.
- Validation + reveal feedback: `role="alert" aria-live="polite"` (matches existing).
- Reveal toggle: `aria-pressed={isRevealed}`, accessible name = visible text ("Reveal"/"Hide") — Label-in-Name; reveal never triggered on hover/focus/mount.
- Masked PII span: `role="img"` + `aria-label` describing the masked value (mirror `ssnMaskedAriaLabel`).
- Selects: native `<select>` (keyboard-complete by default, per `EntityTypeSelect`); reference combobox must be keyboard-navigable + announce results.
- Required-asterisk `*`: `aria-hidden="true"` (the `aria-required` carries the semantic).
- Color is never the sole signal: advisory amber pills carry text + icon; errors carry text.
- Focus: teal `--ring` focus-visible glow (existing `[data-form-control]:focus-visible`); `prefers-contrast: more` strengthens it (already handled).
