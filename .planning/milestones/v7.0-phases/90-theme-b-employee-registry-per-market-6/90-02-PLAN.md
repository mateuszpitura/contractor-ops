PESEL (Pattern 2): weights [1,3,7,9,1,3,7,9,1,3], sum mod 10, check=(10-(sum%10))%10. PLUS embedded-DOB: century-encoded month — CENTURY_OFFSET {0:1900,20:2000,40:2100,60:2200,80:1800}; month=mmRaw%20; year=offset+yy; validate Date round-trips. Vector 44051401359 → check 9 ✓.
Steuer-IdNr (Pattern 1): replace [\s/]; ^[1-9]\d{10}$; uniqueness rule "exactly one digit appears 2 OR 3 times in positions 1-10, all others exactly once"; check via mod11_10CheckDigit(first 10 digits) === digit[10]. Vector 36574261809 ✓; reject 36554266806.
NI (Pattern 4): ^[A-Z]{2}\d{6}[A-D]$; neither prefix letter ∈ {D,F,I,Q,U,V}; 2nd letter ≠ O; prefix ∉ {BG,GB,KN,NK,NT,TN,ZZ}.
UK tax code (Pattern 5): /^(?:[SC])?(?:(?:\d{1,4}[LMNT])|(?:K\d{1,4})|0T|BR|D0|D1|NT)(?:\s?(?:W1|M1|X))?$/i.
Saudi ID (Pattern 3): ^[12]\d{9}$; standard Luhn mod-10; return 1 if leading '1', 2 if leading '2', false otherwise.
Emirates ID: strict format ^784-\d{4}-\d{7}-\d$ is the GATE; Luhn (sum×9)%10 over first 14 digits is ADVISORY — return a structured result (format-valid bool + checksum-advisory bool), checksum NEVER turns a format-valid ID into a hard fail (Pitfall 1, LOW confidence).
GOSI: ^\d{9}$ lenient (MEDIUM confidence, adviser-verify). WPS Establishment ID: ^\d{1,13}$ lenient.
