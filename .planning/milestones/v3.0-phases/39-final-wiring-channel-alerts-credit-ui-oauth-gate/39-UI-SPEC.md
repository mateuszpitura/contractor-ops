## Interaction Flows

### Flow 1: OCR Credit Exhaustion (Admin)

1. Admin uploads invoice in `invoice-upload-area.tsx`
2. OCR mutation fires automatically
3. Server returns `PRECONDITION_FAILED` with message "OCR credits exhausted"
4. Catch block detects credit exhaustion via `isCreditExhaustedError()` helper
5. Sets `creditExhausted` state to `true`
6. `CreditExhaustedInline` renders inline below the upload area
7. Admin clicks "Upgrade plan" -- navigates to `/settings?tab=billing`
8. OR Admin clicks "Buy credits" -- navigates to `/settings?tab=billing`

### Flow 2: OCR Credit Exhaustion (Portal)

Same as Flow 1 but in `portal/invoice-submit-form.tsx`. Identical component, identical behavior.

### Flow 3: FeatureGate on OAuth Provider Sections

1. STARTER-tier user navigates to Settings > Integrations
2. `FeatureGate` in each provider section queries `billing.getSubscription`
3. During loading: children render normally (no flash)
4. After load: tier check fails for STARTER
5. `UpgradeInlineBanner` replaces the entire provider section content
6. User sees: "{Feature} requires a Pro plan to use." + "Upgrade plan" button
7. Clicking "Upgrade plan" navigates to `/settings?tab=billing`

### Flow 4: Channel Alert Dispatch (Server-only, no UI)

1. Notification event fires (e.g., INVOICE_RECEIVED)
2. `dispatch()` in `notification-service.ts` processes per-recipient messages as before
3. NEW: After recipient loop, maps notification type to category
4. Looks up `configJson.channelMapping` from IntegrationConnection
5. If channelId found for category, calls `provider.sendChannelAlert()`
6. Channel alert appears as Adaptive Card (Teams) or message (Slack) in configured channel
7. Failure is logged but does not block other notifications
