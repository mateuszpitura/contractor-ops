**1. Create `packages/api/src/routers/equipment.ts`** following the exact pattern from contractor.ts (tenantProcedure, requirePermission, plain() helper, Zod input validation):

**CRUD endpoints (EQUIP-01):**
- `list`: query, permission equipment:read. Input: equipmentListSchema. Returns paginated list with total count. Include current assignment (join EquipmentAssignment where unassignedAt is null, include contractor name). Support search (name, serialNumber ilike), status filter, type filter, assignedContractorId filter, sorting. Use `ctx.db.equipment.findMany` with `where: { organizationId: ctx.organizationId }`.
- `getById`: query, permission equipment:read. Input: z.object({ id: z.string() }). Returns equipment with current assignment (contractor details), all assignments history, shipments with events. Include `_count` for shipments.
- `create`: mutation, permission equipment:create. Input: equipmentCreateSchema. Creates equipment with organizationId from ctx. Returns created equipment. Creates audit log entry with entityType EQUIPMENT.
- `update`: mutation, permission equipment:update. Input: equipmentUpdateSchema. Updates equipment. Creates audit log entry.
- `retire`: mutation, permission equipment:delete. Input: z.object({ id: z.string() }). Sets status to RETIRED. Validates equipment is not currently assigned (must unassign first). Creates audit log entry.

**Assignment endpoints (EQUIP-02, per D-07, D-08):**
- `assign`: mutation, permission equipment:update. Input: equipmentAssignSchema. Validates: equipment status is AVAILABLE (per D-07 must unassign before reassigning), contractor exists in same org. Creates EquipmentAssignment record, updates equipment status to ASSIGNED, updates equipment contractorId-like reference. Creates audit log with entityType EQUIPMENT. Returns updated equipment.
- `unassign`: mutation, permission equipment:update. Input: equipmentUnassignSchema. Validates: equipment has active assignment. Sets `unassignedAt = now()` and `unassignedByUserId` on current assignment. Updates equipment status to AVAILABLE. Creates audit log. Returns updated equipment.

**Shipment endpoints (EQUIP-04, EQUIP-08):**
- `createShipment`: mutation, permission equipment:create. Input: shipmentCreateSchema. Creates Shipment record with CREATED status. Creates initial ShipmentEvent (status: CREATED). Updates equipment status based on direction: OUTBOUND -> IN_TRANSIT (per D-06), RETURN -> RETURN_IN_TRANSIT. Creates audit log with entityType SHIPMENT. Returns created shipment with events.
- `addShipmentEvent`: mutation, permission equipment:update. Input: shipmentEventCreateSchema. Creates ShipmentEvent, updates shipment.currentStatus. Auto-advances equipment status per D-06 mapping:
  - Shipment DELIVERED + direction OUTBOUND -> equipment DELIVERED
  - Shipment RETURNED + direction RETURN -> equipment RETURNED
  - Shipment FAILED -> no equipment status change (manual override)
  Creates audit log. Returns updated shipment with events.
- `getShipment`: query, permission equipment:read. Input: z.object({ id: z.string() }). Returns shipment with all events ordered by occurredAt asc, equipment info.
- `listShipments`: query, permission equipment:read. Input: z.object({ equipmentId: z.string() }). Returns all shipments for equipment with latest event.
- `deleteShipment`: mutation, permission equipment:delete. Input: z.object({ id: z.string() }). Soft deletes or hard deletes shipment and events. Only if shipment status is CREATED (not yet in transit).

**Contractor equipment endpoint (EQUIP-03):**
- `listByContractor`: query, permission equipment:read. Input: z.object({ contractorId: z.string() }). Returns equipment assigned to contractor with current shipment status (condensed view). Joins EquipmentAssignment (active) -> Equipment -> latest Shipment.

**Status transition constant** (per D-05, D-06):
Define `EQUIPMENT_STATUS_TRANSITIONS` map and `SHIPMENT_TO_EQUIPMENT_STATUS` map as constants at top of file. Validate transitions before applying.
