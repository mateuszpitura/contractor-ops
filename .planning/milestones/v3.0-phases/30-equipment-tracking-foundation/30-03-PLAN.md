Create `packages/api/src/services/equipment-workflow.ts` with two exported functions:

**1. `handleEquipmentTaskStart`** (called from workflow.ts startRun, per D-13, D-14, D-16):

```typescript
export async function handleEquipmentTaskStart(
  db: PrismaClient,
  organizationId: string,
  taskRun: { id: string; taskType: string },
  workflowRun: { id: string; contractorId: string | null; templateType: string },
): Promise<void>
```

Logic:
- If `taskRun.taskType !== "EQUIPMENT"` return early
- If `workflowRun.contractorId` is null, log warning and return (no contractor to ship to)
- Set task status to IN_PROGRESS with `startedAt = new Date()`
- Find all equipment currently assigned to this contractor (EquipmentAssignment where contractorId matches and unassignedAt is null, join Equipment)
- Store equipment IDs in `taskRun.resultJson` as `{ equipmentIds: string[], direction: "OUTBOUND" | "RETURN" }`:
  - If workflow template type is ONBOARDING: direction = "OUTBOUND"
  - If workflow template type is OFFBOARDING: direction = "RETURN", also update each equipment's status to RETURN_REQUESTED
- If no equipment assigned: set task to DONE immediately (nothing to ship), log info
- Use Prisma transaction for atomicity

**2. `checkShipmentTaskCompletion`** (called from equipment.ts addShipmentEvent, per D-15, D-16):

```typescript
export async function checkShipmentTaskCompletion(
  db: PrismaClient,
  organizationId: string,
  shipment: {
    id: string;
    workflowTaskRunId: string | null;
    direction: "OUTBOUND" | "RETURN";
    currentStatus: string;
  },
): Promise<void>
```

Logic:
- If `shipment.workflowTaskRunId` is null, return (not linked to workflow)
- Determine target status: direction OUTBOUND -> "DELIVERED", direction RETURN -> "RETURNED" (per D-15)
- If `shipment.currentStatus !== targetStatus`, return (not at target yet)
- Find ALL shipments linked to same workflowTaskRunId in same org
- Check if ALL have reached their respective target status (each shipment checks its own direction)
- If not all complete, return
- Auto-complete the task: `updateMany` where id = workflowTaskRunId AND status = "IN_PROGRESS" -> set status "DONE", completedAt = now() (idempotent — skip if already DONE)
- After task completion, recompute workflow run progress: count total tasks, count DONE tasks, update progressPercent. If all required tasks DONE, set workflow run status to COMPLETED with completedAt
- Log the auto-completion for observability

Both functions should:
- Import PrismaClient type from @contractor-ops/db
- Use `console.info` for success logging, `console.error` for failures
- Never throw — wrap in try/catch, log errors, return silently (fire-and-forget pattern)
- Include JSDoc comments explaining the trigger conditions
