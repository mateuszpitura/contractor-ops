**Helper function `ensureSyncCronSchedule(connectionId: string, organizationId: string)`:**
Per D-12, create a QStash cron schedule for daily sync at 2 AM. This lives in the router file since it's called by bulkImport and triggerSync procedures.

```typescript
async function ensureSyncCronSchedule(connectionId: string, organizationId: string) {
  const qstashClient = new Client({ token: process.env.QSTASH_TOKEN! });
  const scheduleId = `gw-sync-${organizationId}`;

  const connection = await prisma.integrationConnection.findUnique({
    where: { id: connectionId },
    select: { configJson: true },
  });
  const config = (connection?.configJson ?? {}) as Record<string, unknown>;
  if (config.syncScheduleId) return; // Already has a schedule

  const schedule = await qstashClient.schedules.create({
    destination: `${process.env.NEXT_PUBLIC_APP_URL}/api/google-workspace/_sync`,
    body: JSON.stringify({ organizationId, connectionId }),
    cron: "0 2 * * *", // Daily at 2 AM (per D-12)
    scheduleId,
  });

  await prisma.integrationConnection.update({
    where: { id: connectionId },
    data: {
      configJson: { ...config, syncScheduleId: schedule.scheduleId ?? scheduleId },
    },
  });
}
```

**`bulkImport`** — `tenantProcedure.use(requirePermission({ member: ["create"] })).input(directoryImportInputSchema).mutation()`:

**SECURITY (CLAUDE.md compliance — never trust client input):** The `userGroupMemberships` field is accepted from the client for convenience but MUST NOT be trusted for role resolution. The server MUST re-fetch group memberships from Google API for ALL users being imported. Use the adapter's `listUserGroups` method to get authoritative group data server-side:

```typescript
// Re-fetch group memberships server-side (NEVER trust client-supplied group data for RBAC)
const { connection, credentials, adapter } = await getGoogleWorkspaceConnection(ctx.organizationId);
const serverGroupMemberships: Record<string, string[]> = {};
for (const user of input.users) {
  const groups = await adapter.listUserGroups(credentials.accessToken, user.email);
  serverGroupMemberships[user.email] = groups.map(g => g.email);
}
```

Then use `serverGroupMemberships` (NOT `input.userGroupMemberships`) for role resolution:
- For each user, determine final role: `userRoleOverrides[email] ?? firstMatchingGroupRole(serverGroupMemberships[email], groupRoleMappings) ?? defaultRole`
- Call `auth.api.createInvitation({ headers: ctx.headers, body: { email, role, organizationId: ctx.organizationId } })` (per D-11)
- Collect successes and failures (partial failure handling per research)
- Store import metadata in `IntegrationConnection.configJson`: `{ lastImportAt: new Date(), lastImportCount: succeeded.length }`
- After successful import, call `ensureSyncCronSchedule(connection.id, ctx.organizationId)` to set up daily sync (per D-12)
- Return `{ succeeded: Array<{ email, role }>, failed: Array<{ email, error }> }`

**`triggerSync`** — `tenantProcedure.use(requirePermission({ member: ["read"] })).mutation()`:
