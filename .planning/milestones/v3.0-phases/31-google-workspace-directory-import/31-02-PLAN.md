**3. Create `google-workspace-provider-section.tsx`**

Follow LinearProviderSection pattern. Per D-03, separate section. Per D-04, auto-open wizard after OAuth.

```typescript
export function GoogleWorkspaceProviderSection() {
  const t = useTranslations("Settings.integrations.googleWorkspace");
  const [wizardOpen, setWizardOpen] = useState(false);
  const searchParams = useSearchParams();

  // Check connection health
  const healthQuery = useQuery(
    trpc.integration.getHealth.queryOptions({ provider: "google-workspace" })
  );
  const health = healthQuery.data;
  const isConnected = health?.status === "CONNECTED";

  // D-04: After OAuth redirect, auto-open import wizard
  useEffect(() => {
    if (searchParams.get("google-workspace") === "connected") {
      setWizardOpen(true);
    }
  }, [searchParams]);

  return (
    <div className="space-y-4">
      <ProviderConnectionCard
        provider="google-workspace"
        displayName="Google Workspace"
        icon={<GoogleWorkspaceLogo className="size-8" />}
        description={isConnected ? t("descriptionConnected") : t("descriptionDisconnected")}
      />

      {isConnected && (
        <SyncStatusSection onImportClick={() => setWizardOpen(true)} />
      )}

      <DirectoryImportWizard
        open={wizardOpen}
        onOpenChange={setWizardOpen}
      />
    </div>
  );
}
```


3-step wizard inside a Dialog (max-w-3xl / 720px per UI-SPEC):

**Step indicator** at top: horizontal breadcrumb with 3 steps ("Preview directory", "Assign roles", "Review and import"). Active step: `text-primary border-b-2 border-primary`. Completed: checkmark icon + `text-primary`. Future: `text-muted-foreground`. Use `aria-current="step"` on active.

**Step 1 (Preview):**
- On open, query `trpc.googleWorkspace.listDirectory`
- Show Skeleton table during loading
- Render DirectorySummaryBar + DirectoryPreviewTable
- "Next: Roles" button (disabled if no users selected)
- Per UI-SPEC: no Cancel button, dialog [X] serves as dismiss
- Handle error state: show error banner with copy from UI-SPEC copywriting contract

**Step 2 (Roles):**
- On entering step 2, call `trpc.googleWorkspace.listUserGroups.useMutation()` with selected user emails
- Show RoleAssignmentControls for default role
- Show GroupRoleMappingStep with resolved groups
- "Back" and "Next: Review" buttons

**Step 3 (Confirm):**
- Compute role breakdown: count users per role considering overrides > group mappings > default
- Render ImportConfirmStep
- On confirm: call `trpc.googleWorkspace.bulkImport.useMutation()` with:
  - `users`: selected users mapped to `{ email, name, googleUserId }`
  - `defaultRole`: from step 2
  - `groupRoleMappings`: from step 2
  - `userRoleOverrides`: any per-user overrides
  - `userGroupMemberships`: from listUserGroups response (map user email to group emails). Note: the server re-fetches this data for RBAC security, but including it enables optimistic role breakdown display.
- On success: sonner toast "Successfully imported {N} users", close dialog, invalidate queries
- On partial failure: show error state per UI-SPEC with "{succeeded} of {total} users imported. {failed} failed -- review errors and retry."

Dialog becomes full-screen sheet on mobile (<768px) per responsive behavior spec. Use `DialogContent` with `className="max-w-3xl"` on desktop.
