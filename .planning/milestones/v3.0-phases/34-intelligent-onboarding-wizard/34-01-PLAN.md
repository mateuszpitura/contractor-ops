    **Create `packages/api/src/routers/onboarding-import.ts`:**

    Use `tenantProcedure` (any org member can onboard). Import Zod schemas from `@contractor-ops/validators`.

    Endpoints:

    1. `listSources` (query): Fetch all IntegrationConnection records for org. Return 4-element array: one per provider (JIRA, LINEAR, GOOGLE_WORKSPACE, SLACK) with `connected` based on whether a connection exists and has valid token, `selected: false` default.

    2. `fetchPeople` (query, input: fetchPeopleInputSchema): For each selected source, get IntegrationConnection, refresh token if needed via adapter `refreshToken()`, call `fetchUsersFromSource`. Use `Promise.allSettled` so individual tool failures don't block others. Get existing org members via `prisma.member.findMany` + user emails. Call `mergeByEmail`. Return merged list.

    3. `fetchProjects` (query, input: { sources: sourceProviderSchema[] }): For JIRA: call `GET /rest/api/3/project` then `GET /rest/api/3/project/{id}/statuses` for each. For LINEAR: call `linearGraphQL` for teams with states. Return combined array matching `fetchProjectsOutputSchema`.

    4. `startImport` (mutation, input: startImportInputSchema):
       - Create import job record in org Settings metadata: `{ importJob: { jobId: cuid(), status: "processing", totalItems, completedItems: 0, failedItems: [] } }`
       - Process people: iterate non-skipped people, call `auth.api.createInvitation({ headers: ctx.headers, body: { email, role, organizationId } })` with try/catch per item. Update Settings metadata progress after each item.
       - Process projects: call `createWorkflowTemplatesFromProjects` for non-skipped projects
       - Update import job status to "completed" (or "failed" if all failed)
       - Return `{ jobId }`
       - NOTE: For MVP, process synchronously within the mutation but update progress in DB after each item so `getProgress` polling works. QStash can be added later for true async if import sizes warrant it.

    5. `getProgress` (query, input: { jobId: string }): Read org Settings metadata, return import job matching jobId. Match `importProgressOutputSchema`.

    6. `retryFailedItem` (mutation, input: retryItemInputSchema): Read failed items from Settings metadata, find item by email, re-attempt `createInvitation`, update metadata (remove from failedItems, increment completedItems on success).
