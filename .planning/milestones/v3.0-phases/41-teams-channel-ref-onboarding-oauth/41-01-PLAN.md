In `storeConversationReference` function (around lines 104-113), the channel-scoped ref is currently keyed by `tenantId`:

```typescript
const teamId = (ref.conversation as Record<string, unknown> | undefined)
  ?.tenantId as string | undefined;
if (
  teamId &&
  ref.conversation?.conversationType === "channel"
) {
  teamConversationReferences[teamId] = ref as ConversationReference;
}
```

Change this to key by `ref.conversation.id` instead of `tenantId`. The `conversation.id` is the channel thread ID (e.g., "19:xxx@thread.tacv2") which is what `sendChannelAlert` passes as `params.channelId`.

Replace the block with:

```typescript
const channelId = ref.conversation?.id;
if (
  channelId &&
  ref.conversation?.conversationType === "channel"
) {
  teamConversationReferences[channelId] = ref as ConversationReference;
}
```

Remove the `as Record<string, unknown>` cast since `ref.conversation?.id` is a standard Bot Framework property (it's on the `ConversationAccount` type).

Update the JSDoc comment on `storeConversationReference` (line 69-72) to mention that channel refs are keyed by `conversation.id` (channel thread ID), not tenant ID.
