    **1. Create `packages/api/src/services/courier/inpost-webhook-handler.ts`**:
    - `verifyInPostSignature(rawBody: string, headers: Record<string, string>, secret: string): boolean` — HMAC-SHA256 verification using `crypto.createHmac("sha256", secret).update(rawBody).digest("hex")`, compare with `x-inpost-signature` header. If no secret configured, log warning and return true (graceful degradation per research open question 1)
    - `handleInPostWebhook(db: PrismaClient, organizationId: string, payload: InPostWebhookPayload): Promise<void>`:
      1. Validate payload with `inpostWebhookPayloadSchema.safeParse(payload)` — return early on failure with console.warn
      2. Find shipment by `externalId === String(payload.shipment_id)` AND `organizationId`
      3. If not found, try matching by `trackingNumber === payload.tracking_number` (fallback)
      4. Call `mapInPostStatus(payload.status)` — if null, log warning and return
      5. **Deduplication**: Check if `ShipmentEvent` already exists with same `shipmentId` and `status` — if yes, return early (per pitfall 3)
      6. Create `ShipmentEvent` with mapped status, update `Shipment.currentStatus`
      7. Auto-advance equipment status using the same `SHIPMENT_TO_EQUIPMENT_STATUS` mapping from equipment.ts (import pattern)
      8. Fire-and-forget: call `checkShipmentTaskCompletion` for workflow auto-completion
      9. If mapped status is in `NOTIFICATION_STATUSES`, fire notification via `dispatch()` (per D-06)

    **2. Create `packages/api/src/services/courier/inpost-polling-service.ts`**:
    - `pollInPostShipmentStatuses(db: PrismaClient, organizationId: string): Promise<{ checked: number; updated: number }>`:
      1. Load `CourierConfig` for carrier "inpost" and this org
      2. Decrypt API token using credential-service pattern
      3. Find all shipments where `carrier = "InPost"` AND `currentStatus NOT IN ("DELIVERED", "FAILED", "RETURNED")` AND `externalId IS NOT NULL`
      4. For each shipment (batch of up to 50):

        b. Map status via `mapInPostStatus`
        c. If mapped status differs from `currentStatus`, call `handleInPostWebhook` logic (reuse the dedup + event creation + auto-advancement)
      5. Return count of checked and updated shipments
      6. Log summary: `[inpost-polling] Org ${organizationId}: checked ${checked}, updated ${updated}`
