<objective>
Linear bidirectional sync backend: status mapping service (with PENDING_MAPPING->CONNECTED transition per D-03), issue creation/sync service, webhook handler, workflow integration hooks, and Wave 0 test stubs for API package.

Purpose: This is the core sync engine. Without it, connecting Linear does nothing -- issues are not created, statuses do not sync. This plan mirrors the Jira sync services (jira-status-mapping.ts, jira-issue-sync.ts, jira-webhook-handler.ts) for Linear's GraphQL API. The saveStatusMapping function also handles the critical D-03 flow: transitioning the connection from PENDING_MAPPING (set during OAuth) to CONNECTED when the admin saves their first mapping.
Output: Three service files + wiring into workflow router for auto-issue creation on task start + test stubs for behavioral verification.
</objective>

**1. Create `packages/api/src/services/linear-status-mapping.ts`** mirroring jira-status-mapping.ts:

Export these functions:
- `getStatusMapping(connectionId: string, teamId: string): Promise<LinearStatusMappingEntry[]>` -- Read IntegrationConnection, extract `(configJson as any).statusMappings?.[teamId] ?? []`, validate with z.array(linearStatusMappingEntrySchema).
- `saveStatusMapping(connectionId: string, teamId: string, mappings: LinearStatusMappingEntry[]): Promise<void>` -- Update IntegrationConnection.configJson, merging into existing statusMappings object. Uses prisma.integrationConnection.update with JSON merge. **CRITICAL (D-03 PENDING_MAPPING transition):** After saving the mappings, check if the connection's current status is `PENDING_MAPPING`. If so, also update the connection status to `CONNECTED` in the same prisma call. This completes the D-03 flow: OAuth sets PENDING_MAPPING -> admin saves status mapping -> connection becomes CONNECTED and bidirectional sync activates. The implementation should be:
  ```typescript
  const connection = await prisma.integrationConnection.findUniqueOrThrow({ where: { id: connectionId } });
  const existingConfig = (connection.configJson as Record<string, unknown>) ?? {};
  const updatedConfig = {
    ...existingConfig,
    statusMappings: { ...(existingConfig.statusMappings as Record<string, unknown> ?? {}), [teamId]: mappings },
  };
  await prisma.integrationConnection.update({
    where: { id: connectionId },
    data: {
      configJson: updatedConfig as Prisma.InputJsonValue,
      // Transition PENDING_MAPPING -> CONNECTED on first mapping save (D-03)
      ...(connection.status === "PENDING_MAPPING" ? { status: "CONNECTED" } : {}),
    },
  });
  ```
- `resolveLinearStateId(connectionId: string, teamId: string, workflowStatus: string): Promise<string | null>` -- Look up the mappings for this team, find entry where `workflowStatus` matches, return `linearStateId`. Return null if not mapped.
- `resolveInternalStatus(connectionId: string, teamId: string, linearStateId: string): Promise<string | null>` -- Reverse lookup: find entry where `linearStateId` matches, return `workflowStatus`. If not found, check `configJson.stateCache[teamId]` for the state name, log as unmapped per D-04, return null.

**1. Create `packages/api/src/services/linear-webhook-handler.ts`** mirroring jira-webhook-handler.ts:

Export `processLinearWebhook(payload: unknown, connectionId: string): Promise<void>`:

1. Validate payload with `linearWebhookPayloadSchema.safeParse(payload)`. If invalid, log warning and return.
2. Only process `action: "update"` with `updatedFrom?.stateId` present (state change). For `action: "create"` or `action: "remove"`, log and return early.
3. Find ExternalLink by: externalType="LINEAR_ISSUE", externalId=parsed.data.identifier (e.g., "ENG-123"). If not found, return early (unlinked issue -- normal per Pitfall 5).
4. Parse ExternalLink.metadataJson as LinearIssueMetadata.
5. Check loop prevention: if lastSyncOrigin="APP" and lastSyncAt within 30s, suppress this webhook (per D-04 / Pitfall 3).
6. Check dedup: if webhookTimestamp is within DEDUP_WINDOW_MS of lastSyncAt, skip.
7. Get IntegrationConnection. Extract teamId from parsed.data.teamId.
8. Resolve internal status: call `resolveInternalStatus(connectionId, teamId, parsed.data.stateId)`. If null, log as unmapped per D-04 into WebhookDelivery record and return.
9. Resolve the state name from `configJson.stateCache[teamId][stateId]`. If not in cache, call `linearGraphQL` to fetch `workflowState(id: stateId) { id name type }` and update cache.
10. Update the linked workflow TaskRun status to the resolved internal status. Use prisma.taskRun.update.
11. Update ExternalLink metadataJson: status=new state name, statusType=new state type, lastSyncOrigin="LINEAR", lastSyncAt=now.
12. Log to IntegrationSyncLog: direction=INBOUND, syncType="STATUS_UPDATE", entityType=TASK_RUN, entityId=taskRun.id, status=SUCCESS.

Also export `registerLinearWebhook(connectionId: string, teamId: string): Promise<string>`:
1. Decrypt credentials.
2. Call Linear GraphQL mutation: `webhookCreate(input: { url: "${APP_URL}/api/webhooks/linear", teamId: "${teamId}", resourceTypes: ["Issue"], enabled: true })`. The webhookCreate mutation returns `{ success, webhook { id enabled } }`.
3. Store webhook ID in `configJson.webhooks[teamId]`.
4. Return webhook ID.

And `deregisterLinearWebhook(connectionId: string, teamId: string): Promise<void>`:
1. Read webhook ID from `configJson.webhooks[teamId]`.
2. Call `webhookDelete(id: webhookId)` mutation.
3. Remove from configJson.
