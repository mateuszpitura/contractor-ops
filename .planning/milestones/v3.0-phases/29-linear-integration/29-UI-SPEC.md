# Phase 29 — UI Design Contract

> Visual and interaction contract for the Linear integration phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding, status dot margin |
| sm | 8px | Compact element spacing, chip internal gap |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Section padding, dialog section gaps |
| xl | 32px | Layout gaps between provider cards |
| 2xl | 48px | Major section breaks (integrations tab sections) |
| 3xl | 64px | Page-level spacing |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 16px | 600 (semibold) | 1.2 |
| Display | 20px | 600 (semibold) | 1.2 |

Labels are visually distinguished from body text via color (`text-muted-foreground`) or uppercase transform, not via a separate font weight.

Note: Issue identifier text on chips uses JetBrains Mono at 12px / 600 weight (matching existing Jira chip `font-mono text-xs font-semibold`).

## Color

**Primary visual anchor:** the "Connect Linear" CTA button when disconnected; the status badge + workspace name when connected.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.985 0.003 85)` / `--background` | Page background, surfaces |
| Secondary (30%) | `oklch(1 0 0)` / `--card` | Provider cards, dialogs, popovers |
| Accent (10%) | `oklch(0.44 0.145 178)` / `--primary` (Deep Teal) | Primary CTAs: "Connect Linear", "Save Mapping" buttons |
| Destructive | `oklch(0.58 0.23 18)` / `--destructive` | "Disconnect Linear" button text, error badge |

### Provider Brand Colors

| Provider | Color | CSS Variable | Usage |
|----------|-------|-------------|-------|
| Linear | `oklch(0.58 0.14 290)` | `--linear-brand` | Linear issue chip background tint, Linear logo accent |
| Jira (existing) | `oklch(0.55 0.18 260)` / `--info` | Jira issue chip (unchanged, for contrast reference) |

Accent reserved for: "Connect Linear" CTA button, "Save Mapping" CTA button, focus rings on interactive elements, primary-colored status badge for connected state.

### Status Dot Colors (on issue chips)

Reuse existing semantic tokens from globals.css:

| Status Category | Token | Chip Dot Class |
|----------------|-------|----------------|
| Not started (backlog/triage) | `--muted-foreground` | `bg-muted-foreground` |
| In progress (started/active) | `--info` | `bg-info` |
| Completed (done/closed) | `--success` | `bg-success` |
| Cancelled | `--destructive` | `bg-destructive` |

This phase replicates existing Jira integration UI patterns with Linear branding. All components use existing shadcn primitives.

### New Components

|-----------|----------------------|----------------|

| `LinearIssueChip` | Tooltip | Mirrors `JiraIssueChip` |
| `LinearLogo` | (SVG component) | Mirrors `JiraLogo` |

### Reused Components (no changes)

| Component | Usage in This Phase |
|-----------|-------------------|
| `ProviderConnectionCard` | Linear connection card in integrations tab (provider="linear") |
| `ProviderDetailSheet` | Linear sync/webhook log detail view |
| `IntegrationsTab` | Add Linear provider section to grid |

### Component Specifications

#### LinearProviderSection

Structure mirrors `JiraProviderSection` exactly:
1. `ProviderConnectionCard` with provider="linear", Linear logo at 32x32, status badge
2. When connected: "Configure Status Mapping" outline button
3. When connected + scope expansion needed: warning banner (amber border, AlertTriangle icon)
4. `LinearStatusMappingDialog` opens on button click

#### LinearStatusMappingDialog

Structure mirrors `JiraStatusMappingDialog`:
- Dialog max width: `sm:max-w-2xl`
- Title: "Status Mapping"
- Description: "Map workflow task statuses to Linear workflow states for {teamName}."
- Team selector: shadcn Select with "Select a team" placeholder
- Mapping table: 6 rows (TODO, IN_PROGRESS, DONE, BLOCKED, SKIPPED, CANCELLED) x 2 columns (Workflow Status, Linear State)
- Unmapped rows show AlertTriangle icon with tooltip: "Not mapped -- status changes for this state will be ignored"
- Footer: "Discard Changes" (outline) + "Save Mapping" (primary, disabled until changes exist)
- Smart defaults pre-populate on first load per D-02

#### LinearIssueChip

Structure mirrors `JiraIssueChip` with Linear branding per D-08/D-09:
- Container: `inline-flex items-center gap-1 rounded-md border px-2 py-1 max-w-[200px]`
- Background tint: `bg-[oklch(0.58_0.14_290/8%)]` (Linear purple at 8% opacity) instead of `bg-muted/50`
- Border: `border-[oklch(0.58_0.14_290/20%)]` (Linear purple at 20% opacity) instead of `border-border`
- Hover: `hover:bg-[oklch(0.58_0.14_290/14%)]`
- Status dot: 6px circle with color from status dot mapping above
- Issue identifier: `font-mono text-xs font-semibold text-primary` (e.g., "ENG-123")
- Status text: `text-xs text-muted-foreground truncate`
- Tooltip on hover: shows issue title/summary
- Click: opens Linear issue URL in new tab
- Aria label: "Open Linear issue {identifier} in new tab"

#### LinearLogo

- SVG component accepting `className` prop
- Uses Linear's official logomark (geometric angular mark)
- Default render at 32x32 in provider cards, 16x16 inline

## Interaction States

### Provider Connection Card (Linear)

| State | Visual |
|-------|--------|
| Loading | Skeleton: 32x32 icon placeholder + 16px text placeholder + 64px body placeholder |
| Disconnected | Card with Linear logo, description text, "Connect Linear" primary button |
| Connected | Card with logo, status badge (emerald), connected workspace name, connection date, token expiry, "Manage" + "Disconnect Linear" outline buttons |
| Re-auth required | Amber status badge, warning text, "Reconnect Linear" + "Disconnect Linear" buttons |
| Error | Destructive status badge, error text, "Reconnect Linear" button |

### Status Mapping Dialog

| State | Visual |
|-------|--------|
| Loading teams | Select trigger shows Loader2 spinner |
| No team selected | Table hidden, only team selector visible |
| Loading statuses | Table cells show Loader2 spinners |
| Mapping in progress | Select dropdowns populated, unmapped rows flagged with warning icon |
| Smart defaults applied | All selects pre-populated with best-guess mappings, "Save Mapping" enabled |
| Saving | "Save Mapping" button shows Loader2 spinner, disabled |
| Save success | Dialog closes, toast appears |
| Save error | Toast error appears, dialog remains open |

### Issue Chip

| State | Visual |
|-------|--------|
| Default | Linear purple tinted chip with status dot + identifier + status text |
| Hover | Background opacity increases to 14%, cursor pointer |
| Focus | Focus ring (--ring token) |
| Tooltip visible | Full issue title shown on hover/focus |

## Accessibility

- All interactive elements keyboard-navigable (Tab, Enter, Space, Escape)
- Dialog traps focus when open, returns focus on close
- Select dropdowns announce selected value to screen readers
- Issue chips use `<a>` tag with descriptive `aria-label`
- Status dots are decorative (aria-hidden), status conveyed via text
- Warning icons paired with visible warning text (not icon-only)
- Color alone never conveys status -- always paired with text label
- Minimum contrast ratio 4.5:1 for all text against backgrounds
