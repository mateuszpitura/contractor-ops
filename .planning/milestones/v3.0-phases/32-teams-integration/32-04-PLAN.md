3. **TeamsChannelMappingCard** (`teams-channel-mapping-card.tsx`):
   Per D-09, D-10, and UI-SPEC Channel Mapping Flow:
   - `"use client"` directive.
   - `useTranslations("Settings.integrations.teams")` for copy.
   - Fetch teams list: `trpc.teams.getTeams.useQuery()`. If only one team, auto-select it. If multiple, show a team selector first.
   - Fetch channels for selected team: `trpc.teams.getChannels.useQuery({ teamId })`.
   - Fetch current mapping: `trpc.teams.getChannelMapping.useQuery()`.
   - Render a Card with:
     - Heading "Channel mapping" (`text-lg font-semibold`)
     - Description "Configure which channel receives each notification type." (`text-sm text-muted-foreground`)
     - 5 rows, each with:
       - Label (font-semibold): "Approvals", "Invoices", "Contracts", "Tasks", "Equipment"
       - Select dropdown populated with channels from the selected team
       - Placeholder "Select a channel"
     - Refresh icon button (RefreshCw from lucide-react) to re-fetch channels. Show Loader2 spinner while fetching.
     - "Save mapping" Button (accent color). On click: call `trpc.teams.saveChannelMapping.useMutation()` with the mapping object. Show Loader2 during save. Sonner toast "Channel mapping saved." on success, "Failed to save channel mapping. Try again." on error.
   - Loading state: Skeleton components for dropdowns while channels load.
   - Empty state: When no channels returned, show "No channels available. Ensure the bot is added to at least one team." message.
   - Error state: On channel fetch failure, show "Could not load Teams channels. Verify the bot is installed in your Teams workspace and try again."
   - Local state for selections -- user picks channels, then commits with Save.
   - Channel mapping row layout: Label and Select side-by-side on desktop, stacked on mobile (`flex flex-col sm:flex-row sm:items-center sm:justify-between gap-2`).
   - Accessibility: Each Select has `aria-label="{category} notification channel"` per UI-SPEC.
