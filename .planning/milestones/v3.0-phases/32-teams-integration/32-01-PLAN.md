Per D-03, refactor the `dispatch()` function in `notification-service.ts`:

1. **Replace direct Slack imports.** Remove the direct imports of `getSlackUserIdForUser`, `sendApprovalCard`, `sendReminderDM` from `./slack-client.js`. Instead import `getConnectedMessagingProviders` from `./messaging/index.js` and the param types from `./messaging/types.js`.

2. **Remove the `sendSlackDM` helper function entirely.** Its logic moves into the provider iteration loop.

3. **Replace the Slack-specific block** (lines ~289-298 in current code, the `if (prefs.channelSlack)` block) with a provider iteration loop:
   ```typescript
   // Messaging provider dispatch (Slack, Teams, future platforms)
   const providers = await getConnectedMessagingProviders(event.organizationId);
   for (const provider of providers) {
     const prefKey = provider.platform === "slack" ? "channelSlack" : "channelTeams";
     if (!prefs[prefKey]) continue;

     try {
       const recipientId = await provider.getUserId(event.organizationId, userId);
       if (!recipientId) continue;

       if (event.type === "APPROVAL_REQUEST") {
         const meta = event.metadata ?? {};
         await provider.sendApprovalCard({
           organizationId: event.organizationId,
           recipientId,
           invoiceNumber: (meta.invoiceNumber as string) ?? "",
           contractorName: (meta.contractorName as string) ?? "",
           amount: String(meta.amount ?? ""),
           currency: (meta.currency as string) ?? "",
           dueDate: (meta.slaDeadline as string) ?? "",
           invoiceId: (meta.invoiceId as string) ?? "",
           flowId: (meta.flowId as string) ?? "",
         });
       } else {
         await provider.sendReminderDM({
           organizationId: event.organizationId,
           recipientId,
           text: `*${event.title}*\n${event.body}`,
         });
       }
     } catch (error) {
       console.error(
         `[notification-service] ${provider.platform} send failed for user=${userId}:`,
         error,
       );
     }
   }
   ```

4. **Update `getOrCreatePreferences`** to include `channelTeams: false` in the default creation (Teams defaults to off until user enables it, unlike Slack which defaults to true for backward compatibility).

5. **Keep the email block unchanged** — email dispatch is independent of messaging providers.

6. **Keep the `buildEntityUrl` and `buildPreferencesUrl` helpers unchanged.**
