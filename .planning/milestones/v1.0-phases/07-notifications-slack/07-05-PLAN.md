Update notification-service.ts to replace placeholder senders with real implementations per D-11:

sendNotificationEmail(userId, event):
1. Look up user email from database (query User by userId).
2. Call renderNotificationEmail(event.type, metadata) to get subject + react component.
3. Build ctaUrl from NEXT_PUBLIC_APP_URL + entity route. Build preferencesUrl to /settings?tab=notifications per D-07.
4. Call resend.emails.send with from notifications@contractorhub.io, subject, react template, List-Unsubscribe header.
5. Wrap in try/catch, log error on failure.

sendSlackDM(userId, event):
1. Call getSlackUserIdForUser. If null, skip silently.
2. For APPROVAL_REQUEST: call sendApprovalCard with invoice metadata (invoiceNumber, contractorName, amount, currency, slaDeadline, invoiceId, flowId).
3. For other events: call sendReminderDM with text block of title + body.
4. Wrap in try/catch, log error on failure.

Wire dispatch into approval router:
- In submitForApproval mutation, after flow creation: dispatch APPROVAL_REQUEST to first approver with invoice metadata including invoiceNumber, contractorName, amount, currency, slaDeadline, invoiceId, flowId.
- In advanceFlow mutation, after decision: dispatch APPROVAL_DECISION to invoice owner with decision and approver name. If flow advances to next level, also dispatch APPROVAL_REQUEST to next approver.

Wire dispatch into workflow router:
- In start mutation, after task assignment: dispatch TASK_ASSIGNED to assignee with taskTitle, workflowName, contractorName.
- In reassignTask mutation: dispatch TASK_ASSIGNED to new assignee.

Wire dispatch into invoice router:
- In create mutation, after creation: dispatch INVOICE_RECEIVED to finance team users (query members with FINANCE_ADMIN role). Include invoiceNumber, contractorName, amount, currency.

Helper getFinanceTeamUserIds(orgId): Query org members with FINANCE_ADMIN role, return user IDs array.

CONTRACT_EXPIRING is NOT dispatched inline -- handled by cron reminder route (Plan 02). No contract router changes needed.

Import notificationService in each router. All dispatch calls should be fire-and-forget (do not await or catch should not block the main operation).
