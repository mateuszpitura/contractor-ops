**Notification dispatch service (packages/api/src/services/notification-service.ts):**

Create the central dispatch service per D-11 (inline dispatch, no queue).

Interface:
```typescript
interface NotificationEvent {
  organizationId: string;
  type: "APPROVAL_REQUEST" | "APPROVAL_DECISION" | "TASK_ASSIGNED" | "TASK_OVERDUE" | "CONTRACT_EXPIRING" | "INVOICE_RECEIVED";
  recipientUserIds: string[];
  title: string;
  body: string;
  entityType: EntityType;
  entityId: string;
  metadata?: Record<string, unknown>;
}
```

Implement `dispatch(event: NotificationEvent)`:
1. For each recipientUserId, call `getOrCreatePreferences(userId, orgId, type)`.
2. If channelInApp (always true by default), create Notification row with channel=IN_APP, status=SENT, sentAt=now().
3. Deduplication: Before creating, check if a Notification exists for (userId + type + entityId) created within last 60 seconds. Skip if duplicate.
4. If channelEmail, call placeholder `sendNotificationEmail(userId, event)` -- logs a TODO for now. Plan 02 implements the actual email sending.
5. If channelSlack, call placeholder `sendSlackDM(userId, event)` -- logs a TODO for now. Plan 02 implements the actual Slack sending.
6. Wrap all external calls (email/Slack) in try/catch so failures don't break the main operation. Log errors.

Implement `getOrCreatePreferences(userId, orgId, type)`:
- findFirst by userId + notificationType.
- If not found, create with defaults: channelEmail=true, channelSlack=true, channelInApp=true, digestMode=false.
- Return the preference record.

Use the `plain()` JSON serialize pattern from Phase 02 decision if needed for tRPC returns.

**Notification router (packages/api/src/routers/notification.ts):**

Follow existing router pattern from approval.ts. Use `tenantProcedure` for all procedures.

Procedures:
- `list` (query): Input notificationListSchema. Query Notification where organizationId=ctx.orgId, userId=ctx.userId, filtered by type/status/unreadOnly. Order by createdAt DESC. Paginate with skip/take. Return { items, total, page, totalPages }.
- `unreadCount` (query): No input. Count Notification where userId=ctx.userId, status IN (PENDING, SENT) (i.e., not READ). Return { count: number }.
- `markRead` (mutation): Input notificationMarkReadSchema. Update Notification set readAt=now(), status=READ where id=input.notificationId AND userId=ctx.userId. Return success.
- `markAllRead` (mutation): No input. UpdateMany Notification set readAt=now(), status=READ where userId=ctx.userId, organizationId=ctx.orgId, readAt IS NULL.
- `getPreferences` (query): No input. Find all UserNotificationPreference for userId=ctx.userId. If fewer than 6 rows, call getOrCreatePreferences for missing types. Return array of 6 preference objects.
- `updatePreferences` (mutation): Input notificationPreferenceUpdateSchema. For each preference in input, upsert UserNotificationPreference by userId + notificationType. channelInApp always set to true regardless of input.

**Reminder router (packages/api/src/routers/reminder.ts):**

Use `tenantProcedure` with `requirePermission({ organization: ['update'] })` for create/update/delete (admin only). List is tenant-scoped for all users.

Procedures:
- `list` (query): Find all ReminderRule where organizationId=ctx.orgId, ordered by createdAt DESC. Return array.
- `create` (mutation, admin): Input reminderRuleCreateSchema. Create ReminderRule with organizationId=ctx.orgId. Return created rule.
- `update` (mutation, admin): Input z.object({ id: z.string() }).merge(reminderRuleUpdateSchema). Update where id AND organizationId. Return updated rule.
- `delete` (mutation, admin): Input z.object({ id: z.string() }). Delete where id AND organizationId. Also delete related ReminderInstances.
- `toggleActive` (mutation, admin): Input reminderRuleToggleSchema. Update active field. If deactivating, cancel pending ReminderInstances (set status=CANCELLED).

**Integration router (packages/api/src/routers/integration.ts):**

Use `tenantProcedure` with `requirePermission({ organization: ['update'] })` for admin-only operations (per D-08).

Procedures:
- `getSlackStatus` (query): Find IntegrationConnection where organizationId=ctx.orgId, provider=SLACK. Return { connected: boolean, status, displayName, connectedAt, connectedByUser } or null if not found.
- `getOAuthUrl` (query, admin): Generate Slack OAuth authorize URL with client_id from env, bot scopes (chat:write,users:read,users:read.email,im:write), redirect_uri, and HMAC-signed state containing orgId + userId + timestamp per research pitfall 6. Return { url: string }.
- `disconnect` (mutation, admin): Update IntegrationConnection set status=DISCONNECTED, clear credentialsRef. Return success.
- `listUserMappings` (query): Find all ExternalLink where organizationId, integrationConnectionId (Slack), externalType='SLACK_USER'. Join with org members to show matched/unmatched. Return array with user info + slack info + status.
- `linkUser` (mutation, admin): Input slackUserLinkSchema. Create ExternalLink mapping userId to Slack user externalId. entityType=USER.
- `unlinkUser` (mutation, admin): Input slackUserUnlinkSchema. Delete ExternalLink by id where organizationId.
- `syncUsers` (mutation, admin): Placeholder that will be called after OAuth to auto-match by email (per D-10). For now, accepts no input and returns { matched: number, total: number }.
