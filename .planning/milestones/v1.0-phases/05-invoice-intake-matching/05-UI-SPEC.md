## Spacing Scale

Inherited from Phase 1. No changes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding, status chip internal padding |
| sm | 8px | Compact element spacing, match card field gaps, chip bar inter-chip gap |
| md | 16px | Default element spacing, form field gaps, gap between metadata sections, gap between invoice list rows |
| lg | 24px | Section padding within cards, match card section padding, metadata form section gaps |
| xl | 32px | Layout gaps between major sections, detail page content padding, gap between PDF viewer and metadata panel |
| 2xl | 48px | Page horizontal padding (desktop) |
| 3xl | 64px | Not used in Phase 5 |

Exceptions:
- Side-by-side detail layout gutter: 32px between PDF viewer (60%) and metadata panel (40%)
- PDF viewer minimum height: 640px (ensures enough visible area for A4 invoice preview)
- Status chip bar height: 40px (consistent touch target height)
- Status chip minimum width: 80px (prevents chips from being too narrow with short labels)
- Duplicate warning banner padding: 16px vertical, 24px horizontal (prominent but not overwhelming)
- Match confidence indicator: 8px diameter dot (green/yellow/red)
- Side panel width: 480px (consistent with Phase 2, 3, and 4)
- Side panel animation: 200ms ease-out slide from right (consistent with previous phases)
- Upload progress bar height: 4px per file (thinner than page-level progress, stacked vertically)

## Interaction Patterns

### Invoices Page -- Status Chip Bar

- **Layout:** Horizontal row of Badge-style chips above the table. Each chip: label + count in parentheses. All / Received / Matched / Unmatched / Discrepancy / Pending Approval / Approved / Ready for Payment.
- **Active state:** Selected chip uses `--primary` at 10% opacity background with `--primary` text. Unselected chips use `--muted` background with `--muted-foreground` text.
- **Click:** Clicking a chip filters the table to that status. "All" clears the filter. Active filter synced to URL via nuqs `parseAsString` for deep-linking.
- **Counts:** Each chip shows live count fetched from `invoice.statusCounts` tRPC query. Counts update after any status change.
- **Overflow:** On smaller screens (< 1280px), chips horizontally scroll with fade gradient on edges.

### Invoices Page -- Data Table

- **Columns:** Invoice # (monospace), Contractor (link), Issue date, Due date, Net amount (right-aligned, formatted with locale), Gross amount (right-aligned), Currency, Status (badge), Match status (dot + label), Source (icon).
- **Default sort:** Due date ascending (soonest due first).
- **Sorting:** Click column header to toggle ascending/descending/none. Single-column sort.
- **Row click:** Opens side panel (Sheet from right, 480px). Does NOT navigate.
- **Row double-click or "Open" button in side panel:** Navigates to `/invoices/[id]`.
- **Search:** Debounce 300ms, minimum 2 characters. Server-side full-text search across invoice number, contractor name, contractor NIP.
- **Filters:** Status (multi-select via chip bar above), Contractor (search-select), Match status (multi-select), Source (upload/email), Date range (issue date). Active filters as badges below search bar.
- **Pagination:** Bottom of table. Page size (10, 25, 50). Previous/Next. "Page X of Y".
- **Overdue highlighting:** Rows where due date is past and status is not Paid/Void have `--destructive` at 5% opacity background tint. Due date cell text in `--destructive`.
- **Amounts display:** All monetary amounts formatted as locale currency with 2 decimal places. Internally stored as integer grosze, displayed divided by 100.

### Invoice Upload (Multi-File Batch)

- **Trigger:** "Upload invoices" CTA opens the DropZone (expanded inline on /invoices page, or via dialog on contractor profile tab).
- **DropZone:** Reused from Phase 3 with `accept: 'application/pdf'` and `multiple: true`.
- **File selection:** On file selection, upload fires immediately per file via presigned URL XHR (established Phase 3 pattern). Each file creates a separate Invoice record in RECEIVED status.
- **Progress display:** Stacked list of files below DropZone during upload. Each file: file name (truncated to 40 chars), file size, progress bar (4px, `--primary` fill), status icon (Loader2 spinning during upload, CheckCircle2 on success, XCircle on error).
- **Completion:** When all files complete, Sonner toast: "{count} invoices uploaded. Review your drafts." with link to filter table to RECEIVED status.
- **Error handling:** Individual file failures show error icon + retry button on that file row. Other files continue.

### Invoice Detail Page (Side-by-Side Layout)

- **URL:** `/invoices/[id]`
- **Layout:** CSS grid with two columns. Left: 60% width, PDF viewer (sticky, full viewport height minus header). Right: 40% width, scrollable content column.
- **Left panel (PDF viewer):** Browser-native `<object>` tag (reused PdfPreview from Phase 3). Falls back to download link if browser cannot render. Minimum height: 640px.
- **Right panel (scrollable):** Stacked sections:
  1. Header: Invoice number (Heading, monospace), status badge, source icon. Breadcrumb: Invoices / FV/2026/03/001.
  2. Duplicate warning banner (conditional, only if duplicate detected). Left red border accent (3px `--destructive`). Banner with heading, body, "View original" link, "Not a duplicate" ghost button.
  3. Match card (always visible). Shows match result or manual matching UI. See Match Card section below.
  4. Metadata form (editable while RECEIVED status). All fields from copywriting contract. React Hook Form + Zod resolver.
  5. Action bar: "Save draft" (secondary) + "Submit for matching" (primary, only in RECEIVED status) + overflow menu (Void invoice).
- **Read-only mode:** When invoice status is beyond RECEIVED, metadata form fields become read-only (disabled inputs with `--muted` background). Action bar changes accordingly.

### Match Card (on Invoice Detail Page)

- **Auto-matched state:** Card with `--muted` background. Sections:
  - Match confidence: colored dot (8px) + label (Strong/Partial/Weak). Score percentage in `--muted-foreground`.
  - Matched contractor: name as link to contractor profile, NIP in monospace.
  - Matched contract: contract number as link to contract detail, contract type badge.
  - Deviation display: "Expected: 10,000.00 PLN" vs "Actual: 10,500.00 PLN" in monospace. Deviation: "+5.0%" in green (within threshold) or red (above threshold) with alert icon.
  - Flags: Inline badge-style flags for issues (no active contract, expired contract, currency mismatch). Each flag: warning/destructive badge with icon.
- **Unmatched state:** Card shows "No match found" with `--warning` at 10% opacity background. Below: contractor search picker (Command component with debounced search by name/NIP, 300ms). After contractor selected: contract picker (Select filtered to selected contractor's active contracts). "Confirm match" primary CTA. Both pickers show loading skeletons while searching.
- **Manual match confirmed:** Card updates to show matched entities with "Manually matched" label (blue `--info` badge).

### Duplicate Detection Banner

- **Position:** Top of right panel on invoice detail page, below header, above match card.
- **Appearance:** Card with 3px left border in `--destructive`. Warning icon (AlertTriangle, Lucide) + heading + body text with link to original invoice.
- **Actions:** "View original" (link, opens original in new tab) + "Not a duplicate" (ghost button, dismisses banner and records dismissal).
- **Persistence:** Once dismissed, banner does not reappear for this invoice.

### Invoice Side Panel (Row Click)

- **Open:** Slide from right, 200ms ease-out. Backdrop overlay at 20% opacity.
- **Close:** Click backdrop, press Escape, or click X button.
- **Content:** Invoice number (Heading, monospace), status badge, match status dot + label. Contractor name as link. Amounts section: Net, VAT, Gross (formatted). Dates: Issue date, Due date. Source icon + label. "Open invoice" Indigo CTA at bottom.

### Contractor Profile -- Invoices Tab

- **Layout:** TanStack Table pre-filtered to contractor ID. Same columns as /invoices page minus Contractor column.
- **CTA:** "Upload invoice" button top-right. Opens DropZone dialog with contractor pre-linked (entityType=INVOICE, entityId set after creation).
- **Empty state:** See copywriting contract above.

### Email Intake Settings (Settings Page)

- **Location:** Settings page, new "Invoice Matching" section (Card component, same pattern as ExpiryReminderDefaults from Phase 3).
- **Content:**
