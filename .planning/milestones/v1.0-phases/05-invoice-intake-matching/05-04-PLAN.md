  <action>
    1. match-card.tsx: Card component showing match results or manual matching UI.
       Props: invoice (with contractor, contract, matchResults, flagsJson), onMatchConfirmed callback.

       Auto-matched state (when matchStatus is MATCHED, PARTIAL, or DISCREPANCY):
       - Card with muted background.
       - Match confidence section: 8px colored dot (inline-block w-2 h-2 rounded-full) + label. Score >= 90 = green dot + "Strong match". Score 50-89 = warning dot + "Partial match". Score < 50 = destructive dot + "Weak match". Score percentage in muted-foreground.
       - Matched contractor: contractor legalName as Link (text-primary) to /contractors/[id], NIP in font-mono text-muted-foreground.
       - Matched contract: contract title as Link to /contracts/[id], contract type as Badge.
       - Deviation display: "Expected: {formatted}" vs "Actual: {formatted}" in font-mono. Deviation: "+5.0%" in green (within threshold) or destructive (above threshold) with AlertTriangle icon.
       - Flags: Inline badges for each flag in flagsJson. "No active contract found" = warning badge. "Matched contract is expired" = destructive badge. "Currency mismatch" = destructive badge. Each with appropriate icon.

       Unmatched state (when matchStatus is UNMATCHED):
       - Card with warning/10% opacity background.
       - "No match found" heading.
       - Contractor search picker: Command component (shadcn) with debounced search (300ms). Calls trpc.invoice.searchContractors. Shows results: legalName, NIP, status badge. On select: sets selectedContractorId state.
       - Contract picker: Select component filtered to selected contractor's active contracts. Calls trpc.invoice.contractsForContractor when contractor selected. Shows: contract title, type, status.
       - "Confirm match" primary CTA: calls trpc.invoice.manualMatch with selectedContractorId and selectedContractId. On success: toast "Match confirmed", calls onMatchConfirmed.
       - Both pickers show loading skeletons while searching.

       Manual match confirmed state (when matchStatus is MANUALLY_CONFIRMED):
       - Same as auto-matched display but with blue info Badge "Manually matched".

    2. duplicate-warning.tsx: Alert-style banner for duplicate detection.
       Props: duplicateInvoiceId (string | null), invoiceNumber (string), onDismiss callback.
       - Only renders when flagsJson contains "DUPLICATE_SUSPECTED".
       - Card with 3px left border in destructive color (border-l-[3px] border-destructive).
       - AlertTriangle icon (lucide) + Heading "Possible duplicate detected".
       - Body: "This invoice may be a duplicate of {invoiceNumber}." (need to fetch duplicate invoice number if only ID available, or pass it as prop).
       - "View original" link: navigates to /invoices/[duplicateInvoiceId] in new tab (target="_blank").
       - "Not a duplicate" ghost button: calls trpc.invoice.dismissDuplicate, then calls onDismiss callback. No confirmation needed (inline dismissal).
       - Once dismissed, banner disappears (local state or server state via flagsJson update).

    All text via useTranslations("Invoices") with exact copy from UI-SPEC copywriting contract.
