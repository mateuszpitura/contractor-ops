  <action>
    Install resend SDK in api package: pnpm add resend --filter @contractor-ops/api

    Create apps/web/src/app/api/webhooks/resend-inbound/route.ts as a Next.js API route (NOT tRPC):

    1. Import Resend SDK, prisma from db package, R2 service from api package, crypto for hashing.

    2. Export async function POST(request: NextRequest):

    a) Verify webhook signature:
       - Read raw body via request.text()
       - Extract svix-id, svix-timestamp, svix-signature headers
       - Use resend.webhooks.verify(payload, headers) wrapped in try/catch
       - On failure: return NextResponse.json({ error: "Invalid signature" }, { status: 401 })
       - Fail loudly if RESEND_WEBHOOK_SECRET is not set: return 500 with "Webhook secret not configured"

    b) Check event type:
       - If event.type !== "email.received", return NextResponse.json({ received: true }) with 200

    c) Parse org slug from recipient:
       - Helper function parseOrgSlugFromEmail(toAddress: string): string | null
       - Handle "Display Name <email@domain>" format
       - Extract domain, check suffix ".contractorhub.io", extract slug before suffix
       - Lookup org: prisma.organization.findFirst where slug === parsed slug
       - If no org found, log warning and return 200 (don't retry)

    d) Fetch email details via Resend Receiving API:
       - const emailDetails = await resend.emails.receiving.get(event.data.email_id)
       - Get attachments array

    e) Filter PDF attachments:
       - Filter attachments where content_type === "application/pdf"
       - If no PDF attachments found, log "No PDF attachments in email from {from}" and return 200
       - Non-PDF attachments: if at least one PDF exists, treat non-PDFs as SUPPORTING_ATTACHMENT

    f) For each PDF attachment (fire-and-forget async pattern for speed, but await within a Promise.all for the immediate response):
       - Fetch attachment data via resend.emails.receiving.attachments.get({ id: att.id, emailId: event.data.email_id })
       - Download the PDF content from download_url via fetch
       - Generate R2 key: `${orgId}/invoices/${cuid()}.pdf`
       - Upload to R2 using the S3 client PutObjectCommand (not presigned URL since this is server-side)
       - Create Document record (fileName: att.filename, mimeType: "application/pdf", sizeBytes: att.size, storageKey: R2 key, uploadStatus: CONFIRMED, virusScanStatus: PENDING)
       - Fire-and-forget virus scan (same pattern as document.ts confirmUpload)
       - Create Invoice record: source EMAIL_INTAKE, status RECEIVED, matchStatus UNMATCHED, submittedByEmail: event.data.from, receivedAt: new Date()
       - Create InvoiceFile linking invoice to document with role SOURCE_ORIGINAL
       - Create DocumentLink with entityType INVOICE, entityId invoice.id
       - For non-PDF attachments of same email: create as SUPPORTING_ATTACHMENT role

    g) Return NextResponse.json({ processed: true, count: pdfCount })

    Update .env.example:
    - Add RESEND_API_KEY=your-resend-api-key
    - Add RESEND_WEBHOOK_SECRET=your-resend-webhook-secret
    - Add comment: # Resend - Email intake for invoice processing
