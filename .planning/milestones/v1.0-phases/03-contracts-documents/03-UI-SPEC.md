## Interaction Patterns

### Contract List Table

- **Columns:** Title, Contractor, Type, Status, Start date, End date, Rate, Currency, Billing cycle, Owner, Compliance risk. Default sort: end date ascending (expiring soonest first).
- **Column visibility:** Dropdown menu with checkboxes for each column. Persisted per user via `localStorage`.
- **Sorting:** Click column header to toggle ascending/descending/none. Single-column sort only.
- **Row selection:** Checkbox in first column. Select-all checkbox in header selects current page only. Bulk action bar appears above table when rows selected.
- **Row click:** Opens side panel (Sheet from right, 480px). Does NOT navigate away from the list.
- **Search:** Debounce 300ms, minimum 2 characters. Server-side PostgreSQL full-text search across title, contractor name, type. Loading indicator in search input during query.
- **Filters:** Popover panels for each filter. Multi-select for status, type, owner. Date range picker for end date. Single-select for compliance risk. Active filters shown as badges below filter bar with "x" to remove. "Clear all" link when any filter active.
- **Pagination:** Bottom of table. Page size selector (10, 25, 50). Previous/Next buttons. "Page X of Y" indicator.
- **Bulk actions:** Toolbar appears above table when 1+ rows selected. Contains: export contracts (dropdown: CSV/XLSX), terminate (confirmation dialog).

### Side Panel

- **Open:** Slide from right, 200ms ease-out. Backdrop overlay at 20% opacity.
- **Close:** Click backdrop, press Escape, or click X button.
- **Content:** Contract title (Heading), status badge, contractor name as link. Compact card with key fields: type, start/end dates, rate, currency, billing cycle, payment terms. Documents count with icon. Key dates section with days-remaining calculation. "Open contract" CTA at bottom.

### Contract Creation Wizard

- **Trigger:** "New contract" button on /contracts page, "Add contract" button on contractor profile, top bar quick action. Opens as Dialog (640px max-width).
- **Pre-fill:** When opened from contractor profile, contractor field auto-selected and read-only. Financial terms (rate, currency, billing cycle) pre-filled from contractor billing profile with "Pre-filled from contractor billing profile" hint text below affected fields. User can override all pre-filled values.
- **Step navigation:** Progress indicator at top showing 3 steps. Completed steps show checkmark. Current step highlighted with `--primary`. Future steps grayed out.
- **Step validation:** Each step validates on "Next" click. Invalid fields show inline error below the field. Step does not advance until valid.
- **Step 3 (Documents):** Drop zone for document upload. User can upload 0 or more documents. "Skip, add documents later" link allows proceeding without documents. Upload happens immediately on file selection (not deferred to form submit). Each uploaded file shows: filename, size, upload progress bar, scan status badge.
- **Dismiss:** If no fields modified, "Close" button. If any field modified, "Discard changes" button triggers confirmation dialog ("Discard changes? All entered data will be lost." with "Discard changes" as destructive CTA and "Keep editing" as secondary).
- **Success:** Dialog closes, toast "Contract created", list refreshes, new contract appears at top.

### Document Upload (All Contexts)

- **Drop zone:** Dashed border container. UploadCloud icon centered, "Drag files here or" text, "Browse files" button. Accepts multiple files.
- **Dragover state:** Border changes to `--primary` (solid 2px), background shifts to `--primary` at 3% opacity. UploadCloud icon scales to 1.1x.
- **File validation:** Immediate MIME-type check on file selection. Invalid files show inline error with red text and are not uploaded. Files exceeding 25 MB show size error.
- **Upload progress:** Each file shows as a row below the drop zone: file icon (by type), filename, file size, progress bar (0-100%), cancel button (X). On complete: progress bar replaced by scan status badge.
- **Scan status:** Displayed as inline badge next to filename. Scanning: spinning Loader2 icon + "Scanning..." text in muted color. Clean: ShieldCheck icon in green. Infected: ShieldAlert icon in red + "Threat detected" text, file is quarantined and cannot be downloaded. Failed: ShieldQuestion icon in amber + "Scan failed" text.
- **Multi-file:** Multiple files can be uploaded simultaneously. Each has independent progress and status.

### Document Versioning

- **Upload new version:** "Upload new version" button on existing document card. Opens file picker (not full drop zone). Same validation and progress as initial upload.
- **Version display:** Current version shown prominently on document card. "View version history" link expands to show all prior versions with version number, upload date, uploader. Prior versions remain downloadable.
- **Version number:** Auto-incremented integer (v1, v2, v3...). Displayed as "Version {n}" badge on document card.

### Contract Detail Page

- **URL:** `/contracts/{id}` with `?tab={tabName}` for deep linking via nuqs.
- **Header:** Full-width. Contract title (Heading), status badge, contractor name as link. Action buttons right-aligned in dropdown menu (Edit, Add amendment, Upload document visible; Terminate, Supersede in overflow).
- **Tabs:** Horizontal tab bar below header. Active tab underlined with `--primary` (2px bottom border). 4 tabs: Overview, Documents, Amendments, Activity.
- **Overview tab:** 2-column grid on desktop (>= 1280px), single column below. Cards: Contract details (type, auto-renewal, notice period), Financial terms (rate, currency, billing cycle, payment terms), Key dates (start, end, notice deadline, days remaining with color coding), Linked contractor (name, NIP, status — clickable link to contractor profile).
- **Documents tab:** Document cards in a list. Each card: file type icon, filename, version badge, upload date, uploader name, scan status badge, Download button, "Upload new version" link. Drop zone at top for new uploads. Empty state when no documents.
- **Amendments tab:** Vertical timeline. Nodes ordered chronologically (newest at top). Original contract at bottom of timeline. Each node: date, amendment title/number, change summary text. Current version has larger dot in `--primary`. Click node to expand: full amendment details, linked documents, effective date.
- **Activity tab:** Vertical timeline (same pattern as contractor profile). Events: contract created, status changed, document uploaded, amendment added, contract edited. Each entry: icon + action text + relative timestamp.

### Inline PDF Preview

- **Trigger:** Click "Preview" button on PDF document card, or click PDF filename.
- **Display:** Dialog (max-width 960px, max-height 80vh) with embedded PDF viewer using browser-native `<object>` or `<embed>` tag. Download button in dialog header. Close button (X) top-right.
- **Non-PDF files:** No preview available. Show metadata card with file type icon, filename, size, upload date. Download button only.

### Amendment Creation

- **Trigger:** "Add amendment" button on contract detail page header or Amendments tab.
- **Dialog:** Standard form dialog (not wizard). Fields: amendment title, effective date, change summary (textarea), document upload (optional drop zone). On submit: amendment created, timeline updated, contract version incremented.
- **Success:** Dialog closes, toast "Amendment added", Amendments tab refreshes.

### Contractor Profile Tab Integration

- **Contracts tab:** Replaces Phase 2 placeholder. Shows mini TanStack Table filtered to current contractor. Columns: Title, Status, Start date, End date, Rate. "Add contract" CTA top-right (opens wizard with contractor pre-filled). Click row navigates to `/contracts/{id}`.
- **Documents tab:** Replaces Phase 2 placeholder. Shows document cards filtered to current contractor (across all contracts + standalone). "Upload document" CTA top-right. Drop zone for new uploads.

### Expiry Reminders (Settings Integration)

- **Location:** Organization Settings > General section (or dedicated "Contracts" settings subsection).
- **UI:** Multi-select tag input for default reminder intervals (30, 60, 90 days). Common presets as quick-add buttons. Per-contract override: shown in contract detail Key Dates card as "Reminders: 30, 60, 90 days before expiry" with "Edit" link opening inline editor.
- **Note:** Actual notification delivery deferred to Phase 7. This phase implements the configuration storage and displays reminder intervals in the UI. Dashboard surfacing of expiring contracts is Phase 9.

### Loading States

- **Contract list:** 8 skeleton rows with column-width-matched bars. Search bar and filter bar render immediately.
- **Side panel:** Skeleton card with 6 field placeholders.
- **Contract detail:** Header skeleton (title bar + badge placeholders) + tab bar renders immediately + skeleton content for active tab.
- **Document upload:** Progress bar fills during upload. No skeleton needed.
- **Wizard:** No skeleton needed — form renders empty.
- **PDF preview:** Skeleton rectangle in dialog until PDF loads.
