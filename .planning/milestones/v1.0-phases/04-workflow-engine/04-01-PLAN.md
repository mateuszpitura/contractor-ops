Create `packages/api/src/routers/workflow.ts` following the contract.ts router pattern exactly (tenantProcedure, requirePermission, plain() helper, router from ../init.js).

**Helper functions (defined at top of file, above router):**

1. `plain<T>(data: T): T` — JSON.parse(JSON.stringify(data)) pattern (same as contract.ts).

2. `evaluateCondition(condition: ConditionGroup | null, context: { contractor: any; contract?: any }): boolean` — Pure function. If condition is null or has no rules, return true. For each rule, get nested field value using dot-notation (e.g., "contractor.type" -> context.contractor.type). Apply operator: equals (===), notEquals (!==), contains (String.includes), startsWith (String.startsWith). Combine results with AND (every) or OR (some).

3. `resolveAssignee(task, contractor, contract, orgId, tx): Promise<string | null>` — Switch on task.assigneeMode: FIXED_USER returns task.assigneeUserId; ROLE_BASED queries tx.member.findFirst where organizationId=orgId, role=task.assigneeRole, user.banned=false; CONTRACTOR_OWNER returns contractor.internalOwnerUserId; CONTRACT_OWNER returns contract?.internalOwnerUserId; PROJECT_MANAGER returns null (fallback).

4. `calculateProgress(tasks): { done: number; total: number; percent: number }` — Filter out condition-skipped tasks (status=SKIPPED AND resultJson.skipReason==="condition_not_met"). Count DONE + user-SKIPPED as done. Return { done, total, percent }.

5. `TASK_TRANSITIONS` map: TODO->["IN_PROGRESS","SKIPPED","CANCELLED"], IN_PROGRESS->["DONE","SKIPPED","CANCELLED"], BLOCKED->["TODO"], DONE->[], SKIPPED->[], CANCELLED->[], OVERDUE->["DONE","SKIPPED","CANCELLED"].

**Router procedures:**

Template CRUD (admin: requirePermission({ workflow: ["create"] }) or ["update"] or ["delete"]):
- `createTemplate`: mutation, input templateCreateSchema. Use prisma.$transaction: create WorkflowTemplate (organizationId, name, type, description, version:1, status:DRAFT, appliesToEntityType:CONTRACTOR, createdByUserId:ctx.user.id), then create WorkflowTaskTemplate records for each task in input.tasks with organizationId and workflowTemplateId. Return plain(template with include tasks).
- `updateTemplate`: mutation, input templateUpdateSchema. Use prisma.$transaction: update template fields, delete existing tasks (deleteMany where workflowTemplateId), recreate tasks from input.tasks (upsert pattern: delete all + create fresh to handle reordering and removals cleanly). Return plain(updated template with tasks).
- `getTemplate`: query, input z.object({ id: z.string() }), requirePermission({ workflow: ["read"] }). findUniqueOrThrow with include tasks (orderBy sortOrder asc). Return plain().
- `listTemplates`: query, input templateListSchema, requirePermission({ workflow: ["read"] }). findMany with organizationId filter, optional status filter, optional search (name contains, case insensitive). Count total. Return plain({ items, total, page, pageSize }).
- `deleteTemplate`: mutation, input z.object({ id: z.string() }). requirePermission({ workflow: ["delete"] }). Only allow if status=DRAFT and no runs exist. Delete template + tasks in transaction.
- `duplicateTemplate`: mutation, input z.object({ id: z.string() }). requirePermission({ workflow: ["create"] }). Read source template+tasks, create new with name "{name} (copy)", status DRAFT, recreate all tasks.

Workflow run operations:
- `startRun`: mutation, input startRunSchema, requirePermission({ workflow: ["execute"] }). Use prisma.$transaction: load template (must be ACTIVE), load contractor, optionally load contract. Create WorkflowRun. For each task template (ordered by sortOrder): evaluate condition (if not met, create with status=SKIPPED, resultJson={skipReason:"condition_not_met"}); resolve assignee; compute dueAt using addDays(run.startedAt, task.dueOffsetDays); map dependsOnTaskTemplateId to dependsOnTaskRunId via taskIdMap; set status to BLOCKED if has dependency, else TODO. After all tasks created, compute progressPercent and update run. Return plain(run with tasks).
- `cancelRun`: mutation, input cancelRunSchema, requirePermission({ workflow: ["update"] }). Update run status to CANCELLED, set cancelledAt and cancelReason. Update all non-terminal tasks to CANCELLED.
- `getRun`: query, input z.object({ id: z.string() }), requirePermission({ workflow: ["read"] }). findUniqueOrThrow with include: tasks (orderBy created), comments (include author, orderBy createdAt asc), workflowTemplate, contractor, contract. Add computed isOverdue field to each task (status in TODO/IN_PROGRESS and dueAt < now()). Return plain().
- `listRuns`: query, input workflowRunListSchema, requirePermission({ workflow: ["read"] }). findMany with filters, include workflowTemplate (select name, type), contractor (select name, companyName), tasks for progress computation. Apply overdueOnly filter. Paginate. Return plain({ items, total, page, pageSize }).
- `myTasks`: query, input myTasksListSchema, requirePermission({ workflow: ["read"] }). Find WorkflowTaskRun where assigneeUserId=ctx.user.id, status in [TODO, IN_PROGRESS, BLOCKED]. Include workflowRun (select id, status, contractor name). Optional overdueOnly filter (dueAt < now()). Order by: overdue first (dueAt asc nulls last). Return plain({ items, total }).

Task actions:
- `completeTask`: mutation, input taskActionSchema, requirePermission({ workflow: ["execute"] }). In transaction: load task, validate transition (TODO/IN_PROGRESS -> DONE), set status=DONE, completedAt=now(), completedByUserId=ctx.user.id. Then find dependent tasks (dependsOnTaskRunId = this task id), update their status from BLOCKED to TODO. Recompute run progressPercent. If all active tasks done, set run status=COMPLETED, completedAt=now().
- `skipTask`: mutation, input skipTaskSchema, requirePermission({ workflow: ["execute"] }). Validate transition. Set status=SKIPPED, resultJson={skipReason: input.reason}. Unblock dependents same as complete. Recompute progress.
- `reassignTask`: mutation, input reassignTaskSchema, requirePermission({ workflow: ["update"] }). Update assigneeUserId to input.newAssigneeUserId.

Comments and attachments:
- `addComment`: mutation, input addCommentSchema, requirePermission({ workflow: ["update"] }). Create WorkflowComment with organizationId, workflowRunId, workflowTaskRunId (optional), authorUserId=ctx.user.id, body. Return plain(comment with author).
- `listComments`: query, input z.object({ workflowRunId: z.string(), workflowTaskRunId: z.string().optional() }). Find comments with author include, order by createdAt asc.

Overdue count for sidebar badge:
- `overdueCount`: query, no input, requirePermission({ workflow: ["read"] }). Count WorkflowTaskRun where organizationId=ctx.organizationId, assigneeUserId=ctx.user.id, status in [TODO, IN_PROGRESS], dueAt lt new Date(). Return { count }.
