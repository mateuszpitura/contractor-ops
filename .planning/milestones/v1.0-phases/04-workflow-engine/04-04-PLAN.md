**Create task-card-run.tsx:**
Collapsible Card for a single task in the run checklist. Props: task (WorkflowTaskRun with computed isOverdue), runId, currentUserId.

Collapsed view: status icon (left, per UI-SPEC task status icons table -- Circle for TODO, CircleDot for IN_PROGRESS, CheckCircle2 for DONE, Lock for BLOCKED, SkipForward for SKIPPED, XCircle for CANCELLED, AlertCircle for OVERDUE), task title (Body weight), task type Badge (icon per UI-SPEC type table: FileText for DOCUMENT_COLLECTION, CheckCircle for APPROVAL, KeyRound for ACCESS_GRANT/REVOKE, Banknote for FINANCE_SETUP, Monitor for EQUIPMENT, BookOpen for KNOWLEDGE_TRANSFER, Calendar for MEETING, ClipboardList for MANUAL, Bell for NOTIFICATION), assignee avatar + name (Label size), due date (Label size, destructive color if overdue with "Overdue" label).

Inline action buttons (visible on hover desktop, always visible mobile). Show based on task status and assignment:
- TODO or IN_PROGRESS + assigned to current user: "Complete" (primary Button), "Skip" (ghost Button), "Reassign" (ghost Button)
- BLOCKED: No actions, Tooltip "Waiting for: {dependency task title}"
- DONE: No actions, show "Completed by {name}" + timestamp
- SKIPPED (user): Show "Skipped" + reason on expand
- SKIPPED (condition): Show "Skipped (condition not met)", entire card at 50% opacity
- Tasks assigned to other users: "Reassign" only (if user has workflow update permission)

Skip action: ghost Button opens Popover with Textarea (placeholder "Why is this task being skipped?"), "Skip task" destructive CTA. On confirm: trpc.workflow.skipTask({ taskRunId, reason }), toast, invalidate.

Reassign action: ghost Button opens Popover with user Select (from trpc.user.list), "Reassign task" CTA. On confirm: trpc.workflow.reassignTask({ taskRunId, newAssigneeUserId }), toast "Task reassigned to {name}", invalidate.

Complete action: primary Button. On click: trpc.workflow.completeTask({ taskRunId }), toast "Task completed", invalidate.

Expanded view (click card to expand): full description text, TaskComments component, TaskAttachments component, metadata (created timestamp, completed by + timestamp if done).
