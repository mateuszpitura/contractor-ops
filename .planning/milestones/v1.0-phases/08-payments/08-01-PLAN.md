**1. Create `packages/api/src/routers/payment.ts`:**

Use the established pattern: import `router` from `../init.js`, use `tenantProcedure` with `requirePermission()` middleware. Import `plain()` helper for stripping Prisma types from returns. Import validators from `@contractor-ops/validators`. Import export services and bank statement services.

Define valid status transitions map:
```typescript
const VALID_TRANSITIONS: Record<string, string[]> = {
  DRAFT: ["LOCKED", "CANCELLED"],
  LOCKED: ["EXPORTED", "CANCELLED"],
  EXPORTED: ["COMPLETED", "FAILED", "CANCELLED"],
  COMPLETED: [],
  FAILED: ["DRAFT"],
  CANCELLED: [],
};
```

**Procedures:**

a) `readyForPayment` (query, requirePermission({ payment: ["read"] })):
Input: readyForPaymentListSchema. Query invoices WHERE organizationId = ctx.organizationId AND paymentStatus = "READY". Apply optional filters: currency, dueDateFrom/To (dueDate gte/lte), contractorId. Include contractor (select name, taxId), billingProfile (select iban, swiftBic, bankName, currency), contract (select contractNumber). Cursor-based pagination. Return { items, nextCursor }.

b) `create` (mutation, requirePermission({ payment: ["create"] })):
Input: paymentRunCreateSchema. Wrap in prisma.$transaction(async tx => { ... }):
  - If groupByCurrency: group invoiceIds by their currency, create one run per currency group. For each group:
  - Generate run number: query tx.paymentRun.findFirst({ where: { organizationId, runNumber: { startsWith: `PR-${year}-` } }, orderBy: { runNumber: "desc" }, select: { runNumber: true } }). Parse seq, increment, pad to 3 digits.
  - Verify all invoices have paymentStatus "READY" and belong to organizationId. Throw BAD_REQUEST if any invoice has paymentStatus !== "READY".
  - Create PaymentRun with status DRAFT, calculated totalGrosze (sum of invoice amountToPayGrosze), invoiceCount.
  - For each invoiceId: create PaymentRunItem (amountGrosze from invoice.amountToPayGrosze, currency from invoice.currency, contractorId from invoice.contractorId, billingProfileId from invoice.billingProfileId). Update invoice.paymentStatus to "IN_RUN".
  - Return created run(s) via plain().

c) `get` (query, requirePermission({ payment: ["read"] })):
Input: z.object({ runId: z.string().cuid() }). Query paymentRun by id and organizationId. Include items with invoice (select invoiceNumber, dueDate), contractor (select name, taxId), billingProfile (select iban, swiftBic, bankName). Include exports. Return via plain().

d) `list` (query, requirePermission({ payment: ["read"] })):
Input: paymentRunListSchema. Query paymentRuns WHERE organizationId, apply status filter, dateFrom/dateTo on createdAt. Cursor-based pagination. OrderBy sortBy/sortOrder. Return { items, nextCursor } via plain().

e) `lockAndExport` (mutation, requirePermission({ payment: ["export"] })):
Input: paymentRunLockSchema. In prisma.$transaction:
  - Fetch run with items, verify status is "DRAFT" or "LOCKED" (if relocking after format change).
  - Validate transition DRAFT->LOCKED or LOCKED->EXPORTED.
  - Fetch org settingsJson for transfer title template (default "{invoice_number}") and bankAccount info (iban, bic, name).
  - Build ExportItem[] from run items (join with invoice, contractor, billingProfile data).
  - Resolve transfer titles using resolveTransferTitle for each item.
  - Call appropriate generator: exportFormat === "CSV" -> generateCsv, "BANK_FILE" -> generateElixir, "SEPA_XML" -> generateSepaXml.
  - Update run status to "EXPORTED", set exportFormat, exportedAt.
  - Create PaymentExport record (format, status GENERATED, generatedByUserId).
  - Return { run (plain()), fileBase64: buffer.toString("base64"), fileName: `${runNumber}.${ext}` } where ext = csv/txt/xml per format.

f) `updateItemStatus` (mutation, requirePermission({ payment: ["create"] })):
Input: paymentRunItemStatusSchema. Fetch item, verify run belongs to org. Update item status, set paymentReference if provided, failureReason if FAILED, markedPaidAt if PAID. If PAID: update invoice.paymentStatus to "PAID". If FAILED: update invoice.paymentStatus to "READY" (auto-release per D-11). Check if all items in run are terminal -> auto-complete run.

g) `markAllPaid` (mutation, requirePermission({ payment: ["create"] })):
Input: markAllPaidSchema. In prisma.$transaction: fetch all PENDING items for run, update each to PAID with optional batchReference as paymentReference. Update each invoice.paymentStatus to "PAID". Update run status to "COMPLETED", set completedAt. Return updated run via plain().

h) `cancel` (mutation, requirePermission({ payment: ["create"] })):
Input: paymentRunCancelSchema. Fetch run, verify org. Check status is in VALID_TRANSITIONS cancellable states. If EXPORTED: verify user has admin role (check ctx.session role against "admin"). In prisma.$transaction: update all run items' invoices paymentStatus back to "READY". Update run status to "CANCELLED". Return via plain().

i) `importStatement` (mutation, requirePermission({ payment: ["create"] })):
Input: z.object({ runId: z.string().cuid(), fileContent: z.string(), fileName: z.string() }). Fetch run (must be EXPORTED). Parse statement via parseBankStatement. Fetch run items with billingProfile IBAN. Match via matchStatementToRun. Return { matches: MatchResult[], transactions: ParsedTransaction[] }.

j) `confirmStatementMatches` (mutation, requirePermission({ payment: ["create"] })):
Input: bankStatementConfirmSchema. In prisma.$transaction: for each match, update PaymentRunItem to PAID, set markedPaidAt, update invoice.paymentStatus to "PAID". Check if all items terminal -> auto-complete run. Return updated run via plain().

k) `removeFromRun` (mutation, requirePermission({ payment: ["create"] })):
Input: removeFromRunSchema. Per D-04 (run starts as DRAFT, user can add/remove invoices, then explicitly locks). In prisma.$transaction:
  - Fetch run by runId and organizationId, verify status is "DRAFT". Throw BAD_REQUEST if run is not in DRAFT status ("Can only remove invoices from DRAFT runs").
  - Find the PaymentRunItem for this invoiceId in this run. Throw NOT_FOUND if item does not exist.
  - Delete the PaymentRunItem.
  - Update invoice.paymentStatus back to "READY".
  - Recalculate run totalGrosze (sum remaining items' amountGrosze) and invoiceCount (count remaining items).
  - Update run with new totalGrosze and invoiceCount.
  - If no items remain, auto-cancel the run (set status to "CANCELLED").
  - Return updated run via plain().

l) `listByContractor` (query, requirePermission({ payment: ["read"] })):
Input: z.object({ contractorId: z.string().cuid() }). Query paymentRunItems WHERE organizationId = ctx.organizationId AND contractorId = input.contractorId. Include paymentRun (select runNumber), invoice (select invoiceNumber). OrderBy createdAt desc. Take 100. Return items mapped through plain(). This procedure is consumed by Plan 08-03's contractor profile Payments tab.
