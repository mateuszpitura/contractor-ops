> Visual and interaction contract for Onboarding & Polish. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialized in Phase 1) |
| Preset | base-nova with Zinc neutral + Indigo accent override |
| Component library | Base UI (via shadcn/ui base-nova style) |
| Icon library | Lucide React |
| Font | Inter (variable, via `next/font/google`) |

**shadcn components needed for Phase 10 (new additions beyond Phase 9):**
- None. All required shadcn primitives are already installed from prior phases. `command.tsx` (cmdk) is already present.

**New dependency for Phase 10:**
- `xlsx` -- already in dependencies (used for report export in Phase 9). Used for CSV/XLSX parsing in import wizard.

**Custom components built in Phase 10 (not from any registry):**
- Import wizard dialog -- multi-step Dialog reusing `wizard-dialog.tsx` pattern. Steps: Upload, Column mapping, Validation preview, Duplicate resolution, Confirmation.
- Column mapping grid -- two-column grid with source header (readonly) + target field dropdown (Select). Auto-mapped columns show green check icon.
- Validation preview table -- TanStack Table showing parsed rows with per-cell error indicators (red border + tooltip). Valid/invalid row counts in header.
- Duplicate resolution table -- TanStack Table showing conflict rows with radio group per row: Skip / Update existing / Create new.
- Onboarding checklist widget -- compact Card on dashboard with 5 steps, progress bar, collapsible. Persists in settingsJson.
- Onboarding full wizard -- standalone page/dialog with 5-step flow: Org details, Invite team, Import/add contractor, Configure approvals, Connect Slack.
- Empty state component -- reusable component: icon (48px) + heading (20px semibold) + body (14px) + primary CTA button + optional secondary action link.
- Command palette -- CommandDialog (cmdk) with search input, recent items section, pinned favorites, entity results with type badges, quick actions, navigation targets.
- Global search bar -- Input in top bar header with Search icon, opens command palette on click or Cmd+K.

**Reused from Phase 1-9 (already installed):**
- `table`, `badge`, `card`, `dialog`, `sheet`, `button`, `input`, `label`, `select`, `skeleton`, `tabs`, `checkbox`, `popover`, `progress`, `scroll-area`, `separator`, `breadcrumb`, `dropdown-menu`, `tooltip`, `sonner`, `alert-dialog`, `command`, `radio-group`

## Interaction Patterns

### Import Wizard Dialog

- **Trigger:** "Import" button in Contractors/Contracts list page toolbar (next to "Add" button) + onboarding checklist step 3 secondary action + command palette "Import" action.
- **Dialog:** 720px wide, uses existing wizard-dialog.tsx pattern with StepIndicator.
- **Step 1 -- Upload:** DropZone component (reuse `drop-zone.tsx`). Accept: `.csv`, `.xlsx`. Max file size: 10MB. Entity type RadioGroup below drop zone. File selected: show file name + size + "Remove" ghost button. Proceed: parse file client-side with xlsx library, extract headers and first 5 sample rows. Loading: skeleton grid while parsing.
- **Step 2 -- Column mapping:** Auto-match by header name similarity (case-insensitive, common aliases like "Company Name" -> legalName, "NIP" -> taxId). Each row: source header (readonly) + sample value preview (12px, `--muted-foreground`, first non-empty value) + Select dropdown with target schema fields + match indicator icon. Unmatched sources default to "Skip (do not import)". Required fields (legalName, taxId for contractors) show asterisk. Cannot proceed if required fields unmapped.
- **Step 3 -- Validation:** Server-side validation via tRPC `import.validate` mutation. Stats bar: valid/invalid/total counts. Table with all rows, invalid cells highlighted. Toggle between "Show all" and "Show errors only". Invalid rows excluded from import (D-02). "Back" to fix mapping, "Next" to proceed with valid rows only.
- **Step 4 -- Duplicate resolution:** Only shown if duplicates detected (D-03). Per-row RadioGroup: Skip / Update existing / Create new. Bulk actions: "Skip all" / "Update all". Default: Skip. Table columns: identifier (NIP), name from file, name from existing record, action.
- **Step 5 -- Confirmation:** Summary card with breakdown counts. "Import" primary button triggers server-side processing. For large files (>100 rows per D-04): async processing with Progress bar + polling via tRPC `import.status` query. Completion: success icon + stats + navigation CTA.
- **Navigation:** Back/Next buttons at dialog footer. Step indicator shows completed/current/future. "Discard" confirmation if closing dialog mid-flow (reuse existing discard pattern from contractor wizard).

### Onboarding Checklist Widget

- **Visibility:** Shown on dashboard when `settingsJson.onboardingCompletedSteps` has fewer than 5 items and checklist not dismissed. Admin-only (permission: settings:write).
- **Placement:** Dashboard right column, first widget (above approval queue widget). When checklist is present, dashboard layout adjusts.
- **Interaction:** Click step CTA navigates to relevant page/dialog. On completing the action (e.g., adding first contractor), `onboardingCompletedSteps` array updated via `settings.update` mutation. Step transitions to completed state on next dashboard load.
- **Collapse:** "Dismiss" sets `onboardingDismissed: true` in settingsJson. Collapses to inline bar "Continue setup -- 3 of 5 complete" with expand chevron. Clicking expand chevron re-shows full checklist.
- **Re-entry:** Settings page shows "Setup Guide" link when onboarding not fully complete. Opens onboarding wizard in full-page mode.
- **Completion:** When all 5 steps completed (including optional ones marked as skipped): brief celebration (confetti or simple "Setup complete" toast), widget auto-hides.

### Empty States

- **Component:** Reusable `EmptyState` component accepting: icon (LucideIcon), heading (string), body (string), primaryAction (label + onClick/href), secondaryAction (optional label + onClick/href).
- **Placement:** Replaces table/list content when entity count is 0. Centered vertically within content area with `min-h-[40vh]` to prevent layout collapse.
- **Smart sequencing (D-11):** Component accepts `prerequisite` prop. If prerequisite entity missing, override CTA with prerequisite action. Logic: check contractor count via tRPC -- if 0, most empty states redirect to "Add contractor" instead of their default CTA.
- **Tone (D-12):** Friendly, encouraging, action-oriented. Never blame the user. Always provide a clear next step.
- **Integration points:** Contractors list, Contracts list, Invoices list, Workflows list (all 3 tabs: Active, My Tasks, Templates), Payments list, Approvals queue, Notifications page, Dashboard widgets (deadlines, approval queue, activity feed).

### Command Palette

- **Trigger:** Cmd+K (Mac) / Ctrl+K (Windows) keyboard shortcut registered globally. Also triggered by clicking global search bar in top bar.
- **Dialog:** Uses existing `CommandDialog` component from `command.tsx`. 560px wide, positioned top-third of viewport.
- **Empty state (no query):** Show "Recent" section (last 8 viewed entities from localStorage, each with type badge + name + relative timestamp) + "Pinned" section (user-pinned favorites from settingsJson, with star icon). If no recent/pinned: show "Actions" section directly.
- **Search behavior:** 200ms debounce. Searches across: (1) contractors by name/company/NIP via tRPC `search.global`, (2) contracts by title/contractor, (3) invoices by number/contractor, (4) page names (client-side match), (5) action labels (client-side match). Results flat-listed ranked by relevance (D-16), each with type badge.
- **Result interaction:** Arrow keys navigate, Enter selects. Entity results navigate to entity detail page. Page results navigate to page. Action results trigger action (e.g., "New contractor" opens contractor wizard). Palette closes on selection.

- **No results:** "No results found" centered in list area with `SearchX` icon in `--muted-foreground`.

### Global Search Bar
