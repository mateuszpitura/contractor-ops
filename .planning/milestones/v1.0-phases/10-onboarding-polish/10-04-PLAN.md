    5. When no query typed (empty state per D-15):
       - "Recent" CommandGroup: read from localStorage recent items, show each with type badge + name + relative time (e.g., "2m ago")
       - "Pinned" CommandGroup: read from settingsJson `pinnedItems` (if available, otherwise skip)
       - "Actions" CommandGroup: "New contractor" (Plus icon), "New contract" (Plus icon), "Upload invoice" (Upload icon), "Start workflow" (Play icon)
       - "Pages" CommandGroup: all navigationItems, each with their icon
    6. When query typed (per D-16 flat results):
       - Entity results from tRPC: flat list, each with type Badge (colored per UI-SPEC type badge table) + name (14px font-semibold) + subtitle (14px text-muted-foreground)
       - Type badge colors: Contractor = `bg-primary/10 text-primary`, Contract = `bg-chart-2/10 text-chart-2`, Invoice = `bg-warning/10 text-warning`
       - Page matches: client-side filter navigationItems by query, show with ArrowRight icon
       - Action matches: client-side filter actions by query
    7. Result interaction:
       - Entity: navigate to entity detail URL (`/contractors/{id}`, `/contracts/{id}`, `/invoices/{id}`), call `addRecentItem()`, close palette
       - Page: navigate to page href, close palette
       - Action: trigger action (e.g., navigate to `/contractors` with `?action=new` or programmatically open wizard)
    8. Pin/unpin: show star icon on hover of entity results. Store pinned items in localStorage `"contractor-ops:pinned-items"` as `Array<{type, id, name}>`. Pinned items show in "Pinned" section
    9. Footer: keyboard hints "Enter to select" + "Arrow keys to navigate" + "Esc to close" in 12px monospace text-muted-foreground
    10. Loading: 4 Skeleton rows while search query is in flight
    11. No results: CommandEmpty with "No results found"
