    Create the complete Prisma 7 multi-file schema covering ALL tables from db-schema.md. Use Prisma 7's multi-file schema support (all .prisma files in prisma/schema/ directory).

    **prisma.config.ts (Prisma 7 config file):**
    ```typescript
    import path from "node:path";
    import type { PrismaConfig } from "prisma";

    export default {
      earlyAccess: true,
      schema: path.join("prisma", "schema"),
    } satisfies PrismaConfig;
    ```

    **schema.prisma (datasource + generator):**
    - datasource db: provider "postgresql", url env("DATABASE_URL")
    - generator client: provider "prisma-client-js", output "../../generated/prisma/client", previewFeatures ["driverAdapters"]

    **auth.prisma:** Better Auth tables (User, Session, Account, Verification). Follow Better Auth's expected table structure. User model includes: id (String @id), name, email (unique), emailVerified, image, createdAt, updatedAt. Session model: id, expiresAt, token (unique), createdAt, updatedAt, ipAddress, userAgent, userId. Account model: id, accountId, providerId, userId, accessToken, refreshToken, idToken, accessTokenExpiresAt, refreshTokenExpiresAt, scope, password, createdAt, updatedAt. Verification model: id, identifier, value, expiresAt, createdAt, updatedAt.

    **organization.prisma:** Organization, Member, Invitation tables (Better Auth organization plugin). Organization: id, name, slug (unique), logo, metadata, createdAt. Member: id, organizationId, userId, role, teamId (nullable, FK to Team), createdAt. Invitation: id, organizationId, email, role, status, expiresAt, inviterId, createdAt. Plus custom Organization fields from db-schema.md: legalName, countryCode (CHAR 2), defaultCurrency (CHAR 3), timezone, language, fiscalYearStartMonth, status (enum: ACTIVE, SUSPENDED, TRIAL, ARCHIVED), billingEmail, settingsJson (Json?), updatedAt, deletedAt. Also Team, Project, CostCenter tables from db-schema.md.

    **contractor.prisma:** Contractor, ContractorContact, ContractorBillingProfile, ContractorAssignment, ContractorTag, ContractorTagLink, ContractorComplianceItem, ComplianceRequirementTemplate. All fields from db-schema.md sections 4.5-4.12. Contractor status enum: DRAFT, ONBOARDING, ACTIVE, OFFBOARDING, INACTIVE, ARCHIVED. ALL monetary fields as Int (grosze). Every tenant-scoped model has organizationId String field with index.

    **contract.prisma:** Contract, ContractAmendment, ContractRatePeriod, Document, DocumentLink. All fields from db-schema.md sections 4.13-4.17. Contract status enum: DRAFT, ACTIVE, EXPIRING, EXPIRED, TERMINATED, SUPERSEDED. Document fields: id, fileName, fileKey (R2 key), mimeType, sizeBytes, uploadedById, organizationId, createdAt, deletedAt. DocumentLink: polymorphic linking (entityType enum + entityId).

    **invoice.prisma:** Invoice, InvoiceFile, InvoiceLine, InvoiceMatchResult. All fields from db-schema.md sections 4.18-4.21. Invoice status enum: RECEIVED, MATCHED, UNMATCHED, DISCREPANCY, PENDING_APPROVAL, APPROVED, REJECTED, READY_FOR_PAYMENT, PAID, CANCELLED. InvoiceLine: per-line VAT rate as String ("23", "8", "5", "0", "ZW", "NP"), all amounts as Int (grosze): unitPriceGrosze, netAmountGrosze, vatAmountGrosze, grossAmountGrosze.

    **workflow.prisma:** WorkflowTemplate, WorkflowTaskTemplate, WorkflowRun, WorkflowTaskRun, WorkflowComment, WorkflowAttachment. All fields from db-schema.md sections 4.22-4.27.

    **approval.prisma:** ApprovalChainConfig, ApprovalFlow, ApprovalStep, ApprovalDecision. All fields from db-schema.md sections 4.28-4.31.

    **payment.prisma:** PaymentRun, PaymentRunItem, PaymentExport. All fields from db-schema.md sections 4.32-4.34.

    **audit.prisma:** AuditLog, OutboxEvent. All fields from db-schema.md sections 4.35-4.36. AuditLog: id, organizationId, userId, action, entityType, entityId, entityName, changes (Json), ipAddress, userAgent, createdAt.

    **notification.prisma:** Notification, UserNotificationPreference, Comment, ReminderRule, ReminderInstance. From db-schema.md sections 4.37-4.41.

    **integration.prisma:** IntegrationConnection, IntegrationSyncLog, ExternalLink, WebhookDelivery. From db-schema.md sections 4.42-4.45.

    **CRITICAL RULES:**
    - Every tenant-scoped model MUST have `organizationId String` with `@@index([organizationId])` (exceptions: User, Session, Account, Verification which are global)
    - All monetary fields use `Int` (grosze, not Decimal or Float)
    - All models with soft-delete have `deletedAt DateTime?`
    - Use `@id @default(cuid())` for all primary keys (String type)
    - Use `@default(now())` for createdAt, `@updatedAt` for updatedAt
    - Add composite indexes from db-schema.md section 6

    **packages/db/src/client.ts:** Prisma client with Neon adapter (exact pattern from RESEARCH.md Pattern 1). Import from "../../generated/prisma/client". Create singleton with global caching for dev.

    **packages/db/src/tenant.ts:** Tenant isolation via Client Extension with AsyncLocalStorage (exact pattern from RESEARCH.md Pattern 2). globalModels array: ["User", "Session", "Account", "Verification"]. Throws error if tenant context not initialized.

    **packages/db/src/soft-delete.ts:** Soft delete via Client Extension (exact pattern from RESEARCH.md Pattern 3). Intercepts delete/deleteMany/findMany/findFirst/count to filter by deletedAt.

    **packages/db/src/index.ts:** Re-export prisma client, withTenantScope, withSoftDelete, tenantStore, and a composed `createTenantClient(orgId)` helper that chains both extensions.

    Run `pnpm --filter @contractor-ops/db db:generate` to verify schema compiles.
