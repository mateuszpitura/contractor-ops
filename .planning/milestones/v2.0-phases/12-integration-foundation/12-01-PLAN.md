**8. Create `packages/integrations/src/services/credential-service.ts`:**
Extract and generalize the AES-256-GCM pattern from `packages/api/src/services/slack-client.ts`. Per D-01, use per-provider encryption keys: env var format `${SLUG_UPPER}_ENCRYPTION_KEY` (e.g., `SLACK_ENCRYPTION_KEY`, `JIRA_ENCRYPTION_KEY`).

Functions:
- `getProviderEncryptionKey(providerSlug: string): Buffer` — reads `${slug.toUpperCase()}_ENCRYPTION_KEY` from env, returns Buffer from hex. Throws descriptive error if missing.
- `encryptCredentials(blob: CredentialBlob, providerSlug: string): string` — JSON.stringify blob, encrypt with AES-256-GCM, return `iv:authTag:ciphertext` hex format (identical format to existing Slack pattern).
- `decryptCredentials(encrypted: string, providerSlug: string): CredentialBlob` — split on `:`, decrypt, JSON.parse.

Use `node:crypto` built-in. Constants: `ALGORITHM = "aes-256-gcm"`, `IV_LENGTH = 12`, `AUTH_TAG_LENGTH = 16`.

**9. Create `packages/integrations/src/registry.ts`:**
```typescript
const adapters = new Map<string, IntegrationProviderAdapter>();

export function registerAdapter(adapter: IntegrationProviderAdapter): void {
  adapters.set(adapter.slug.toLowerCase(), adapter);
}

export function getAdapter(slug: string): IntegrationProviderAdapter | undefined {
  return adapters.get(slug.toLowerCase());
}

export function getAllAdapters(): IntegrationProviderAdapter[] {
  return Array.from(adapters.values());
}
```
