**1. Create calendar-event-service.ts:**

Core helper — `findCalendarConnections(prisma, organizationId, userId?)`:
- Find all CONNECTED IntegrationConnections where provider IN ["GOOGLE_CALENDAR", "OUTLOOK_CALENDAR"]
- Filter: personal connections (userId = given userId) + org connections (userId = null)
- Return array of connections with decrypted credentials (via `getDecryptedCredentials`)
- Per D-12: events push to both personal and org calendar when both connected

**createCalendarEvent(prisma, input: { organizationId, userId, entityType, entityId, summary, description, startDateTime, endDateTime, attendees? }):**
- Find calendar connections via findCalendarConnections
- For each connection:
  - If provider is GOOGLE_CALENDAR: call `GoogleCalendarAdapter.createEvent(accessToken, { summary, description, startDateTime, endDateTime, attendees })`
  - If provider is OUTLOOK_CALENDAR: call `OutlookCalendarAdapter.createEvent(accessToken, { subject: summary, bodyHtml: description, startDateTime, endDateTime, attendees })`
  - Create ExternalLink storing eventId:
    - `entityType`, `entityId`, `organizationId`, `integrationConnectionId: connection.id`
    - `externalType: "GOOGLE_CALENDAR_EVENT"` or `"OUTLOOK_CALENDAR_EVENT"`
    - `externalId: eventId`
    - `externalUrl: htmlLink or webLink`
    - `metadataJson: { eventId, title: summary, startTime, endTime, link, etag (Google only), provider }`
- Fire-and-forget pattern: wrap in try/catch, log errors but don't throw (D-11, Jira outbound sync precedent)

**updateCalendarEvent(prisma, input: { organizationId, entityType, entityId, summary, description, startDateTime, endDateTime }):**
- Find ExternalLinks matching entityType + entityId with externalType IN ["GOOGLE_CALENDAR_EVENT", "OUTLOOK_CALENDAR_EVENT"]
- For each ExternalLink:
  - Get connection, decrypt credentials
  - If Google: call updateEvent with etag from metadataJson
  - If Outlook: call updateEvent
  - Update ExternalLink.metadataJson with new data
- Fire-and-forget pattern

**deleteCalendarEvent(prisma, input: { organizationId, entityType, entityId }):**
- Find and delete all calendar ExternalLinks for entity
- For each: call provider deleteEvent, then delete ExternalLink
- Fire-and-forget pattern

**2. Create calendar-deadline-sync.ts:**

**syncContractExpiryDeadline(prisma, input: { organizationId, contractId, contractName, contractorName, expiryDate, userId }):**
- Event title: `[Contractor Ops] Contract expiry: ${contractorName} - ${contractName}` (per D-15)
- Description: `Contract "${contractName}" with ${contractorName} expires on ${expiryDate}.\n\nView in Contractor Ops: ${baseUrl}/contracts/${contractId}`
- startDateTime: expiryDate at 09:00 UTC, endDateTime: expiryDate at 09:30 UTC
- Call createCalendarEvent with entityType: "CONTRACT", entityId: contractId
- If ExternalLink already exists for this entity: call updateCalendarEvent instead

**syncApprovalSlaDeadline(prisma, input: { organizationId, approvalFlowId, itemType, itemName, deadline, userId }):**
- Event title: `[Contractor Ops] Approval deadline: ${itemType} - ${itemName}` (per D-15)
- Description with deep link to approval flow
- Call create/update pattern same as above, entityType: "APPROVAL_FLOW"

**syncPaymentDueDeadline(prisma, input: { organizationId, invoiceId, invoiceNumber, contractorName, dueDate, userId }):**
- Event title: `[Contractor Ops] Payment due: ${contractorName} - ${invoiceNumber}` (per D-15)
- Description with deep link to invoice
- Call create/update pattern, entityType: "INVOICE"

**createTaskCalendarEvent(prisma, input: { organizationId, workflowTaskRunId, config: CalendarTaskConfig, contractorName, contractName, taskName, userId }):**
- Substitute template variables: replace `{contractor}` with contractorName, `{contract}` with contractName, `{task}` with taskName in config.titleTemplate
- Prepend `[Contractor Ops] ` to title
- Calculate endDateTime from duration: "30m" -> +30min, "1h" -> +1h, "2h" -> +2h, "4h" -> +4h, "full_day" -> end of day
- startDateTime: now (task activation time)
- Attendees: config.attendees (contractor and assignee added automatically by caller)
- Call createCalendarEvent with entityType: "WORKFLOW_TASK_RUN"

**1. Create packages/api/src/routers/calendar.ts:**

Follow jira.ts router pattern. Create `calendarRouter` using `router({})`.

Procedures:

**`listConnections` (query):**
- Uses `tenantProcedure`
- Input: none
- Returns all IntegrationConnections for org where provider IN ["GOOGLE_CALENDAR", "OUTLOOK_CALENDAR"]
- Include both org-level (userId null) and personal (userId = ctx.userId) connections
- Select: id, provider, status, displayName, connectedAt, userId, tokenExpiresAt

**`listPersonalConnections` (query):**
- Uses `tenantProcedure` (authed)
- Input: none
- Returns IntegrationConnections where userId = ctx.userId AND provider IN ["GOOGLE_CALENDAR", "OUTLOOK_CALENDAR"]

**`disconnect` (mutation):**
- Uses `tenantProcedure`
- Input: `z.object({ connectionId: z.string().cuid() })`
- Verify connection belongs to org AND (userId = ctx.userId OR user is admin for org connections)
- Update connection status to DISCONNECTED
- Do NOT delete calendar events already created (per UI-SPEC disconnect copy: "Existing events on your calendar will not be removed")

**`listEvents` (query):**
- Uses `tenantProcedure`
- Input: none
- Count ExternalLinks where organizationId = ctx.organizationId AND externalType IN ["GOOGLE_CALENDAR_EVENT", "OUTLOOK_CALENDAR_EVENT"]
- Return `{ count: number }`

**`syncContractDeadline` (mutation):**
- Uses `tenantProcedure`
- Input: `z.object({ contractId: z.string().cuid() })`
- Fetch contract with contractor relation, call `syncContractExpiryDeadline` if expiryDate exists
- Fire-and-forget async (void return)

**`syncPaymentDeadline` (mutation):**
- Uses `tenantProcedure`
- Input: `z.object({ invoiceId: z.string().cuid() })`
- Fetch invoice with contractor relation, call `syncPaymentDueDeadline` if dueDate exists

**`getTaskConfig` (query):**
- Uses `tenantProcedure`
- Input: `z.object({ taskTemplateId: z.string().cuid() })`
- Fetch WorkflowTaskTemplate.configJson, parse calendarTaskConfigSchema from it
- Return parsed config or default (calendarEnabled: false)

**`saveTaskConfig` (mutation):**
- Uses `tenantProcedure`
- Input: `z.object({ taskTemplateId: z.string().cuid(), config: calendarTaskConfigSchema })`
- Read existing configJson, merge calendar fields, update WorkflowTaskTemplate.configJson
- Preserve existing jira config fields when merging
