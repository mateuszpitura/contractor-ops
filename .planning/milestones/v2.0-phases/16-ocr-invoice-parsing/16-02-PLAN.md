**1. Create `apps/web/src/components/ocr/extraction-status-bar.tsx`** (per UI-SPEC ExtractionStatusBar):

Props: `{ status: "PENDING" | "PROCESSING" | "EXTRACTED" | "PARTIAL" | "FAILED"; fieldCount?: number; totalFields?: number; errorMessage?: string; onRetry?: () => void }`.

Horizontal bar spanning full width. Slide-down + fade-in animation (200ms ease-out per UI-SPEC). Renders based on status:
- PENDING: hidden (return null)
- PROCESSING: info Badge "Processing" + Loader2 spinner (animate-spin) + "Extracting invoice data..." text
- EXTRACTED: success Badge "Extracted" + "Extraction complete -- {fieldCount} fields extracted" text
- PARTIAL: warning Badge "Partial" + "Partial extraction -- {fieldCount} of {totalFields} fields extracted. Please fill in the remaining fields." text
- FAILED: destructive Badge "Failed" + errorMessage text + "Re-run OCR" Button (outline variant, onClick=onRetry)

Use Card as container with appropriate border color matching status.

**2. Create `apps/web/src/components/ocr/ocr-processing-overlay.tsx`** (per UI-SPEC):

Props: `{ progress?: number }`. progress is 0-100 (optional, for determinate progress).

Renders as absolute-positioned overlay over parent container (parent must be relative). Semi-transparent background (bg-card/60 backdrop-blur-sm).

Center: Loader2 spinner (24px, animate-spin) + Progress bar (if progress provided) + "Analyzing invoice..." heading (text-sm font-semibold) + "This usually takes a few seconds" subtext (text-sm text-muted-foreground).

Below overlay: 8 skeleton field rows (Label skeleton 80px wide + Input skeleton full width) to show the form structure underneath.

**3. Create `apps/web/src/components/ocr/line-items-table.tsx`** (per D-08, UI-SPEC):

```typescript
{
```

Renders a Table with columns: Description, Qty, Unit, Unit Price, Net, VAT Rate, VAT Amount, Gross, Confidence, Actions.

"Line Items" heading (text-xl font-semibold) + item count Badge ("{N} items") above the table per UI-SPEC Section 5.

Height expand/collapse animation (200ms ease) when adding/removing rows per UI-SPEC animations.
