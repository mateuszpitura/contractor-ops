1. Install docusign-esign SDK: `cd packages/integrations && pnpm add docusign-esign@8.6.0`

2. Create `packages/integrations/src/adapters/docusign-adapter.ts`:
- Class `DocuSignAdapter` extends `BaseAdapter` and implements `ESignAdapter`
- Properties: `slug = "docusign"`, `displayName = "DocuSign"`, `supportsOAuth = true`, `supportsWebhooks = true`, `supportsEmbeddedSigning = true`
- `getOAuthConfig()`: returns `{ clientIdEnvVar: "DOCUSIGN_CLIENT_ID", clientSecretEnvVar: "DOCUSIGN_CLIENT_SECRET", authorizationUrl: "https://account-d.docusign.com/oauth/auth", tokenUrl: "https://account-d.docusign.com/oauth/token", scopes: ["signature"], redirectPath: "/api/oauth/docusign/callback" }`
- `exchangeCodeForTokens(code, redirectUri)`: POST to DocuSign token URL with authorization_code grant, return `CredentialBlob` with access_token, refresh_token, expires_at
- `refreshToken(credentials)`: POST to DocuSign token URL with refresh_token grant
- Helper `getApiClient(connectionId)`: fetches IntegrationConnection, decrypts credentials via `decryptCredentials()`, creates `docusign.ApiClient`, sets basePath from `configJson.basePath` (stored during OAuth), adds Authorization header, returns `{ apiClient, accountId: configJson.accountId }`
- `createEnvelope(connectionId, request)`: Use docusign-esign SDK. Build EnvelopeDefinition with document (base64), signers with routingOrder and clientUserId (set to email for embedded). Set status = "sent". Configure notification.expirations if expiresInDays provided. Configure notification.reminders if reminderIntervalDays provided. Return SigningEnvelopeResult.
- `getEmbeddedSigningUrl(connectionId, envelopeId, recipientEmail, returnUrl)`: Call createRecipientView with returnUrl, authenticationMethod "none", clientUserId = recipientEmail, frameAncestors = [origin from env APP_URL, "https://apps-d.docusign.com"], messageOrigins = [APP_URL origin]. Return { url, expiresAt: 5 min from now }.
- `getSignedDocument(connectionId, envelopeId)`: Call envelopesApi.getDocument with documentId "combined" to get all docs as one PDF. Return base64-encoded result.
- `getEnvelopeStatus(connectionId, envelopeId)`: Call envelopesApi.getEnvelope + listRecipients. Map to SigningEnvelopeResult.
- `voidEnvelope(connectionId, envelopeId, reason)`: Call envelopesApi.update with status "voided" and voidedReason.
- `resendToRecipient(connectionId, envelopeId, recipientEmail)`: Call envelopesApi.updateRecipients with resend for specific recipient.
- `normalizeWebhookEvent(payload)`: Parse DocuSign Connect JSON payload. Map envelopeStatus + recipientStatus to NormalizedSigningEvent. DocuSign event types: "envelope-sent", "envelope-delivered", "envelope-completed", "envelope-declined", "envelope-voided", "recipient-sent", "recipient-delivered", "recipient-completed", "recipient-declined", "recipient-authenticationfailed". Map these to internal event types.
- `verifyWebhookSignature(rawBody, headers)`: HMAC-SHA256 verification using `DOCUSIGN_WEBHOOK_SECRET` env var against `x-docusign-signature-1` header.

3. Create `packages/integrations/src/adapters/autenti-adapter.ts`:
- Class `AutentiAdapter` extends `BaseAdapter` and implements `ESignAdapter`
- Properties: `slug = "autenti"`, `displayName = "Autenti"`, `supportsOAuth = true`, `supportsWebhooks = true`, `supportsEmbeddedSigning = false`
- `getOAuthConfig()`: returns `{ clientIdEnvVar: "AUTENTI_CLIENT_ID", clientSecretEnvVar: "AUTENTI_CLIENT_SECRET", authorizationUrl: "https://app.autenti.com/oauth/authorize", tokenUrl: "https://app.autenti.com/oauth/token", scopes: ["document-process:write", "document-process:read"], redirectPath: "/api/oauth/autenti/callback" }`
- `exchangeCodeForTokens(code, redirectUri)`: Standard OAuth2 authorization_code exchange
- `refreshToken(credentials)`: Standard OAuth2 refresh_token grant
- Helper `autentiFetch(connectionId, path, options)`: fetches connection, decrypts credentials, makes authenticated fetch to `https://api.autenti.com/api/v2${path}`, handles errors
- `createEnvelope(connectionId, request)`: Multi-step REST flow per RESEARCH.md Pattern 5:
  a. POST /document-process with title, description, processLanguage "pl"
  b. POST /document-process/{id}/files with multipart form data (PDF)
  c. For each signer: POST /document-process/{id}/participants with party details, role "signer", constraints [{ constrainedActions: ["SIGN"], classifiers: ["SIGNATURE_BASIC_AUTENTI"] }]
  d. POST /document-process/{id}/actions with event_type "SEND"
  Return SigningEnvelopeResult with externalEnvelopeId = document process ID.
- `getEmbeddedSigningUrl()`: Throw error "Autenti does not support embedded signing. Use redirect flow." (supportsEmbeddedSigning = false guards callers)
- `getSignedDocument(connectionId, envelopeId)`: GET /document-process/{id}/files?filePurpose=SIGNED, download first signed file, return as base64
- `getEnvelopeStatus(connectionId, envelopeId)`: GET /document-process/{id}, map Autenti statuses (DRAFT, IN_PROGRESS, SIGNED, REJECTED, EXPIRED) to SigningEnvelopeResult
- `voidEnvelope(connectionId, envelopeId, reason)`: POST /document-process/{id}/actions with event_type "WITHDRAW"
- `resendToRecipient()`: POST /document-process/{id}/actions with event_type "REMIND" (if supported, otherwise no-op with log)
- `normalizeWebhookEvent(payload)`: Parse Autenti "Document Change" webhook. Map document-process status changes to NormalizedSigningEvent. Autenti statuses: DRAFT, IN_PROGRESS, SIGNED, REJECTED, EXPIRED, WITHDRAWN.
- `verifyWebhookSignature(rawBody, headers)`: Verify Autenti webhook signature using `AUTENTI_WEBHOOK_SECRET` env var

4. Update `packages/integrations/src/adapters/register-all.ts`:
- Add imports for DocuSignAdapter and AutentiAdapter
- Add `registerAdapter(new DocuSignAdapter())` and `registerAdapter(new AutentiAdapter())`

Create `packages/integrations/src/services/esign-service.ts`:

This service provides provider-agnostic signing operations. It resolves the correct adapter from the registry and delegates.

Exports:
1. `getESignAdapter(provider: "DOCUSIGN" | "AUTENTI"): ESignAdapter` — looks up adapter from registry by slug (lowercase provider), casts to ESignAdapter. Throws if adapter not found or does not implement ESignAdapter methods.

2. `async createSigningEnvelope(params: { provider: "DOCUSIGN" | "AUTENTI"; connectionId: string; request: SigningEnvelopeRequest }): Promise<SigningEnvelopeResult>` — resolves adapter via getESignAdapter, calls adapter.createEnvelope. Returns result.

3. `async getEmbeddedSigningUrl(params: { provider: "DOCUSIGN" | "AUTENTI"; connectionId: string; envelopeId: string; recipientEmail: string; returnUrl: string }): Promise<EmbeddedSigningUrlResult | null>` — checks adapter.supportsEmbeddedSigning first. If false, returns null (caller should use redirect fallback per D-03). If true, calls adapter.getEmbeddedSigningUrl.

4. `async downloadSignedDocument(params: { provider: "DOCUSIGN" | "AUTENTI"; connectionId: string; envelopeId: string }): Promise<SignedDocumentResult>` — calls adapter.getSignedDocument.

5. `async voidSigningEnvelope(params: { provider: "DOCUSIGN" | "AUTENTI"; connectionId: string; envelopeId: string; reason: string }): Promise<void>` — calls adapter.voidEnvelope.

6. `async resendSigningNotification(params: { provider: "DOCUSIGN" | "AUTENTI"; connectionId: string; envelopeId: string; recipientEmail: string }): Promise<void>` — calls adapter.resendToRecipient.

7. `normalizeSigningEvent(provider: "DOCUSIGN" | "AUTENTI", payload: unknown): NormalizedSigningEvent` — calls adapter.normalizeWebhookEvent.

Also export the types from esign.ts for convenience: `export type { ESignAdapter, SigningEnvelopeRequest, SignerInfo, SigningEnvelopeResult, EmbeddedSigningUrlResult, SignedDocumentResult, NormalizedSigningEvent } from "../types/esign.js";`

Update `packages/integrations/src/index.ts` (or equivalent barrel export) to export from esign-service.
