# Phase 15 — UI Design Contract

> Visual and interaction contract for e-sign integration: DocuSign + Autenti contract signing with multi-party support, embedded signing, and auto-storage. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |
|----------|-------|

| Preset | base-nova (neutral base color) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, signer list row gaps |
| md | 16px | Default element spacing, dialog section padding |
| lg | 24px | Section padding, dialog content spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical rhythm |
| 3xl | 64px | Not used in Phase 15 |

Exceptions: Embedded signing modal uses full viewport height minus 56px top bar (100dvh - 56px). Minimum touch target 48px for signer reorder drag handles and signing action buttons.

## Layout Contract

### Contract Detail Header (D-06)
- "Send for Signature" button added to DetailHeader action area, left of the existing "Actions" dropdown
- Button: primary variant, size sm, PenLine icon (16px) + "Send for Signature" label
- Visible when: contract status is DRAFT or ACTIVE, and at least one e-sign provider is connected
- Disabled with tooltip when: no signable document attached ("Upload a document first")

### Send for Signature Dialog (D-07)
- Trigger: SendForSignatureButton click
- Dialog max-width: 640px. ScrollArea body with max-height 80vh.
- Section 1 -- Provider: Label "Signing Provider" + ProviderPicker Select. 16px bottom margin.
- Section 2 -- Signers: Label "Signers" + SignerList. Default order: contractor first, countersigner second (D-09). "Add countersigner" link if no org rep selected. CountersignerPicker for org member selection (D-10). Drag handle on each row for reorder. 16px bottom margin.
- Section 3 -- Message: Label "Message to Signers" + Textarea (3 rows, placeholder: "Optional message included in the signing email"). 16px bottom margin.
- Section 4 -- Document: Label "Document" + DocumentPreview showing the contract PDF. File name + page count. 16px bottom margin.
- Section 5 -- Options: Two fields side by side (8px gap). "Expires After" -- Select with options: 7 days, 14 days, 30 days, 60 days (default: 14 days). "Send Reminders" -- Select with options: None, Every 3 days, Every 7 days (default: Every 7 days). 24px bottom margin.
- Footer: "Send for Signature" primary button (right-aligned) + "Discard" outline button. Send button shows loading spinner when submitting.

### Documents Tab Per-Document Action (D-06)
- Each document row in DocumentsTab gets a "Send for Signature" action in its dropdown menu
- Icon: PenLine. Label: "Send for Signature"
- Opens the same SendForSignatureDialog, pre-populated with that specific document
- Available only when at least one e-sign provider is connected

### Signing Progress on Contract Detail (D-08)
- New section below contract detail header, above tabs. Visible only when contract has an active signing envelope.
- Layout: Card with 16px padding. Horizontal layout on desktop, stacked on mobile.
- Left: SigningProgressBar showing step indicators for each signer
- Right: Status text "Waiting for {signer name} to sign" or "All parties have signed"
- Signer steps: Circle with initial letter inside. States:
  - Waiting: muted border, muted-foreground text
  - Current (awaiting action): primary border with pulse animation, primary text
  - Signed: success background, white check icon
  - Declined: destructive background, white X icon
- Connector line between steps: 2px solid, muted color (success color for completed segments)

### Contract Status Integration (D-08)
- Contract list table: existing status Badge now shows "Pending Signature" (warning variant), "Signature Declined" (destructive variant), "Signature Expired" (destructive variant)
- Contract detail header: existing Badge reflects signing-related statuses with same color mapping
- Status badge mapping:

| Status | Variant | Color Token | Label |
|--------|---------|-------------|-------|
| PENDING_SIGNATURE | warning | `--warning` | "Pending Signature" |
| SIGNATURE_DECLINED | destructive | `--destructive` | "Signature Declined" |
| SIGNATURE_EXPIRED | destructive | `--destructive` | "Signature Expired" |

### Embedded Signing Modal (D-03, D-05)
- Full-viewport overlay (z-50). Background: white.
- Top bar: 56px height, border-bottom, flex between.
  - Left: Document title at `text-sm font-semibold`
  - Right: Close button (X icon, ghost variant, 32x32px, `aria-label="Close signing modal"`)
- Body: iframe element, width 100%, height calc(100dvh - 56px)
- Fallback (when embedded not supported): centered Card (max-width 480px) with provider logo, "You will be redirected to {provider} to complete signing", "Continue to {provider}" primary button, "Return to Contract" outline button
- On signing complete (postMessage/callback from provider): close modal, show success toast, refresh contract detail

### Portal Pending Signatures (D-04)
- Location: Portal dashboard, new section above existing content
- Section heading: "Pending Signatures" at `text-sm font-semibold` + Badge with count
- Visible only when contractor has pending signatures. Hidden when count is 0.
- Each pending item: Card with 16px padding. Horizontal layout.
  - Left column: Contract title at `text-sm font-semibold`, organization name at `text-sm text-muted-foreground`, "Sent {relative date}" at `text-sm text-muted-foreground`
  - Right column: "Sign Now" primary button (size sm)
- Max 5 items shown. If more: "View all pending signatures" link below.
- Clicking "Sign Now" opens EmbeddedSigningModal (same component as admin, portal variant)

### Signing Audit Trail (D-08, SIGN-04)
- Trigger: "View Signing History" link/button on contract detail, within signing progress section or Activity tab
- Opens as Sheet (right side panel), width 400px
- Header: "Signing History" at `text-xl font-semibold`
- Body: Chronological list (newest first) of signing events
- Each event row: 8px vertical padding, border-bottom
  - Timestamp: `text-sm text-muted-foreground` (relative time, full datetime on hover via Tooltip)
  - Event: `text-sm` description (e.g., "Envelope sent to John Doe", "Jane Smith signed the document", "Signed PDF saved to documents")
  - Actor: `text-sm text-muted-foreground` (who performed the action)
- Event icons (left-aligned, 16px, muted-foreground):
  - Sent: Send icon
  - Viewed: Eye icon
  - Signed: PenLine icon (success color)
  - Declined: XCircle icon (destructive color)
  - Voided: Ban icon (destructive color)
  - Completed: CheckCircle2 icon (success color)
  - PDF Saved: FileDown icon

### Void/Resend Actions (D-11)
- Location: Contract detail signing progress section, or Actions dropdown
- "Void Envelope" -- destructive dropdown item. Opens VoidEnvelopeDialog (AlertDialog).
- "Resend to {signer}" -- outline dropdown item. Available when a signer has not completed signing.
- Both actions only available when envelope is in progress (PENDING_SIGNATURE status).
