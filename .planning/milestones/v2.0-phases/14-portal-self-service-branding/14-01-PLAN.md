**A. Create `packages/api/src/services/portal-change-request.ts`:**

Service with three exported functions using raw prisma client:

1. `createChangeRequest(contractorId: string, organizationId: string, requestedChanges: Record<string, unknown>, previousValues: Record<string, unknown>): Promise<ContractorChangeRequest>`
   - Check for existing PENDING request: `prisma.contractorChangeRequest.findFirst({ where: { contractorId, organizationId, status: "PENDING" } })`
   - If found, throw TRPCError code "CONFLICT" message "You already have a pending change request"
   - Create and return new ContractorChangeRequest

2. `approveChangeRequest(requestId: string, organizationId: string, reviewerId: string, comment?: string): Promise<void>`
   - Find the request (PENDING status, matching org)
   - If not found, throw TRPCError code "NOT_FOUND"
   - Get contractor's default billing profile
   - Apply changes in $transaction:
     - Update billing profile with requestedChanges fields (bankName, swiftBic, taxId, bankAccountMasked, bankAccountEncrypted)
     - Update ContractorChangeRequest status to APPROVED, reviewedById, reviewedAt: new Date(), reviewComment
   - Re-read current values before applying (pitfall 3 from research)

3. `rejectChangeRequest(requestId: string, organizationId: string, reviewerId: string, comment?: string): Promise<void>`
   - Find the request (PENDING status, matching org)
   - If not found, throw TRPCError code "NOT_FOUND"
   - Update status to REJECTED, reviewedById, reviewedAt: new Date(), reviewComment

**B. Add 8 new endpoints to `packages/api/src/routers/portal.ts`:**

All use portalProcedure (authenticated). All use raw prisma client with explicit organizationId. All return via plain() wrapper.

1. `getProfile` — query, no input. Select contractor fields (displayName, email, phone, addressLine1, addressLine2, city, postalCode, countryCode, taxId) + default billing profile (bankAccountMasked, bankName, swiftBic, taxId — NEVER bankAccountEncrypted). Also fetch pending ContractorChangeRequest for this contractor+org. Return { ...contractor, billingProfile, pendingChangeRequest }.

2. `updateContactInfo` — mutation, input z.object({ displayName: z.string().min(1).max(200), phone: z.string().max(50).optional().nullable(), addressLine1: z.string().max(200).optional().nullable(), addressLine2: z.string().max(200).optional().nullable(), city: z.string().max(100).optional().nullable(), postalCode: z.string().max(20).optional().nullable(), countryCode: z.string().length(2).optional().nullable() }). Directly update contractor record. Per D-01: contact info changes take effect immediately.

3. `submitFinancialChangeRequest` — mutation, input z.object({ bankAccountNumber: z.string().optional(), bankName: z.string().optional(), swiftBic: z.string().max(11).optional(), taxId: z.string().optional() }). Read current billing profile values as previousValues. Call createChangeRequest service. Per D-01: financial fields require approval.
   - For bankAccountNumber: encrypt server-side and store both encrypted and masked versions in requestedChanges JSON (research open question 1 recommendation). Use the encryption pattern from the existing billing profile creation.

4. `getNotificationPreferences` — query, no input. Read existing ContractorNotificationPreference rows for contractor+org. For any of the 5 categories missing, return default (emailEnabled: true). Categories: INVOICE_UPDATES, PAYMENT_CONFIRMATIONS, CONTRACT_CHANGES, DOCUMENT_UPLOADS, SECURITY_ALERTS. Per D-06.

5. `updateNotificationPreference` — mutation, input z.object({ category: z.enum(["INVOICE_UPDATES", "PAYMENT_CONFIRMATIONS", "CONTRACT_CHANGES", "DOCUMENT_UPLOADS", "SECURITY_ALERTS"]), emailEnabled: z.boolean() }). If category is SECURITY_ALERTS and emailEnabled is false, throw TRPCError code "BAD_REQUEST" message "Security alerts cannot be disabled". Upsert the preference row. Per D-07.

6. `getOrgBranding` — query, no input. Read organization.settingsJson and organization.logo for ctx.organizationId. Extract brandColor from settingsJson. Return { logo, brandColor }. Per D-12.
