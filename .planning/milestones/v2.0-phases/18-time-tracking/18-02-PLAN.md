**1. Create `packages/api/src/services/clockify-sync.ts`:**

On-demand fetch per D-09. No background polling/webhooks.

```typescript
import type { PrismaClient } from "@contractor-ops/db";
import { TRPCError } from "@trpc/server";

interface ClockifyTimeEntry {
  id: string;
  description: string;
  timeInterval: {
    start: string; // ISO 8601
    end: string;
    duration: string; // PT format e.g. "PT1H30M"
  };
  projectId: string | null;
  project?: { name: string } | null;
}

// Parse ISO 8601 duration (PTxHyMzS) to minutes
function parseDurationToMinutes(duration: string): number {
  const match = duration.match(/PT(?:(\d+)H)?(?:(\d+)M)?(?:(\d+)S)?/);
  if (!match) return 0;
  const hours = parseInt(match[1] || "0", 10);
  const minutes = parseInt(match[2] || "0", 10);
  const seconds = parseInt(match[3] || "0", 10);
  return hours * 60 + minutes + (seconds >= 30 ? 1 : 0);
}

export async function syncClockifyEntries(
  prisma: PrismaClient,
  organizationId: string,
  contractorId: string,
  contractId: string,  // Explicit contract assignment - caller resolves from user context
  timesheetId: string,
  connectionId: string,
  startDate: string, // YYYY-MM-DD
  endDate: string,   // YYYY-MM-DD
): Promise<{ imported: number; skipped: number }> {
  // 1. Get connection + decrypt credentials
  // 2. Extract config: { workspaceId, userId, region }
  // 3. Build base URL from CLOCKIFY_REGIONS[region]
  // 4. Paginate GET /workspaces/{wid}/user/{uid}/time-entries?start={ISO}&end={ISO}&page={n}&page-size=100
  // 5. For each entry: upsert TimeEntry with source=CLOCKIFY, externalId=clockify entry id
  //    metadataJson = { clockifyProjectId, clockifyProjectName, clockifyDescription }
  //    minutes = parseDurationToMinutes(entry.timeInterval.duration)
  //    entryDate = date portion of entry.timeInterval.start
  // 6. Use @@unique(organizationId, contractorId, source, externalId) for dedup (Pitfall 5)
  // 7. Recalculate timesheet totalMinutes
  // 8. Return { imported: newCount, skipped: existingCount }
}
```

Key details:
- Paginate with page parameter, page-size=100
- Check response array length < page-size to detect last page
- Use `start` and `end` query params with ISO 8601 datetime format: `${startDate}T00:00:00Z` and `${endDate}T23:59:59Z`
- contractId is a required parameter passed by the caller (portal route). The portal syncExternal procedure determines the contractId from user context (the contractor selects which contract to import entries for in the sync UI). Fallback when only one active contract: use that contract. When multiple active contracts and no explicit contractId: throw a TRPCError with BAD_REQUEST code and message "Multiple active contracts found. Please specify which contract to import entries for." The user can then retry via the UI with the correct contract selected
- All imported entries are read-only per D-11 (source=CLOCKIFY enforced in time-entry.ts service)

Two-step fetch per D-10 and Pitfall 4:

```typescript
export async function syncJiraWorklogs(
  prisma: PrismaClient,
  organizationId: string,
  contractorId: string,
  timesheetId: string,
  connectionId: string,
  startDate: string,
  endDate: string,
): Promise<{ imported: number; skipped: number }> {
  // 1. Get connection + decrypt credentials (OAuth access token)
  // 2. Extract config: { cloudId, accountId } from configJson
  //    - cloudId from accessible-resources call during OAuth setup
  //    - accountId from ExternalLink (entityType=CONTRACTOR, externalType=JIRA_USER)
  // 3. Base URL: https://api.atlassian.com/ex/jira/{cloudId}/rest/api/3
  //
  // Step A: JQL search for issues with user's worklogs
  //   GET /search?jql=worklogDate>="{startDate}" AND worklogDate<="{endDate}" AND worklogAuthor="{accountId}"&fields=key,summary&maxResults=100&startAt=0
  //   Paginate with startAt (Pitfall 4)
  //
  // Step B: For each issue, fetch worklogs
  //   GET /issue/{issueKey}/worklog?startAt=0&maxResults=1000
  //   Filter worklogs by author.accountId === our accountId (endpoint returns ALL authors)
  //   Filter by date range (started field)
  //
  // Step C: Map to TimeEntry
  //   externalId = worklog.id (string)
  //   minutes = worklog.timeSpentSeconds / 60 (rounded)
  //   entryDate = date portion of worklog.started
  //   description = worklog.comment?.content text or issueKey + issueSummary
  //   metadataJson = { issueKey, issueSummary, worklogId: worklog.id }
  //   source = JIRA
  //
  // 4. Upsert with @@unique dedup (Pitfall 5)
  // 5. Recalculate timesheet totalMinutes
  // 6. Return { imported, skipped }
}
```

Both services:
- Use `decryptCredentials` from integrations package for secure credential access
- Log sync via IntegrationSyncLog (direction=INBOUND, syncType="time_entries" or "worklogs")
- Handle API errors gracefully: 401 -> throw UNAUTHORIZED with "reconnect" message, 429 -> respect Retry-After header
- Use Authorization: Bearer {accessToken} for Jira, X-Api-Key: {apiKey} for Clockify
