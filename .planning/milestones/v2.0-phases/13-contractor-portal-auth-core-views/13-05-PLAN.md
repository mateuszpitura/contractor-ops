<objective>
Build the invoice views and submission flow: invoice list with status badges, invoice detail with three-layer status tracking (timeline + activity log), invoice submission form with contract picker and PDF upload, and post-submission success page.

Purpose: Invoice submission is the primary write operation in the portal (PORT-03). Three-layer status tracking gives contractors full visibility into their invoice lifecycle (PORT-04, D-10). This is the core value proposition of the portal -- contractors submit and track invoices.

Output: Invoice list/detail pages, submission form with upload, success page, StatusTimeline and ActivityLog components.

```typescript
const STEPS = [
  { key: "submitted", label: "Submitted" },
  { key: "review", label: "In Review" },
  { key: "approved", label: "Approved" },
  { key: "scheduled", label: "Payment Scheduled" },
  { key: "paid", label: "Paid" },
] as const;
```

Props: `{ status: string; approvalStatus: string; paymentStatus: string; rejectedAt?: Date | null }`.

Logic to compute active step index:
```typescript
function getActiveStep(invoice: Props): number {
  if (invoice.paymentStatus === "PAID") return 4;
  if (invoice.paymentStatus === "IN_RUN") return 3;
  if (invoice.approvalStatus === "APPROVED") return 2;
  if (invoice.status === "UNDER_REVIEW" || invoice.status === "APPROVAL_PENDING") return 1;
  return 0; // RECEIVED = submitted
}
```

Rejected state detection: if `rejectedAt` is set, show step at index 1 (In Review) as destructive.

Visual rendering -- HORIZONTAL on desktop (md+), VERTICAL on mobile:
- Desktop (`hidden md:flex`): flex-row, items-center. For each step:
  - Circle (24px): filled with appropriate color if active/past, muted border if future.
    - Past steps: `bg-green-600` (success) with Check icon (white, 14px).
    - Active step: `bg-primary ring-2 ring-primary/30` (accent with ring animation via `animate-pulse` on ring only).
    - Future steps: `border-2 border-muted-foreground/30 bg-background`.
    - Rejected (at review step): `bg-destructive` with X icon.
  - Label below circle: `text-[13px]`, colored to match step state.
  - Connector line between circles: `h-0.5 flex-1 mx-2`. Filled line (`bg-green-600`) for completed, muted (`bg-border`) for future.

- Mobile (`md:hidden`): flex-col. Vertical timeline with circles on left, labels on right, vertical connector lines.

1. Create `apps/web/src/components/portal/invoice-submit-form.tsx` ("use client"):

Per UI-SPEC Invoice Submission and D-06, D-07, D-08:

Zod validation schema:
```typescript
const invoiceSubmitSchema = z.object({
  contractId: z.string().min(1, "Please select a contract"),
  invoiceNumber: z.string().min(1, "Invoice number is required").max(100),
  issueDate: z.string().min(1, "Issue date is required"), // date input returns string
  dueDate: z.string().min(1, "Due date is required"),
  netAmount: z.string().min(1, "Net amount is required").refine(v => !isNaN(Number(v)) && Number(v) > 0, "Must be a positive number"),
  grossAmount: z.string().min(1, "Gross amount is required").refine(v => !isNaN(Number(v)) && Number(v) > 0, "Must be a positive number"),
});
```

Component structure (single-page form with logical sections per UI-SPEC):

**Section 1 -- Contract Selection (D-08):**
- Fetch active contracts: `api.portal.getActiveContracts.useQuery()`.
- Select component with contract options: `{title} ({rateValueGrosze/100} {currency}/{rateType})`.
- Auto-select if only 1 active contract: `useEffect(() => { if (contracts.length === 1) form.setValue("contractId", contracts[0].id); }, [contracts])`.
- When contract selected, show expected amount hint below select: "Expected: {rateValueGrosze/100} {currency}/{billingModel}".

**Section 2 -- Upload (D-06, D-07):**
- Use existing DropZone component for single PDF upload. Accept only `application/pdf`.
- On file drop:
  a. Call `api.portal.getUploadUrl.useMutation()` with filename and contentType.
  b. Upload file to presigned URL via `fetch(uploadUrl, { method: "PUT", body: file, headers: { "Content-Type": "application/pdf" } })`.
  c. Show Progress bar during upload.
  d. On success, store `documentId`, `storageKey`, `originalFileName`, `fileSizeBytes` in component state.
  e. Show uploaded file info: filename + size + "Remove" button to clear and re-upload.
- Track upload state: idle | uploading (with progress) | uploaded | error.

**Section 3 -- Metadata (D-06):**
- Invoice number: Input type="text", required.
- Issue date: Input type="date", required.
- Due date: Input type="date", required. Validate due date >= issue date.
- Net amount: Input type="number" with step="0.01". Label includes currency from selected contract.
- Gross amount: Input type="number" with step="0.01". Label includes currency.
- Convert amounts to grosze on submit: `Math.round(parseFloat(value) * 100)`.

**Section 4 -- Review Summary:**
- Collapsible Card (default open) showing: contract name, invoice number, dates, amounts, uploaded filename.
- Separator between sections.

**Submit button:** "Submit Invoice" (primary, full-width on mobile). Disabled when: form invalid, no file uploaded, or submitting.

**On submit:**
- Call `api.portal.submitInvoice.useMutation()` with all data.
- Convert amounts to grosze: `netAmountGrosze: Math.round(parseFloat(netAmount) * 100)`, same for gross.
- Convert date strings to Date objects.
- On success: redirect to `/portal/invoices/submit/success?invoiceId={id}&invoiceNumber={number}`.
- On error: show toast via sonner with error message. Keep form data intact for retry (per UI-SPEC error copy: "Failed to submit invoice. Your data has been saved. Please try again.").
