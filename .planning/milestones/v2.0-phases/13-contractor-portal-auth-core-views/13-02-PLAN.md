2. `portalProcedure` -- for authenticated portal endpoints. Middleware:
   a. Extract `portal_session` cookie value from `ctx.headers.get("cookie")`. Parse cookies manually (split on `; `, find `portal_session=` prefix, extract value). The cookie name MUST be `portal_session` (not `better-auth.session_token` -- distinct per Pitfall 1).

   e. Wrap the `next()` call inside `tenantStore.run({ organizationId: session.organizationId }, () => ...)` to enable automatic tenant scoping on all downstream Prisma queries.
   f. Pass extended context: `{ portalSession: session, contractorId: session.contractorId, organizationId: session.organizationId, contractor: session.contractor }`.

Cookie parsing function (inline helper):
```typescript
function parsePortalCookie(cookieHeader: string): string | null {
  const cookies = cookieHeader.split("; ");
  for (const cookie of cookies) {
    if (cookie.startsWith("portal_session=")) {
      return cookie.slice("portal_session=".length);
    }
  }
  return null;
}
```

Create `packages/api/src/routers/portal.ts` with the following endpoints. Use `router` from `../init.js`. Import `portalProcedure` and `portalPublicProcedure` from `../middleware/portal-auth.js`.

**Auth endpoints (portalPublicProcedure):**

1. `requestMagicLink` mutation:
   - Input: `z.object({ email: z.string().email() })`
   - Normalize email: `toLowerCase().trim()`
   - Call `findContractorsByEmail(email)` -- if results exist, call `createMagicLinkToken(email)` then `sendPortalMagicLink({ email, token, baseUrl })`. Derive baseUrl from request headers (Origin or X-Forwarded-Host).
   - ALWAYS return `{ success: true }` regardless of whether contractor was found (prevent email enumeration per Pitfall 2 / D-16).

2. `verifyMagicLink` mutation:
   - Input: `z.object({ token: z.string() })`
   - Call `verifyMagicLinkToken(token)`.
   - If null, throw TRPCError BAD_REQUEST with message "Invalid or expired link".
   - Call `findContractorsByEmail(email)` to get contractor records.
   - If 0 contractors found, throw TRPCError NOT_FOUND.
   - If 1 contractor, auto-create PortalSession via `createPortalSession({ contractorId, organizationId, email, ipAddress, userAgent })`. Return `{ session: { rawToken, expiresAt }, orgs: null, needsOrgPicker: false }`.
   - If 2+ contractors, return `{ session: null, orgs: contractors.map(c => ({ contractorId: c.id, organizationId: c.organizationId, orgName: c.organization.name, orgLogo: c.organization.logo })), needsOrgPicker: true, email }`.

3. `selectOrg` mutation (for multi-org contractors after verification):
   - Input: `z.object({ email: z.string().email(), contractorId: z.string(), organizationId: z.string() })`
   - Verify the contractor record exists and matches the email.
   - Create PortalSession. Return `{ rawToken, expiresAt }`.

4. `logout` mutation (portalProcedure -- authenticated):
   - Extract raw token from cookie, call `deletePortalSession(rawToken)`.
   - Return `{ success: true }`.

**Read endpoints (portalProcedure -- all double-scope: org via tenantStore + contractorId from ctx):**

5. `overview` query (D-02):
   - Count active contracts: `prisma.contract.count({ where: { contractorId: ctx.contractorId, status: { in: ["ACTIVE", "EXPIRING"] } } })`
   - Count pending invoices: `prisma.invoice.count({ where: { contractorId: ctx.contractorId, status: { in: ["RECEIVED", "UNDER_REVIEW", "APPROVAL_PENDING"] } } })`
   - Sum recent payments (last 90 days): `prisma.invoice.findMany({ where: { contractorId: ctx.contractorId, paymentStatus: "PAID", paidAt: { gte: 90daysAgo } }, select: { totalGrosze: true, currency: true } })`
   - Next deadline: earliest dueDate from unpaid invoices or earliest endDate from expiring contracts
   - Recent activity: last 5 invoices with status changes (derive from Invoice timestamps: receivedAt, reviewedAt, approvedAt, paidAt, rejectedAt)
   - Return: `{ activeContracts: number, pendingInvoices: number, recentPaymentsGrosze: number, recentPaymentsCurrency: string, upcomingDeadline: Date | null, recentActivity: ActivityEntry[] }`

6. `listContracts` query (PORT-02):
   - `prisma.contract.findMany({ where: { contractorId: ctx.contractorId, status: { in: ["ACTIVE", "EXPIRING", "EXPIRED"] } }, select: { id, title, type, status, startDate, endDate, currency, billingModel, rateType, rateValueGrosze, contractNumber }, orderBy: { startDate: "desc" } })`
   - Do NOT include: internalOwnerUserId, teamId, projectId, costCenterId, notes, metadataJson (per D-11 / Pitfall 3)

7. `getContract` query (PORT-02):
   - Input: `z.object({ id: z.string() })`
   - `prisma.contract.findFirst({ where: { id, contractorId: ctx.contractorId }, select: { id, title, type, status, startDate, endDate, currency, billingModel, rateType, rateValueGrosze, paymentTermsDays, contractNumber, autoRenewal, noticePeriodDays, ratePeriods: { select: { rateType, rateValueGrosze, currency, validFrom, validTo } } } })`
   - Also fetch attached documents: `prisma.documentLink.findMany({ where: { entityType: "CONTRACT", entityId: id }, include: { document: { select: { id, originalFileName, mimeType, fileSizeBytes, documentType, storageKey } } } })`
   - For each document, generate presigned download URL via `createPresignedDownloadUrl(doc.storageKey)`.
   - Return contract + documents with download URLs. Exclude storageKey from response.

8. `listInvoices` query (PORT-04, D-10):
   - `prisma.invoice.findMany({ where: { contractorId: ctx.contractorId, deletedAt: null }, select: { id, invoiceNumber, contractId, totalGrosze, currency, issueDate, receivedAt, status, matchStatus, approvalStatus, paymentStatus, paidAt }, orderBy: { receivedAt: "desc" } })`
   - Include contract title via join: `contract: { select: { title: true } }`
   - Return list with status badges derivable from status/approvalStatus/paymentStatus fields.

9. `getInvoice` query (PORT-04, D-10, D-11, D-12):
   - Input: `z.object({ id: z.string() })`
   - Fetch invoice with `contractorId` scope. Select: id, invoiceNumber, issueDate, dueDate, subtotalGrosze, totalGrosze, currency, status, approvalStatus, paymentStatus, receivedAt, reviewedAt, approvedAt, paidAt, rejectedAt, rejectionReason.
   - Include contract: `{ select: { id, title } }`.
   - Include files: `{ include: { document: { select: { id, originalFileName, storageKey } } } }`. Generate download URL for each.
   - Include payment info: `prisma.paymentRunItem.findFirst({ where: { invoiceId: id, contractorId: ctx.contractorId }, select: { paidAt: true, amountGrosze: true, currency: true } })`. Only return date + amount (D-12).
   - Build activity log from timestamps (D-11): filter to contractor-visible events only:
     - receivedAt -> "Invoice submitted"
     - reviewedAt -> "Under review"
     - approvedAt -> "Invoice approved"
     - rejectedAt -> "Invoice rejected" + rejectionReason
     - paidAt -> "Payment completed"
   - Do NOT expose: internal reviewer names, approval chain details, batch IDs, costCenterId.

10. `listDocuments` query (PORT-05):
    - Find DocumentLinks where `entityType = "CONTRACTOR"` and `entityId = ctx.contractorId`.
    - Also find DocumentLinks where `entityType = "CONTRACT"` and entityId is in contractor's contract IDs.
    - Include document: `{ select: { id, originalFileName, mimeType, fileSizeBytes, documentType, createdAt, storageKey } }`.
    - Generate presigned download URL for each. Exclude storageKey from response.
    - Return: `{ id, name: originalFileName, type: documentType, mimeType, sizeBytes, addedAt: createdAt, downloadUrl }[]`

11. `listPayments` query (PORT-04, D-12):
    - `prisma.paymentRunItem.findMany({ where: { contractorId: ctx.contractorId, status: "COMPLETED" }, select: { id, invoiceId, amountGrosze, currency, paidAt }, orderBy: { paidAt: "desc" } })`
    - Include invoice: `{ select: { invoiceNumber: true } }`.
    - Return: `{ id, invoiceNumber, amountGrosze, currency, paidAt }[]`. No batch IDs (D-12).

**Write endpoints (portalProcedure):**

12. `getUploadUrl` mutation (for invoice PDF upload):
    - Input: `z.object({ filename: z.string(), contentType: z.string().refine(ct => ct === "application/pdf", "Only PDF files") })`
    - Generate document ID (cuid), storage key via `generateStorageKey(ctx.organizationId, docId, filename)`.
    - Generate presigned upload URL via `createPresignedUploadUrl(key, contentType)`.
    - Return `{ uploadUrl, documentId: docId, storageKey: key }`.

13. `submitInvoice` mutation (PORT-03, D-06, D-07, D-08):
    - Input schema:
      ```
      z.object({
        contractId: z.string(),
        invoiceNumber: z.string().min(1).max(100),
        issueDate: z.date(),
        dueDate: z.date(),
        netAmountGrosze: z.number().int().positive(),
        grossAmountGrosze: z.number().int().positive(),
        documentId: z.string(),
        storageKey: z.string(),
        originalFileName: z.string(),
        fileSizeBytes: z.number().int().positive(),
        checksumSha256: z.string().optional(),
      })
      ```
    - Verify contract belongs to this contractor and is ACTIVE: `prisma.contract.findFirst({ where: { id: input.contractId, contractorId: ctx.contractorId, status: "ACTIVE" } })`. If not found, throw TRPCError NOT_FOUND.
    - Create Document record for the uploaded PDF: `prisma.document.create({ data: { id: input.documentId, storageKey: input.storageKey, originalFileName: input.originalFileName, mimeType: "application/pdf", fileSizeBytes: input.fileSizeBytes, documentType: "INVOICE", source: "USER_UPLOAD", checksumSha256: input.checksumSha256 ?? "" } })`.
    - Create Invoice with source PORTAL: `prisma.invoice.create({ data: { contractorId: ctx.contractorId, contractId: input.contractId, invoiceNumber: input.invoiceNumber, source: "PORTAL", issueDate: input.issueDate, dueDate: input.dueDate, subtotalGrosze: input.netAmountGrosze, totalGrosze: input.grossAmountGrosze, amountToPayGrosze: input.grossAmountGrosze, currency: contract.currency, status: "RECEIVED", matchStatus: "UNMATCHED", approvalStatus: "NOT_STARTED", paymentStatus: "NOT_READY", submittedByEmail: ctx.portalSession.email } })`.
    - Create InvoiceFile link: `prisma.invoiceFile.create({ data: { invoiceId, documentId: input.documentId, role: "SOURCE_ORIGINAL" } })`.
    - Return: `{ invoiceId: invoice.id, invoiceNumber: invoice.invoiceNumber, status: invoice.status }`.

14. `getActiveContracts` query (for invoice form contract dropdown, D-08):
    - `prisma.contract.findMany({ where: { contractorId: ctx.contractorId, status: "ACTIVE" }, select: { id, title, currency, rateValueGrosze, rateType, billingModel } })`
    - Return list for Select dropdown. Auto-select hint: include count.

15. `getSession` query (for portal layout -- current contractor + org info):
    - Return `{ contractor: { id, displayName, email }, organization: { id, name, logo } }` from ctx.portalSession and ctx.contractor.
