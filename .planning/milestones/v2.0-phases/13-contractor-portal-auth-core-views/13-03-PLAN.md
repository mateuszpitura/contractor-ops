3. Create `apps/web/src/app/[locale]/(portal)/login/verify/page.tsx` ("use client"):

This page handles the magic link callback. URL: `/portal/login/verify?token=xxx`.

Flow:
- Read `token` from URL search params via `useSearchParams()`.
- On mount (useEffect), call `portal.verifyMagicLink.useMutation()` with the token.
- While verifying: show centered spinner with "Verifying your sign-in link..." text.
- If error (invalid/expired token): show error state with "This link has expired. Request a new sign-in link." and "Back to login" link.
- If `needsOrgPicker === false` (single org): set `portal_session` cookie and redirect to `/portal`.
- If `needsOrgPicker === true` (multi-org): render OrgPicker component with the orgs list.
- On org selection, call `portal.selectOrg.useMutation({ email, contractorId, organizationId })`. On success, set cookie and redirect.

Cookie setting: After receiving rawToken from verifyMagicLink or selectOrg response, set the cookie:
```typescript
document.cookie = `portal_session=${rawToken}; path=/; max-age=${7 * 24 * 60 * 60}; samesite=lax${window.location.protocol === 'https:' ? '; secure' : ''}`;
```

Then redirect: `router.push("/portal")` using `useRouter()` from next/navigation.

Important: The cookie MUST be named `portal_session` (matches what portal-auth.ts middleware parses). Set as httpOnly=false here because it's set from client-side JS. The server middleware reads it. For production hardening, consider moving cookie setting to a server action or API route that sets httpOnly=true -- but for now, client-side setting is acceptable since the token is hashed server-side anyway.

Actually -- better approach: Have the verifyMagicLink and selectOrg tRPC mutations return the raw token, then make a separate `POST /api/portal/set-session` Next.js API route that receives the token and sets an httpOnly cookie. This is more secure. Create this route:

`apps/web/src/app/api/portal/set-session/route.ts`:
```typescript
import { NextRequest, NextResponse } from "next/server";

export async function POST(req: NextRequest) {
  const { token, expiresAt } = await req.json();
  const response = NextResponse.json({ success: true });
  response.cookies.set("portal_session", token, {
    httpOnly: true,
    secure: process.env.NODE_ENV === "production",
    sameSite: "lax",
    path: "/",
    expires: new Date(expiresAt),
  });
  return response;
}
```

Then in verify page, after receiving token, call `fetch("/api/portal/set-session", { method: "POST", body: JSON.stringify({ token, expiresAt }) })` before redirecting.

Also create a corresponding clear-session route for logout:
`apps/web/src/app/api/portal/clear-session/route.ts`:
```typescript
import { NextResponse } from "next/server";

export async function POST() {
  const response = NextResponse.json({ success: true });
  response.cookies.delete("portal_session");
  return response;
}
```
