  - "done": `bg-success` (use CSS variable --success)

**jira-activity-summary.tsx** (per UI-SPEC D-10):

Props: `{ contractorId: string }`

Implementation:
1. Query: `trpc.jira.recentActivity` with { contractorId, limit: 5 }
2. If no data or empty array, return null (don't render the card at all)
3. Card: `border rounded-lg p-4` (no shadow)
4. Header row: JiraLogo (16px via `className="size-4"`) + "Recent Jira Activity" (text-sm font-semibold)
5. List: up to 5 items, each a flex row:
   - JiraIssueChip (left)
   - Issue summary text (center, `text-sm text-muted-foreground truncate flex-1 mx-2`)
   - Relative timestamp (right, `text-xs text-muted-foreground whitespace-nowrap`)
6. Loading: 3 skeleton rows per UI-SPEC (chip skeleton 120px + text skeleton 200px + timestamp skeleton 60px)
7. Use `animate-fade-up` on mount (400ms ease-out) per UI-SPEC

Format relative timestamps: use Intl.RelativeTimeFormat or simple helper (X min ago, X hours ago, X days ago).

Props: `{ taskTemplateId: string }`

Implementation:
1. Query trpc.jira.connectionStatus to check if Jira is connected

3. Query trpc.jira.getTaskConfig with { taskTemplateId }
4. State: local useState for jiraEnabled toggle
5. Layout: single-row flex: `flex items-center gap-3`

   - Mapping summary (center, flex-1): show "PROJ / Bug" if configured, "Not configured" in text-muted-foreground if not
   - "Configure Jira" ghost button (right): opens JiraProjectMappingDialog
6. When switch toggled and mapping exists, call trpc.jira.saveTaskConfig mutation immediately
7. Import JiraProjectMappingDialog from "./jira-project-mapping-dialog"
