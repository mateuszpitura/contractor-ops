# Phase 19 — UI Design Contract

> Visual and interaction contract for Jira integration: OAuth connection, workflow-to-issue creation with configurable mapping, bidirectional status sync, and linked Jira issue display on workflow and contractor views. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps within Jira chips, badge inline padding, chip vertical padding |
| sm | 8px | Status mapping row gaps, chip horizontal padding, compact list spacing |
| md | 16px | Form field vertical spacing, card content padding, mapping table cell padding |
| lg | 24px | Section padding within mapping dialog, card internal padding |
| xl | 32px | Layout gaps between settings sections, dialog section breaks |
| 2xl | 48px | Page-level vertical rhythm |
| 3xl | 64px | Not used in Phase 19 |

Exceptions: Jira chip components use 4px vertical padding (`py-1`) and 8px horizontal padding (`px-2`) for compact inline display. Status mapping rows use 8px vertical padding per row for table density.

## Layout Contract

### Focal Point

The primary screen focal points for Phase 19 are:

1. **Settings > Integrations tab**: The Jira provider card uses the standard ProviderConnectionCard with OAuth connect. When connected, a "Configure Mapping" button provides access to status mapping.
2. **Workflow task template editor**: The JiraTaskConfig inline section shows the Jira integration toggle and mapping summary per task template.
3. **Contractor profile > Workflows tab**: The JiraActivitySummary card at the top shows recent Jira activity. Individual workflow run rows display inline JiraIssueChip components.
4. **Workflow detail side panel**: Linked Jira issues appear as JiraIssueChip components below the progress section.

### Jira Provider Card (JIRA-01, D-03)

- Follows existing ProviderConnectionCard pattern exactly (consistent with Slack/KSeF cards)
- Card header: Jira icon (custom SVG, 32px, Jira blue #0052CC) + "Jira" title + status badge (Connected/Disconnected/Error)
- Card body when disconnected: description text + "Connect Jira" primary button (triggers OAuth 2.0 3LO)
- Card body when connected: connected workspace name + connected date + token expiry + "Manage" outline button + "Disconnect" destructive outline button
- When connected, below the card: "Configure Status Mapping" outline button opens JiraStatusMappingDialog
- OAuth scopes display: `read:jira-work`, `write:jira-work` shown in muted text on detail sheet

### Jira Status Mapping Dialog (D-04, D-05)

- Trigger: "Configure Status Mapping" button below Jira provider card (only when connected)
- Dialog width: `sm:max-w-2xl` (672px) to accommodate two-column mapping table
- Header: "Status Mapping" title with project name subtitle
- Table layout:
  - Left column "Workflow Status": read-only labels showing each WorkflowTaskStatus value (TODO, IN_PROGRESS, DONE, BLOCKED, CANCELLED, SKIPPED)
  - Right column "Jira Transition": Select dropdown per row, populated from Jira project transitions API
  - Unmapped rows: AlertTriangle icon (14px, warning color) + "Not mapped" placeholder text in Select
- Footer: "Discard Changes" outline button (left) + "Save Mapping" primary button (right, disabled until at least one mapping changed)
- Loading state: Select dropdowns show Loader2 spinner while fetching Jira transitions

### Jira Task Template Config (D-01, D-02)

- Location: inline within each WorkflowTaskTemplate configuration view
- Single-row flex layout: Switch (left) + mapping summary text (center, flex-1) + "Configure Jira" ghost button (right)
- Switch label: "Create Jira issue when task activates"
- When mapping exists: summary shows "PROJ-NAME / Bug" (project + issue type) in text-sm
- When no mapping: shows "Not configured" in text-sm text-muted-foreground, "Configure Jira" button highlighted
- "Configure Jira" button opens JiraProjectMappingDialog for this specific task template
- Switch disabled when no mapping is configured (must configure first)

### Jira Issue Chips on Workflow Views (D-09, D-10)

- **Workflow run rows** (WorkflowsTab): JiraIssueChip renders inline after the status badge, before the progress counter. Multiple chips separated by 4px gap. Max 3 chips visible, overflow shows "+N more" badge.
- **Workflow side panel** (WorkflowSidePanel): New "Linked Issues" section after the progress section, before the contractor section. Section header: "Linked Issues" (`text-sm font-semibold text-muted-foreground uppercase tracking-wider`, matching existing section header pattern). Each linked task shows: task name (text-sm) + JiraIssueChip(s) on the same row.
- **Contractor Workflows tab**: JiraActivitySummary card renders above the workflow runs list. Card uses subtle border (not elevated) to not compete with the main content.

### Jira Activity Summary (D-10)

- Position: top of contractor's Workflows tab, above the header row with "Start Workflow" CTA
- Card: `border` only (no shadow), `rounded-lg`, `p-4`
- Header row: Jira icon (16px, Jira blue) + "Recent Jira Activity" (text-sm font-semibold)
- List: up to 5 items, each a flex row with JiraIssueChip (left) + issue summary (center, truncated) + relative timestamp (right, text-xs text-muted-foreground)
- Empty state: not rendered at all when no linked issues (no empty card shown)
- Collapse behavior: not collapsible, always visible when data exists

## Status Badge Mapping

### Connection Status (on provider card)

| Status | Variant | Color Token | Label |
|--------|---------|-------------|-------|
| CONNECTED | success | `bg-emerald-500/10 text-emerald-500` | "Connected" |

### Jira Issue Status (on chips)

| Category | Dot Class | Text Color | Examples |
|----------|-----------|------------|----------|
| To Do | `status-dot-muted` | `text-muted-foreground` | Open, Backlog, To Do |
| In Progress | `status-dot-info` | `text-info` | In Progress, In Review, Testing |
| Done | `status-dot-success` | `text-success` | Done, Closed, Resolved |

Jira categorizes all statuses into these three categories via its API. The chip uses the category, not individual status names, for color.

## Jira Icon

The Jira logo is a custom SVG component (not a Lucide icon). Render as a 32px icon on the provider card and 16px for inline usage (activity summary header). Use Jira's official blue (#0052CC) for the icon fill. SVG source: standard Jira Cloud mark (two overlapping triangles). The icon component should accept a `className` prop for sizing.

```
JiraLogo component: SVG viewBox="0 0 32 32", two gradient triangles forming the Jira mark.
Fill: #0052CC (Jira blue) for both dark and light mode -- brand color, not theme-dependent.
```
