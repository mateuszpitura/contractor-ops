       ```prisma
       model PeppolParticipant {
         id                String                  @id @default(cuid())
         organizationId    String
         participantId     String                  // e.g., "0192:123456789012345"
         schemeId          String                  // e.g., "0192"
         identifierValue   String                  // e.g., "123456789012345" (TRN)
         aspProvider       String                  // "storecove" etc.
         aspRegistrationId String?                 // External ID from ASP
         status            PeppolParticipantStatus @default(PENDING)
         registeredAt      DateTime?
         createdAt         DateTime                @default(now())
         updatedAt         DateTime                @updatedAt

         organization      Organization @relation(fields: [organizationId], references: [id])
         transmissions     PeppolTransmission[]

         @@unique([organizationId, participantId])
         @@index([organizationId])
       }

       model PeppolTransmission {
         id                  String                    @id @default(cuid())
         organizationId      String
         peppolParticipantId String
         invoiceId           String?
         direction           SyncDirection             // INBOUND or OUTBOUND
         aspTransmissionId   String?                   // External ID from ASP
         documentTypeId      String?
         status              PeppolTransmissionStatus  @default(PENDING)
         xmlPayload          String?                   @db.Text
         errorMessage        String?
         transmittedAt       DateTime?
         deliveredAt         DateTime?
         createdAt           DateTime                  @default(now())
         updatedAt           DateTime                  @updatedAt

         organization        Organization @relation(fields: [organizationId], references: [id])
         participant         PeppolParticipant @relation(fields: [peppolParticipantId], references: [id])

         @@index([organizationId])
         @@index([organizationId, status])
         @@index([organizationId, invoiceId])
         @@index([aspTransmissionId])
       }

       enum PeppolParticipantStatus {
         PENDING
         REGISTERED
         ACTIVE
         SUSPENDED
         DEREGISTERED
       }

       enum PeppolTransmissionStatus {
         PENDING
         TRANSMITTED
         DELIVERED
         FAILED
         REJECTED
       }
       ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → tRPC router | User input (TRN, API key) must be validated |
| tRPC → Prisma | Queries must be scoped to organizationId (tenant isolation) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-05 | Spoofing | peppol.connect | mitigate | Validate TRN format with Zod regex; require protectedProcedure auth |
| T-49-06 | Information Disclosure | peppol.getStatus | mitigate | Scope all queries to ctx.organizationId — never return other orgs' participants |
| T-49-07 | Elevation of Privilege | peppol.disconnect | mitigate | Check org ownership of participant before deregistering; use protectedProcedure |
| T-49-08 | Information Disclosure | ASP credentials | mitigate | Encrypt API key via existing encryptCredentials before storing in credentialsRef; never return decrypted key in query responses |
</threat_model>
