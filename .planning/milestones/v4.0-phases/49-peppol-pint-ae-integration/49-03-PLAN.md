<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Storecove API → Platform | API responses are untrusted external input |
| Storecove Webhook → Platform | Webhook payloads must be signature-verified |
| QStash → API routes | QStash signature must be verified on all routes |
| Inbound XML → Parser | XML from Peppol network may contain malicious content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-09 | Spoofing | Storecove webhook | mitigate | HMAC-SHA256 signature verification in StorecoveAdapter.verifyWebhookSignature; reject invalid signatures with 401 |
| T-49-10 | Spoofing | QStash routes | mitigate | Verify QStash signature on all /api/peppol/* routes using @upstash/qstash Receiver |
| T-49-11 | Tampering | Inbound XML | mitigate | Validate parsed XML against PINT-AE business rules via PeppolAEProfile.validate() before creating Invoice record |
| T-49-12 | Repudiation | Transmission audit | mitigate | Store full XML payload in PeppolTransmission.xmlPayload for audit trail; log all status transitions via IntegrationSyncLog |
| T-49-13 | Denial of Service | Poll endpoint | mitigate | QStash CRON controls polling frequency (15min); rate limit applies to ASP API calls; timeout on fetch calls (30s) |
| T-49-14 | Information Disclosure | Credential decrypt | mitigate | Decrypt ASP credentials only in route handlers (not logged); never include in error messages or response bodies |
</threat_model>
