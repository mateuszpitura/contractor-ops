## Phase-Specific UI Components

### 1. Peppol Connection Wizard (Settings > Integrations > Peppol)

**Location:** Settings > Integrations page, triggered by "Connect Peppol" card action
**Component:** Multi-step `Dialog` wizard with step indicator
**Behavior:**
- 5 steps: TRN Entry -> ASP Selection -> API Credentials -> Register Participant -> Confirm
- Progress saved between steps (user can close and resume)
- Step validation before advancing (next button disabled until step criteria met)
- Final step shows registration status polling

**Layout:**
```
┌─────────────────────────────────────────────────────┐
│  Connect to Peppol Network                     [X]  │
│─────────────────────────────────────────────────────│
│                                                      │
│  ● ─── ● ─── ○ ─── ○ ─── ○                         │
│  TRN   ASP   Credentials  Register  Confirm         │
│                                                      │
│  Step 2: Select ASP Provider                         │
│                                                      │
│  ┌─────────────────────────────────┐                │
│  │ ● Storecove                     │                │
│  │   Certified Peppol ASP          │                │
│  │   60+ countries supported       │                │
│  └─────────────────────────────────┘                │
│                                                      │
│  More providers coming soon.                         │
│                                                      │
│            [Back]            [Next →]                │
└─────────────────────────────────────────────────────┘
```

**Step 1 — TRN Entry:**
```
Tax Registration Number (TRN)
[________________________]
 15-digit UAE TRN

Peppol Participant ID will be: 0192:{TRN}
```
- Input: `Input` (shadcn) with 15-character max, numeric validation
- Real-time preview of computed Participant ID below input
- Label: `text-sm font-medium`, hint: `text-sm text-muted-foreground`

**Step 3 — API Credentials:**
```
Storecove API Key
[________________________]  [Show/Hide 👁]
 Obtain from Storecove dashboard

Environment
  ● Sandbox (testing)
  ○ Production
```
- API key input: `Input` with `type="password"` and toggle visibility
- Environment: `RadioGroup` (shadcn)
- Credential stored securely on save (encrypted per existing pattern)

**Step 4 — Register Participant:**
```
Registering your organization on the Peppol network...

Participant ID: 0192:123456789012345
ASP Provider: Storecove
Environment: Sandbox

[████████████░░░░░░░░] Registering...

Status: Submitted — waiting for confirmation
```
- Progress indicator: `Progress` (shadcn) with animated indeterminate state
- Status polling via tRPC subscription or interval query (3s)
- Success: green checkmark + "Registered" status
- Failure: destructive alert with error message and "Retry" button

**Step 5 — Confirm:**
```
✓ Peppol Connection Active

Participant ID: 0192:123456789012345
ASP: Storecove (Sandbox)
Registered: 2026-04-11

You can now send and receive e-invoices
through the Peppol network.

                              [Done]
```

## Interaction Patterns

### Peppol Connection Wizard
- Multi-step wizard with server-side step persistence (resume after close)
- Step validation: TRN format (15 digits), API key non-empty, environment selected
- Registration step polls tRPC `peppol.getParticipantStatus` every 3 seconds
- Timeout after 60 seconds with "Registration is taking longer than expected" message and manual refresh option

### Outbound Invoice Transmission
- "Send via Peppol" action appears on invoices for orgs with active Peppol connection
- Triggers async QStash job — button shows loading spinner, then switches to "Queued" badge
- Transmission status updates via tRPC query on invoice detail page (polling interval: 5s while pending)
- Failed transmissions show retry button with error context

### Inbound Invoice Processing
- New invoices from Peppol appear in intake queue with `PEPPOL` source badge
- Standard intake queue flow applies (matching, review, approval)
- Info banner on invoice detail shows Peppol-specific metadata (sender ID, document type)

### Connection Disconnect
- Destructive action with confirmation dialog
- Triggers ASP deregistration API call
- On success: card reverts to empty state, related invoice transmissions retain their status (historical)

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Wizard step navigation keyboard | Tab between steps, Enter to advance, Escape to close dialog |
| Step indicator screen reader | `aria-label="Step {N} of 5: {step_name}"` with `aria-current="step"` on active |
| Connection status screen reader | `aria-label="Peppol connection status: {status}"` on status dot |
| Transmission timeline screen reader | `role="list"` with `aria-label="Transmission timeline"`, each step as `role="listitem"` |
| QR code alt text | `alt="UAE FTA QR code for invoice {number}"` |
| Color contrast | All semantic status colors pass WCAG AA on their respective card backgrounds |
| Status indicators | Text labels accompany all status dots (not dot-only) |
| Form validation | `aria-invalid="true"` and `aria-describedby` linking to error messages |
