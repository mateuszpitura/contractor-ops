<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ASP API → Platform | ASP credentials and API responses are untrusted external input |
| Inbound XML → Parser | XML from Peppol network is untrusted — could contain malicious content |
| QR data → parseQR | QR code data is untrusted external input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-01 | Spoofing | ASP adapter | mitigate | Validate ASP API responses with Zod schemas; verify webhook signatures before processing |
| T-49-02 | Tampering | XML parser | mitigate | Use fast-xml-parser with strict options (no entity expansion, no external DTD); validate parsed XML against PINT-AE business rules before creating EInvoice |
| T-49-03 | Information Disclosure | ASP credentials | mitigate | Store API keys via existing credentialsRef encryption pattern; never log credentials; redact in error messages |
| T-49-04 | Denial of Service | XML parser | mitigate | Set maxEntityCount and maxNodeCount limits in fast-xml-parser options; reject oversized XML payloads (>5MB) |
</threat_model>
