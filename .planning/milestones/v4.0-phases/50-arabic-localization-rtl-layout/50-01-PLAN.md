  **Per D-01 (Tailwind CSS logical properties) and D-02 (big-bang conversion):**

  Convert ALL physical directional Tailwind classes to logical equivalents in every file listed. This is the big-bang conversion of UI primitives — these components are the foundation all other components build on.

  **Conversion map (apply to every className string):**

  | Physical | Logical |
  |----------|---------|
  | `pl-` | `ps-` |
  | `pr-` | `pe-` |
  | `ml-` | `ms-` |
  | `mr-` | `me-` |
  | `left-` | `start-` |
  | `right-` | `end-` |
  | `text-left` | `text-start` |
  | `text-right` | `text-end` |
  | `rounded-l-` | `rounded-s-` |
  | `rounded-r-` | `rounded-e-` |
  | `border-l-` | `border-s-` |
  | `border-r-` | `border-e-` |
  | `float-left` | `float-start` |
  | `float-right` | `float-end` |
  | `scroll-ml-` | `scroll-ms-` |
  | `scroll-mr-` | `scroll-me-` |

  **IMPORTANT exceptions — do NOT convert:**
  - `px-` (padding x-axis, not padding-left) — this is already bidirectional
  - `mx-` (margin x-axis) — already bidirectional
  - Physical properties used for decorative/absolute positioning that must stay fixed regardless of direction (rare in UI primitives — flag if found)
  - `translate-x-` — keep as-is, these are transform-based

  **For each file:**
  1. Read the file completely
  2. Find all className strings containing physical directional classes
  3. Replace each physical class with its logical equivalent
  4. Verify no `pl-`, `pr-`, `ml-`, `mr-`, `text-left`, `text-right`, `left-`, `right-` remain (except in non-Tailwind contexts like CSS custom properties or JavaScript variable names)
  5. Ensure the file still has valid JSX/TSX syntax

  **sidebar.tsx specifics (10 occurrences):** Contains `left-0`, `right-0` positioning, `pl-*`, `pr-*` padding, `border-l`, `border-r`. Convert all to logical equivalents.

  **dropdown-menu.tsx specifics (11 occurrences):** Contains `text-left`, `text-right`, `pl-*`, `pr-*`, `left-*`, `right-*`. All must become logical.

  **select.tsx specifics (4 occurrences):** Contains `pr-*`, `text-left`, `right-*` positioning.
