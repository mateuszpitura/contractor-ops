Services provide grant, revoke, query-current-state, and query-history operations.
</objective>

<threat_model>
## Security Threat Model (ASVS L1)

| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-51-01 | Consent records tampered after creation | HIGH | Append-only design — no UPDATE/DELETE operations exposed. ConsentRecord has no updatedAt field. Revocations create new records. |
| T-51-02 | Unauthorized consent modification | HIGH | All consent mutations require authenticated tenantProcedure + requirePermission middleware. |
| T-51-03 | Missing consent audit trail | MEDIUM | Every grant/revoke captures userId, ipAddress, userAgent, timestamp. Records are never deleted. |
| T-51-04 | Purpose enum injection | LOW | ConsentPurpose is a Prisma enum — only valid values accepted at DB level. Zod schema validates at API boundary. |
</threat_model>

```prisma
// ConsentRecord, PrivacyNotice — PDPL compliance (Phase 51)

model ConsentRecord {
  id             String         @id @default(cuid())
  organizationId String
  userId         String
  purpose        ConsentPurpose
  granted        Boolean
  version        Int            @default(1)
  grantedAt      DateTime?
  revokedAt      DateTime?
  ipAddress      String?
  userAgent      String?
  createdAt      DateTime       @default(now())

  organization   Organization   @relation(fields: [organizationId], references: [id], onDelete: Cascade)
  user           User           @relation("UserConsentRecords", fields: [userId], references: [id], onDelete: Cascade)

  // Immutable — no updatedAt (per D-02, matches AuditLog pattern)
  @@index([organizationId])
  @@index([organizationId, userId, purpose])
  @@index([organizationId, purpose, createdAt])
}

model PrivacyNotice {
  id             String   @id @default(cuid())
  organizationId String
  jurisdiction   String   @db.Char(2)
  version        Int      @default(1)
  contentJson    Json
  effectiveFrom  DateTime
  createdAt      DateTime @default(now())

  organization   Organization @relation(fields: [organizationId], references: [id], onDelete: Cascade)

  @@index([organizationId, jurisdiction])
  @@unique([organizationId, jurisdiction, version])
}

enum ConsentPurpose {
  CONTRACTOR_DATA_PROCESSING
  INVOICE_PAYMENT_PROCESSING
  ANALYTICS_REPORTING
  CROSS_BORDER_TRANSFER
  INTEGRATION_DATA_SHARING
  COMMUNICATION_NOTIFICATIONS
}
```

Add relations to Organization model in `organization.prisma`:
```prisma
  consentRecords   ConsentRecord[]
  privacyNotices   PrivacyNotice[]
```

Add relation to User model in `auth.prisma`:
```prisma

```

Per D-02: ConsentRecord is append-only — revocations create new records with `granted: false` and `revokedAt` set. Never update existing records.

```typescript
import { z } from "zod";

export const consentPurposeEnum = z.enum([
  "CONTRACTOR_DATA_PROCESSING",
  "INVOICE_PAYMENT_PROCESSING",
  "ANALYTICS_REPORTING",
  "CROSS_BORDER_TRANSFER",
  "INTEGRATION_DATA_SHARING",
  "COMMUNICATION_NOTIFICATIONS",
]);

export type ConsentPurpose = z.infer<typeof consentPurposeEnum>;

// Purposes that MUST be accepted during onboarding (blocking)
export const REQUIRED_PURPOSES: ConsentPurpose[] = [
  "CONTRACTOR_DATA_PROCESSING",
  "INVOICE_PAYMENT_PROCESSING",
  "COMMUNICATION_NOTIFICATIONS",
];

// Purposes that are optional during onboarding
export const OPTIONAL_PURPOSES: ConsentPurpose[] = [
  "ANALYTICS_REPORTING",
  "CROSS_BORDER_TRANSFER",
  "INTEGRATION_DATA_SHARING",
];

export const grantConsentSchema = z.object({
  purpose: consentPurposeEnum,
  granted: z.boolean(),
});

export const bulkGrantConsentSchema = z.object({
  consents: z.array(grantConsentSchema).min(1),
});

export const consentQuerySchema = z.object({
  purpose: consentPurposeEnum.optional(),
});

// Jurisdiction codes that trigger PDPL compliance
export const PDPL_JURISDICTIONS = ["AE", "SA"] as const;
export type PdplJurisdiction = (typeof PDPL_JURISDICTIONS)[number];

export function isPdplJurisdiction(countryCode: string | null | undefined): countryCode is PdplJurisdiction {
  return PDPL_JURISDICTIONS.includes(countryCode as PdplJurisdiction);
}
```

**2. Create `packages/api/src/services/privacy-notice.ts`:**

Service with:
- `getPrivacyNotice(organizationId, jurisdiction)` — returns latest version of privacy notice for jurisdiction. If none exists for org, creates from default template for AE or SA.
- `getDefaultNoticeContent(jurisdiction: "AE" | "SA")` — returns structured JSON with privacy notice sections: controller identity, purposes, legal basis, data categories, transfer destinations, retention periods, data subject rights, contact information. UAE version references Federal Decree-Law No. 45/2021. Saudi version references Royal Decree M/19 PDPL.
- `createPrivacyNotice(organizationId, jurisdiction, contentJson)` — creates new version, auto-increments version number.

Cache with `CacheKeys` pattern: `consent:notice:${orgId}:${jurisdiction}`.

**3. Create `packages/api/src/services/consent-record.ts`:**

Service with:
- `grantConsent(orgId, userId, purpose, ipAddress?, userAgent?)` — creates ConsentRecord with `granted: true`, `grantedAt: now()`. Computes version by counting existing records for (orgId, userId, purpose) + 1.
- `revokeConsent(orgId, userId, purpose, ipAddress?, userAgent?)` — creates NEW ConsentRecord with `granted: false`, `revokedAt: now()`. Does NOT modify existing records (per D-02 append-only).
- `getCurrentConsent(orgId, userId)` — returns Map<ConsentPurpose, {granted, version, lastUpdated}> by querying latest record per purpose.
- `getConsentHistory(orgId, userId, purpose?)` — returns all records for audit, ordered by createdAt DESC.
- `hasRequiredConsents(orgId, userId)` — returns boolean: true if all REQUIRED_PURPOSES have latest record with `granted: true`.
- `bulkGrantConsent(orgId, userId, consents: {purpose, granted}[], ipAddress?, userAgent?)` — batch create records in a transaction.
