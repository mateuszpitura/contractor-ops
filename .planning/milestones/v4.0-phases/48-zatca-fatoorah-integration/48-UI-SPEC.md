# Phase 48 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (Tailwind v4 + CSS custom properties) |
| Preset | Custom "Precision Craft" theme in globals.css (OKLCh color format) |
| Component library | base-ui (shadcn components in `@/components/ui/`) |
| Icon library | lucide-react |
| Font | Outfit (body) + Bricolage Grotesque (display) + JetBrains Mono (code/amounts) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, wizard step indicators |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, wizard step content |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label | 14px (text-sm) | 500 (medium) | 1.5 |
| Heading | 16px (text-base) | 600 (semibold) | 1.4 |
| Display | 20px (text-xl) | 600 (semibold) | 1.3 |

Source: Consistent with Phase 47 design system. Heading at 16px for wizard step titles, section titles within settings. Display at 20px for the ZATCA integration page heading.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.985 0.003 85)` / `--background` | Page background, wizard content area |
| Secondary (30%) | `oklch(1 0 0)` / `--card` | Wizard step cards, invoice ZATCA status cards, onboarding form surfaces |
| Accent (10%) | `oklch(0.44 0.145 178)` / `--primary` | "Complete Onboarding" CTA, active wizard step indicator, "Submit to ZATCA" action |
| Destructive | `oklch(0.58 0.23 18)` / `--destructive` | Rejection status badge, certificate expiry warning, failed submission indicator |

Accent reserved for: "Complete Onboarding" final wizard CTA, "Submit to ZATCA" manual submission button, active wizard step number circle, sandbox/production toggle when production is selected.

Semantic colors used:
- `--success` / `oklch(0.55 0.17 150)` — CLEARED/REPORTED status badge, compliance checks passed indicator, production certificate active
- `--warning` / `oklch(0.70 0.16 65)` — PENDING submission status, certificate approaching expiry, sandbox mode indicator
- `--info` / `oklch(0.55 0.18 260)` — Sandbox environment banner, compliance check in-progress indicator

## Interaction Patterns

### Onboarding Wizard
- Wizard state persisted to `IntegrationConnection.configJson` after each step
- User can leave and resume at any completed step
- "Cancel" at any step returns to integrations page (progress preserved)
- Step transitions use subtle slide animation (200ms ease-out)
- All async operations (CSR gen, CSID request, compliance checks) show inline loading states

### Invoice ZATCA Submission
- Submission triggered automatically via QStash when invoice is created for a ZATCA-connected org
- Status badge updates via tRPC subscription or polling (10s interval)
- "Resubmit" available only for REJECTED status — triggers new QStash job
- Manual "Submit to ZATCA" button available in invoice detail if auto-submission was disabled

### QR Code Display
- QR code image generated server-side and stored as invoice file attachment
- Displayed inline in ZATCA submission detail panel
- Decoded fields shown beside QR for transparency
- QR code also embedded in downloadable invoice PDF

### Certificate Management
- Certificate expiry tracked and shown in ZATCA compliance widget
- Warning banner at 30 days, critical at 7 days
- Renewal flow reuses Steps 2-5 of the onboarding wizard

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Wizard step keyboard nav | Arrow keys to navigate steps, Enter to select, step indicator has `role="tablist"` |
| Wizard step screen reader | Each step has `aria-current="step"` for active, `aria-label` with step number and name |
| Status badges screen reader | `aria-label` with full status text (e.g., "ZATCA status: Cleared on April 11, 2026") |
| Compliance check results | Results list uses `role="list"` with status announced per item |
| QR code alt text | `alt="ZATCA QR code containing seller name, VAT number, invoice total, and VAT amount"` |
| Environment toggle | `RadioGroup` has built-in keyboard support and aria labels |
| Loading states | Spinner has `aria-label="Loading"` and live region announces completion |
| Color contrast | All badge variants pass WCAG AA on their respective backgrounds |
| Hash/UUID values | `aria-label` with "Invoice UUID" or "Previous hash" prefix for screen readers |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Desktop (>1024px) | Full wizard layout with horizontal stepper, side-by-side QR and details |
| Tablet (768-1024px) | Wizard stepper becomes compact (numbers only, no labels below), QR and details stack |
| Mobile (<768px) | Wizard stepper becomes vertical progress list, all cards full-width, environment toggle stacks vertically, ZATCA submission detail panel simplified (hash values truncated with copy button) |
