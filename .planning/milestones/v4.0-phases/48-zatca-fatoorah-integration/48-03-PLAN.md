Create `packages/einvoice/src/profiles/zatca/qr-code.ts`:

**TLV utilities (exported):**

`encodeTLV(fields: Array<{ tag: number; value: string | Buffer }>): Buffer`
- For each field: write tag (1 byte), length (1 byte if < 128, multi-byte 0x81+1byte for 128-255, 0x82+2bytes for 256+), value (UTF-8 or raw Buffer bytes)
- Concatenate all into single Buffer

`decodeTLV(buffer: Buffer): Array<{ tag: number; value: Buffer }>`
- Parse sequentially: tag (1 byte), length (handle multi-byte), value (length bytes), repeat

**ZatcaTLVQRCode class implementing QRCodeable:**

`generateQR(invoice: EInvoice): Promise<Buffer>`
1. Build TLV fields using ZatcaTlvTag enum:
   - Tag 1 (SELLER_NAME): invoice.supplier.name (UTF-8)
   - Tag 2 (VAT_NUMBER): invoice.supplier.id (UTF-8)
   - Tag 3 (TIMESTAMP): ISO 8601 from invoice.issueDate (UTF-8)
   - Tag 4 (TOTAL_WITH_VAT): (taxInclusiveAmount / 100).toFixed(2) (UTF-8)
   - Tag 5 (VAT_AMOUNT): sum of taxBreakdown taxAmountMinor / 100 (UTF-8)
   - Tag 6 (INVOICE_HASH): extensions.invoiceHash if present (hex -> bytes)
   - Tag 7 (ECDSA_SIGNATURE): extensions.signatureValue if present (raw bytes)
   - Tag 8 (PUBLIC_KEY): extensions.publicKey if present (raw bytes)
   Tags 6-8 only when present in extensions.
2. Validate: reject supplier.name > 1000 chars (DoS protection)
3. encodeTLV(fields)
4. Base64-encode the TLV buffer
5. QRCode.toBuffer(base64String, { type: 'png', width: 200, errorCorrectionLevel: 'M' })
6. Return PNG buffer

`parseQR(data: Buffer): Promise<Partial<EInvoice>>`
1. data is base64 TLV string as UTF-8 buffer
2. Decode base64 to TLV buffer
3. decodeTLV() to get tag-value pairs
4. Map: tag1->supplier.name, tag2->supplier.id, tag3->issueDate, tag4->taxInclusiveAmount (x100), tag5->taxBreakdown[0].taxAmountMinor (x100)
