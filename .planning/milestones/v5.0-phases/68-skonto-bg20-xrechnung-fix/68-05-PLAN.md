<objective>
Wire the Skonto cascade into the `generateZugferdPdf` tRPC procedure (`einvoice.ts:234-301`) so callers of the ZUGFeRD generation endpoint produce PDF/A-3 documents whose embedded factur-x.xml carries BG-20 Payment Terms when a Skonto term is configured.

Purpose: Phase 68 D-06. Plan 04 wired the Skonto term through `generateZugferdPdf` (the einvoice-package generator); Plan 03 wired the cascade through `finalizeEInvoice` (the XRechnung finalize service). The remaining wiring gap is the `generateZugferdPdf` tRPC procedure in the api router — it does not eager-fetch `skontoTerms`, does not call `resolveSkontoTerm`, and passes only `{ invoice: envelope }` into the einvoice-package generator. This plan closes that final wiring gap symmetric to Plan 03 (same cascade pattern, same `payment.ts:1213-1255` reference, same inline mapper).

Output:
- Extended Prisma `include` in `ctx.db.invoice.findFirst` (currently `einvoice.ts:241-263`) to fetch `skontoTerms` + `contractor.billingProfiles`.
- Inline `toSkontoTermData(prismaRow)` mapper inside the procedure (parallel to Plan 03 but local to this router file).
- Cascade-resolution block computing `effectiveSkonto = resolveSkontoTerm(invoiceTerm, profileDefault)`.
- `generateZugferdPdf({ invoice: envelope, skontoTerm: effectiveSkonto })` (and `leitwegId` if the procedure already resolves one — verify during execution).
- Layer C router test (extending `einvoice.generate-zugferd.test.ts`) with three cascade-branch assertions.
</objective>
