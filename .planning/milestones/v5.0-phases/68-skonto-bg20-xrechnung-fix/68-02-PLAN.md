<objective>
Plumb the Skonto term through the `XRechnungDEProfile` profile-level API so that callers passing `{ skontoTerm: { ... } }` get structured BG-20 Payment Terms in the resulting XRechnung CII XML — the wiring layer the v5.0 audit I-1 found broken.

Purpose: Phase 68 D-01 + D-02. The CII-level helper `generateXRechnungCii(invoice, leitwegId, skontoTerm)` already accepts a third arg and `buildPaymentTerms` already produces correct BG-20 output (Phase 63 D-23 work, locked by `generator.test.ts:341-408`). The defect is that `XRechnungDEProfile.generate(invoice, opts)` never forwards `opts.skontoTerm` because `XRechnungGenerateOptions` doesn't have a `skontoTerm` field. This plan adds the field, wires it through both `generate` and `generateAndValidate`, and lands a new `profile.test.ts` (Layer A per D-08) that locks the wrapper-level wiring against future regression.

Output:
- `XRechnungGenerateOptions.skontoTerm?: SkontoTermInput | null` field added.
- `SkontoTermInput` re-exported from `xrechnung-de/index.ts`.
- `XRechnungDEProfile.generate` and `generateAndValidate` forward the third positional argument.
- New `profile.test.ts` with two new fixtures asserting both BG-20 emission AND KoSIT 3-layer `'VALID'` status across with-Skonto and no-Skonto branches (D-09).
</objective>
