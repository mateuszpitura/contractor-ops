`country-fields.ts` is the server-side truth for contractor profile mutations. The `countryFieldsSchemaMap[organization.countryCode].parse(payload)` call runs inside tRPC procedures (Plan 06 will verify). Client RHF uses the same schema via `zodResolver`.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-11 | Tampering (type confusion across countries) | `countryFieldsSchemaMap` dispatch | mitigate | Dispatch is by `countryCode` string from `session.organization` (server-derived); never by user-submitted country param (ASVS V5, V13). |
| T-56-12 | Tampering (schema bypass via unknown country) | `validateCountryFields` | accept | Falls through to `return {}` for unknown countries — acceptable since contractor creation is scoped to an existing org whose country is validated at org-creation time. |
| T-56-13 | Information Disclosure (error leakage) | Zod error messages | mitigate | User-friendly format hints only ("Example: 93/815/08152"); no regex source, no internal state (ASVS V7). UI-SPEC §Copywriting error states defines every string. |
| T-56-14 | Tampering (Handelsregister court injection) | `handelsregister.court` input | mitigate | `.refine` restricts to `HANDELSREGISTER_COURTS` code set; prevents arbitrary free-text that could be used for log-injection or downstream template injection. |
</threat_model>
