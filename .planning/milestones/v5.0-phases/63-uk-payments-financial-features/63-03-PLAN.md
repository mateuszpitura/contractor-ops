**`resolveStatutoryRate(rateHistory: RateHistoryEntry[], debtOverdueDate: Date): { referenceDate: Date; boeRate: number; statutoryRate: number }`:**
- Per LPCDA Section 4(1): if debt becomes overdue Jan-Jun, reference date = 31 Dec of previous year. If Jul-Dec, reference date = 30 Jun of current year.
- Find the BoE rate in effect on the reference date (latest entry with effectiveFrom <= referenceDate).
- Return `{ referenceDate, boeRate, statutoryRate: boeRate + 8 }`.

**`getCompensationTier(invoiceTotalMinor: number): number`:**
- Per LPCDA Section 5A and 2013 Regulations:
  - `< 100000` (< GBP1000) -> `4000` (GBP40)
  - `100000..999999` (GBP1000-GBP9999.99) -> `7000` (GBP70)
  - `>= 1000000` (>= GBP10000) -> `10000` (GBP100)

**`calculateLateInterest(input: LateInterestInput): LateInterestResult`:**
1. Scope check: if `contractorCountryCode !== 'GB'` -> `{ applicable: false, reason: 'NON_GB_INVOICE' }`. If `!isBusinessCustomer` -> `{ applicable: false, reason: 'B2C_TRANSACTION' }`. If `invoiceCurrency !== 'GBP'` -> `{ applicable: false, reason: 'NON_GBP_CURRENCY' }`.
2. Calculate due date: use `invoiceDueDate` directly (caller resolves payment term cascade).
3. If `asOf <= invoiceDueDate` -> return applicable=true, 0 interest, 0 days.
4. Compute `daysOverdue = Math.floor((asOf.getTime() - invoiceDueDate.getTime()) / 86400000)`.
5. Resolve statutory rate via `resolveStatutoryRate(rateHistory, daysAfterDueDate=invoiceDueDate + 1 day)` -- the debt start date is the day after the due date.
6. Calculate `principalOutstandingMinor = invoiceTotalMinor - sum(payments.amountMinor)` (clamped to >= 0).
7. Daily simple interest: `dailyInterestMinor = Math.round(principalOutstandingMinor * (statutoryRate / 100) / 365)`.
8. `accruedInterestMinor = dailyInterestMinor * daysOverdue`.
9. Compensation: use `compensationTierMinor` if provided (already snapshotted), else compute via `getCompensationTier(invoiceTotalMinor)`.
10. Check waivers: find active waiver (non-revoked). If waiveType === 'STATUTORY_INTEREST' -> zero interest. If 'COMPENSATION' -> zero compensation. If 'BOTH' -> zero both.
11. `totalStatutoryClaimMinor = accruedInterestMinor + compensationTierMinor`.

Write tests FIRST (RED), then implement (GREEN). Test the statutory period edge cases heavily per Pitfall 1 from RESEARCH.md.

Export `async function pollBoeBaseRate(): Promise<{ updated: boolean; currentRate: number | null; error?: string }>`:
1. Fetch the BoE IUDBEDR series CSV from `BOE_CSV_URL` with query params: `csv.x=yes`, `Datefrom={today-30d}`, `Dateto={today}`, `SeriesCodes=IUDBEDR`, `CSVF=TN`, `UsingCodes=Y`, `Filter=N`. Use `fetch()` with User-Agent header `contractor-ops/1.0 (BoE rate poller)`.
2. Parse the CSV response: skip header rows (first few lines are metadata), find data rows with format `DD Mon YYYY, rate`. Extract the most recent rate.
3. Query `db.boEBaseRateHistory.findFirst({ orderBy: { effectiveFrom: 'desc' } })` to get the latest stored rate.
4. If the fetched rate differs from the stored rate, insert a new `BoEBaseRateHistory` row with `effectiveFrom = today`, `ratePercent = fetchedRate`, `source = 'BOE_API'`.
5. On any fetch/parse error: log warning via `log.warn({ err, url: BOE_CSV_URL }, 'BoE rate poll failed — manual entry still possible')`. Return `{ updated: false, currentRate: null, error: err.message }`. Do NOT throw.
6. Return `{ updated: true/false, currentRate }`.

Use existing `GovApiRateLimiter` from `@contractor-ops/gov-api` with `apiName: 'boe-base-rate'` to rate-limit calls. Use existing `GovApiAuditLogger` to log the poll attempt.
