All values inherit the existing 4-pt-multiple scale via Tailwind v4 (`gap-1=4px`, `gap-2=8px`, `gap-4=16px`, `gap-6=24px`, `gap-8=32px`, `gap-12=48px`, `gap-16=64px`).

| Token | Value | Phase-60 Usage |
|-------|-------|----------------|
| xs | 4px | Icon-to-label gap inside chips/badges (alert chip, "Reassessment recommended" pill) |
| sm | 8px | Tile internal padding for KPI labels; chip + icon vertical rhythm |
| md | 16px | Default card padding (`p-4`); tile-to-tile gap in the per-market 4-tile grid |

## Typography

Inherits the locked fluid-typography scale from `globals.css:306-323`. Phase 60 declares exactly **4 sizes** and **2 weights** for its surfaces (additional scale steps exist globally but are not used by Phase-60 UI).

| Role | Scale Token | Computed Range | Weight | Line Height | Phase-60 Usage |
|------|-------------|----------------|--------|-------------|----------------|
| Body | `text-sm` | 13px → 14px (clamp) | 400 (regular) | `1.25rem` | Tile descriptions, trigger reason metadata, form labels, empty-state body, CSV column captions, "Last scanned …" caption, chip labels |
| Label / heading (tile) | `text-base` | 14px → 15px (clamp) | 600 (semibold) | `1.5rem` | Tile titles ("Coverage", "Risk distribution", "Active alerts"), table column headers, form-field labels, DRV-clearance row primary text |
| Heading (card) | `text-xl` | 18px → 22px (clamp) | 600 (semibold) | `1.75rem` | Per-market card title ("UK — IR35", "Germany — Scheinselbständigkeit"), dialog heading |
| Display (page + KPI) | `text-3xl` | 28px → 40px (clamp) | 600 (semibold) | `2.75rem` | Page H1 ("Classification health"), KPI numerals (coverage %, alert count, overdue count) |

**Total: 4 sizes (sm, base, xl, 3xl) and 2 weights (400 regular, 600 semibold) — strictly within the ≤4 sizes / ≤2 weights design-system budget.** Emphasis for the display role comes from size (`text-3xl`) and Bricolage Grotesque display face, not an additional weight step.

**Font families (locked by `globals.css`):**
- Body / labels / KPIs / all Phase-60 chrome → `var(--font-sans)` = Outfit
- Page H1 only → `var(--font-display)` = Bricolage Grotesque
- DRV reference numbers + audit-log IDs in trigger reason chips → `var(--font-mono)` = JetBrains Mono

**Copywriting variants outside the 4-size set (explicitly not counted as additional scale steps):** the `text-xs` caption role (11–12px) exists in the global scale but Phase-60 surfaces do NOT use it — all meta captions use `text-sm` at muted foreground colour for visual de-emphasis rather than a smaller size. This keeps the phase-60 scale strictly at 4 sizes.

## Color

Inherits the full Precision Craft palette from `globals.css:20-121` (light) and `globals.css:127-224` (dark). Phase 60 maps semantic intent to existing tokens — does NOT introduce new color values.

| Role | Token | Usage in Phase 60 |
|------|-------|-------------------|
| Dominant (60%) | `--background` (warm off-white / deep warm slate) | Page surface, dashboard wrapper |
| Secondary (30%) | `--card`, `--surface-1`, `--surface-2` | Per-market cards, individual KPI tiles, DRV-clearance panel container, reassessment trigger detail row |
| Accent (10%) | `--primary` (Deep Teal `oklch(0.44 0.145 178)`) | **Reserved for:** primary CTAs ("Refresh", "Download CSV", "Start new assessment", "File new clearance"); KPI numeral color when status is healthy; selected-tab indicator on the per-market card; focus rings (`--ring`) on every interactive control |
| Destructive | `--destructive` (refined crimson `oklch(0.58 0.23 18)`) | **Reserved for:** Critical economic-dependency band (≥83.33% billing share); "Dismiss trigger" destructive confirmation button; DRV clearance with `outcome='ABHANGIG'` |
| Success (semantic, not part of 60/30/10) | `--success` (`oklch(0.55 0.17 150)`) | Safe economic-dependency band (<70%); IR35 "Outside" verdict pill; DRV "Selbstandig" verdict; resolved trigger state; "Coverage 100%" KPI tile chrome |
| Warning (semantic) | `--warning` (`oklch(0.7 0.16 65)`) | Warning economic-dependency band (70–83.33%); IR35 "Undetermined" verdict; DRV mid-traffic-light; expiring-within-30d clearance; OPEN reassessment trigger chip |
| Info (semantic) | `--info` (`oklch(0.55 0.18 260)`) | "Last scanned at" caption icon; informational empty-state illustration tint; DRV `outcome='PENDING'` clearance |
| Risk-distribution stacked bar (D-14) | `--success` / `--warning` / `--destructive` | Single contiguous horizontal bar per market card. Order left→right: success (green) → warning (amber) → destructive (red). Inherits the Phase-58 outcome-pill palette exactly — do NOT introduce a parallel chart palette |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. Primary action buttons on the dashboard ("Refresh", "Download CSV", "File new Statusfeststellungsverfahren", "Start new assessment")
2. KPI numeral foreground when the value is in the healthy band (e.g. coverage ≥80%, alert-count = 0)
3. Active per-market card focus indicator (left-border accent rule, 4px, matches Phase-58 verdict-banner pattern at `verdict-banner.tsx:32-37`)
4. Focus rings via `--ring` on tab triggers, table-row keyboard focus, button focus
5. Link colour for inline "View prior SDS" / "View audit log entry" anchors inside trigger reason chips

**Never use accent for:** card backgrounds, body text, decorative dividers, or non-actionable iconography.

**Semantic triad rule (WCAG 1.4.1):** every status communicator (band chip, verdict pill, trigger status badge, clearance outcome label) MUST combine **colour + icon + text** — never colour alone. Pattern locked in `apps/web/src/components/contractors/classification/outcome/verdict-banner.tsx:28-44`. Phase 60 reuses lucide icons:
- safe / Outside / Selbstandig → `CircleCheck`
- warning / Undetermined / mid-traffic-light → `ShieldAlert`
- critical / Inside / Abhangig → `ShieldX`
- pending / unknown → `ShieldQuestion`

## Interaction Contracts

| Surface | Interaction | Behaviour |
|---------|-------------|-----------|
| Dashboard refresh button | Click | React Query cache invalidated for all `classificationDashboard.*` queries; spinner replaces icon for ≥500ms (avoid jank); success → button returns to idle, last-scanned caption updates |
| Risk-distribution segment | Hover / focus | shadcn `tooltip` shows `{count} engagements ({percent}%) — {bandLabel}` |
| Reassessment trigger chip on engagement page | Click | Scrolls to inline trigger panel below the engagement header (no modal) |
| "Start new assessment" CTA | Click | Routes to existing Phase-58 wizard (`/contractors/[id]/engagements/[engagementId]/classification/new`) — Phase 60 does NOT pre-fill |
| "Dismiss trigger" | Click | Opens `alert-dialog`; reason ≥10 chars enables destructive button; on confirm → tRPC mutation → toast on success, optimistic chip removal |
| DRV-clearance row | Click | Opens edit `dialog` (same form as create, pre-filled) |
| DRV-clearance "File new" | Click | Opens create `dialog` |
| CSV download | Click | Triggers signed-URL fetch (300s TTL per CONTEXT.md D-16); button disabled + spinner during generation; browser download starts on URL receipt |
| All notifications in inbox | Click | Routes to engagement detail page anchored at the relevant section (band indicator, trigger panel, or DRV-clearance panel) — uses existing notification entity-pointer infra |

**Loading states:** every tile renders shadcn `Skeleton` placeholders sized to the final layout (no layout shift). Skeleton heights match: KPI numeral skeleton 36px, body skeleton 16px × 2 lines, chart skeleton 24px bar.

**Empty states:** every tile and the DRV-clearance panel use the existing dashboard empty-state pattern (centred icon + heading + body + optional CTA). Icons: `Inbox` (no triggers), `ShieldCheck` (no alerts), `FileText` (no clearance on file).

## Accessibility Contract (WCAG 2.2 AA — locked)

- **Colour contrast:** all text on `--background`, `--card`, and on `--primary` / `--destructive` / `--success` / `--warning` / `--info` swatches meets ≥4.5:1 for body, ≥3:1 for large text. Confirmed by Phase-58 outcome palette being the same OKLCh values reused here.
- **Semantic triad:** every band/verdict/status combines colour + icon + text (1.4.1).
- **Keyboard:** dashboard refresh, CSV download, all CTAs, all tab-like segments of the risk bar reachable via Tab, activatable via Enter/Space, focus ring uses `--ring` (Deep Teal).
- **Focus order:** logical — global header → market card 1 (tile 1→4 → CTAs) → market card 2 → footer.
- **Tooltip vs visible label:** tooltips on risk-bar segments are supplementary; raw counts are also exposed in the CSV export and in the table view of overdue triggers (information not solely conveyed by tooltip — 1.3.1).
- **Live regions:** the "Refresh data" button announces "Dashboard data refreshed" via `aria-live="polite"` on success.
- **Form validation:** DRV-clearance form uses inline `role="alert"` for validation errors (label, message, field association).
- **Modal traps:** dismiss-trigger and DRV-clearance dialogs use shadcn `alert-dialog`/`dialog` (Radix) — focus trap and ESC handling inherited.
- **RTL:** dashboard layout and stacked-bar segment order respect `dir` (Ar locale flips left↔right). `border-s-` / `start/end` Tailwind utilities used throughout — no `border-l-` / `border-r-` in new code.
- **i18n:** every visible string keyed under `Classification.polish.*` in all four locales (en/de/pl/ar). Numbers, percentages, and dates rendered via `Intl.NumberFormat` / `Intl.DateTimeFormat` with the active locale.
