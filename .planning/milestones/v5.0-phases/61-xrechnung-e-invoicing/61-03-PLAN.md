<objective>
Implement the KoSIT three-layer validator (EINV-04). Layer 1 = XSD via libxmljs2 against bundled CII D16B schemas; Layer 2 = EN 16931 Schematron via saxon-js SEF; Layer 3 = XRechnung CIUS Schematron via saxon-js SEF. Normalizes SVRL output into a typed `XRechnungValidationReport` usable by Plan 06's `finalizeEInvoice` mutation and Plan 07's E-invoice tab. Pre-compiles the KoSIT `.sch`/`.xsl` artifacts into `.sef.json` via `scripts/recompile-kosit-schematron.ts`, with SHA-256 checksums locked in-repo.

Purpose: EINV-04 is the load-bearing quality gate for every XRechnung invoice — without it, Plan 06's finalize mutation cannot persist a `validationStatus` and Plan 07 has nothing to render.

Output: a `validate(xml)` method on `XRechnungDEProfile`, validator-bundle populated with pinned SEF + XSD artifacts + checksums, tests passing against four KoSIT test-suite fixtures (2 positive, 2 negative).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted XML → XSD/XSLT/SVRL parsers | Any XML flowing into libxmljs2 / saxon-js / fast-xml-parser is a direct XXE/SSRF vector if misconfigured. |
| KoSIT-provided SEFs → saxon-js runtime | If an attacker substitutes a malicious SEF, `SaxonJS.transform` executes its XSLT — full RCE primitive. |
| Validator-bundle fixtures on disk | Files under `validator-bundle/` are trusted (checksummed at compile-script time). |
