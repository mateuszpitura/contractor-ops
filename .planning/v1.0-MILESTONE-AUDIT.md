## Integration Issues

### Medium Severity

**1. AsyncLocalStorage tenant isolation gap (Phase 01)**
- `tenant.ts` middleware sets `organizationId` in tRPC context but never calls `tenantStore.run()`
- Direct Prisma queries in routers would bypass tenant isolation
- All current queries use `ctx.organizationId` explicitly, so no actual data leak exists
- Fix: Wire `tenantStore.run()` in tenant middleware

**2. Onboarding checklist CTAs link to 404 routes (Phase 10)**
- `/contractors/new` — wizard is modal-based, no dedicated route
- `/settings/users` — actual path is `/settings/members`
- `/settings/approvals` — tab within settings page, not sub-route
- `/settings/integrations` — tab within settings page, not sub-route
- Fix: Convert CTAs to onClick handlers or use correct routes with tab params

### Low Severity

**3. Dashboard sidebar nav href** — `/dashboard` should be `/`
**4. Integrations sidebar nav href** — `/integrations` should be `/settings?tab=integrations`
**5. Cmd+K quick actions** — `?action=new` params not consumed by target pages
**6. Delegate user picker** — Text input for userId (documented v1 simplification)
