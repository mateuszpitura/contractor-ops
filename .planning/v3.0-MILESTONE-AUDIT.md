### Unsatisfied/Partial Requirements (4)

**LIN-04: Status changes in Linear sync to linked workflow task via webhooks**
- Phase verifier: passed (code exists and is correct)
- Integration check: BROKEN — `/api/webhooks/_process/route.ts` has no `if (provider === "linear")` dispatch block. `processLinearWebhook` exists but is never called. Linear webhooks are received and logged but status changes never reach the task.
- Fix: Add Linear dispatch branch in `_process` route (mirrors existing Jira pattern)

**LIN-05: Status changes on workflow task sync to Linear issue**
- Outbound (task→Linear): WORKING — `syncTaskStatusToLinear` called from `workflow.ts` on completeTask/skipTask
- Inbound (Linear→task): BROKEN — depends on LIN-04 fix
- Net: partial — bidirectional requirement is only half-satisfied

**EQUIP-06: DPD API integration for shipment creation, label, and tracking**
- Backend: COMPLETE — `DPDClient`, `createDpdShipment` (PRO-gated), `dpd-polling-service`, status mapper all exist
- UI: NOT MOUNTED — `CarrierShipmentForm` and `CarrierCredentialForm` are built but never imported into any page. Equipment detail page still uses old `ShipmentForm`. Settings has no carrier credential section.
- Fix: Mount `CarrierShipmentForm` in equipment detail and `CarrierCredentialForm` in integrations tab

**EQUIP-07: UPS API integration for shipment creation and tracking**
- Same pattern as EQUIP-06 — backend complete, UI components orphaned

**BILL-09: Feature gating by subscription tier (partial)**
- API gating: WORKING — `requireTier("PRO")` enforced on DPD/UPS endpoints
- UI gating: NOT ACTIVE — `FeatureGate` component built but never wraps any feature. STARTER users hitting PRO endpoints get raw tRPC FORBIDDEN errors instead of graceful upgrade prompts.
- Fix: Wrap tier-locked UI sections with `<FeatureGate>` component
