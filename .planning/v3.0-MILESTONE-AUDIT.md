### Broken Flow Details

**1. Offboarding auto-shipment → task auto-completion (MISSING-02)**

The webhook path is: InPost webhook → handleInPostWebhook → updates Shipment.currentStatus → updates Equipment.status → **stops**. The `checkShipmentTaskCompletion` call that would auto-complete the linked workflow task is missing from both `inpost-webhook-handler.ts` and all polling services. Task auto-completion only works when admin manually calls `addShipmentEvent` via tRPC.

**2. Tier gating inconsistency (MISSING-01)**

Phase 36-03 applied `requireTier("PRO")` to Linear, Jira, Calendar, OCR, and Audit mutation procedures. Teams `saveChannelMapping`, Google Workspace `bulkImport`/`triggerSync`, and all onboarding import endpoints were omitted. STARTER-tier orgs can access these features without upgrade prompts.

**3. CourierClient interface mismatch (MISSING-03)**

`CreateShipmentParams.targetPoint` is InPost-specific (Paczkomat ID). DPD/UPS use `AddressShipmentParams.deliveryAddress`. TypeScript bivariance allows `implements CourierClient` to compile. No runtime breakage today because router call sites are carrier-specific, but `getCourierClient(): CourierClient` return type is a correctness trap.
