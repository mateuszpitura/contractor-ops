## Integration Gaps (From Cross-Phase Checker)

### MISSING-04: sendChannelAlert never called (HIGH)
**Affects:** TEAM-02, TEAM-03
**Broken flow:** Teams channel alert routing

The `sendChannelAlert` method is implemented in both `TeamsMessagingProvider` and `SlackMessagingProvider`, and the `TeamsChannelMappingCard` UI stores category-to-channel mappings in `configJson.channelMapping`. However, `notification-service.ts` dispatch loop only calls `sendApprovalCard` (for APPROVAL_REQUEST) and `sendReminderDM` (for all other types) — never `sendChannelAlert`. Teams channel notifications are completely non-functional regardless of configuration.

**Fix:** Add a connector in `notification-service.ts` dispatch that: (a) looks up `configJson.channelMapping` by notification category, (b) calls `provider.sendChannelAlert({ channelId, ... })`.

### MISSING-05: CreditExhaustedInline never mounted (HIGH)
**Affects:** BILL-06
**Broken flow:** OCR credit exhaustion UI feedback

`credit-exhausted-inline.tsx` exists with upgrade/buy-credits actions but has zero imports outside its definition. When OCR credits run out, the backend correctly returns `PRECONDITION_FAILED`, but the calling UI shows a generic error instead of the inline upgrade prompt. The global `handleTierError` only handles `TIER_REQUIRED`, not credit exhaustion.

**Fix:** Import and render `CreditExhaustedInline` in OCR-triggering components (invoice intake, document upload) when the mutation returns credit exhaustion error.

### MISSING-06: OAuth connect buttons missing FeatureGate (MEDIUM)
**Affects:** BILL-09
**UX gap only — API layer is hard-blocked**

`LinearProviderSection`, `GoogleWorkspaceProviderSection`, and `TeamsProviderSection` allow STARTER users to complete OAuth connection. They then hit `requireTier("PRO")` walls on all mutations. Users connect successfully but can't do anything.

**Fix:** Wrap the OAuth connect CTA in each provider section with `<FeatureGate requiredTier="Pro">`.
