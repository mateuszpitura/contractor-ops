## Root-cause of the false positive
The dossier was generated by literal full-key scanning. Every namespace in this cluster is consumed via the next-intl-compat hook `useTranslations('<Namespace>')` (impl at `apps/web-vite/src/i18n/useTranslations.ts`), where call sites then use **relative** subkeys: `t('back')`, `t('connectButton')`, `t('dashboard.statusLabel')`, etc. The detector saw `Zatca.complianceChecks.back` in `messages/*.json` but no literal `'Zatca.complianceChecks.back'` in source — so it flagged it dead. It is not.

Sample evidence:
- `components/zatca/zatca-integration-container.tsx` uses `useTranslations('Zatca.page')` → calls `t('title')`, `t('description')`, `t('connectButton')`, `t('connectTitle')`, `t('connectDescription')`, `t('backAriaLabel')`, `t('sandboxBanner')`, `t('dashboard.statusLabel')`, `t('dashboard.statusHint')` — all flagged "dead".
- `components/zatca/compliance-checks.tsx` (`Zatca.complianceChecks`) → `t('title')`, `t('description')`, `t('back')`, `t('next')`, `t('runChecks')`, `t('running')`, etc.
- `components/zatca/csr-generation.tsx` (`Zatca.csrGeneration`) → same pattern.
- `components/settings/tax/hooks/use-wht-calculator-section.ts` (`TaxAdmin.calculator`), `use-wht-certificates-section.ts` (`TaxAdmin.certificates`), `use-country-rates-section.ts` (`TaxAdmin.rates`).
- `components/settings/hooks/use-ksef-setup-dialog.ts`, `use-integrations-tab.ts`, `use-ksef-sync-history.ts`, `components/invoices/ksef-duplicate-banner.tsx`, `ksef-metadata-section.tsx` all bind `useTranslations('ksef')`.
- `components/settings/e-invoicing/hooks/use-peppol-participant-card.ts` binds `EInvoice.Settings.PeppolCard`, `EInvoice.PeppolDialog`, `Peppol.capabilities`.

All 47 distinct `useTranslations()` namespaces matching `Zatca|EInvoice|TaxAdmin|ksef|Peppol` were enumerated and every dossier-listed leaf falls under one of them.

## Recommendation
- **Recommended action: HOLD the entire cluster. Do NOT delete any of the 344 keys.**
- Rationale: every dossier leaf is reachable via a live `useTranslations(namespace)` binding in shipped components backed by completed roadmap phases (XRechnung 61, ZUGFeRD 62, KSeF milestone 118–120, ZATCA Phase 70, Peppol via Storecove). The dossier is a methodology artefact — a literal full-key scanner that does not resolve relative `t('subkey')` calls under namespaced hooks. Deleting these keys would break live UI in all 4 locales (en/de/pl/ar) on the next render.
- **Action for the audit pipeline:** before any compliance-cluster deletion, the detector must be re-run with namespace-aware resolution (parse `useTranslations(<ns>)` and synthesize `${ns}.${literalSubkey}` references). Until that is in place, treat compliance namespaces as quarantined.
