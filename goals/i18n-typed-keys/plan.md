# Plan — i18n typed keys cleanup

## Solution approach

Three building blocks, executed in order:

1. **Generated type** — codegen script reads `apps/web/messages/en.json` and emits `apps/web/src/generated/i18n/messages.d.ts` containing a deep `Messages` interface mirroring the JSON structure.
2. **Module augmentation** — `apps/web/src/types/next-intl.d.ts` declares `interface AppConfig { Messages: GeneratedMessages }`, so next-intl's `useTranslations` / `getTranslations` resolve dotted keys strictly. This alone fixes all 53 static-string casts.
3. **`tDyn` helper** — `apps/web/src/i18n/typed-keys.ts` exports a typed dynamic-key helper that validates sub-namespace + leaf at the type level, replacing all 81 template-literal/pass-through casts.

Codegen runs under turbo with `inputs: messages/en.json` and feeds `typecheck` / `build` / `dev`. A Biome rule (or fallback CI grep) blocks future casts.

## Cast inventory (verified)

| Pattern | Count | Resolution |
|---|---|---|
| `t('static.key' as Parameters<typeof t>[0])` | 53 | Augment → drop cast |
| `t(\`subNs.${expr}\` as Parameters<typeof t>[0])` | ~70 | `tDyn(t, 'subNs', expr)` |
| `t(key as Parameters<typeof t>[0])` (pass-through `key: string` param) | ~10 | Narrow caller signature to `MessageKey<NS>` |

### Step 4 — `tDyn` helper

- Create `apps/web/src/i18n/typed-keys.ts`:
  - Re-export utility types:
    ```ts
    import type { useTranslations } from 'next-intl';
    import type { GeneratedMessages } from '@/generated/i18n/messages';

    export type NamespaceKeys<NS> = /* … recursive object-child keys … */;
    export type MessageLeafKeys<NS> = /* … recursive string-leaf keys … */;
    ```
  - Implement `tDyn`:
    ```ts
    export function tDyn<
      T extends ReturnType<typeof useTranslations>,
      Sub extends NamespaceKeys<Parameters<T>>,
      Key extends MessageLeafKeys<NamespaceAt<Parameters<T>, Sub>>
    >(t: T, subNs: Sub, key: Key, values?: Parameters<T>[1]): string;
    ```
  - Implementation: `return t(\`${subNs}.${key}\` as never, values);` (single localized internal cast — acceptable, hidden).
- Add unit tests at `apps/web/src/i18n/__tests__/typed-keys.test.ts`:
  - Valid `tDyn(t, 'status', 'active')` compiles and returns a string.
  - Invalid `tDyn(t, 'status', 'nonExistent')` triggers tsc error (use `// @ts-expect-error`).
  - Invalid `tDyn(t, 'doesNotExist', 'x')` triggers tsc error.

**Files touched:** `apps/web/src/i18n/typed-keys.ts`, `apps/web/src/i18n/__tests__/typed-keys.test.ts`.

### Step 5 — Migrate call sites

Process in batches by folder to keep PR reviewable. For each batch:

1. **Static-string casts** (53 sites): Remove ` as Parameters<typeof t>[0]`. Augmentation already accepts the literal.
2. **Template-literal casts** (~70 sites): Replace `t(\`a.${x}\` as Parameters<typeof t>[0])` with `tDyn(t, 'a', x)`. Where the pattern crosses two segments (`t(\`a.b.${x}\`)`), pass `'a.b'` as `subNs`.
3. **Pass-through wrappers** (~10 sites): Wrappers like `getColumns((key: string) => t(key as Parameters<typeof t>[0]))`. Change wrapper's caller-supplied `key: string` to `key: MessageLeafKeys<NS>` exported from `typed-keys.ts`, then drop the cast at the call site.

## Risks & open questions

- **tsc cost from inflated `Messages` interface (7564-line JSON → deep type tree).** Mitigation: emit as `interface` not `type` (TS handles interfaces better for deep trees), sorted keys, no unions. If tsc memory/time regresses by >25%, fall back to a flat `MessagePath = "a.b.c" | "a.b.d" | …` union — still strict, often faster.
- **`tDyn` generic depth.** Recursive conditional types on a 55-namespace tree may hit TS's instantiation limits. Mitigation: cap recursion in helper types at depth 5 (deeper paths take `string` and runtime-validate). Benchmark with `tsc --extendedDiagnostics` after step 4.
- **Pass-through `(key: string) => t(key as …)` wrappers crossing module boundaries** (e.g. `getColumns` in `apps/web/src/components/invoices/invoice-table/columns.tsx`). Caller may pass keys from another namespace. Need to inspect each: if the wrapper truly accepts any key, accept `MessageLeafKeys<NS>` from the namespace the wrapper's `t` is scoped to; if the wrapper is called from multiple namespaces, hoist the wrapper to accept a typed `t` parameter directly.
- **`scripts/audit-i18n-code-coverage.ts` overlap.** Once `tDyn`+augmentation enforce key validity statically, the audit script becomes a runtime double-check (still valuable for keys built via patterns the type system can't see, e.g. enum-key-derived). Keep it running in CI; no functional change.
- **Biome `useNoRestrictedSyntax` availability in 2.4.14.** Confirm rule exists; if not, ship the fallback grep script in step 7 from the start (zero functional difference, same enforcement).
- **Locale parity.** Augmentation makes `en.json` the canonical type source. Drift in `pl/de/ar` is caught by the existing `pnpm i18n:parity` script — confirm it's wired into CI before declaring done.
- **`next-intl` v4.12 in pnpm tree.** Multiple versions resolved (4.11 + 4.12). Check whether augmentation needs to target both; typically the workspace root resolution determines which `AppConfig` is augmented. Verify with a simple typed-key smoke test before mass migration.

## Out of scope

- No new translation features (pluralization, ICU formatting, new namespaces).
- No migration of `audit-i18n-code-coverage.ts` logic into the type system; it stays as a complementary runtime gate.
