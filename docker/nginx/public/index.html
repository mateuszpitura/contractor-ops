contractor-ops — local dev links

Bring everything up: docker compose --profile infisical --profile dev-tooling --profile unleash up -d

Application servers

  • Web app — Next.js dashboard + portal (pnpm dev)
  • Landing — marketing site (pnpm dev)
  • Public API — Hono Enterprise REST (pnpm dev)
  • Web health probe — Postgres/Redis/QStash/R2/backpressure status

Feature flags

  • Unleash — flag dashboard (profile unleash; default admin admin / unleash4all)
  • Seed flags + generate API token: node scripts/seed-dev-services.mjs

Secrets

  • Infisical — secret manager via nginx gateway (profile infisical; backend on :8080 internally)

Email + storage

  • Mailpit — captured email (SMTP on :1025 — set DEV_SMTP_HOST=127.0.0.1 + DEV_SMTP_PORT=1025 in .env)
  • MinIO Console — S3-compatible storage (API on :9000; admin minioadmin / minioadmin)

Database + cache UIs

  • DBGate — unified Postgres + Redis GUI (parallel rollout — will replace pgAdmin + RedisInsight once verified) (login root / Test1234!; 4 Postgres + 2 Redis connections pre-registered from env)
  • pgAdmin4 — Postgres GUI (login admin@local.dev / Test1234!; 4 servers pre-registered with passwords: app-db (browse contractor_ops_eu/me/cms inside), infisical-db, unleash-db, glitchtip-db)
  • Redis Insight — Redis GUI (app cache: add redis://app-redis:6379; Infisical's own Redis: redis://infisical-redis:6379)

Local cloud substitutes

  • app-db — plain Postgres 17 on localhost:5433 (Neon substitute, hosts both regions) (profile dev-tooling; databases contractor_ops_eu + contractor_ops_me; persistent volume app_db_data; run pnpm db:migrate:all to fan migrations across both)
  • app-redis — plain Redis 8.6 on localhost:6380 (profile dev-tooling; persistent volume app_redis_data)
  • SRH — Upstash-Redis-REST proxy in front of app-redis (UPSTASH_REDIS_REST_URL=http://localhost:8079 · UPSTASH_REDIS_REST_TOKEN=example_token)
  • QStash dev server — Upstash QStash mock (set QSTASH_URL=http://localhost:8089 + token/signing-keys printed at container startup; consumer callbacks must use http://host.docker.internal:3000)
  • GlitchTip — Sentry-compatible error tracking (profile glitchtip; auto-seeded admin admin@glitchtip.local / Test1234! + org/project; grab DSN: docker compose logs glitchtip-seed | grep NEXT_PUBLIC_SENTRY_DSN)

Background services (no UI)

  • clamav — antivirus scanner on TCP :3310 (default profile — required for file uploads; check: docker compose ps clamav)
  • worker — node-cron scheduler (profile worker; runs as standalone Node ESM container)

External / cloud-only (no local container)

  • Neon Postgres — primary DB (DATABASE_URL; local substitute via app-db above)
  • Upstash Redis — rate limiting + idempotency cache (UPSTASH_REDIS_REST_URL; local substitute via SRH above)
  • Upstash QStash — webhook queue (QSTASH_TOKEN; local substitute via QStash dev above)
  • Cloudflare R2 — document/PDF storage (R2_ACCOUNT_ID)
  • Sentry — errors + tracing (NEXT_PUBLIC_SENTRY_DSN; local substitute via GlitchTip above)
  • Axiom — log shipping (AXIOM_TOKEN)

One-shot setup commands

  • pnpm db:push — apply Prisma schema to Neon
  • pnpm db:seed:dev -- --profile=showcase --confirm --progress — seed dev data with progress bar
  • node scripts/seed-dev-services.mjs — create Unleash flags + print env block
